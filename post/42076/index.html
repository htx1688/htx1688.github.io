如何避免用户“到手就砸”？NFT项目可以试试“锁仓时间表”

NFT项目方识别用户忠诚度和长期利益一致者的试纸。

为了激励有长远眼光的投资者加入白名单，并防止白名单地址快速 Flip（获取短期利润就抛售的行为）。 NFT 锁仓时间表（VestingTimeLock）应运而生，类似于代币锁仓时间表，从白名单中 Mint 的 NFT 将面临某种形式的锁仓。为 NFT 项目方筛选忠诚度最高的 NFT 收藏者，为项目增值。

现有的解决方案——质押

目前用于协调质押者利益的最常见机制之一是质押。用自己的原生代币或其他特权将其 NFT 质押到合约中的人将获得质押奖励。

例如，Moonbirds 有一个“软质押”机制，即你的 NFT 不会被锁定，但如果你不参与质押，你将失去累积的奖励点数。这类似于投票托管模式。在激励机制中，你可以累积非代币点数来获得更高层次的奖励，可以通过调整激励机制来激励长期质押者。为了防止或阻止抛售行为，白名单钱包可以根据分级制度发行，以较低价格 Mint 的用户在 Mint 结束后被锁定的时间会稍长一些。

Moonbirds和其嵌套（nesting）页面

NFT 项目 10KTF 还提供了一种通过游戏玩法进行质押的创新方法，正如其名，质押发生在游戏中。通过在游戏任务中给予 NFT 奖励，NFT 持有者能够完成任务并获得该 NFT 作为奖励。

现有方案存在的问题

虽然上述机制确实能够通过累积奖励激励玩家的忠诚度，但它们仍然不能阻止早期白名单玩家的抛售。此外，由于没有现有的关于 NFT 的正式标准，当项目想要在不同的 NFT 市场上上线时，上述的“软质押”机制可能会遇到问题，因为团队将必须与相关的 NFT 市场合作，以确保这些“软质押”的 NFT 不被上线出售。

因此，对于需要锁定供应的项目来说，实现锁仓机制可能是一个更好的解决方案。

设计决策的改进

我们相信 NFT 不仅仅代表了单纯的 PFP，或 Flipper（获取短期利润就抛售的行为的收藏者）快速赚钱的渠道。为了帮助 NFT 领域的长期发展，我们必须考虑如何使 NFT 社区与项目的长期利益保持一致。

以下是我们在提出解决方案时所使用的一些指导要点：

1. NFT 项目的增值价值和 NFT 的增值价值之间的一致性： 这将有助于激励人们长期持有他们的 NFT。然而，通常情况下，NFT 项目的增长可能不会导致持有 NFT 价值的相应增长，这导致人们抛售他们的 NFT。

2. 代币锁仓时间概念在NFT 领域的应用： 我们还考虑将代币锁仓时间的概念（它为后 ICO 时代 NFT 的爆发划定界限）应用到我们的解决方案中。这将有助于防止 NFTFlipper /巨鲸用户滥用伪匿名的概念来排挤 NFT 市场，因为这将使他们能够 Flip。通过这种方式，我们将能够奖励真正的长期信念，而不是抛售行为，这样社区就会受到激励，为项目的长期发展而留下，从而消除抛售倒卖的行为。从而最初的白名单可以与锁仓协同使用，作为 V 神的“灵魂绑定”的一种形式，或表示真正的信仰。

3. V 神的“灵魂绑定” NFT： 我们也相信可以从 V 神的“灵魂绑定”讨论中，以及经过考验的锁仓流程中获得灵感，从而协调质押者之间的激励。灵魂绑定的识别方面变得越来越重要，因为 NFT 开发的用例不仅仅是头像，而是代表真实世界的实体（比如细分的房地产）。

“软硬皆施”的解决方案

为了创建一个真正的信徒社区，我们相信某种形式的锁仓或锁定将适当地调整激励。

因此，我们正在提出一个新的智能合约标准，为白名单钱包执行一种锁仓方式，其中从白名单中产生的 NFT 将面临一种形式的锁仓期（类似于代币锁仓时间表）。这将激励有长期愿景的投资者进入白名单，并防止白名单用户地址迅速 Flip。

我们的解决方案是“软硬皆施”方案，其中我们将引入一个双重奖励系统（定时锁定识别）和惩罚系统（NFT 锁仓），从而更好地协调白名单和 NFT 项目之间的激励机制。

惩罚系统：基本的 NFT锁仓合约

首先，有一个简单的 NFT 锁仓智能合约，其灵感来自于 OpenZepplin 的代币锁仓。在这种情况下，NFT 被锁定在合约中，只有在锁定期结束时（比如 1 年）才可以提取。作为一种基本合约，可能没有太多的效用，因为社区成员只有在锁定期之后才能访问他们的 NFT。此外，这类合约并没有真正的“奖励系统”。

然而，尽管有上述的缺陷，基础的 NFT 锁仓合约仍然是更高级时间锁定模式的基础，我们将构思这些模式来解决前面提到的设计问题。

NFT锁仓——奖励锁定其 NFT 的白名单用户

前面提到的一些设计问题涉及到项目和锁仓之间的激励调整。一方面，项目不希望白名单或早期买家在 Mint 时抛售。另一方面，除了表面的（虚荣心）原因，长期支持者没有任何动力继续维持 NFT。因此，我们应该考虑锁仓的概念，以奖励锁定他们的 NFT 的白名单用户。

下面两张图展示了锁仓机制如何使质押者之间的激励保持一致：

有锁仓期的NFT

例如，一个 NFT 需要 1 个 ETH ，该项目为支持该项目的白名单用户提供 50% 的折扣。在这些白名单用户 Mint 了 NFT 后，所述的 NFT 将进入锁仓合约，在初始锁定时（给定的 NFT 项目）将向开发者发放 0.5 ETH（每个生成的 NFT）。然后，白名单用户必须等待断崖锁仓期（NFT 被锁定的最短时间）结束。

在传统金融中，断崖锁仓期（cliff vesting：每期将较大份额分期兑现）是指在一定日期之前不能授予股份的一段时间。在断崖期之后，合约进入一个锁仓期，在此期间折扣将开始增加。应计折现率取决于联合曲线集（bonding curve set），以及锁仓期限。

不同类型的锁仓曲线会影响用户解锁后获得的折扣。

例如，如果用户锁定了他们的 NFT，并在最大锁定时间的 1/2 之后解锁。不同锁仓曲线下收到的 ETH 金额如下图所示：

  • 在线性锁仓下，0.5 ETH 归开发者，0.5 ETH 归用户；

  • 在曲线锁仓下，0.5 ETH 归开发者，小于 0.5 ETH 归用户。在最后会有更重的权重，呈指数增长。

锁仓曲线类型

锁仓曲线类型包括基于间隔的、线性的、曲线的（凸形和 S 型）锁仓曲线。其中一个曲线更适合项目的用例：

基于间隔锁仓： 希望有一个固定的锁仓时间表，其中折扣在断崖期之后定期解锁的项目。

线性锁仓： 用于不想歧视断崖期后锁定对象锁仓的时间长度，并希望持续增加锁仓期的项目。

曲线锁仓： 用于那些想要奖励长期的锁定对象而惩罚短期锁定的项目。这对于那些想要奖励长期行为的项目可能有效。

1. 凸形锁仓曲线：当锁定对象实现最大折扣或锁定对象实现最大折扣的可能性时，需要一个有最高上限要求的项目。

2. S 形锁仓曲线：用于那些想要激励锁无限锁定的项目，并且不想为锁定设置固定的持续时间。

然而，作为一种机制的锁仓并不足以调整激励机制，因为白名单用户仍然能够在锁仓结束时进行抛售。因此，我们必须引入另一种方法来激励白名单成员随着时间的推移而继续被锁定。

双重奖励系统：锁仓识别

是否有一种方法可以进一步区分真正的项目支持者和 NFTFlipper？

目前，一些项目对质押 NFT 提供激励。虽然这可能有助于暂时发现项目的真正支持者，但这种锁定通常是短期的，从长远来看，这无助于区分支持者和 Flipper。

不同的识别等级

当我们探索在货币领域之外锁定 NFT 的动机时，打开了大量的可能性。

作为一个社区，我们可以在不同的锁定级别上进行区分，并相应地识别支持者（从高到低）：

0 级-从最初的空投开始的白名单锁定；

1 级-锁定时间超过最初的空投期；

2 级-解锁超过最初的空投期；

3 级-解锁并已售出 X 次；

4 级-已解锁，自上次在市场交易以来已被卖出 X 次；

5 级及以上，更多属性。

然后，项目可以为这些锁定对象提供不同的奖励级别，优先考虑最佳奖励（特殊的 NFT 空投，更多代币等），以支持最高级别（0 级）的支持者。例如，项目所有者可以奖励持有> 30 天的持有者，并获得奖励。这种奖励可以是二次空投的折扣，特殊的代币奖励等。

此外，链上和链下的识别可以用于其他项目，通过空投来激励或奖励这些人。对支持者来说，这种识别可能是一种重要的地位象征，它可能超越所持有的 NFT 的货币价值。最终，这种识别对激励社区内的长期思考和买卖行为非常重要。

识别的表现

当持有者锁定了他们的 NFT，而不是简单地失去 NFT，合约可以被扩展为绑定到一个包装（wrapped） NFT 或一个“灵魂绑定” NFT。

例如，对于包装的 NFT，可以通过交易包装的 NFT 来获取真正的 NFT，从而允许社区成员对这些代币的效用有一个引用。“灵魂绑定” NFT 可以代表用户在解锁真正的 NFT 之前锁定他们的 NFT 多长时间。为了创造更多的社交理由去锁定，项目可以使用“名人堂概念”作为一个排行榜，将锁定时间最长的地址作为最忠实的信徒。

例如，当 NFT 在锁仓中被锁定时，锁定对象可能仍然希望展示他们确实拥有这样一个锁定的 NFT。然而，从开发者的角度来看，引用锁定的 NFT 是很奇怪的。因此，“灵魂绑定”代币可以代表用户锁定 NFT 的时间，他们可以进入名人堂排行榜。在解锁时，需要烧毁不可转移的包装代币来解锁 NFT，而锁定对象将不再具有“名人堂”的地位。

识别的不可转让性

为了区分锁定对象，项目可能希望将代币的不可转移性纳入其中，以验证帐户地址是否是真正的支持者。这可能很重要，因为它将允许项目奖励真正的支持者。仍然会有一些人通过挖矿获得这些代币并出售账户买卖不可转让的“地位”代币，但这样的行为将会受到抑制。

锁仓识别的应用

在应用上，项目可以采用链上和链下相结合的方式。在智能合约端，应该有一个功能来检查 NFT 被锁定了多长时间。然后，项目可以跟踪每个地址锁定的时间长度，并奖励他们相应的识别，如名人堂或空投基于锁仓的不可转让的“地位”代币。

结论

在本文中，我们回顾了围绕当前 NFT 发行设计的问题，并提出了一种不常见的替代解决方案——锁仓（时间锁定）。事实上，在锁仓解决方案和质押之间，以及市场上的 NFT 之间，时间锁定解决方案有很多协同作用。项目可以继续使用代币/ NFT 的质押机制，在开始时创造热门和强大的社区，同时为质押者提供不同的特权，以激励长期持有。

除了使用锁仓或其他链上手段来跟踪 NFT 持有者的忠诚度水平，项目还可以考虑链下、市场上的 NFT。参考“自上次转手以来的持有时间”可以帮助项目确定 NFT 所有者对项目的忠诚度水平，并根据他们的忠诚度水平奖励持有者。具有随时间变化的奖励机制将为锁定提供另一种激励。
