随着 SBT「Soulbound Token  - 灵魂绑定代币」概念的出现以及深入探究，许多人对于「无数 SBT 总和形成 Web3 世界身份」这一未来寄予厚望。

2022 年 1 月，以太坊创始人 Vitalik Buterin 撰文提出 SBT 设想：如果我们将 SBT 理解为具有标签性质的代币，一方面 SBT 与用户账户（钱包）深度绑定，另一方面 SBT 的不可转让性也弱化了投机属性，两种特质加持下，随着一个钱包积累越来越多的 SBT，这一钱包所代表的链上身份就越丰富、越值得信任。

回望 Web2，应用层均是围绕「用户身份」所展开的数据价值捕获，而 Web3 的叙事吸引力则在于数据主权回归+数据价值挖掘。 因此围绕去中心化链上身份而展开的数据挖掘和社交延伸，理所当然成为推动 Web2 用户加速迈进 Web3 的必由之路。

作为构建去中心化信誉体系的可行解决思路，SBT 理念迅速聚焦社区目光，在这一赛道中，以「实现 Social Data Oracle」为目标的项目 Port3 Network（以下简称 Port3）作为 BNB Chain MVB V 最新一批孵化项目暂露头角。

该项目基于项目发展思路与 SBT 理念高度契合考虑，于近期推出了首个SBT设计大赛（点击查看活动详情）。8 月 17 日 - 9 月 13 日，Web3 爱好者可尽情释放想象力，设计各种应用场景下的独特 SBT，目前已有 379 名创作者踊跃参与为 Port3 的 SBT 设计形象。该 SBT 将代表着用户在 Port3 体系中的 Pass，实现身份的识别和验证，项目将于 2022 年第四季度正式推出其 SBT 的 Mint 工厂。

  

Port3 的 Social Gateway 宏图

Web3 社交平台的目标接近于：基于开放数据勾勒链上关系图谱并提供更丰富、高效信息撮合服务。实现高效撮合的关键点在于广泛的数据获取和精准的数据分析能力，Port3 要成为 Web3 世界的社交数据网关，已拿出相当具体的实现思路并呈现出「两步走」的态势：

第一步：多维度聚合链上链下数据并进行分析处理，构建从 Raw Data 中提取标准化数据的机制。

用户身份的基础是数据，没有足够多的数据，用户的Web3身份不能够形成。Port3 在早期尽可能多地拓展有效的数据集并寻找应用场景，有了数据，Port3 将可以对用户的身份、偏好习惯、社交联系等进行标签化，以安全、隐私的方式实现 Web3 身份体系。

人们越来越意识到，离开现有的 Web2 平台去开辟独立的社交处女地是不现实的，大量社交行为仍发生在 Web2 中，链上没有供 NFT Mint、智能化空投、DAO 治理等场景直接可用的数据。将 Web2 和 Web3 的数据聚合，再有选择的提供一些场景需要的数据进行切入，是当前最合理的选择。

广义的 Social 行为，不只是你在 SNS 平台发的推文，还包含你在链上链下的方方面面。由于 Social Data 的体量巨大，每条源数据所蕴含的价值较小，需要从海量的 Raw Data 中进行清洗、分析、提取，得到价值浓度更高的标准数据，才能有效服务第三方应用。此外，Social Data 和价格等数据不同，它的来源较为中心化，很难判定第三方提交的数据真假。故而在早期，Port3 通过自建 DAO Tool 矩阵和 SoQuest 等产品，获得具有应用潜力的数据，从而探索出一个标准化的机制。

第二步：基于数据聚合打造去中心化 Social Data Oracle，为各种 Web3 身份、资产应用提供开放式的数据层服务，融入 Incentive Layer（激励层）彻底实现数据权利归用户所有。

打通链下链上数据通道方面，Oracle 可谓是功不可没，从 Chainlink 等 Price Oracle 对于 DeFi 的贡献中可见一斑。离开可靠的价格数据，链上借贷、交易等场景无法展开。未来的 Web3 大浪潮中，离不开 Social 场景和数据的上链。而 Social Data 和 Price Data 要解决的问题具有很大差异，要实现的是海量数据的交换需求，故而 Port3 为解决这一问题专门构建一个 Social Data Oracle 实现链下链上数据的桥接。

Port3 通过聚合 Web2+Web3 数据，在区块链网络中提供开放数据服务，这对于形成用户身份，保护用户隐私，实现各类应用提供了实现的基础。Web3 领域在声誉系统、身份认证、智能空投、社群管理、DAO 治理、广告推荐等方面已经涌现了众多应用，甚至还涉及到了招聘、相亲领域，然而就目前来讲这些场景还只存在小范围的连通，没有呈现广泛的连通性。就像 Price Oracle 之于 DeFi 一样，蓬勃发展的 Web3 场景，也需要自己的 Oracle，才能达到这一连通性，从而形成强大的网络效应。

Social Data Oracle的目标，在于解决三个核心问题：

  • 数据以什么方式组织与连通；

  • 数据的权利如何归属用户；

  • 用户的隐私如何保护。

Port3在这三方面做了深入的思考，并设计了一套去中心化机制予以实现。首先，各种源数据通过多种渠道（上报、爬虫、索引）汇集至Port3。开发者可以定义自己的数据模型，也即定义输入、输出及处理算法，并发布到Data Markets中。Oracle中的Aggregator节点，将运行数据模型定义的算法，并将结果输出到Oracle合约中，可直接供第三方应用读取使用。Data Markets中的数据模型可以相互组合，并被网络中的Readers节点索引，形成二次聚合数据提供给第三方使用。

在此过程中，Oracle 还融入了 Incentive Layer，读取数据需要向 Oracle 网络支付费用，支付的费用将会分发给网络的 Aggregator和Readers节点，一部分将分发给数据的贡献者们。单个用户是最小贡献单位，在获得贡献之前，需要拥有 Port3 的 SoulBound NFT，作为Port3生态的身份认证，这同时也是用户隐私的有效隔离。最终输出至 Oracle 的，只有用户的 Web3 身份和聚合后的目标数据。通过这种方式，用户即使只是参与了一次被 Oracle 读取的链下活动，也会得到分享数据带来的收益，这是 Web2 所从来不曾有的。与此同时，用户将可以通过 Social Data Oracle 的 Readers，索引出所有自己的关联数据，进行统一管理。

△ SoQuest 产品界面

SoQuest 上线后不俗表现

作为 Port3 宏图聚焦的第一个数据场景，SoQuest 的定位是成为 Web3 世界的流量聚合平台，其目标是在 Quest 场景中将收集的数据，开放至 NFT Mint 和 Airdrop 等场景中。SoQuest 已于近期 7 月上线，并获得不俗的数据表现。

SoQuest 上线两周，已有70+ 合作伙伴开通了 SoQuest Space，并举办了 61 场 Campaign 活动，总计18,330+ 用户参与，完成了313,640+ 次 Action 操作，总计有 29,130 的用户连接自己的钱包登录，SoGraph 日活达到 2,100+。

Web3 世界的任何项目方、意见领袖和 DAO，都可通过申请页面构建专属 Space 空间并轻松发起包含各种链上、链下任务的品牌活动，在和 SoQuest 数据互通后，所有参与者完成 SoQuest 任务后可以立即获得奖励，无需快照、地址上传、白名单等繁琐流程。

△ SoQuest 的市场表现

随着越来越多的项目方推出各式活动以吸引用户参与，不断丰富链上、链下数据促使 Port3 标签更多元、身份更精准，加速 Port3 成为真正意义上的 Web3 世界社交数据网关及流量入口迈进。

Port3 Network 的三大模块

Port3 何以在短期内展现出强劲的增长势头？其设计有何特别之处？从项目的设计体系来看，Port3 拥有三个核心模块，分别对应着数据收集、数据查看、数据运用。

△ Port3 的三个核心模块分别对应数据收集、数据查看、数据运用

  • 社交数据聚合渠道：数据收集

通过 DAO Tools，Port3 聚合的数据涵盖 Telegram/Discord 等即时通讯平台社区动态、推特（未来还将支持 YouTube/Instagram 等）社交媒体内容趋势，以及链上实时价格及交易交互，另外通过 Contribute  to Earn 方式，鼓励更多人贡献数据获得平台代币收益及治理权，为 Port3 数据聚合提供了更广泛的数据来源。

目前，Port3 通过 Discord 和 Telegram 的 DAO Tools 矩阵覆盖 330+ 万用户，并支持 10 条热门公链的 340+ 项目，包括 Ethereum、BNB Chain、Solana、Avalanche 和 Fantom 等，项目方可轻松通过 DAO Tools 实现链上交互和社群分析。

  • SoGraph.xyz：数据查看

作为 Port3 终端页面，SoGraph.xyz 是数据聚合的结果展现，可视化的展现了社区动向、项目活动以及市场趋势等关键信息。

对于个人而言，SoGraph.xyz 是关联 Web2、Web3 身份，参与各种活动，查看 Web3 发展趋势的入口，链上链下的整合简化了个人参与项目活动的流程，从繁琐的各个页面跳转解放出来，轻松完成空投、白名单等任务；

SoGraph 的 Dashboard 版本，则为项目方和社群提供更多数据 Insight，基于 ML 的分析算法可以多种数据指标的分析，例如社群用户组成，以及市场 Sentiment。数据看板更好地帮助项目方了解自己的社群，并且能够敏锐捕捉用户的趋势和需求，正确地激励用户。

  • Port3 协议：数据应用

Port3 协议是 Port3 构建开放协作数据网络的基础，也是实现数据服务撮合的核心。

首先，数据聚集并完成隐私保护处理后，数据结果上传至 Port3 数据交易市场；当有人希望从 Port3 读取数据内容时，读取者需要向平台支付 Oracle 代币；当数据被使用，则数据提供者可获得代币奖励。

整个过程中，数据提供者完全掌控自己的数据且受到严格隐私保护，并且通过 Port3 协议激励层获得了数据使用收益以及平台治理权益；与此同时，Port3 协议消除中心化数据中介，第三方数据使用者也以更低成本、更精准的方式获取了有效数据。

Port3 Network 的 RoadMap 及核心竞争力

7 月 28 日，Port3 已经发布 SoQuest 解决方案并提供 API 支持，而在即将到来的 9 月，Port3 将构建 BNB Chain Action Parser，以实现更大规模的链上数据聚合。

2022 第四季度，团队将正式推出 Social Data Oracle v1 版本，实现通过预言机合约共享数据，并发布 SDK 以方便合作伙伴使用，预计在第四季度，Port3 合作伙伴将增至 200+。

Port3 将在 2023 年第一季度将链上 Parser 扩展到以太坊和其他  EVM 兼容链，并发行 Soulbound NFT 标记用户；到第二季度，团队将升级数据分析看板，以展现更多数据分析结果；第三季度，Port3 Network 将探索交易、IDO、DAO 管理、社交平台等更多数据应用场景。

Web3 世界瞬息万变，新叙事不断涌现。但万变不离其宗，围绕去中心化链上身份所展开的数据确权、数据分析、数据服务以及数据收益探索，仍是跨越周期的重点探索方向。

不管是 Web2 世界的 Gleam，还是 Web3 世界的 Quest 平台，市场中都不乏和 SoQuest 类似的产品，那么 Port3 有何优势面对激烈竞争？

与 Web2 世界的任务系统巨头 Gleam 相比较，首先，Port3 实现了 Web2 和 Web3 之间的无缝连接；其次，Port3 将数据的所有权真正地归还给用户，让用户不仅享有隐私保护，还能获得收益。

而与 Web3 世界里众多的 Quest 平台相比较，Port3 只是把 Quest 当成当前最活跃的一个场景，作为 Social Data Oracle 的初始燃料。此外，Port3 拥有其它项目不具备的 DAO Tools 百万用户基础，这也意味着更广泛的数据来源。

基于这些源源不断的数据，Port3 得以探索社交网关和 SBT 的实现，致力于实现真正意义上安全、高效的社交数据聚合，推动 Web2 的用户和数据加速进入 Web3。Port3 是否能真正成为  Web3 世界的社交数据网关，相信时间是最好的证明方法。
