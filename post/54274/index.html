编者按：NFT 的第一个迭代（10k PFPs）结束了，是时候回归 NFT 的基本面并重拾 NFT 的真正使命，比如数字所有权、创作者/用户权益、去中心化的元宇宙等。ERC-6551 是 NFT 领域的重要创新，让我们一探究竟！

ERC-6551 的代币绑定账户 (TBA) 揭示了一个新范式。任何单个不可替代代币（NFT）现在都可以超越其作为单纯数字资产的传统角色，成为功能齐全的钱包。NFT 不再只是拥有漂亮的外表，它们还充满活力。

这一技术突破的影响远远超出了艺术品和数字收藏品的范围。交易、游戏和无数其他行业都准备好迎接这一变革。TBA 驱动的 NFT 新用例为更多创新铺平了道路。它需要彻底重新构想我们如何看待链上所有权、身份和价值。

本文将简要介绍 NFT 的历史，然后深入探讨 ERC-6551 背后的技术复杂性，并对比 ERC-721 和 ERC-6551 ，最后总结 ERC-6551 目前的用例、注意事项和局限性。

NFT 的诞生与演进

2017 年对于以太坊来说是重要的一年，ERC-721 为独特数字资产的代币化铺平了道路， 并为链上所有权和溯源的未来提供了新方向。NFT 乘着 CryptoKitties、NBA TopShot 和 CryptoPunks 等突破性项目引发的流行浪潮，迅速成为主流关注的焦点。

从开发者、创作者到短线交易者，NFT 适合所有人。不久之后，NFT 行业就变得过度饱和，并受到投机和炒作的破坏。到2022 年末，市场信心开始减弱，并开始步入低谷期。

然而，NFT 底层技术忠实的构建者和信徒们仍然在探索下一个“突破”。大约一年后，ERC-6551 的出现为 NFT 注入了新的活力。

什么是 ERC-6551？

ERC-6551 是代币绑定账户的以太坊标准。它于 2023 年 5 月 7 日部署到以太坊主网，由 Future Primitive 团队（Benny Giang、Jayden Windle等人创建）。它使得为每个 ERC-721 NFT 创建智能合约钱包成为可能，将可组合性、动态性和交互性提升到了新水平。

该提案最早于 2023 年 2 月 23 日发布。

? Benny 是 ERC-721 和 CryptoKitties 的创建团队成员之一。

什么是代币绑定账户 (TBA)？

代币绑定账户 (TBA) 是可被ERC-721 代币拥有的智能合约账户创建接口和注册表的钱包。 它们与“拥有”它们的 NFT 直接相关。

它们赋予 NFT 两个重要的特性。第一个是“拥有资产的能力”，无论是 ERC-20、ERC-721 还是 ERC-1155。第二个是“参与社区治理的能力”（例如成为多重签名的签名者、注册自己的 ENS 域或对社区提案进行投票）。

ERC-6551的底层机制

ERC-721 代币、ERC-721 代币所有者、代币绑定账户（TBA）和注册表之间的关系

如前所述，TBA 是单个 ERC-721 NFT 拥有的智能合约钱包。 控制权在 NFT 持有者手中，他们有权通过 TBA 执行链上操作。

ERC-6551 TBA 的机制可以分为 3 个部分：

登记处

登记处运行着两项对于 TBA 的创建至关重要的职能：

  1. createAcount：使用指定的地址为 ERC-721 部署全新的 TBA。

  2. account：生成现有 ERC-721 代币的 TBA 地址。

最小代理合约

然后，每个 TBA 都会作为ERC-1167 最小代理(MPC)分发，并在字节码上附加不可变的常量数据。使用 MPC 有两个原因：

  1. 更便宜： 无需多次部署 NFT 合约，只需克隆合约即可。

  2. 更少的工作量： 只需部署代理合约一次

账户接口

  1. 定义 TBA 当前可用的一组功能 → 定义它可以采取的一组操作。

  2. 限制 NFT 所有者执行调用的能力。这对于安全目的至关重要，因为它可以防止对 TBA 的未经授权的访问。另一方面，帐户界面提供了一种向非所有者帐户授予执行权限的方法。这对于多方 TBA 很有用。

ERC-6551 机制的这三个部分使其能够充当智能合约钱包。

它保留了 ERC-721 的原语，同时减轻了其局限性。

你可能会问，ERC-6551 的局限性到底是什么？

ERC-721 局限性和 ERC-6551 优点

溯源

ERC-721❌仅提供所有权证明。该资产将链接到所有者的帐户，但不会描绘资产的交易历史、效用等。

ERC-6551 ✅ 你的 NFT 现在会讲一个故事，包括完整的交易历史和有用性。它维护自己的无准入注册表和链上活动日志。获得有关 NFT 过去交易和互动的宝贵洞见。

可组合性

ERC-721❌它们存在于真空中，独立于你的任何其他资产。

ERC-6551✅ 是时候将二者绑定了。使用你的 NFT 和相关资产（例如 NFT 和代币）创建一个链上派对。当你出售或转让 ERC-6551 时，其中的所有资产都会随之转出。

身份

ERC-721❌作为单一资产，NFT 没有自己的身份。只有钱包才算作你的“在线身份”。

ERC-6551✅ NFT 拥有自己的身份，并且不再与持有它们的钱包绑定。现在是 NFT 世界的 dApp。它们不再只是“数字收藏品”，它们是你数字角色的延伸。

动态

ERC-721❌ JSON 元数据是静态的。无法添加任何附加值。无法应用进一步的定制。

ERC-721✅ 允许添加资产，这意味着你可以更改 NFT 的价值或外观，而无需更改其元数据。你好，不断变化的艺术 NFT！

互动性

ERC-721❌ 无法与其他链上资产交互。

ERC-6551✅ 能够拥有 ERC20、721 和 1155 等链上资产。它们增强了其性能或外观。你可以通过自己独特的个性和故事让你的 NFT 焕发活力。

价值来源

ERC-721❌源自纯粹的投机和外部和/或链下活动。

ERC-6551✅ 由于其来源和交互能力，它具有更多价值。

ERC-6551 的另一个优势是它与现有以太坊基础设施的向后兼容性。它们本身与 ERC-721 代币兼容，这意味着它们既不需要部署新的合约，也不需要对现有 NFT 进行包装。它们还与 OpenSea、Metamask、Etherscan 等平台和工具兼容。

ERC-6551 的用例

Gaming

TBA 是名副其实的游戏规则改变者。TBA 的可组合性使得能够为单个角色创建游戏内“库存”。玩家将能够将所有游戏内资产转移到角色的“钱包”中，从而获得无缝的玩家体验，并让他们全面了解自己的成就。

社区建设

在线社区（例如 NFT 项目、DAO）成员将有更多的动力参与。他们最初购买的 NFT，无论是 PFP 还是会员卡，都会因收集相关代币、NFT 或其他资产而积累价值。

会员资格、忠诚度计划、声誉系统、链上身份

如果链上身份由给定钱包中的资产和交易组成，那么 TBA 使 NFT 能够拥有自己的身份。他们还可以独立与 dApp 交互。

这为忠诚度计划和声誉系统开辟了无数的可能性。NFT 可以让所有者进入忠诚度计划并享受福利，或者在贷款的情况下，NFT 可以成为可验证信用评级的工具。

ERC-6551 注意事项和局限性

ERC-6551上线时间不久。尽管其技术具有突破性，但前进之路仍然崎岖不平。

❌并非所有 NFT 都受支持

仅 ERC-721 的封装版本兼容。在开始之前，检查兼容性至关重要。

❌缺乏生态系统支持

并非所有人都张开双臂欢迎变革。一些现有的项目和平台还没有被 ERC-6551 所吸引。尤其是那些喜欢ownerOf方法的人*。

❌安全风险

权力越大，责任越大。NFT 价值的增加意味着它们对黑客的攻击将变得更具吸引力。为了确保稳定性并允许使用更高价值的资产，ERC-6551将需要严格的测试。

❌用户体验

ERC-6551 在该领域诞生不久，因此几乎没有得到用户的反馈。虽然概念上并不难理解，但该功能在技术上是全新的。为了让用户理解和浏览代币绑定账户的技术方面，需要更多地关注流程和直观的界面。

结束语：展望未来

NFT 的第一个迭代结束了，但其忠实支持者们一直在等待 NFT 的卷土重来。虽然在应用落地方面尚无明显进展，但 ERC-6551 标准是否正在引领数字资产自然而必要的进步？

ERC-6551 建立在 ERC-721 的基础上，并在功能、所有权和互操作性方面实现了有意义的增强。尽管仍处于试验阶段，但显而易见的是，ERC-6551 和代币绑定账户将在 NFT 的发展中发挥关键作用。数字资产格局正在接近一个新的地平线。借助 ERC-6551 催生一系列新功能，将打开更多创造力、创新和连接之门。
