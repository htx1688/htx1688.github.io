本文来自香港 Web3 嘉年华速记，演讲内容为作者个人分享，不代表吴说观点。

Victor Zhou, 以太坊标准 EIP 编辑评审。

Victor Zhou：大家好！我是 Victor，感谢主持人，感谢大家来收听我的演讲，抱歉我没办法亲自到场，我从加州跟大家进行远程介绍。

今天我要谈一谈开放思维，先介绍一下自己，我是 EIP 编辑，我主要活跃在 ERC 智能合约还有以太坊应用层方面，我是之前 Google 的校友、约翰霍普金斯大学及微软的校友。

很多人问我以太坊的治理模型，我的回答是什么呢？

可以有人会说，看起来以太坊其实治理是非常混乱的，这种治理是开放的，而且会给我们带来很多好处。现在想要问一下观众，你们有多少人有写过智能合约？有多少人写过后台服务？我知道在座有很多开发者。

通常我会问一下我的开发者朋友，你怎么样把这两个设计的问题相差异区分。一个是有 API Write，一个是 API Save，选择的话你选择哪一个呢？答案是我们根本不在乎，因为除了要一致性外，不介意其他的东西。

现在在 Web3 的机制中，但先退后一下，从 Web1 开始服务总是有读方和写方，会让公司允许请求 API，不同写方呼叫不同的服务，就好像你去呼叫 Google API 平台、微信、SDK 等等。在 Web2.0 可以加入更多新的读方和写方。Web3 有新的机制，允许服务被替代。

在 Web3 的机制中要有开放的思维。

共识所带来了价值，尽管解决这个问题，把它叫做呼叫一个 API 的话，可能有内建的 WiFi、蓝牙、CDMA，有 HTTP、SSL、TCP、USB、TypeC 等等，这些都是标准化的连接，让我们连到汽车上、耳机上。

为什么香港是非常伟大的海港呢？有很多集装箱船进出，这也是你们在港口附近很多人可以看到的东西。一开始，集装箱是用于火车运输的，后来变成了非常重要的标准，可以让船来运输，所以船的建设是设计用来装集装箱的。后来在未来发现了有人开始用集装箱建房子了，因为集装箱是标准，非常容易进行整合。

在以太坊，其中的核心部分就是智能合约，智能合约围绕着标准来设立。这些共识是怎样进行整合的？

用一个代码来展示的话，ERC 20 就是转移。有了简化开放的共识就可以基于 ERC 20 建很多非常棒的项目，市场市值增加，非常棒，大家可以看到公司的成长。这是开放整合的威力。

下面介绍开放的四个层次，这也是为什么以太坊在这里上这么成功。但是在这之前补充一下，非常有名的 Richard Stallman 讲的开源软件原则，他说：一个软件真正免费的话必须要四种不同的面自由，自由使用、自由学习、自由分享、自由修改。

开放有哪四个层次呢？加密货币项目都在开放的不同阶段上。一开始最常见的最可见是开放，然后是控制、贡献，不仅仅内部可以做出贡献，外部也可以做出贡献，Github 贡献者、开源贡献者也可以贡献于你的项目。人们也可以进来加入治理，只要他是利益相关方及只要他愿意贡献。最后比较有哲理性的，欢迎竞争。

这四种开放让以太坊非常成功。

我给大家讲一下 ERC 20 和 ERC 21 的基础，看起来好像非常详细，但其实它是非常常见的，在外部和内部有很多不同，会发现用户和客户是在外面， Builder 创建者程序员 / 内部的员工，边界慢慢消失，如果允许用户进来提供帮助的话，其实可以提供很多有用的帮助，这就是 ABCDE 层面的贡献。好像一个小 baby 可以爬梯子一样，安全第一，先从最安全最低的地方开始，可能你点个赞分享活动，可以告诉人们项目是多么好，你多么喜欢。

第二层，要从精神上参与受到吸引，甚至捐钱。

第三层，开始改变致力于解决一些东西，要有承诺，不断提升。

第四层，建设，建立软件，参与会议，最早最上面的一层是主动开始一些东西，所以贡献者在每个层面都有参与。

开放治理和传统治理是不太一样的，传统各个公司的结构是自上而下的，会看到 CEO、CCO 都是董事会选举，开始指派工作，组合团队。另外一方也是去中心化的世界，开始借助越来越多的开放治理。在加密货币之前，非常有名的是有 APACHE 社区，让用户做贡献者，让代码贡献者成为项目管理委员会，成为和 APACHE 的核心成员，来选举委员会，所以你可以作为贡献者，在参与治理被越来越多的信任，开始在决策的过程中有更多决策力量，就好像 WIKIPEDIA 一样，也是去中心化的社群，在以太坊之间就存在的，有很大治理在里面，有很多开源贡献者和 WIKIPEDIA 编辑者参与进来进行治理。

以太坊外面比较混乱，有很多利益相关方，最小的是以太坊基金会，很多时候是学生式的基金会，而且保持中立，作为协调者。还看到有些人是开发者、程序员，他们给客户开发东西，会开发 ERC，开发应用层协议，还有 EVM 运营商。你可能也是这里面的一部分，你可能是持有比特币的群体，最重要的是应用用户在应用使用以太币的时候在链上的应用。

每个人在某种程度上都进行了投票改变治理，这也是为什么以太坊能保持开放治理，在 PoS 出现和去年合并之前，发行 USDT USDC 的公司，宣布他们会支持 PoS，Vitalik 说他放心了，因为他有点担心 DeFi 世界是不会跟随 PoS 的。但是现在作为项目的创办人，你也会担心 DeFi 项目，不会跟你想要走的方向一致，所以每个人都可能用脚投票。竞争的开放是最难的，要允许竞争者进入。

谈到竞争，核心哲理是我们不一定是最好的，我们可能是错误的，也可能会有些妥协，所以必须要做好准备进行提升，进行纠正，甚至被其他东西所替代。比如说比特币的安全机制可以有 51% 攻击，但如果 51% 的权利想要改变比特链的发展方向，其实可以进行分叉。在过去也发生过分叉，以太坊也是接受的。

Wikipedia 也可以接受分叉内容，所以版权跟进，有相应的版权政策。如果你想朝着不同的方向，我们可以支持。

从这个角度来说，我们是可以被替代的，欢迎竞争，也欢迎来自外部的竞争。你可能会意识到，不仅仅是一个，可能有 10 个不同的以太坊 EVM 执行节点，可能有 5、 6 个 cell 机进行了开发。

以太坊基金会领导讲人们应该多元化客户，执行性客户太关注于以太坊的黄金版本，现在有一种“呼吁”，要把客户端多元化。还有链的多元性，尽管以太坊本身就是链，是主链，欢迎也支持其他链在生态系统中。在以太坊的生态系统中，没有人可能会比以太坊更好，或者替代以太坊。如果你想推出自己的 EVM 项目，你只需要给出一个 Chain ID 就可以了。

如果你有这方面兴趣的话，可以让开发商聚在一起分享实施，目标是尽可能容易地使用，依赖于它加大智能合约的内部创新。好像我给你解释以太坊整体的威力，开放的贡献，可以对某一个项目进行贡献，我相信以太坊最大的利益相关方就是应用的用户。ERC 帮助了开放的整合，让人们使用应用，帮助项目发展。这是我想跟大家分享的，让大家保持开放。

以太坊为什么这么成功地进行治理？希望其他项目可以把它作为参考进行学习。
