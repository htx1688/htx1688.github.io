作为以太坊上最经典的 NFT 人物形象之一，加密朋克形象代表了一种精神，拥有它是一种身份的象征，并成为永远具有吸引力的叙事。似乎每一条新链的崛起，加密朋克形象都成为第一个被移植的对象，由此我们看到了各种链上朋克，和众多的朋克衍生品和合成人物形象。

随着比特币生态 NFT 叙事的兴起，BTC 网络中也开始出现各种各样的加密朋克，并受到加密藏家们的热捧。作为首个使用 Ordinals 免费铸造的 10 K 集合 Bitcoin Punks，不到一天的时间内便被一抢而空，其地板价一度突破 4 BTC。

在 Ordinals 之后，BTC 生态的 NFT 基建继续生长，其中值得一提的就是递归铭文「Recursive Inscriptions」。

作为 BTC 生态中的新生事物，Recursive Inscriptions 上线后就迎来了首个递归朋克（Recursive Punks）的启动，在社区热捧下， 1 万枚递归朋克被迅速刻录完毕，后又迅速上线 MagicEden 市场，并在该平台上日内总交易额达到 8.15 个比特币，成为平台上交易总额排名第一的系列，其最高价格曾一度涨至 0.002 比特币，相对刻录成本涨幅最高达 30 倍。此外，有多个具有稀有属性的 punk 以 0.05 比特币的价格成交，其持有人数也在一夜之间增长 2000 多位。

戏剧性的是，用户发现自己刻录的递归朋克显示为空白，检查代码后发现，这是因为递归朋克官方开发者使用了错误的递归铭文语法，将"/content/“错误地写成了”/-/content/:"，从而导致早前使用官方 SVG 文件刻录的用户打到的铭文为无图版。于是社区发起了一个正确的修复版本，即将原来 SVG 文件中的"/-/content/:“替换成”/content/"，由此引发了真假递归朋克之争。

真假朋克之争

6 月 13 日，比特币上的 Recursive Punks 正式启动，并受到社区热捧，总量 10 K 枚 Recursive Punks 在短时间内迅速刻录完毕。然而，当刻录完毕，有用户称自己拥有的递归朋克是空白的，查证后发现这是由于官方发布的 Recursive Punks 采用了已被弃用的旧版本语法"/-/content/:"，根据开发者 Casey Rodarmor 发布的代码更新，正确的写法应该是"/-/content/:"，从而导致基于官方 SVG 文件刻录用户生成的是无图版本的铭文。

Recursive Inscriptions 开发者发布的语法更新

随后，社区组织发起了 v2 版的递归朋克，该版本修正了此前官方版本的语法错误，即将原来 SVG 文件中的"/-/content/:“替换成正确的”/content/"，修正版本刻录相对缓慢，可正确清晰显示递归朋克的图片。该版本在 24 小时后亦被刻录完毕。社区认为，官方发布版本的数据源托管在 github 上，并不符合去中心化精神，这会误导社区，所以创建了真正的递归朋克，也就是 v2 版本的 Recursive Punks。

于是，Recursive Punks 有了两个版本，官方发布的正式版本 v1 和社区发起的 v2 版本，且 v2 版同样得到了 Recursive Punks 官方的认可，Recursive Punks 在一则由官方发布的推文中表示希望协议开发者和社区能够同时认可无图版本和有图版。

不过，不久之后 Recursive Punks 将该推文删除，随即在 MagicEden 上线了无图版本合集。

目前已被删除官方发布的推文

令人惊讶的是，原本无法显示图片的 Recursive Punks 在 MagicEden 上却显示高清大图。经过代码研究发现，原来是 Recursive Punks 官方使用前端渲染的方式实现，而非使用递归铭文的实现方式。

然而，作为新协议 Recursive Inscriptions 上的首发项目，官方发布的版本仍然拥有更强更广的共识，以下数据说明一切。

MagicEden 数据显示，基于错误语法刻录的 Recursive Punks 无图系列在上线日内总交易额达到 8.15 个比特币，成为平台上交易总额排名第一的系列，其最高价格涨到 0.002 比特币，相对刻录成本涨幅最高达 30 倍。此外，有多个具有稀有属性的 punk 以 0.05 比特币的价格成交。其持有人数也在一夜之间增长 2000 多位。

与之对比，社区发起的有图版在上线 NFT 平台 Ordyssey 后，虽然持有地址数在不断增长，但交易总额为 1.3 比特币，日内交易额排名第三。

虽然被社区某些成员斥责为假递归朋克，存储在 github 上，甚至严格意义上这个合集都不属于是递归铭文，但它是由官方更早地发布在协议上，更早地上架 MagicEden 市场，而在比特币生态中 NFT 基础设施尚不成熟和完备的早期阶段，是否能在二级市场中上架以及是否能被索引，都决定了该系列的发展速度甚至成为其存活的关键因素。

这让我们进一步发问，判断真朋克的标准究竟是什么，到底是 first is first 的 v 1 上还是使用了正确递归铭文代码的 v2 上？

纯链上递归朋克竟存储在 Github 上？

有意思的是，这个号称「递归铭文」上首个项目的递归朋克和「递归铭文」一点关系也没有。

那么，「递归铭文」是什么？

事实上，「递归铭文」基于比特币生态的 Ordinals protocol 创建，Ordinals protocol 是 2023 年 1 月由软件工程师 Casey Rodarmor 发起，它支持以一种独特方式在比特币上以最小单位 satoshis（或 sats）上镌刻数据，生成类似于 NFT 的 Ordinals 铭文。

Ordinals 的崛起让比特币网络有了新的可能性，与以太坊相比，在比特币网络中的图片完全存储于链上，具有永久性，「永久存储在链上」的叙事不仅充满浪漫色彩，也让比特币生态从此充满了无限的想象力。

而同样由 Ordinals 创始人发起的 Recursive Inscriptions（递归铭文）基于 Ordinals 之上进一步开发。

受制于客观条件的限制，Ordinals 上的铭文有一个上限——最多只能容纳 4 MB 的容量，而递归铭文可实现更高的存储效率，同时实现互操作性，让铭文大小突破此前比特币区块 4 MB 的容量限制，这意味着可实现直接在比特币区块链上托管大量文件，例如视频游戏、电影或复杂软件。所有这些都可直接在比特币链上实现，无需依赖链下解析，这个创新为 Ordinals 生态带来了无限的创造力，激发了人们的想象力。

然而，作为 Recursive Inscriptions 上发起的首个递归项目「递归朋克」，却因疏忽采用了错误的语法格式导致用户刻录的 Recursive Punks 显示为空白，这意味着这个版本的朋克并非真正意义上的递归铭文项目，自然也就和递归铭文所代表的比特币永久链上存储毫无关系。

后续上线 Magic Eden 后，更是通过修改前端代码以正常显示图片，其代码不仅没有存储在比特币网络上，甚至不在 ipfs，而是存储在中心化托管仓库 github 上。这对于一个以「比特币永久链上存储」叙事为主的项目来说，讽刺味十足。

即便如此，作为 Recursive Inscriptions 上首个递归项目，「Recursive Punks v1 版本」启动时却相当成功，不仅短时间内铸造完毕，其火爆程度甚至一度导致网站宕机，后续上 Magic Eden 后日内总交易额达到 8.15 个比特币，成为平台上交易总额排名第一的系列，其最高价格曾一度涨至 0.002 比特币，相对刻录成本涨幅最高达 30 倍。此外，有多个具有稀有属性的 punk 以 0.05 比特币的价格成交，其持有人数也在一夜之间增长 2000 多位。截止今日交易量为 13 个比特币。同样上线 MagicEden 的 V2 版本交易量则少得多。

新生事物孕育着潜在机会，同样也意味着巨大风险，尤其是还在基础设施并不完善的早期阶段。对于比特币生态来说，虽然 NFT 叙事为其生态注入了新的生机，但其不成熟的基础设施和尚未达成共识的争议有可能让在其之上铸造的加密资产瞬间清零。

尤其是在这种新生协议之上发起的新项目，且不论其项目本身的合法性尚且存在争议（其价值取决于社区判断真假朋克的标准以及未来社区共识的推动和发展），更别说其赖以存在的底层协议「递归铭文」协议层，以及同样存在争议更底层的 Ordinals 均受到了比特币核心开发者的抵制，认为这些协议支持的新用例会占用比特币网络的区块空间，从而将更多人排除在比特币网络之外，不利于推动加密采用，这偏离了中本聪将比特币作为点对点现金系统的愿景。

对底层基础协议的依赖性有可能导致开发者对原协议进行修改，从而影响软件的功能和错误的文件存储哈希。加密货币是一种非常高风险的投资标的，尤其是在所有的底层基础设施尚不完善且未达成社区共识的基础上，投资这样的资产有巨大风险。区块律动提醒大家，投资前请独立思考，DYOR（do your own research）。
