赏金计划为 DAO 释放了去中心化工作的力量。 赏金是 DAO 与其贡献者和社区共享以完成的任务和项目。这些任务的范围可能从文档翻译和会议记录，到在 Optimism 和 Polygon 等第 2 层链之上构建产品。

DAO 正在发现赏金计划如何授权当前和潜在的贡献者参与、组织和完成增值任务。虽然没有制定有效的赏金计划的固定模板，但许多 DAO 正在尝试不同类型的赏金、结构和平台。

赏金类型

在大多数赏金计划中，有三种广泛的赏金类型——DAO 创建的、贡献者驱动的和赞助的。赏金计划可以使用这三种类型的组合来实现其特定目标。

DAO 创建

一些 DAO 中的公会或团队将为某些任务和项目创建赏金。在这种动手实践的方法中，团队花时间确定项目范围，制定愿景和目标，并筛选贡献者以确定适合性。这可能还需要 DAO 任命成员来监督和管理整个赏金过程。

在 Raid Guild，客户经理管理赏金。他们是经验丰富的管理者，他们阐明项目的愿景和目标，评估贡献者与赏金的契合度，并创造一个积极的入职体验。Aragon 通过一对一的筛选过程对一些赏金使用类似的框架来评估贡献者与赏金的匹配度。

贡献者驱动

去中心化为贡献者创造了机会，让他们主动提出可能为 DAO 增加价值的项目。一些 DAO，如 Bankless 和 Forefront，围绕着特定的兴趣领域，为撰写提案的贡献者提供开放式赏金。成员审查并投票资助符合这些开放式赏金标准的具体提案。

例如，Bankless 和 Forefront 通过贡献者驱动的赏金生成优质内容。他们采用开放式的内容征集方式，供贡献者撰写和提交文章。这些文章由每个 DAO 的成员审查和补偿。

赞助

黑客马拉松围绕 Web3 公司和协议赞助的巨额赏金聚集了有才能的个人和团队。在许多情况下，公司会集中资源来激励团队在设定的时间内组织起来去争夺赏金。这些赏金是吸引开发人员在新兴协议和服务之上进行开发的一种方式。

Gitcoin 提供两种类型的黑客马拉松——开放的和独家的。在公开黑客马拉松中，Gitcoin 活动团队会联系赞助商，为赏金和活动的开展提供资金。对于独家黑客马拉松，Web3 公司会与 Gitcoin 活动团队接触，以制作黑客马拉松。在这两种情况下，Gitcoin 都会为黑客马拉松制定说明、赏金和奖品，并将其推广到由 30,000 多名开发人员组成的网络中。

许可与无许可

赏金的范围、技能和报酬各不相同。许多 DAO 根据要求为特定赏金开发了不同级别的访问权限。一些赏金将需要 DAO 的明确批准才能完成（许可），而另一些赏金则不需要批准（无许可）。

无许可

无许可赏金为 DAO 的潜在贡献者创造了较低的进入门槛。这些允许人们通过完成为 DAO 提供价值以换取补偿的小额赏金来表达兴趣。无许可赏金无需事先获得 DAO 的批准，并且可能包括日常任务，例如记录会议记录、翻译页面和完成小型的 Pull Requests。

例如，Juicebox 和 Aragon 使用无需许可的赏金来完成日常任务，如记会议记录和翻译页面。DAOhaus 还有一个无需许可的 GitHub，其中充满了新贡献者可以解决的问题。

许可

许可使 DAO 能够为具有更高复杂性和更大补偿的赏金增加一层安全性。在许多情况下，这些赏金是由具有既定记录的当前 DAO 成员完成的。代币门控（Token-gating）是一种常用技术，通过要求特定的代币和 Discord 角色来为有权限的赏金把关。

Gitcoin 的赏金平台通过要求人们在查看公司发布的赏金之前连接他们的 GitHub 个人资料来利用许可。这允许公司通过审查他们的 GitHub 来验证开发人员的技能和意图，从而在接受他们的赏金工作之前建立起一层信任。

优秀赏金帖子的组成部分

一个优秀的悬赏帖为成功建立了潜在的贡献者。它们简明扼要地概述了期望、可交付成果、截止日期和补偿。许多帖子都会进行优化，使其具有可读性，并分享完成任务所需的所有相关信息和资源。1Hive 的 bug 赏金计划是一个很好的例子。

概述

帖子的开头应该对赏金进行高度概括。1Hive Bug Bounty 的帖子简明扼要地介绍了赏金计划，包括范围、评估过程、补偿结构以及存放资金的钱包。

任务/要求

下一部分概述了完成赏金所需的特定任务或要求。这应该使潜在的贡献者更容易完成和提交赏金。1Hive 用要点来构建其要求区域，以便于浏览，并概述了其期望。

补偿

每一笔赏金都应该得到补偿（Compensation）。许多 DAO 使用 Web2 市场价格来决定赏金的补偿。1Hive 提供了基于 CVSS 风险评级量表的详细补偿结构，以及以前的 bug 赏金的评分实例。这种透明度有助于潜在的贡献者确定他们是否愿意完成赏金。

补偿示例

提供以前补偿提交的示例可以为人们提供格式化和索取赏金的指南。这也可以减少想要完成首次赏金的人的摩擦。1Hive 为潜在贡献者提供了两个示例。

赏金计划技术栈

用于赏金计划的 DAO 工具仍处于初期和发展阶段。虽然没有特定的技术堆栈，但 DAO 使用了一些通用平台。其中包括 Discord、Notion、Dework 和 GitHub。除了这些平台之外，Gitcoin 还为开发人员创建了自己的赏金平台，Bankless 开发了自己的内部平台，以便在未来与更广泛的 Web3 社区共享。

Discord

Discord 实际上是 DAO 的交流平台。Discord 可以作为赏金通信的纽带。这可能包括分享新的赏金、团队组织、提交内容以及对许可赏金的代币门控。

在 DAOhaus Discord 中，选择“贡献者”角色的人会解锁一个专用于潜在和当前贡献者的特定频道。该频道允许人们与成员交流，询问有关特定无需许可的赏金的问题，并分享已完成的悬赏。

Notion

Notion 是一个项目管理和笔记软件，DAO 可以使用它来管理赏金计划。Notion 允许 DAO 发布、跟踪和共享开放的赏金。Clarity 和 Charmverse 是两个 Web3 解决方案，它们使用类似于 Notion 的界面，具有钱包连接、代币门控和赏金支付等功能。

Forefront 是一个使用 Notion 进行开放社区赏金的 DAO 示例。该页面共享无许可的赏金，任何人都可以处理并通过表格提交，以获得FF代币的补偿。

Dework

Dework 是一个带有代币支付、认证和赏金的 Web3 原生项目管理平台。该平台专门解决赏金计划管理问题，并可以轻松管理从开始到结束的整个工作流程。

许多 DAO 在启动赏金计划时开始尝试使用 Dework。BanklessDAO Marketing Guild 和 Aragon Network DAO 是 DAO 测试 Dework 技术和非技术赏金的例子。

GitHub

GitHub 是一个用于版本控制和协作的代码托管平台。GitHub 是面向开发人员的 DAO 管理赏金计划、审查和接受修复以及共享必要文档的一种方式。

Gitcoin

Gitcoin 是一个平台，开发人员通过在 Python、Rust、Ruby、JavaScript、Solidity、HTML、CSS、Design 等开源软件上工作获得报酬。Gitcoin 是为组织机构在其平台上发布开发者奖励而设计的。Gitcoin 要求开发人员连接他们的 GitHub 个人资料，以允许组织在使用或接受赏金提交之前审查开发人员。

建立一个赏金计划

赏金计划是一种强大的工具，可以让贡献者运用他们的技能为 DAO 创造价值。构建一个有效的赏金计划需要仔细考虑为 DAO 带来最大价值的赏金类型、流程和平台。虽然没有一个适合所有的赏金计划，但许多 DAO 可以从规划和实施赏金计划中受益。
