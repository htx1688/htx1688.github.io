原创 | Odaily星球日报

作者 | 夫如何

随着区块链底层基础设施和应用的持续发展，Web3 正成为下一代互联网的应许之地。然而，Web3 在实际应用中仍然面临诸多挑战，如用户体验、可扩展性和政策监管环境等。

上述核心问题中，后两者在稳中求进——可扩展性依靠技术的推进，目前以太坊为中心的多 L2 宇宙正逐渐形成，新一代 L1 争抢龙头应用的部署，因而网络延迟、高 gas 费等问题，不像过去那般限制用户的进入和留存；各主要国家的加密法例也在向更清晰化的方向发展和磨合。由此，用户体验仍是当下亟需解决的关键问题。

Web3 应用对于 Web2 用户来说仍然存在一定的学习曲线和技术门槛。缺乏直观、友好的用户界面和简化的操作流程限制了更大规模的采用。换句话说，Web3 缺乏真正的“杀手级”应用平台，能够让“小白”无需理解 Web3 的底层技术逻辑，也可以享受到与 Web2 同等的体验。杀手级应用需要诞生在 Web3 入口处，让更多怀有不同需求的用户无痕进入、体验 Web3。

目前最贴近这一存在的，便是 Web3 钱包。钱包集成了多链的多应用，让链上活动变得高效；AA 钱包也是今年的创业热门赛道。不过，尽管自身产品在用户体验上有所提升，当需要交互其它 DeFi 应用时，依然要面对复杂的流程和门槛。

那有没有可能将钱包从单纯的账户属性延伸出多链应用的统一入口平台呢？ Odaily星球日报近期采访的 ParaX 或许会给出答案。

ParaX 是什么？

ParaX 由 ParaSpace 和 Parallel 合并而来。创始人 Yubo 以连续创业者、个人风投身份深耕行业多年，此前创立的 Parallel Finance 和 ParaSpace 分别是 DeFi 和 NFT 领域知名项目，曾获 Polychain Capital、Coinbase、Sequoia 等顶级风投青睐。如今，Yubo 将导入此前积累的经验、用户、资金，为 ParaX 完成冷启动。

ParaX 采用了基于 ERC-4337 的账户抽象（AA）模型和跨域意图的大语言模型，同时配合功能完善、可定制的 UI 界面，使用户更方便地参与 Web3 生态系统，旨在为 Web3 的大规模采用铺平道路。

多链账户抽象（MAA）

多链账户抽象（MAA）是通过弱化账户中每条链的账户属性，将其汇总成统一的账户信息体现，类似于中心化交易所的账户。

该模型使用户在使用 Web3 应用时可以更加便捷地管理和控制他们的多链资产和身份。用户可以拥有一个统一的身份，跨多个应用和平台进行身份验证和授权，从而增加了用户对自己数据的掌控权。

ParaX 的多链账户抽象通过在不同目标链上建立智能合约来搜集用户的链上交易和资产数据，统一呈现在 ParaX 的账户界面中。

基于跨域意图的大语言模型

基于跨域意图的大语言模型（Cross Domain Intent-based LLM）是以大量的 DeFi 相关数据模型对大语言模型（LLM）进行训练，从而实现用户针对不同场景的交易意图。

其中数据模型包含合约代码、主流协议文档等，旨在通过自然语言理解用户意图，翻译成想要达成的目标及限制，最后再生成相应的系列交易，并在多链的合约账户上执行，实现横跨不同的应用场景的交易模式。

比如 Uniswap 负责交易需求，Maker 包含借贷需求，如果用户想结合这些需求，则可以使用这一方案。用户就算相关知识、操作路径、风险意识不太清晰，AI 也可以协助用户自动化执行以上的操作和需求。

此 AI 模型将意图转化为可行动的请求，封装和隐藏区块链技术细节，从而降低用户进入 Web3 的门槛。

元用户界面(MUI)

ParaX 的 UI 设计采用简约风格，但又不失功能性。ParaX 作为平台将与 100 多款 DApp 集成，例如 Uniswap、Opensea、Lens、Aave 等知名应用，打造一站式 Web3 入口。下图为 ParaX 测试版界面：

同时 ParaX 提供两种 UI 界面——简易版及专业版。

● 简易版：只提供用户所需功能，省略中间过程，降低使用门槛。

● 专业版：提供相应 DApp 的具体名称，有基础的用户可以根据平时喜好和需求自行选择 DApp。

底层布局：ParaVM

Yubo 向Odaily星球日报表示：“ParaVM 作为 ParaX 的底层基础设施，将采用 zkVM 的技术标准，团队根据实际情况来选择自行开发或与现有项目合作，意向合作方有 RiskZero、Axiom、HyperOracle。”

作为底层基础设施，ParaVM 通过“链下证明、链上服务”于 ParaX 的各应用中运行，如自动套利应用，其类似跨链版 YFI。用户的资金可以在事先的白名单协议中进行自动调仓，并自动选择高利率的协议获利。

ParaVM 作为 ParaX 的特定底层，其开发完全服务于 ParaX 应用的要求，这将极大地提高证明及验证速度。

ParaVM 的诞生意味着 ParaX 将打通“基础设施-应用协议-应用前端”的纵向结构，依托作为前端的 ParaX 的Web3流量入口，吸引开发人员基于 ParaVM 二层开发或迁移应用，丰富 ParaX 生态，最终形成完整闭环。

目前 ParaVM 还在开发的过程中，具体落地进展，需保持关注。

三大“聚合”机制   

ParaX 通过无痕聚合不同 DeFi 和 NFTFi 协议并收集相关数据，为用户提供更优的路由路径。

1.  自动市场套利机制：呈现各借贷协议的收益水平，再择选最合理的路由路径，展示最高 APY。

2.  全链信用贷款机制：所有链上的资产都可以抵押作为担保。

3.  多元借贷利率的市场模型机制：汇总多个知名项目的借贷利率及风险指标，形成风险利率曲线，同时根据不同资产类别划定抵押品评级标准，贷款人可针对不同资产及其评级标准选择合适区间贷款。

模糊 Web2 与 Web3 的界限

Web2 用户进入 Web3 时，首当其冲要面临钱包助记词的知识点。助记词“不助记”使得一些新用户倒在 Web3 门口。ParaX 可使新用户通过传统的入口进入，生成密钥绑定传统账户，使用户无需理解助记词，又不折损安全性。

为什么是 ParaX？

ParaX 想要成为 Web3 流量入口，不仅要具备本身的创新性，同时团队的经验及能力也是项目能否长久运营的关键。ParaX 是由 Parallel Finance 及其子公司 ParaSpace 合并而来的应用平台，ParaX 团队具备丰富的行业经验和资源。

Parallel Finance

Parallel Finance 最初是基于 Polkadot 网络的 DeFi 平行链，旨在降低进入 DeFi 门槛。该协议提供了一系列去中心化应用，包括流动性质押、自动做市商、去中心化货币市场、流动性众贷、流协议、钱包、流动性挖矿和跨链功能。

ParaSpace

ParaSpace 是一个点对池 NFT 借贷协议，允许用户抵押和借出 NFT 和同质化代币。

在 ParaSpace， 用户可利用未被充分使用的资金来进一步投资，并从中赚取收益。简单理解就是用户可以将 NFT 和 Token 打包，用打包后的组合抵押借贷，以提高资产利用率。

目前，ParaSpace 支持目前 BAYC、MAYC、BAKC、CryptoPunks、Otherdeed、Azuki、CloneX、Doodles 等共计 12 种蓝筹 NFT 资产，并支持 APE、ETH、DAI、WETH、USDC、USDT 等共计 8 种 ERC-20 资产。其中，Yuga 系列资产（BAYC、MAYC、BAKC）的市场占比率处在领先地位。

ParaX 团队背景

ParaX 的团队背景较为亮眼，创始人 Yubo 是一位科技企业家、风险投资家。离开斯坦福大学后，他在 2021 年创立了 Parallel Finance，将“DeFi 普及化”作为目标。该协议已成为 Polkadot 生态系统中重要的应用之一。

团队成员大多来自 Google、Meta、Amazon 等全球知名公司，团队顾问是 Dan Boneh 和 Naval Ravikant，前者是斯坦福大学教授和a16z和以太坊基金会的顾问，后者是著名的天使投资人。

此后，Parallel Finance 孵化出 ParaSpace，在新形成的 NFT 金融生态系统中再次创新。

Parallel Finance 和 ParaSpace 的成功推出先后吸引了包括 Polychain Capital、Coinbase Ventures、Starkware、Sequoia 等顶级资本的青睐，总融资超过 3000 万美元。

同时，在建设这两个项目的过程中，团队也与 Chainlink、LayerZero、Delegate 等业内知名项目建立良好的合作关系。

通过对 ParaX 前身的介绍，不难发现，Web3应用层两大核心点——DeFi 和 NFTFi，ParaX 之前都有涉猎，且取得不错的成绩。

Yubo 向Odaily星球日报表示：“ParaX 并不是新的项目，而是将原有的项目资源整合与品牌升级，以平台的模式将 Parallel Finance 和 ParaSpace 作为平台上的应用。而我们从业的初心并未改变——将去中心化应用带入 10 亿人手中。”

Parallel Finance 和 ParaSpace 的现有规模，为 ParaX 建设Web3超级流量入口提供坚实的用户和资金基础。从这一角度来看，ParaX 具备了快速启动的前置条件。

ParaSpace 和 ParaX 的空投计划

此次资源整合及品牌升级后，ParaX 将整合 Parallel Finance 和 ParaSpace 的用户持有代币情况和链上活动与新平台 ParaX 的用户链上活动共同作为空头计划的标准。用户可以通过积分和推荐仪表板查看在 Parallel Finance、ParaSpace 和 ParaX 上的活动和积分。这些积分将用于兑换治理代币。

在 Parallel Finance 上，用户的代币持有和链上行为作为空投的标准。

在 ParaSpace 上，用户可以通过借贷、质押 APE 代币、参与市场活动等方式获得积分。

在 ParaX 上，用户将根据使用钱包的合格行为获得奖励积分，包括创建钱包、交易、与其他应用程序互动以及带来的总锁定价值和交易量。

由此可见，ParaX 沿用了过去的经济体系，并将 token 打通，使老用户获得更多的权益。

具体空投细则，请关注项目方官网及推特。

总结

如果对比 Web2 的进化之路，ParaX 的项目发展路径很像微信，微信以 IM 即时通讯作为前期流量点，后期开放开发标准，依托小程序形成应用平台。

但 Web2 中有数据分享和 API 限制，难有统一入口，只能自建；而 Web3 有着天然的开放性优势，即不受以公司为主体的流量孤岛限制，可以通过集成多链 DApp 形成应用平台。

ParaX 作为行业的深耕者，其优秀的项目经历、团队背景和对路线方向的准确预判，或将为Web3大规模采用创造基础条件。
