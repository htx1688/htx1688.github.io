去中心化是区块链技术带来的关键创新，也是 web3 协议最重要的特征之一。因此，对于更准确地评估和比较各种 web3 协议的去中心化程度， web3 参与者、政策制定者和监管机构必须共同形成更统一而细致的对于去中心化的理解。这样做不仅能够更好地使 web3 的监管和政策适应去中心化的特性，理解和考虑去中心化如何降低风险，还能够最终激励 web3 的构建者追求去中心化，以实现最大化 web3 承诺的公共利益。

为此，我们定义了三种去中心化类型，并针对每种类型提出了相关要素，适用于代 币化区块链协议 （例如Bitcoin、Ethereum、Polygon、Solana、Optimism、Arbitrum、zksync 等 layer1 和 layer2 区块链）以及部署在区块链上的代币化智能合约协议 （例如 Uniswap、Aave、Compound、Curve 等）。我们还提供了两个表格，一个用于代币化区块链协议（图1），另一个用于代币化智能合约协议（图2），这些表格将有助于列举去中心化的组成部分，以提供更具体和标准化的定义。

对于区块链协议或智能合约协议的去中心化分析，必须考虑与该协议相关的全部情况。我们在此提出的要素是为了为这种分析制定一个方向。

为什么需要去中心化？

Web3 引领了互联网的全新时代——“阅读、写作、拥有”的时代。支撑 web3 的技术实现了“无需信任的计算”，不再需要依赖中心化实体来浏览互联网和数据库。这使得我们能够开发更为复杂和高级的协议，既能提供现代互联网的功能，又能够由用户真正拥有。 例如，一个去中心化的社交媒体协议可以支持多个应用程序的构建，并通过代币拥有权将对该协议的拥有权和控制权分配给广大的开发者和用户。

去中心化是 web3 协议的关键特征， 它推动了这种范式转变。去中心化将推动创造一个民主化的互联网，并实现三个重要的变革：促进竞争，保护自由，回报利益相关者。

首先， 去中心化使得 web3 系统具备可靠的中立性（1）（它们不能歧视任何个人利益相关者或利益相关者群体），这对于激励开发者在生态系统中构建应用至关重要。同时，它们也具备可组合性（2）（就像乐高积木一样，可以混合和匹配软件组件）。因此，web3 系统更像是公共基础设施， 而不是专有的技术平台。与 web2 的封闭软件不同，web3 协议提供了分布式的互联网基础设施，任何人都可以在其上构建和创造互联网业务。重要的是，在 web3 中，这可以在不需要原始协议部署者的许可或使用集中控制接口的情况下完成。例如，可以将 Twitter 与一个提供由公众通过代币拥有权控制的社交媒体底层数据架构的 web3 协议进行对比。在这样的系统中，任何人都可以在协议之上构建自己的客户端或应用程序，并获得对其用户网络的访问。

这个概念比较抽象，可以考虑一下 web3 生态系统的示意图，其中包括一个去中心化的区块链、由代币持有者 DAO 管理的去中心化智能合约协议，以及几个使用传统实体形式作为独立业务运营的专有客户端。每个区块链和智能合约协议都作为去中心化的互联网基础设施，企业可以在其上构建、竞争和创新。

其次， 去中心化要求在 web 3协议中广泛分配控制权和参与权，确保网络的发展和使用反映了各种利益相关者的意见， 而不仅仅是创建这些协议的公司。通过适当构建的促进去中心化的协议，可以限制权力集中在一个或少数几个公司手中的情况。因此，去中心化应该限制企业或个人对门槛的控制权，并确保对协议的任何更改与持有代币并最终管理网络的广泛用户生态系统保持一致。

第三， 去中心化使得可以设计出更注重利益相关者资本主义的系统——这些系统旨在更公平地满足所有参与者的利益，而不是只服务于特定的一部分人。以代币激励的利益相关者资本主义将所有权和控制权分配给更广泛的利益相关者，而不是将股权持有者置于其他所有利益相关者（包括客户和员工）之上。因此，web3 协议和网络成为了一个多样的设计空间，可以更公平地满足所有利益相关者的利益。这样的去中心化协议提供了更稳定的互联网基础设施，让更广泛的利益相关者能够有信心地在其上进行建设。

去中心化类型

我们可以从三个不同但相互关联的角度来看待去中心化：技术、经济和法律。这三个角度都很重要，但往往存在着相互竞争的利益，因此在最大化整体去中心化和效用方面面临着复杂的设计挑战。

#技术去中心化（T）  

技术去中心化主要涉及 web3 系统的安全性和结构机制。 可编程的区块链和自治的智能合约协议可以通过提供自主、无需许可、无需信任和可验证的生态系统来支持技术去中心化，从而实现价值的转移。产品和服务可以在无需依赖可信、集中化中介的情况下部署和运行，打开了广阔的可能性。

对于区块链协议来说，技术去中心化是一个极具挑战性的问题，需要在多个相互竞争的力量之间取得平衡。但对于智能合约协议来说，通过使智能合约不可变（即任何人都无法控制和升级），这种类型的去中心化可以相对快速和容易地实现。

#经济去中心化（E）

区块链和智能合约协议利用自身的本地代币，开启了这些开源和去中心化系统具备自己的去中心化经济（即自治的自由市场经济）的潜力，使更多人能够参与并从这些去中心化生态系统中受益。

通过谨慎的设计决策，web3 系统的构建者可以促进去中心化经济的形成，从各种来源交换和积累价值（包括信息价值、经济价值、投票权等）。如果被正确构建，去中心化生态系统可以利用代币激励参与者为生态系统贡献价值，并根据其贡献更公平地分配价值给系统的利益相关者。为了实现这一点，web3 系统需要将权力、控制和所有权赋予系统利益相关者（通过空投、代币分发、去中心化治理等方式）。因此，整个生态系统的价值将归属于更广泛的参与者，而不是集中在一个中心实体和其股东手 中。

利益相关者（开发者、贡献者和消费者）之间持续的激励平衡可以推动更多的价值贡献到整个系统中，造福于所有人 。换句话说，这就是现代网络效应（5）所带来的所有好处，但却没有中心化控制和封闭经济的弊端。

#法律去中心化（L）  

法律去中心化取决于一个系统的去中心化是否消除了特定监管所要解决的风险。

例如，技术上去中心化的区块链和智能合约协议可以消除与可信中介相关的风险。因此，这样的系统在涉及针对可信中介的法规时，技术上的去中心化也意味着它们在法律上是去中心化的。

对于技术和经济上的去中心化来说，还可以消除其他风险，包括与 web3 系统的代币及其潜在价值相关的风险。这种去中心化将消除将美国证券法适用于代币交易的需要，否则可能会严重限制代币的广泛分发。

根据 SEC 的指导，我们可以将法律去中心化定义为一个 web3 系统能够消除重要信息不对称的潜在风险，并不依赖他人的重要管理工作来推动该企业的成功或失败的程度。达到这个阈值后，该系统可能被认为是“足够去中心化”，因此无需将美国证券法适用于该系统的代币。作为一个阈值问题，这要求特定代币不会赋予持有人与发行方或其附属机构的持续努力、资产、收入或资源相关的任何合同权利。

去中心化要素

在使用本地代币的 web3 系统中，必须全面考虑技术、经济和法律三种去中心化类型。这些类型之间相互影响，改变其中一种可能会影响其他两种。 举例来说，去中心化经济可以推动系统朝着法律去中心化的方向发展，通过优先考虑利益相关者的去中心化所有权、来自去中心化来源的价值积累以及将价值分配给去中心化的利益相关者。所有这些因素降低了信息不对称的风险，减少了对个人管理努力的依赖。

相反，如果一个 web3 系统的数字资产价值取决于原始开发团队的持续管理努力，那么该系统在技术、经济和法律三个层面上的去中心化可能会受到威胁。例如，管理团队的离职可能对数字资产的价格产生巨大的下行压力，使系统更容易受到 51% （6）攻击的影响。

鉴于这种相互影响，我们将去中心化分解为许多可能影响它的因素。以上图 1 和图 2分别提供了对代币化共识区块链协议和代币化智能合约协议最重要的要素的全面列表。这些要素根据类型（技术、经济和法律）和类别（计算、开发、治理、价值积累以及使用和可访问性）进行了细分。

去中心化是一个评估过程，不是基于绝对标准，而是基于包括任何 web3 系统的全部情况在内的一系列情况。因素的相对重要性将根据 web3 系统和评估者的目的而改变。此外，不同类型去中心化之间的优先权取舍可能因项目和人而异。

本文前两张配图的两个矩阵应该是提供更具体和标准化的去中心化定义的有用工具。希望这将使 web3 参与者能够为构建更多去中心化的项目做出贡献，同时使决策者和监管机构能够设计认识到去中心化的力量可以降低和消除风险的监管框架。
