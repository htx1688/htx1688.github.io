本文来自Blec report

2023 年 2 月 23 日，Coinbase 宣布将在以太坊上开发一个名为 Base 的 L2 网络，并称其是“为构建去中心化应用程序提供安全、低成本、开发者友好的方式”。

如果你不熟悉以太坊 L2 网络的概念，可以把它们想象成构建在以太坊之上的“链”，本质上是在以太坊的权益证明（PoS）网络之上添加另一个交易“层”。这些 L2 通常使用 ETH 作为基础货币，并提供与以太坊之间的“跨链桥”，用于转移资金。

最著名的以太坊 L2 网络有 Arbitrum、Optimism、zkSync 和 Aztec。Coinbase 选择利用 Optimism 软件堆栈来构建自己的 L2。

从基础设施的角度来看，所有现有的L2网络都可以被认为是相当中心化的。它们都利用了中心化的“排序器”（sequencer），这些节点构建和执行L2区块，同时还将用户的操作从L2传输到L1。本质上，这些排序器负责处理L2上的所有用户交易。

由于像 Arbitrum 和 Optimism 这样的网络的中心化性质，人们对这两个网络背后的公司应对其处理的交易内容承担多大责任提出了疑问。由于 Arbitrum 和 Optimism“合法货币媒介”的角色（尚未）得到任何政府的承认，它们在很大程度上逃脱了监管。

然而，Coinbase 多年来一直是公认的监管与加密间的“媒介”，并在 FinCEN（美国财政部反洗钱办公室）注册为“金融服务业务”。可以从 Coinbase 官方网站上了解更多关于 Coinbase 在合规方面的信息。

Coinbase 必须遵守许多金融服务和消费者保护法律，包括：

  • 《银行保密法》要求 Coinbase 核实客户身份，保存长达 5 年的货币交易记录，并报告某些交易

  • 《美国爱国者法案》要求 Coinbase 指定一名合规官，以确保遵守所有适用法律，制定程序和控制措施，确保合规，开展培训，并定期审查合规计划。

但如今，事情变得有点令人担忧。Coinbase 已经通过 Bankless 宣布，它将在发布时运行 Base 的唯一“排序器”。

这意味着 Coinbase，一家被要求遵守《美国爱国者法案》和《银行保密法》的持牌货币转账机构，将全权负责处理所有 Base L2 交易。

这就引出了一个问题：在允许跨链之前，所有跨链到 Base 的资金都需要 Coinbase （KYC）的身份验证吗？如果是这样，那么 Base 将是首个只需KYC（first-ever KYC-only）的以太坊L2。

在采访、推文和博客文章中，Base 负责人 Jesse Pollak 和 Bankless 等 Coinbase 发言方一直非常谨慎地表示，Base 将免权限（permissionless）在其基础上进行构建。然而，他们从来没有明确说过将可以免权限使用 Base，Coinbase 很明显在“打太极”。事实上，在 Jesse Pollak 关于 Base 的整整 1 个多小时的采访中，关于 KYC 或用户免权限的问题从未出现。

我通过 Twitter 和 Discord 多次直接向 Coinbase 和 Jesse Pollak 提出了以下问题：

  • 是否需要通过 KYC 合规才能将资金转到 Base？

  • 如果不需要，那么作为唯一的排序器，Coinbase 计划如何遵守《银行保密法》和《美国爱国者法案》规定的现有义务？

到目前为止，我收到的唯一回复是来自 Base 负责人 Jesse Pollak 的互动：

在这个回答中，Pollak 巧妙地回避了我的问题。他再次表示，Coinbase 打算让网络“免权限、去中心化、开放、全球、可访问”。听起来他似乎是在表示任何人都可以使用 Base，但他也没有直接表明，听起来更像是经过 Coinbase 合规团队仔细审查过的“声明”。而这份声明他在其博客文章中并没有提到的，我们需要特别注意。

为什么 Coinbase 在回答这些问题时会闪烁其词？他们似乎根本不希望 DeFi 用户考虑这些问题，也不担心开发者在开始构建网络时是否会因为没有读懂 Coinbase 的“言外之意”而混淆实际情况。

通过对 Coinbase 披露的信息整合，我们可以得出以下几点信息：

  • 自 2012 年成立以来，Coinbase 一直是公认的货币转账机构，受《银行保密法》和《美国爱国者法案》的约束。

  • 根据法律规定，Coinbase 必须对其所有储户进行 KYC 身份验证，以便美国政府能够充分跟踪和监视其储户的资金。

  • Coinbase 将成为 Base L2的唯一“排序器”，这意味着它将全权负责通过网络的所有交易。

  • 作为唯一的“排序器”，Coinbase 有能力以任何它认为合适的方式阻止或重新排序 Base 上的交易。这是审查 Base 交易的完美方式。

  • 允许开放跨链到 Base（无论是从以太坊L1还是其他L2和侧链）将意味着来自未知来源的非 KYC 资金将由 Coinbase 的“排序器”处理，这将导致 Coinbase 违反美国法律。

  • 因此，我们完全有理由相信 Coinbase 只允许用户将资金从 Coinbase 交易所本身转移到 Base，因为 Coinbase 交易所的所有资金都已经过 Coinbase 的 KYC 验证。

Coinbase 未能证实或否认这一有充分依据的结论。或许直到吸引了足够多的开发人员和项目在 Base 上构建，Coinbase 才可能公布答案。

为什么“需要通过 KYC 合规才能将资金转到 Base”这一点很重要？DeFi 在 2019-2020 年被宣传为“反击银行”的一种方式。这是加密行业的一个共同的主题（或价值观），直到我们发现 2020 年初开发团队及其投资者保留了不少集中控制权（通过管理私钥）。从那时起，迫使 DeFi 团队对他们对用户存款的集中控制水平保持诚实就成了一场持续的斗争。现在，如果我们看到引入一个只需 KYC（KYC-only）的链，那么集中控制权离移交给政府和他们所支持的银行就仅差“一步之遥”。
