原文标题：Beyond 721 & 1155: Revisiting The NFT Design Space

原文作者：Natalie Mullins

原文编译：bayemon.eth，ChainCatcher

对于牛市中的 NFT 市场，人们通常会关注谷歌搜索词“NFT”的趋势，将其视为采用和流行程度的指标。然而，讽刺的是，当某个词从语言中完全消失时，才可能是大规模采用的真正信号。当“NFT”三个字母淡出搜索框，就意味着市场中绝大多数人已经对 NFT 概念有了清晰的认知，人们将不再需要使用“NFT”这个术语，因为“NFT”已经会在人们茶余饭后的闲谈中细化为具体项目。

来源：The Block

然而在熊市中，人们则会将关注重点放在 NFT 的“效用”上。许多人抱怨说，自 ERC-721 首次提出以来，已经过去六年时间，而 NFT 市场却始终缺乏非投机性的用例和应用。

尽管这些不同的观点都非常重要，但牛熊之前的循环往复，观点间的持续争论，有时候会蒙蔽 NFT 发展过程中更为重要的问题。本文旨在重新审视 NFT 实用性的基本原理，并主要介绍近年来出现的用于 NFT 实用性扩展，并带来丰富用户体验的创新标准。

跳出市场，建立大局观

正如政府可以冻结你的银行账户、扣押资金并阻止你参与（合法）银行系统交互一样，中心化的内容平台和数据垄断者在数字领域具有同等的权利。这不仅对网络抗审查产生可怕的影响，而且还带会带来更大的挑战，如极具反竞争性的市场动态，阻碍了全面的创新，以及在面对对有价值的人工智能训练数据需求增加的背景下，对数据隐私和商业化的新问题。

那么，为什么 NFT 的实用性主要集中在（昂贵的）艺术品和收藏品的来源证明上呢？

通常情况下，在一个新面世的技术出现之后，由于人们需要更多时间去消化、理解、或头脑风暴出这项技术更多的可能性，结构最为清晰简单的应用案例往往首先起步。NFT 的实用性和应用之所以发展缓慢，还有一个主要原因是过去几年区块链的可扩展性挑战，限制了 NFT 的实用性。例如，最新的 ERC-6551 标准，允许任何 ERC-721 代币绑定智能合约账户，然而额外的功能往往伴随着额外的成本。只要创建、分发和参与 NFT 项目对大多数人来说仍然成本高昂，参与低成本或“非投机”的 NFT 投资就违背了他们作为“理性经济人”的标准。

然而，由于整个行业在可扩展性方面的巨额投资终于开始见效，这种情况正在发生改变。以太坊上的 Layer 2 解决方案蒸蒸日上，而 Solana 也在其独特的架构和在代币标准创新方面的领导地位上大力发展。

扩展 NFT 实用性：新颖的代币标准

（注意：这些标准并不一定是互斥的）

NFT 即应用：xNFT

xNFT，又称“可执行”NFT，它是一个 Solana 原生的 NFT 标准，该标准以 Backpack 钱包一为操作系统和密钥管理层，与基于 xNFT 的应用程序进行交互。xNFT 可用于构建收藏品（如 Mad Lads 系列的 PFP）或应用程序，但由于它们也是某些代码“执行权的象征”，开发人员可以通过 xNFT 构建体验更加丰富的项目。

抵押 Mad Labs xNFT 会产生与 NFT 锚定的奖励积分

xNFT app store

最独特的是，xNFT 类似于使用链上的软件许可证，开发人员能够轻松使用开源代码结构，并控制其 xNFT 应用程序的安装次数、安装者身份、安装费用，以及跟踪升级和用户交互情况。

Backpack 团队最近还推出了一个叫做"Soul Abstraction"的新功能，不仅为 xNFT 提供了类似于 NFT 托管的功能，还允许开发人员无需经过 Backpack 团队的许可或技术支持，自由地构建附加功能或向 xNFT 及其持有者空投福利。

NFT 规模经济：压缩 NFTs（cNFTs）

顾名思义，压缩 NFTs 是由 Solana 实验室、Solana 基金会和 Metaplex 的工程师开发的，通过状态压缩实现的 NFT 新范式。该想法源于社区关于将 Instagram 之类的应用程序数据上链所需条件的讨论，旨在让“单位边际存储成本尽可能趋近于零”。

简单地说，状态压缩的工作方式是将 NFT 数据的 Merkle 根存储在链上，而将实际的数据放在链外。即使链外数据被篡改，Merkle 根也会与链上存储的数据不同，从而维护安全性。而 NFT 数据的正确性，则通过 Metaplex 的 Bubblegum 合约来验证。

是否真的需要将成本降到 1000 倍以下呢？事实上，将这个问题提上日程需要等到 NFT 成为真正无处不在的“小图片”。“万物皆可代币经济学"已经成为一种流行趋势，而 cNFT 做的则是将互联网的数据上链并转化为"持久的主权数字对象”，从而成为经济可行的选择，并且能够大规模实现。

这正是在可扩展性方面继续降低铸造和发行成本非常重要的主要原因之一。像 Arbitrum 和 Optimism 这样的 Layer 2 解决方案已经在降低交易成本方面取得了令人瞩目的进展，它们相对于 Layer 1 ，将交易成本向下压缩了 10 倍。

这些改进绝非微不足道，以 Instagram 为例，即使是 0.10 美元至 0.50 美元的费用对于许多使用情况来说也是过于昂贵了。而实际上，需要付出更为昂贵的成本才能解决这看起来“微乎其微”的使用成本。值得注意的是，Layer 2 想要解决这些还为时尚早，不过这并不妨碍我们在下面重点介绍一些 Layer 2 NFT 创新的早期举措。

cNFTs 被用于提高用户体验的例子：

DRiP

DRiP 由之前创建 Solana Spaces 的团队打造，旨在构建一个加密原生的、以创作者为核心的内容分发平台。通过订阅他们的主要频道 Showcase，用户将在每个星期三收到 DRiP 团队的免费空投，他们还能自主订阅更多包括 Degen Poet、Degenerate Ape Academy、Vault Music、Floor 等在内的创作者频道。以获取创作者在 cNFT 空投的各种各样的数字藏品。

Dialect

Web3消息平台 Dialect 还为可交易的 emoji 表情包铸造了数十万个 cNFT。用户可以利用 Dialect 的智能消息功能，在消息界面内直接购买和出售资产。

Helium

作为从自己的 Layer 1 网络迁移到 Solana 的应用，Helium 现在正在使用 cNFT 作为代理，为其近 100 万个唯一物理热点设备上链。热点设备运营商可以在登录到与热点设备关联的钱包后认领 NFT。

Metaplex

在 Mailchimp 帐户被关停后，NFT 基础设施提供商 Metaplex 转而使用 cNFTs 发送邀请函给其新的创作者工作室。Metaplex 现在正在与 Underdog Protocol 合作开发一种更便宜、更符合加密原生特性的替代工具。

具有“超能力”的 NFT - 动态 NFT、私有 NFT、NFT 托管、NFT 融合

动态 NFT

能够根据特定的链上或链下事件更新元数据，是一个微妙但具有颠覆性的超能力。拥有这种能力的动态 NFT 在几乎所有情况下都比其静止形态更有意义。此外，为了让部署和管理过程更加无缝和标准化，动态 NFT 还强调对加密原生数据库和基础设施的极高要求。

私有 NFT

私有 NFT 最早的例子之一可以追溯到是 Secret Network 的 SNIP-721 标准，Quentin Tarantino 在 2021 年使用该标准将“Unleased Pulp Fiction Scene ”作为 NFT 发布。

其他解决方案也涉及到 Lit Protocol 访问控制和 Arweave，以实现对加密元数据的权限访问。举例来讲，基于 Solana 的音乐平台 Vault Music 就使用了这种解决方案，为向其用户提供私有 NFT。

NFT 托管与 NFT 融合

另一个 Solana 原生标准，NFT 托管是 Metaplex 代币元数据合约的扩展。该标准允许任何 NFT 作为钱包运作并持有自己的代币。它实现了两种不同类型的托管账户方式，即由 NFT 持有人管理的自有托管（TOE）和由特定创造者管理的自有托管（COE）。近期发布的 ERC-6551 ，实际上也标志着这一功能现在进入如今的 EVM 生态系统。

NFT 融合则是为了 NFT 托管功能的扩展方案，它使 NFT 能够捆绑在一起以创建新的 NFT，并根据它所持有的资产而改变。

这两个标准都为构建游戏的沉浸式体验打下了坚实的基础，游戏玩家在这两种方案引入后，可以自由体验代币交互和等级提升。

AIGC NFT：基于人工智能产出内容为信任和证明机制的 NFTs

EIP-7007 提出了一个 ERC-721 封装器，其中包括使用 zkML 技术检查提示和证明组合有效性的函数。元数据模式还提供了存储关于 AIGC-NFT 的信息的结构，例如提示、内容、所有权证明等。实际上，机器学习模型的创建者可以将其模型和相应的 ZKP 验证器发布到以太坊上，用户可以声明一个输入提示，发布一个推理任务，并最终以 NFT 的形式接收输出。该 EIP 的提出者认为这是一种帮助实现小型模型商业化的方法。

金融领域的 NFT：DeFi 与 RWA

虽然 DeFi 可能看起来是一个不太可能找到新型 NFT 应用案例的地方，但事实证明，许多金融资产，尤其是衍生品，是非同质化的，（这也正是 Uniswap 使用 ERC-721 标准来定义V3 LP 头寸的原因）。为此，NFT 在 DeFi 中有许多非显而易见的用例：

Panoptic（集中流动性市场制造商和 NFT）

Panoptic 是一个基于 Uniswap V3 LP 头寸的加密原生永续期权协议。期权由于需要提供抵押品而具有很高的资本效率问题，因此为了实现低抵押率的期权，团队转向 NFT 寻求解决方案。

Panoptic 实质上用半同质化头寸管理器替代了 Uniswap 的非同质化 LP 头寸管理器，使用的是 ERC-1155 接口而不是 ERC-721 。这使他们能够将多个期权合并为单个 NFT，以创建"自定义风险头寸"，从而更容易计算一组相互关联期权的抵押要求。因此这对于具有整体风险定义配置文件的多腿期权尤其有用，即使其中的单个期权理论上可能面临无限损失，也可以具体评估期权整体风险。

Pantopic 使用 ERC-1155 的 tokenID 来指代期权头寸的信息

Homebase（RWA）

Homebase 是一个允许用户以低至 100 美元的价格投资于代币化住宅房地产的平台。将交易平台上的资产转化为 NFT，为每个代币都有自己独特的元数据 URI，其中有关资产的信息被存储在区块链上并定期更新。

虽然 Homebase 和其他 RWA/NFT 融资平台在今天更像是“过家家”一样的小把戏，但它们实际上恰恰突出了专门为 NFT 建立的金融基础设施的重要性。大多数资产和许多金融头寸实际上是非同质化或半同质化的。这些示例还突显了通过将更多金融元数据上链可以实现的可能性（关于此问题的更多信息将在以后讨论）。

基础设施的挑战

尽管上述探讨的用例令人兴奋，但无论是对于激励机制还是数据标准，当前 NFT 元数据的严重分散仍然是实现 NFT 的全部潜力的重要障碍。如今，NFT 存在于各种不同的 Layer 1 链和 Rollup 链上，元数据存储位置、是否兼容 EVM、存储方式等方面，都是元数据分散化的具体体现。

关键是，随着 NFT 成为越来越多内容、资产和体验的容器，为了给元数据层带来更加可组合性和功能，对于一个分散化的、基于Web3的数据库的需求则越来越迫切。理想情况下而言，这样的数据库尽可能释放“困于中心化高墙”的数据，让其能在实体之间更精细地交换。

这样的产品中有一个有前景的例子是 Tableland，它是一个专门为Web3开发构建的 SQLite 数据库解决方案。可以将 Tableland 视为位于 Filecoin 存储和 EVM 智能合约逻辑的中间层，使得数据库可以直接从智能合约进行编程。例如，开发人员可以允许特定 NFT 的所有者或根据某些链上事件的结果来更改某些单元格和/或行。尽管在适应更新的代币标准和生态系统方面仍有许多工作要做，但也不能否认 Tableland 已经找到了正确的方向。

深入了解：这里可以查看 Tableland 策划的数据库和元数据解决方案清单。

随着人工智能和数字经济走到人类生活的前沿，在提取/控制数据和保护数字产权之间穿针引线将变得越来越困难。NFT 可能是突破数据控制与保护数字产权间壁垒的最好机会。
