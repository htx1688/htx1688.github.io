原文标题：《一览 Web3 基础设施框架，哪些基础设施应用值得关注》

撰文：Eric Hu

互联网从根本上改变了社会、个人和经济。 作为解锁前所未有的知识转移的工具，它引领了信息时代，成为支持应用程序、网络和服务的基础平台。

网络随着时间的推移而发展，并将继续发展——今天，我们正在从「Web2」转向「Web3」，这是互联网的下一次发展。 让 10 个人来定义 Web3，你会得到 10 多个不同的答案，但互联网正在发生变化，这是多种技术趋于成熟、全球化和社会转变的结果。

Web3 体现了许多原则 - 但它有望成为一个更加分散和协作的数据和价值互联网。 区块链和加密是核心，但只是这个新互联网采用的众多技术之一。

区块链和加密实验的爆炸式增长展示了新颖的结构和商业模式：强大的网络效应、更加不信任地协调利益相关者以实现共同目标、调整激励机制、转变治理模式、定义商定的所有权，也许最重要的是——互连数据具有价值，所有这些都以数字原生方式完成。

尽管区块链和加密有助于创建新的应用程序和业务，但重要的是要强调 Web3 是前端应用程序 / UX 以及后端技术的演变。 Web3 突出了技术堆栈中更大的可组合性和更智能的网络。无论是通过智能合约还是机器学习驱动的决策，升级后的计算网络更适合自动化和管理高吞吐量分析或监控技术（NLP、计算机视觉、物联网）与数据密集型用例（AR/VR、自动驾驶，智慧城市）。与 2000 年代带宽爆炸以支持不断增长的互联网并行发展，硬件方面的 5G 部署和边缘计算将融合为处理能力（无论是通过扩展的区块链驱动的计算和 / 或下一代云）为应用程序和服务打开了新的可能性。

需要强调的另一点是新技术成功所不可或缺的文化和社会转变（即时机）。过去几年，在新冠肺炎大流行的加速下，支持了许多趋势。我们已经看到数字化转型、无现金支付和向云迁移的显着增长。作为一个社会，我们越来越习惯于以数字方式进行交易和体验我们的生活。加密货币已进入主流文化，并已成长为一种真正的（尽管仍是新生的）资产类别。说了这么多年「机构什么时候入场」，现在机构终于入场了。

今天的「Web3」仍然很笨重，充满了摩擦。下一步是入职体验、更好的 UX/UI、适当的监管以及卓越或新颖的用例。但是，根据当今的情况对一项技术下注是愚蠢的。有大量的基础工作正在奠定基础。首先从适当的构建区块开始。

在过去的 2 年里，出现了创新的爆炸式增长——从构建第 1 层或扩展协议的团队、新的应用程序和项目，以及新颖的治理或代币模型。人才和创新走到哪里，资本就跟到哪里。 2021 年，250 亿美元的资金流入了基于区块链的项目。在这些项目中，那些被归类为「基础设施」的项目是最关键的。

那么，为什么今天是基础设施是关键，而不是应用程序？

正如 Signal 的创始人 Moxie 恰当地指出的那样——没有人愿意运行自己的服务器。 随着 Web3 的成熟，与 Web 的其他迭代一样，抽象复杂性是增长的关键驱动力。 托管服务和工具使团队能够发挥自己的优势并进行创新，而无需将大量资源用于后端（安全性、可靠性、可扩展性）。 当下一个十亿用户加入 Web3 时，大多数人不必关心协议使用什么区块链或代币标准，或者跳过障碍（和恐惧！）与应用程序交互。 这就是为什么适当的基础设施、架构和工具是关键的原因。 这在 Web3 中的前景是广阔的，所以总而言之，我们可以使用 Web3 基础设施的框架作为正在发生的事情的心智模型。

Web 的演变和 Web3 原则

Web 演进概述

在我们深入研究基础设施之前，多谈谈 Web3 原则可能会很有用——毕竟，这就是所有这些项目的构建目标。

  • 数字化：今天，我们正处于从物理信息和资产到数字化的过渡状态。 Web3 存在于数据流时代。 物理数据可以通过数字信息进行测量、组合和执行。 后端计算将处理来自各种来源的信息——AR/VR、物联网传感器和自主设备等等。

  • 新兴技术的融合和成熟：没有技术存在于真空中。 加密和区块链技术是架构和价值观的关键部分。 但其他技术也在成熟——5G、人工智能、边缘计算、物联网。 结合更高效、更智能地处理更大的数据集、灵活的基础设施和新的商业模式（例如代币）将为下一代应用程序提供一个基础。

  • 数据和资产的所有权：Web3 将有助于实现数据、资产和工作的真正所有权。区块链是革命性的，因为它们可以追踪和证明数字所有权。数字资产——无论是法定货币、加密货币还是 NFT，都可以通过钱包进行用户控制。可追溯性（并且对隐私的了解为零）允许衡量和控制资产或个人的贡献。结合数字身份，这是强大的。

  • 增加控制和治理的去中心化：加密货币和区块链从根本上基于去中心化、信任和协调的问题。代币不仅代表资产，而且通常代表投票权或会员证。去中心化将始终是一个范围内实体存在的规模。中心化有其自身的好处，但随着 DAO 的第一次迭代，公司治理、网络控制和分布式决策的新模式正在形成。更加分散的控制将使平台的建设者、运营商和用户能够拥有并拥有平台发展的发言权。

  • 可组合性：在互联网出现之前，我们拥有的都是围墙内联网。 区块链也在越来越多地出现这种情况，互操作性成为重点。 通过开源代码，Web3 的构建考虑了可组合性。 这意味着团队可以采用现有的项目或计划并在它们之上进行构建。 这允许跨应用程序和通过技术堆栈进行更快的开发和通信（互操作性）。

更成熟的 Web3 将为转变网络上的数据、价值和所有权之间的关系奠定基础。 公司、用户和网络可以更自由地共享或与数据交互或转移价值。 所以 Web3 有很多事情要做，而今天我们正在观察一个处于成长阵痛中的空间。 这就是为什么人们关心为 Web3 奠定必要基础的基础设施服务。

那么，基础设施的重要性是什么？

基础设施是发展的基石。今天，我们将其视为道路、桥梁和公用事业，以及核心的「数字基础设施」。 「数字基础设施」是指当今互联网的管道和服务。在过去的几年里，数字基础设施已经从信号塔或光纤网络等硬资产发展为云、数据中心和托管网络软件等「上层」技术。

对于 Web3，我们现在将基础设施定义为允许大规模部署、构建和使用 Web3 应用程序的工具、服务和架构。毕竟，企业和零售业都必须以更加顺畅和可扩展的方式使用 Web3。 Web3 基础设施有多种形式——尤其是在去中心化范围内的一种。很大一部分 Web3 项目本身就是去中心化的。也就是说，使用代币或基于社区的治理。然而，去中心化是一个范围，并且有充分的理由——在资金、用户体验和运营效率方面存在权衡。在 Web3 基础设施中，项目和团队正在构建各个领域，其中许多采用中心化方法，通过传统股权融资筹集资金。

Web3 基础设施的一些特点使其对建设者和投资者具有吸引力。 更可预测的收入、与现有平台 / 软件的相似性、更有经验的管理团队以及对生态系统的使命关键性。 这导致了来自 Web3 本地 /Crypto 资本的资金，以及来自传统金融参与者的增长或风险投资。 因此，从 2021 年开始，这些 Web3 基础设施轨道已经开始形成。

框架的指导性问题

当我们查看 Web3 的基础架构时，几乎没有什么指导性问题支持该框架作为一种在其不断发展的过程中思考前景的方式：

  • 项目团队通常使用哪些 Web3 服务和工具（「架构」）以及哪些数据、存储或计算（「核心基础设施」）系统支持这些？

  • Web3 中的建设者或机构存在哪些运营挑战？ 已经开发了哪些「关键任务」或抽象不必要的复杂性的解决方案？

  • 某些用例（例如 DeFi、NFT）已经并且需要建立哪些基础服务才能大规模成功？

  • 现有公司和项目在哪些能力上投入资源——无论是通过并购还是有机建设？

  • Web3「基础设施」的组件如何相互交互？ 每个组件的建立程度如何（即领域中的项目数量、通用标准、解决方案的成熟度）？

在 Web3 中，人们可以考虑在基础计算（区块链）层之上的三类服务和产品中的数字基础设施——数据、价值 / 流动性和 Interplayers/ 区块链支持服务。

计算层（区块链和受区块链启发的技术）：Web3 的基础是计算层。这包括区块链和受区块链启发的网络（第 1 层、子网 / 侧链和扩展解决方案），其他一切都建立在其之上。由于当今有许多区块链，因此出现了跨链 / 全链协议以及链上消息传递项目，以允许价值和信息从一条链交换到另一条链。这是很重要的一点——区块链网络以一种更加无需信任的方式将数据和价值在各个利益相关者之间的传输交织在一起。关于这个正在开发的计算 / 区块链层有大量信息，但可以说这是花费了大量时间并将继续投资的地方。

在基础区块链层之上，我们可以将服务、工具、架构和基础设施分为三个类别：

  • 数据参与者：为团队构建和运营 Web3 项目提供核心基础架构、架构和工具

  • 价值和流动性参与者：资本在 Web3 应用程序和参与者之间流动的构建块，摩擦更小（即高昂的 gas 和滑点费用）

  • 「Interplayers」或区块链支持基础设施：支持和维护区块链 / 计算网络的利益相关者

请注意，应用程序和项目如何利用或构建 web3 基础设施存在着无穷无尽的细微差别，但我们将其保持在高水平。 （更多深度内容请参见附录 1）

数据基础设施：Web3 将是语义学的演变。 这意味着机器和计算机将需要分析、发送和计算大量数据。

对于数据基础设施，有更多「核心」或已建立的数据基础设施和「新兴」服务和工具构成了开发团队的架构。 此处的数据基础设施包括使区块链数据易于使用或在其上构建的各种工具、服务和组件。

这些以数据为中心的构建块的简要概述：

  • 核心数据基础设施：管理与区块链交互的节点和 API 提供商、去中心化数据存储、用于跟踪交易的区块浏览器以及用于高效访问数据的索引 / 查询服务

  • 新兴 / 相邻数据基础设施：为开发应用程序交互的工具和托管服务； 这包括从开发人员环境到协助代码传送、代码审计、SDK（例如隐私 / 零知识工具包）和数字身份服务

  • 协助 Web3 用例的工具和托管服务：NFT 相关和 DAO / 社区运营服务。 示例包括 NFT 分析或低代码 NFT 创建、组织财务管理、投票平台和社区入职工具。

  • 价值和流动性基础设施：Web3 的一项关键创新是在最基本的层面上将信息与价值联系在一起——这可能是由于商定和可验证的所有权。

对于 Web3，价值与运营网络的核心功能交织在一起，无论是通过治理、安全还是实用代币激励。但没有流动性的价值并不值钱。对于流入 Web3 的所有类型的资本，价值需要得到保护、出入、分配、转换、访问，并允许在没有太多摩擦的情况下在生态系统中流动（即 gas、滑点或使用费不应该对活动具有极大的破坏性）。

因此，许多参与者提供了「价值或流动性」的构建块：

今天的「核心」价值基础设施包括：

  • 钱包：Web3 的网关，钱包是用户与区块链、代币和应用程序交互的主要界面。最近，我们看到钱包变得更像「浏览器」，这是入职的一个很好的步骤

  • 托管人保护（机构或大型零售）资本，交易所转换整个生态系统和代币的价值，以及现有支付基础设施（例如法定货币、信用卡）的出入通道

新兴价值和流动性基础设施包括：

  • 用于投资组合管理的仪表板和分析； DeFi 或交易所聚合器用于增强流动性和机构交易基础设施以实现高效市场

  • 协助资本运营方面的相邻服务 - 主要是税务、合规、监管和网络安全（以保持价值）服务

  • 「Interplayers」和支持区块链的基础设施：我们已经强调，Web3 的核心租户是区块链技术允许的价值和数据的结合。 因此，有支持区块链计算层的基础设施，这推动了价值和数据服务获得成功的能力。

Interplayer / 区块链支撑基础设施包括几个子类别：

  • 网络维护：加密货币矿工（用于工作量证明网络）、Staking 服务和验证者服务允许更轻松的网络支持或维护；这些有助于交易处理、投票（通过委托）和网络激励共享（例如区块奖励）

  • 链上分析：这些服务如今在合规或交易用例中很流行，为计算层中的活动提供仪表板和跟踪。对当今加密应用程序中的众多黑客或安全漏洞非常有帮助。

  • 新兴的支持基础设施：链上通信和消息传递协议受到越来越多的关注，正如今年夏天在 EthCC 5 中所强调的那样，随着它们的不断发展，我们可以在此处包括 Omnichain 协议或桥接器。

相互结合 - 说明性 NFT 项目：

在 Web3 基础设施上使用这种心智模型，让我们通过一个社区管理的 NFT 项目的说明性示例来看看所有东西是如何组合在一起的。 （对于更多技术性读者，附录 1 中对框架的组件有更深入的了解）

  • 首先，让我们选择以太坊网络来启动 NFT 项目。矿工和质押提供者用于通过处理交易来保护和支持网络。

  • 现在，我们需要构建 NFT。该团队可能会使用 NFT 的开发人员工具包或在标准开发环境中编写自己的智能合约。他们将利用像 IPFS 这样的数据存储提供商来存储实际的 NFT 信息。如果这是一个大型或复杂的项目，可能会在启动之前聘请智能合约审计公司。在发布时，该团队将寻求在 Opensea 等市场上提供该系列。

  • 社区参与者将需要一个钱包来存储 NFT 并使用 ETH 支付。他们可能会进入交易所或通过另一个入口（如 Moonpay）用卡支付。请注意，加密货币交易所、钱包和市场都利用区块浏览器和 API 提供商与以太坊区块链进行信息交流（例如，当您将 ETH 兑换成 NFT 时）。

  • 最后，社区和治理。 NFT 项目团队可能会使用 MPC 或多重签名钱包或托管解决方案作为他们的金库。诸如计数之类的操作工具可用于管理他们的社区投票提案。同时，成熟的参与者可能会利用链上或 NFT 分析来衡量活动、创建仪表板和管理自己的投资组合。

结论性想法：

今天可能并不明显，但即使回顾几年，Web3 的增长和创新速度也令人难以置信。在过去的几年里，我们接受了一个全新的资产类别，一些最聪明的人、最大的公司和大量资本加入了这场创新。大规模采用将改善监管、入职、新应用（有实际收入）、商业模式、社会结构和技术基础设施。

对于批评者来说，我们还没有到达那里。但是有很多东西正在建造中并且令人兴奋。我们正在实时体验一个新行业的发展，并且随着社会学习在数字时代进行互动。肯定需要批评。但随着建设者解决从隐私到可扩展性的每一次迭代问题，我们已经看到了新治理模式、数字或物理混合业务以及激励对齐网络的力量的实验成果。现在是构建的最佳时机，我很高兴看到事情的发展方向。

附录 1：Web3 基础设施框架 - 深入了解

数据基础设施：

核心数据基础设施：

  • 节点和 API 基础设施：Web3 应用程序功能的核心，节点提供商通过基于使用的收入模型提供 API 和 RPC（远程过程调用，允许程序与区块链对话）。 例： Alchemy 和 Infura

  • 数据存储：由于效率和成本的原因，许多类型的数据放在区块链上没有意义（例如视频和大多数前端内容）。 去中心化存储解决方案以及为支持扩展使用而构建的服务填补了这一空白。 例：IPFS 和 Arweave； Pinata 和 Ceramic

  • 数据流 - 查询、索引和预言机：数据分布在区块链中的节点之间。 查询和索引协议对于使这些数据易于通过 API 使用至关重要。 同样，预言机将来自外部系统的数据提供给区块链（例如天气数据、代币价格等），充当智能合约的链上 API。 例：The Graph，Chainlink

  • 区块浏览器：区块链浏览器是区块链网络中显示资金流或交易状态的数据的搜索引擎。 Metamask 等钱包将使用 API 调用来阻止浏览器显示最近交易等项目。例：Etherscan, Blockchain.com

新兴数据服务和工具：

  • 开发者平台 / 工具包：开发者工具帮助团队创建、测试和调试他们的软件。这包括可以更轻松地开发 SDK 等应用程序的工具。尽管为 EVM 兼容的链建立了更多的开发者环境，但随着开发者在非 EVM 链上构建应用程序，这些链使用了类似的工具。例：Hardhat, Truffle, Aleo

  • 智能合约审计：作为区块链开发过程的关键部分，审计员被依赖于在公开部署之前确保代码的稳健性和安全性。例：Open Zeppelin, Trail of Bits

  • 数字身份：Web3 的核心租户取决于数字身份——其中可能包括 KYC 或许可访问等关键属性。这里的项目正在构建服务，例如带有钱包的单点登录 (SSO) 或 Web3 身份，如「.eth」域名。例：ENS，Spruce

  • NFT 工具和服务：随着 NFT 用例的增长，出现了各种各样的工具和服务来支持生态系统。项目范围从分析到开发人员工作室、NFT 工具包、聚合器和市场。例：Opensea, Rarity Sniper, NFTrade, LooksRare

  • 社区建设和运营：随着 DAO（去中心化自治组织）越来越受欢迎，许多人在物流和运营方面遇到了挑战。已经出现了一些项目来帮助协作、发现、运营、资金管理、启动和分析。例：Coordinape、RabbitHole、Snapshot、DeepDAO、SuperDAO、Tally

价值和流动性基础设施：

核心价值基础设施：

  • 钱包提供商：作为 Web3 大部分的主要接口，钱包不仅可以存储价值（数字资产），还可以实现链上身份和与 dapp（例如 DeFi）的交互。 根据安全和用户体验方面的特定需求，存在几种类型。 例：Metamask，Rainbow，Coinbase 钱包

  • 托管人：合格的托管人对于广泛采用数字资产至关重要。 资产的可信保管、记录保存和管理是机构进入 Web3 的第一步。 例：Anchorage Digital、Fireblocks、Gemini、Qredo

  • 交易所和市场：没有流动性就无法维持价值。 交易所和市场是必不可少的。 随着公司寻求建立一整套资产管理产品，交易所领域发生了很多并购。 例：Coinbase、FTX、Uniswap

  • 法定货币出入口：许多用户通过中心化交易所入口（即用法定货币兑换加密货币），但是随着 Web3 的发展，法定货币和加密货币轨道必须变得更加无缝。 今天的平台为 dapps 提供集成，允许使用信用卡购买 NFT 等操作。 例：Moonpay，Wyre

新兴价值和流动性服务：

  • 流动性和 Defi 工具：DeFi 经历了多次迭代，生态系统中适当的流动性和投资组合管理的重要性不容小觑。 项目范围从仪表板到去中心化交易所聚合器再到交易基础设施。 例：1inch，Bloxroute Labs，Talos

  • 税务服务：随着税务指南的制定，Web3 或 Defi 活动的准确税务和会计工具对任何组织都至关重要。 例：Koinly，Taxbit

  • 合规性：对于 DeFi 尤其重要，这里的项目专注于交易监控、KYC、AML 和监管合规。 例：Elliptic，Coinfirm

  • 安全性：Web3 中的当前网络安全服务仍处于初期阶段，主要是赏金平台或众包威胁检测。 例：Immunefi，Polyswarm

Interplay 基础设施（支持区块链的基础设施和架构）

  • 加密货币挖矿和 Staking 提供商：支持 Web3 中计算层（区块链和类似技术）的基础设施。这些处理交易并通过挖矿或质押进行验证来帮助保护网络。这些提供商运行节点基础设施，并允许用户通过分配资金来参与质押奖励。例：Core Scientific、Foundry、Whinstone、Blockdaemon、Figment

  • 链上分析：区块链包含大量数据，为从政府到贸易商的生态系统中的所有参与者提供有用的信息。指标包括交易数据、钱包和用户活动。链上分析服务聚合有用的指标并提供仪表板以可视化区块链数据，以进行尽职调查、发现、市场情绪等。例：Chainalysis，Nansen，Dune Analytics

  • 跨链桥、链上消息和全链协议：在传统的互联网基础设施中，「消息 / 网络」是一个关键概念。由于 Web3 可能包含多链 / 计算的未来，因此允许区块链互操作的协议是统一生态系统的基础。这可能以多种形式出现，从单链桥到多链桥，再到互操作性协议。链上消息传递也正在开发中（例如，在钱包之间，将交易与直接消息传递相结合以用于对账等用例）。 例：Wormhole、CCIP (Chainlink)、Layer Zero、WalletConnect、EPNS ( 以太坊消息通知服务 )
