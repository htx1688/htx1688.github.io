BTC闪电网络是BTC主网上运行的Layer 2，它允许通过处理远离主区块链的交易来更快地支付比特币，同时保持相同的去中心化和安全性。闪电网络处理交易速度是BTC主网处理交易速度的14万倍以上，并且交易费用极低。跨区块链也是BTC闪电网络的一大特点，跨链原子交换可以通过异构区块链共识规则立即发生在链下，只要链可以支持相同的加密哈希函数，就可以在不信任第三方托管人的情况下跨区块链进行交易。这给了BTC闪电网络DEX（去中心化交易所）生长空间。下面我们介绍BTC闪电网络DEX的解决方案。

Portal

对于DEX来说，最根本的是智能合约的可用性。 智能合约一直难以在BTC网络上得到实施，对此Portal的解决方法是在BTC的主网和Layer 2上搭建一个Layer 3。Portal的底层协议为Fabric peer to peer Network。Fabric是一个Layer 3的点对点计算市场。这个系统通过使用智能合约在不同区块链（BTC、ETH等）上达成短暂计算的基础设施。

图片来源：https://docsend.com/view/pdk55ecxmuewiy8f

Portal通过零知识证明系统保证订单薄得以执行， 使交易订单统一和匹配来聚合去中心化金融应用上交易员的流动性。

Portal推出“atomic swap”的点对点合约，通过它搭建的点对点交易所可以拥有中心化替代品的速度、可用性和流动性。

经典的Tier Nolan Atomic swap模式

图片来源：https://docsend.com/view/pdk55ecxmuewiy8f

Portal 的L2和L3原子交换技术可用于信息数据和计算的通用市场。 Portal的目标是用自由的数据和计算市场来取代易于在设计和架构上集中化的 Web 服务器模型。Portal 使用的底层协议Fabric扩展了比特币的功能，而不需要任何 BIPs (Bitcoin Improvement Proposals，)。

1）信息数据市场，以去中心化方式存储数据的名为Tiny的L3合约取代了 Web 服务器模型，其中费用提供了对信息审查的抵抗力。

2）计算市场，用户能够部署复杂的程序，以便由同行网络盲（blinded）执行。当“计算请求”得到答复时，Portal swaps支付绑定的比特币。合同确保任何一方的履约和付款证明。

图片来源：https://portaldefi.com/

Portal支持点对点、去中心化、抗审查的应用程序作为比特币的Layers。 这个系统允许安全地建立和执行点对点协议，包括金融合约（现货、借贷、众筹和加密货币衍生品）或非金融合约（沟通网络和社交媒体等）。

目前Portal只存在一个L2 swap协议，Arwen。Arwen被设计用于中心化交易所和他们的用户。在它的模式下，中心化交易所不是一个“交易所”而只是一个交易柜台或者OTC柜台。在Arwen上，用户和交易所均需要将他们的币锁定在链上托管合约上，并将他们的托管通过交换他们各自的公钥哈希（PKH）进行配对。因此所有交易涉及标记转账的链下通行。其中仅有二者间的交互，二者额度更新完毕托管也就结束了。整个转账过程快速且接近零成本。

Arwen协议的缺点：

  • 仍然依赖中心化交易所，协议仍然无法百分百保证交易安全，协议目前也没有理性的经济激励机制来减轻风险；

  • Arwen专门设计出来针对中心化交易所，那些匿名的点对点跨链交易所无法使用Arwen；

  • 与交易所商业模型不兼容，Arwen需要中心化交易所变成交易柜台和做市商，这改变了他们现存的商业模型。

Mintlayer

为了提高BTC网络的代币互操作性并实现原生BTC交换，Mintlayer用侧链的形势来达成这个目的。它还提供了多个功能来加强网络安全、增加节点包容性并确保长期可持续性。Mintlayer是一个建立在比特币之上的代币化层，支持与BTC的Layer 1、Layer 2闪电网络进行交互。

图片来源：https://docs.mintlayer.org/whitepaper/1-blockchain-architecture

Mintlayer 区块链锚定到比特币：每个 Mintlayer 区块都有一个比特币区块的引用。在持续 1008 个比特币区块的每一轮 Mintlayer 中，参与者从质押者中选出，以合作创建链上的区块。

决定积极参与网络共识的用户需要运行一个节点并质押足够多的 MLT 代币以通过最低门槛。每一轮，算法都会自动执行一次选举，用户可以被选为“参与者”。代币权益越高，被选中的机会就越大。在每一轮的特定时间，每个参与者都被要求建立和传播一个区块，将收取所有交易费用。Mintlayer 链的长期安全性由比特币区块链上的检查点系统保证。

图片来源：https://docs.mintlayer.org/whitepaper/1-blockchain-architecture

这可以将 Mintlayer 区块链用作比特币或以太坊等其他区块链的第二层。 中央实体可以在 Mintlayer 上包装代币：它们将资金锁定在主区块链上，从而创建相应数量的 MLS-01 或 MLS-02 代币。独立的预言机可以证明主链上存在相应数量的锁定代币。通过这种方式，可以在 Mintlayer上创建机密代币，例如 m-BTC 或 m-ETH。

Mintlayer  还可以实施 强联邦（Strong Federations，如Liquid）来发行打包的代币和机密资产。 联邦循环更新交易的锁定时间（比特币上的 nLockTime），冻结多重签名 UTXO 中的资金，并在 Mintlayer 上创建相应的代币。代币总是有可能被挂钩：在这种情况下，Mintlayer 代币被烧毁，而原始加密货币在主网络上被解锁并可供原始所有者使用。

开发者可以自由使用 ACL、可编程池和预言机的组合，在 Mintlayer 链上创建更复杂的智能合约，并将 MLS-01 代币的发行或存在与锁定在其他区块链上的相应数量绑定（甚至自动绑定） MLS 代币通过具有数字领域之外存在的资产的预言机）。

Mintlayer 使用比特币 UTXO 结构，而不是 Ethereum、Ripple、Stellar 或其他的帐户基础模型。

选择 UTXO 模型是因为三个基本特征：

1）它与比特币中已经实现的技术兼容，例如原子交换和闪电网络。

2）它更加注重隐私，因为单个钱包通常使用多个地址，因此很难甚至有时无法区分哪些地址属于同一用户。

3）付款可以在单笔交易中批量（汇总），从而节省大量空间，否则每次付款进行单笔交易所需的空间。

MLS-01 代币和 MLT 仅在 Mintlayer 侧链上转移，无需占用比特币区块链。

MLS-01 标准端口不同的交易功能：

1）访问控制列表 (ACL) ，可以对特定加密货币的交易实施进行特定限制。 ACL 有助于根据公司政策或其他法律要求发行或更新安全令牌，类似于在以太坊的 ERC-20 上的做法：接收地址可以被列入白名单或黑名单。但是，应用程序可能更广泛：它们可能涉及实用代币的使用条件或确定代币的不可替代性。例如，使每笔交易无法在最小或最大阈值下转移一定数量的代币，或者强制实施限制代币在特定时间的可转移性的时间锁定。

2）Mintlayer 钱包管理的所有代币都启用了闪电网络。 由于 Mintlayer 的代码库继承自比特币并进行了一些小改动，因此闪电网络钱包功能使用与比特币相同的 BOLT 规范。预计在 Mintlayer 网络上交换的大多数证券代币将不会使用闪电网络，因为难以为交易频率低且流动性低于货币用途的代币开发一个分布良好的渠道网络。尽管如此，闪电网络作为 Tether 交易和其他稳定币的用例确实有很大的成功机会，特别是因为它们经常在有更大机会创建均衡渠道的托管服务/交易所之间转移。

3）点对点批量处理。 Mintlayer 在钱包内开发了两种不同的工具来激励批处理的使用：独特的用户界面，可帮助交易所等服务/企业大规模批量交易（所有 MLS-01 代币）；一种点对点批处理系统，灵感来自于 Wasabi 的 coinjoin，用户可以在其中将一个或多个令牌发送到一个或多个目标地址，同时将交易与其他用户结合起来。用户可以聚合使用不同 MLS-01 代币进行的交易，并使用签名聚合将每次支付的大小减少 70%。

4）Mintlayer 通过引入一个可靠的跨链比特币原子交换绕过了经典原子交换时间锁（比特币上的 nLockTime 参数）不可靠等问题。 由于其结构为比特币侧链，因此每个 Mintlayer 区块都带有对比特币区块链的引用。将硬币返回给发送者的两个交易的 nLockTime 是根据两条链上计算的比特币块的确切数量计算的。原子交换技术是分散交换概念的基础。Mintlayer 为跨链 DEX 交易（在两个 MLS-01 代币之间）或链内交易（在 BTC 和 MLS-01 代币之间）实现它。钱包还提供CPFP（child pays for parent）功能，如果由于费用太低而卡在内存池中，可以加速原子交换交易。例如，钱包可以与瞭望塔（甚至由用户自己控制，或由服务提供商/预言机控制）协作，在交易支出输出延迟时通知用户，并建议适当的 CPFP 交易费用金额。

5）无gas的经济环境。除非用户另有选择，否则 Mintlayer 上任何交易的默认费用都设置为 MLT。 在这种情况下，费用会自动以用户转移的代币表示。在钱包中可用的代币中，用户仍然可以决定使用哪个代币来支付费用。预计大多数区块签名者将接受所有主要的 MLS-01 代币。（考虑到收费经济是一个自由市场，区块提议者有动机接受任何代币，只要它在市场上具有公认的价值。相反，留下的交易可能会被下一个区块签名者获取。）这种博弈论机制可能导致拒绝非流动性代币，构成有效的垃圾邮件过滤器。在准备交易时，钱包 UI 会根据分析 Mintlayer 的内存池和过去交易的算法建议每个代币的费用金额。区块制造者还可以发出信号，他们是否会在他们将生产的下一个区块中接受新的代币。

专注于机密交易（Confidential transactions）的MLS-02的原理如下： 通过使用环签名实现Bullet-proof机密交易，可以在网络上发布代币，这是代币化标准 MLS-02 的特征。启用机密交易的代币的发行是隐私和匿名的保证。但是，除非确实需要隐私，否则不鼓励这样做，因为它需要权衡：CT 占用更多空间（更高的费用），需要更多资源进行验证，并且不能在多笔支付的交易中批量处理。由于挂钩机制，可以发行其他加密货币的机密版本，例如 m-BTC 或 m-Ether，以更高的匿名性转移已经存在的加密货币。

即使在没有实现 CT 方法的情况下发行代币，Mintlayer 的 UTXO 模型也会授予一定程度的机密性：

1）一个账户/钱包可以有多个独立地址；

2）闪电网络交易提供更高的隐私性；

3）点对点批处理有效地混合了不同用户的输入、输出和签名，使链分析变得更加困难。

依靠上述功能，Mintlayer可以使用合法合规的技术架构对一级和二级市场的股权、房地产和其他资产进行代币化。它支持效用代币的税收、投资支出和其他代币经济学模型。

总结

Portal和Mintlayer也可以相互合作，为用户提供一种私密、安全且经济实惠的方式与DeFi生态系统进行交互。 在Mintlayer上添加侧链资产将增加Portal DEX上可用的交易对数量，通过Portal的抗审查跨链交易所使其资产可供更广泛的客户群访问，这也使Mintlayer受益。

当前BTC的市值占加密货币总市值的40%以上，而以太坊作为DEX主要存在的链，其代币ETH市值占加密货币总市值的20%不到。DEX还有很大的原生流动性没有被挖掘。 BTC闪电网络以及搭建在BTC Layer 1和Layer 2之上的如Portal和Mintlayer这些项目，正在将这些原生流动性引入DEX的潮流之中。对于比特币DEX的发展，我们会保持持续关注。
