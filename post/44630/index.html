作者：月之暗面

导言

推特作为社交巨头终于花落马斯克之手，开启了Web 2社交平台向Web 3社交协议的进化之旅，在一系列大刀阔斧的改革进程下，我们依稀可以窥见未来的大致方向：引入更多的区块链技术，增强推特的造血能力，减少无用冗员造成的财政负担等等。

但这并非是事情的全部，Web 3社交协议的实现方式也并非一种，至少还有从头重构去中心化网络，进而从其上诞生原生去中心化化社交协议的方式，如果说推特是渐进式改革，那么Jack Dorsey 布局的Web 5就是一种革命式的重构。

Jack Dorsey 在今年5月退出推特董事会之后，全身心扑在比特币之上，以他另一家公司支付平台Block（原名Square）为名，养了闪电网络的开发团队，以及更为宏伟的Web 5网络开发人员。Block三季度总净收入为 45.2 亿美元，同比增长 17%，其中比特币交易额为 17.6 亿美元，毛利润总计近 3700 万美元。

2022年6月11日Jack Dorsey 在Consensus大会上宣布将构建Web 5去中心化网络平台，将成为下一代的网络基础设施，直接取代目前的Web 2和Web 3概念，因此也被称为Web 2 + Web 3 = Web 5。

目前，5个月的时间过去了，Block旗下的TBD公司开发的Web 5平台也逐渐成型，已经完成了跟Circle的出入金实验合作，以及使用闪电网络接入比特币的货币层建设，Web 5各要件已经基本成型，包括DID、DWN（去中心化节点）、DWA（去中心化网络应用）以及等等。

最新的进展是10月18日发布的VC（Verifiable Credentials，可信验证凭证），至此，一个基本完整的网络设施已经完备，并且是完全不同于加密货币加链上项目的模式，而是整体构建“网络”这一Web的原意，这也是Jack Dorsey 版的Web 5引人注目的最大亮点，此外作为比特币最大主义者的Jack也选择比特币而非以太坊等进行建设。

Jack Dorsey 所理解的Web 5

围绕比特币，JackDorsey在Block公司进行了组合式布局，其中支付公司Block做比特币支付，旗下Spiral负责比特币闪电网络开发套件LDK和比特币开发套件BDK，以及TBD进行建立在比特币货币层上的Web 5。

与其说下一代网络是Web 5，不如说是将比特币引入网络之中，并对其进行数据所有权改造，这也是最初的比特币想要做而未完成的事，Jack Dorsey 不愧是继中本聪之后的最坚定的比特币信仰者。

在Jack Dorsey 的事业版图中，Block部分是实际产生收入的公司实体，旗下的Cash App已经是较为主流的比特币支付应用，季度交易量稳定在15亿美元以上，目前已经可以支持比特币支付薪酬，以及使用闪电网络进行转账等操作。

而另一家TBD公司正在进行的Web 5开发，某种程度上是串联起现有的Cash App以及LDK、BDK等开发工具，助推比特币应用场景的扩展，进而让Web 5发生在比特币网络之上，已经在去年推出了基于比特币网络的交易平台tbDEX的白皮书。

Jack Dorsey 此前曾认为，目前的Web 3是鸠占鹊巢般的词汇，是被VC用资金裹挟和催熟的早产儿，并且和我们熟知的网络并无太大关系，与此相对应的是Elon Musk也是目前阶段的Web 3怀疑论者，更典型的是万维网的发明人Tim Berners-Lee认为加密货币不是Web 3的全部。

因此，必须明确的是Jack Dorsey 理想中的Web 3也好，Web 5也罢，首先必须是一种网络设施，能够进行通信、交流是其第一特征，其次必须跟VC有较大程度的隔离，最好的方式是以开源的方式进行，以最大程度捍卫网络的开放性。

而在其上搭建的应用层，比如未来的去中心化社交协议，应该是数据可支配，关系可迁移，身份可识别的一种网络通信模式，高度区别于现今的社交平台。

所以Jack Dorsey旗下开发的均属于开源项目，这也是最符合Web 1.0时代的开源开放精神的继承者，类似于Linux的全球项目才是他梦想中的下一代互联网。

可以发现，对于Jack Dorsey 而言，Web 5这个词本身并没有那么重要，以及跟Web 3的关系，或者Web 4是不是特指移动互联网，都不如Web 5的本质特征更为重要：一个数据属于用户的网络才是Web 5概念的本质含义。

而在其上搭建的社交媒体，也应该是从网络通信和社交关系的角度去蔓延，可以称之为Web 5原生社交模式，而非对目前的社交关系的模仿和迁移，从这个角度，才可以去探索未来的网络以及建构在其上的DWA（去中心化网络应用）究竟该是何种样貌。

在理清楚概念的牵涉以及本质特征后，接下来是探索目前的开发进展，得益于开源开放的uilld方式，PANews得以实际观察Web 5发展的全过程。

Web 5架构：去中心化网络开发平台

在TBD的开发设想中，Web 5是一种网络通信设施，是互联网的最底层模型，其本身就是一种网络，目前的万维网才是Web 5的对标物，而不是跟以太坊或者比特币一样的Layer 1公链，这也是其最容易被误解的地方。

在其内部，可以将其划分为作为入口功能的DID/Wallet部分，以为作为VC（可验证凭证）部分的验证套件，信息在DWNs（去中心化节点）之间安全流转，而这些统一构建成DWP（去中心化网络平台）。

在网络之上，是DWA（去中心化网络应用），而比特币、闪电网络以及Circle构成了真正的加密货币流转模块，可真正将加密货币和网络通讯链接去起来，这也是Web 5结合二者的真正途径。

DID：前端匿名化

去中心化标识符是W3C（万维网联盟）的一个国际标准，用于由个人创建、拥有和控制的标识符，不依赖中心化实体。

在TBD的设计中，DID可以被嵌在加密钱包之内，作为个人用户进入去中心化网络的前端和入口，如同Vitalik所言，对DeFi前端进行KYC缺乏意义，既无法组织真正作恶的黑客和项目方，也会让用户感到不安，去中心化应该用去中心化的方式组织安全措施。

钱包是作为实现DID功能的最好载体，价值和信息的同一化是Web 5的典型特质。而前端匿名化才能从源头确保整个网络的去中心化，从这个角度而言，DID是跟Tor、Linux和比特币精神一脉相承的继承者。

VC：去中心化凭证

VC（Verifiable Credentials）可验证凭证是在接受信息时，需要网络节点对其进行核验，确保传递的是正确的原始信息。

凭证在现实生活中并不少见，如我们常见的身份证、驾驶证都是其中的代表，而在目前的互联网中中，主要使用的电子签名、CA证书等，比如网站、邮箱在大规模使用，以此来证明自身的可信性。

这些证书的主要问题并不在于安全性低，而是中心化认证机制对个人的控制权，比如Zlibrary在11月4日被美国政府封禁，则基本丧失在互联网的链接能力，与之对应的还有科研文献分享平台Sci-Hub等多家网站。

VC本质上起到的依旧是验证信息，但是是通过去中心化的节点来实现，而非某个中心化实体。

从流程上而言，用户在使用DID验证自身后，VC会验证该持有人发出的信息本身，以此做到信息发出者和信息的同向匿名。例如，近期摩根大通进行了DeFi的操作，就使用了VC以使仅获得许可的地址访问自己的钱包。

DWN：信息安全流转

DID和VC是信息发出的准备阶段，而DWN（Decentralized Web Node）去中心化网络节点，则负责信息的传递过程的安全，将三者统合而构建为所谓的DWP（Decentralized Web Platform）去中心化网络平台。

本质上，DWN是一个去中心化的数据存储和消息中继节点，用户可以有多个节点，可以在匿名状态下保持信息在节点内的安全流转。

DWA：未来应用另一种方式

而在网络层之上，也会存在对应的应用组织方式，比如TBD定义的DWA（Decentralized Web App），本质上是对现有网页的去中心化改造。

如果网络所依赖的前后端都是去中心化的，那么网页自然也是去中心化的，这里的问题在于未来的应用究竟是何种模式？

  • DAPP，依赖于公链或者各类Layer 2

  • DWA，依赖于Web 5所构建的去中心化网络

  • 手机APP，依赖于手机等硬件终端，但会做去中心化实现，如Solona手机或者iOS支持NFT标准的APP

  • VR APP，依赖于Meta等研发的VR终端，并在其上运行

目前的趋势是DAPP火热在加密世界，手机App用户量最广，VR设备主要使用场景被限制在游戏领域，而DWA仍在起步阶段，只能说这会是一个长期的竞争格局。

Web 5会是去中心化社交的未来吗？

比特币可以不属于任何人，但一定要属于Jack Dorsey，多年来他以一种宗教般的热忱推广比特币，并且是为数不多的建设者，勤勤恳恳为闪电网络、支付进行商业建设和技术开发，甚至Web 5也选择了比特币作为价值层。

纵观互联网的发展史中，GUI（图形化界面）取代了终端命令行，移动互联网取代了桌面浏览器，当然这种取代，更像是包裹其中而非全然替代，只是在开发模式和商业化进程中谁占据主流的问题。

那么我们要回答的问题是接下来应该如何组织社交，目前有三种流派和范式：Lens Protocol的复刻模式，Twitter/Mask的改造模式，以及最为底层的Web 5原生模式。

Twitter坐拥4亿用户，一举一动都会对社交模式产生巨大影响，而Lens Protocol是通过把现有的社交模式重新用区块链的方式再实现一遍，而TBD推出的Web 5，是用去中心化的方式思考下一代网络。

这个逻辑可以解构为，先建设网络本身，进而会有原生协议诞生在其上，从而完成对既有网络的兼并，这也是Web 1 – Web 2 – Web 3发生的故事，目前的以太坊在新路线架构图之后，瞄准的就是10万+ TPS，以支持加密原生应用的日常使用。

当然，在目前的实际状况下，Web 5本身仍在建设中，其上的应用还未真正运行起来，但是纵观历史趋势，这依然是个值得重视的方向，甚至会从根本上改变目前的加密世界。

目前的公链及Dapp竞争，更强调金融及安全，对网络本身的作用，如通信、链接等支持较少，TBD的Web 5发展在此方面进行了相当程度的探索，这也会是一个长期的方向。
