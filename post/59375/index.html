专为以太坊上的金融应用场景设计、基于混合zk-OP架构的Layer 2 网络Layer N 获得了Founders Fund领投的500万美元的种子轮融资。

尽管链上金融生态已经发展出较为丰富的场景，但总体来说，去中心化交易总量仅占所有加密货币交易量的 1%。现有的链上解决方案面临一些限制，在延迟、性能、成本或可组合性方面难以与线下金融交易所相比。

这周三，由 PayPal 的创始人 Peter Thiel 支持的传统风投机构 Founders Fund 投出了近期的第二笔加密货币领域的投资，一个专门为以太坊上的金融应用场景设计、基于混合 zk-OP 架构的 Layer 2 网络 Layer N 获得了 Founders Fund 领投的 500 万美元的种子轮融资。

Layer N 宣称其特点在于引入了一个能够容纳 n 个汇总的共享环境，每个汇总都有自己的超高性能执行环境和近乎即时的交叉汇总消息传递功能，可以实现无缝的可组合性。基于此，Layer N 能实现高吞吐量、低延迟和高可组合性（资本效率），以支持能与中心化交易所速度相匹敌的链上金融交易活动。

图源：Layer N

Layer N 的模块化架构

Layer N 为其每个汇总实例引入了一种新颖的模块化 Layer N 架构，该架构能够针对整个堆栈中的特定任务利用高度优化的解决方案，并且能够轻松即插即用不同的组件以实现最大的可组合性。

图源：Layer N

混合 zk-OP 架构

Hybrid Rollup 是目前较受关注的以太坊扩容方案，通过混合 Optimistic Rollups 与 ZK Rollup 通过结合两种技术，可最大限度地优化产品功能。Layer N 使用的架构就是与 RiscZero 合作开发的可支持混合 zk-OP 架构的零知识欺诈证明系统。

在设计 Rollup 时，设计者需要考虑如何在满足安全性和信任性问题的同时提高 Layer 1 的可扩展性。目前，完整的 ZK Rollup 面临这缓慢的证明时间和昂贵的计算成本的限制。Layer N 采用了一种新颖的方法，即利用零知识证明和 RISC Zero 的零知识虚拟机，通过采用混合 zk-OP 架构，只需要在存在欺诈可能性时生成证明（ZKFP）。

图源：Layer N

对于传统 optimsim 汇总解决方案来说，用户发现欺诈并提交欺诈证明后进行提款的时间通常在 7 天左右，这在金融活动中会成为一个影响效率的限制问题。尽管 ZKFP 并没有完全解决这个问题，但 ZKFP 不是 ETH 上冗长的来回二分协议，而是允许单个来回交易来证明/反驳欺诈。通过其「一次性」方法，它们能够大幅减少提款时间。

因此，基于事件驱动的混合 zk-OP 架构可以实现类似于 web2 的性能和比传统 optimsim 汇总解决方案更短的防欺诈时间。

数据可用性

除了成本高以外，汇总解决方案的另一个重要瓶颈是块空间问题。传统汇总方式效率低下是因为使用以太坊的区块空间来存储汇总数据会导致汇总交易价格非常高，面对区块空间的日益激烈的竞争以及可用区块空间的稀缺，Coinbase 或 Binance 等复杂应用程序就很难完全迁移到以太坊上。

Layer N 选择 EigenDA 来实现高效的数据可用性架构，Eigen DA 是 EigenLayer 开发的产品，是以太坊超大规模数据可用性层，旨在实现更便宜、更一致的费用，以及更高的数据可用性带宽。与其他链下 DA 解决方案不同，在 Eigen DA 中，以太坊验证器通过重新抵押（restaking）继续保护数据，可以使「链下 DA 风险」得到了缓解。现有的区块链无论验证者的数量如何都具有固定的容量，而 EigenDA 的容量随着验证者的数量而扩展。

这项创新有可能克服区块链固有的传统带宽限制，使 Layer N 能够水平扩展并保持较低的交易费用。Layer N 称利用 EigenDA 后交易费用可以大幅降低至以太坊主网上的约 1/1000。与 EigenLayer 合作, Layer N 可以利用 ETH restaking 的去中心化网络提供一个用于排序交易的抗审查、可自我管理的 MEV 环境。通过分配排序节点的随机子集来对不同的交易子集进行排序，以实现网络的水平扩展。

去中心化排序器

由于集中式定序器模型启动速度快且易于实施，因此 目前 Layer N 的启动将通过单个定序器进行。在未来，Layer N 计划去中心化其定序器集。

共享定序器使 rollup 生态系统的流动性碎片化问题有了解决的可能性。通常分散在不同链上的 DAPP 需要在拓展性能和用户体验之间做权衡，还要考虑增加流动性的需求与同其他 DApp 合作、以实现生态系统发展的需求之间的矛盾。

通过 Layer N 构建的汇总解决方案可以共享相同的定序器，确保它们的状态逐块地一起解决。这样，由以太坊保护的本机交叉汇总通信就可以实现，而无需外部桥或包装的代币系统。EVM Rollup 技术可以让 DApp 在 Layer N 生态系统中以原子方式直接与 Nord Rollup 交互，有效消除第三方桥接的风险。

此外，IRC（Rollup 间通信协议）用来连接所有 Rollup 实例的统一消息传递层，使每个 Rollup 能够无缝地将消息传输到其汇总并桥接流动性。共享定序器支持的 IRC 可以解决现有的桥接安全和用户体验问题，最终用户不会察觉到正在操作哪个 Layer N 汇总，在使用应用程序时不必考虑桥接的复杂性。

量身定制：帮助交易平台更高效

在 Layer N 上，每个汇总解决方案都是针对特定目的量身定制的。例如，Nord (订单簿汇总) 进行了针对交易场景的优化，能够反映领先金融交易所的架构。在合约中，N-EVM (EVM rollup) 针对智能合约以及与其他汇总解决方案的无缝可组合性进行了优化，可以使开发人员和团队能够始终找到适合其特定需求的环境。

Nord 是 Layer N 计划推出的第一个解决方案，是一个专为满足链上交易需求而构建的基础设施。在 Layer N 的介绍中，Nord 具有定制的高效 Rust 订单执行环境。「比现有扩展解决方案高 10-100 倍的吞吐量，以及实现亚毫秒级的延迟」，可以处理复杂的交叉保证金和交叉抵押，同时保持与 N-EVM 等智能合约层的完全可组合性以利用其流动性。

基于这些特点，Nord 可以优化传统链上交易基础设施的性能，为用户提供在中心化交易所获得的功能和用户体验，构建高效的跨保证金、跨抵押品和高效的清算引擎等系统。Nord 的链上基础设施与现有的中心化交易所相当。Romanov 表示，Node 将允许去中心化交易所推出自己的汇总，以实现类似 Binance/Coinbase 的性能，并补充说「没有其他产品可以与之相比（延迟和吞吐量方面）。」

Layer N 还计划推出公共 EVM 汇总方案 N-EVM。N-EVM 基于 reth 构建，针对智能合约应用程序和无缝汇总通信进行了优化，从而实现了一流的性能以及与 Nord 和其他 Layer N 汇总及其流动性的深度可组合性。

团队及投资情况

据 BlockBeats 了解，此前 Layer N 曾考虑为 Solana 构建扩展网络，但在去年 11 月加密货币交易所 FTX 崩盘后放弃了。当时 FTX Ventures、Solana Ventures 等都是 Layer N 的投资者，但该轮融资在 FTX 崩盘后被取消。

本周三，Layer N 宣布完成 500 万美元种子轮融资，由 Peter Thiel 支持的 Founders Fund 和去中心化投资集团 dao5、Kraken Ventures 和加密货币基金 Variant 前合伙人 Spencer Noon 共同领投。

Layer N 联合创始人 Dima Romanov 和 David Cao 表示，现在的首要任务是吸引加密货币交易者和去中心化交易所等协议，「Layer N 可以提供以太坊作为其底层区块链的相对安全性和可靠性」。在他们看来，Layer N 的长期目标，是能够允许币安和 Coinbase 等中心化交易所在区块链之上构建去中心化、非托管产品。
