我们需要更复杂和动态的元数据让 NFT 在未来开放和去中心化的金融系统中发挥更大的作用。

原文：《dNFT, Metadata-Enabled》by Nico Pei，Blockchain Capital

编译：饼干，链捕手

背景

提到 NFT，大多数人首先想到的可能仍然是 Bored Ape 或 Punks——从 2020 年的一文不值上升到价值数百万美元。从 2021 年初到 2022 年，耐克、LVMH、 Tiffany & Co 要么启动了自己的 NFT 项目，要么收购了他们的 NFT 合作伙伴。

随着 NFT 交易量的迅速增长，我们已经在 NFT 领域探索一年半的时间，好奇的人可能会问：这项技术还能实现什么？

让我们退后一步，思考一下 NFT 真正在链上做什么。可替代代币将用户映射到余额，而 NFT 将代币映射到用户（用户 → 余额 vs NFT_id → 用户）。因此，如果要评估 NFT 的潜力，正确问题是：我们有多少金融工具是不可替代的？

不可替代性

为了评估哪些资产类别是不可替代的，让我们看一下大型公司——沃尔玛的资产负债表。

明确地说，可替代性是指资产的可互换性——我的比特币和你的比特币一样。如果严格定义，不可替代性是指众多资产类别之一。值得注意的是，在真正不可替代资产（例如一件艺术品或一栋房子）和真正可替代资产（例如 BTC/USD/苹果公司的普通股）之间存在一个完整的区间范围。

一些资产/负债可以是不可替代的，因为它们是由到期日、利率、执行价格、基础资产、交易对手等定义的。但是，这些资产/负债的部分/份额也可以交易，将为资产带来流动性，因此可以对这些原本纯粹不可替代的资产进行更有效的价格发现。通常，它们通过银行和场外交易进行协商和构建。它们的流动性通一般低于可替代资产。不可替代资产的明显例子可以是房地产、场外交易期权、场外交易期货、两方协商的奇异衍生品、艺术品和收藏品等。

另一方面，可替代资产是指由标的资产/现金和现金等价物定义的资产/负债。它们通常由政府、大型公司发行，或者是现金和现金等价物抵押发行。可替代资产通过交易所交易，但也可以场外交易，并且由于做市商的服务而具有显着的流动性。

世界上有这么多不可替代的金融工具，在一个开放和去中心化的金融系统中，如何在链上表示这些资产？NFT 是解决方案吗？

简短的回答是肯定的，但我们需要采用更复杂和动态的元数据形式。

NFT 作为数据容器

NFT 本质上由两个组成部分组成：TokenID 和元数据。TokenID 是代表所有权的唯一标识符。元数据是与 NFT 相关的对象。元数据可以包括名称、描述、存储媒体文件的 CID 或任何其他可以是整数、字符串或对象的属性。CID 下的媒体文件存储 NFT 的内容——它可以是图片、视频、网站、一段代码（例如，生成艺术）或任何任意数据。

最容易理解的元数据示例是当你在 OpenSea 上查看 NFT 时，OpenSea 页面上显示的名称和描述不是由创建者或 OpenSea 团队手动指定的，而是从元数据中查询的。

NFT 可以以智能合约中指定的方式与协议交互，元数据包含定义资产的所有关键数据。因此，通过增加更多的可编程性和功能，NFT 将超越艺术品和收藏品的形式并改变我们使用元数据的方式。

大多数现有 NFT 的元数据包含指向媒体文件的名称、描述和 CID。然而，由于上述元数据的灵活性，元数据可用于指定很多东西：到期日、标的资产、行权价、收益、交易对手、固定利率、折旧率、游戏角色级别，可访问性等。

元数据还可以包含功能属性，例如指定游戏内的角色级别，或记录会员通行证上二维码的扫描次数，或指定财务信息中不可替代的详细状况。

例如，Uniswap v3 使用 NFT 来代表流动性提供者 (LP) 的头寸。NFT 在其元数据中指定 LP 的费用等级、范围、poolID、头寸大小和赚取的费用。下面是 Uniswap v3 合约的代码，它定义了代表 LP 头寸的 NFT 元数据。

Uniswap v3 NFT 开启了 NFT 超越艺术品和收藏品的世界：NFT 可以成为不可替代金融头寸的收据。NFT 作为流动性池中提供流动性的收据以及提供的流动性所赚取的费用。当用户出售 NFT 时，他们就是在出售自己的头寸。当用户将 NFT 发送回 Uniswap 协议时，他们可以收回自己所拥有的东西——他们提供的流动性资产和在整个周期中赚取的费用。

对于静态 NFT，元数据是在生成时定义的，之后不可更改元数据。无聊猿 NFT 仅包含无聊猿图片且无法更改。拥有静态元数据非常适合艺术品和收藏品 NFT。

然而，为了在艺术品和收藏品之外建立一个强大且开放的 NFT 系统，元数据中的许多属性都需要更新。例如，在流动性提供周期中赚取的费用会增多，头寸的基础资产可能会发生变化，玩家在游戏中的表现会影响他们的角色水平，持有人可能希望通过支付溢价来推迟到期日。

动态 NFT 在这里发挥作用。

介绍动态 NFT (dNFT)

动态 NFT (dNFT) 也被称为 NFT 2.0，其元数据可以根据智能合约中写入的指令和外部条件或动作而改变。动态 NFT 具有唯一且静态的 tokenID（所有权）和动态元数据（或其他动态元素，例如某些外部条件触发铸造/燃烧）。唯一的 tokenID 用作 dNFT 的标识符，而它们的动态元数据可以根据外部条件发生变化，包括链上或链下事件、动作、结果或数据更新。

dNFT 本质上需要两条信息来了解如何更新元数据：

1. 向底层 NFT 提供关于何时以及如何更改元数据的说明

2. 访问相关的外部数据源

指令写在智能合约中，数据源来自链上数据或由防篡改预言机提供的链下数据。

dNFT 可以根据外部条件以各种方式更新。这些升级包括描述更新、财务头寸大小和流动性范围更新、隐藏特征解锁和升级、图像更改、实用程序更改和授予访问权限等，其中大部分的类别是财务头寸 NFT、链上游戏和用于票务和令牌门控内容和社区的实用 NFT。

除了元数据变化之外的动态元素也可以存在，这使得 dNFT 可以根据特定的外部条件进行铸造，例如在现实生活中的球队赢得比赛时铸造某个 dNFT。

dNFT 还可以包含用户交互生成的“隐藏特征”。例如，扫描二维码后可能会禁用转账功能，这对于将 NFT 用作门票非常有用。

演示：dNFT 的应用场景比 ERC-20 和静态 NFT 更多

Solv 协议——在金融凭证中应用的 dNFT

Solv 协议使用基于 ERC-3525 标准的 dNFT 版本，被称为半同质代币，允许任何有特定发行需求的用户，无论是希望筹集资金的协议还是想要销售结构化产品的机构，以灵活的配置买卖衍生品。

Solv推出了三种产品：归属凭证、可转换凭证和债券凭证。归属凭证允许 ERC-20 代币锁定在已发行的 NFT 中，并由所有者归属。可转换凭证和债券凭证允许协议发行具有固定收益率或基于看涨期权的零息债券。

使用 dNFT 来表示这些资产具有许多优点。

首先，它带来了更好的开发者体验。如果使用 ERC-20 范围代币而不是 dNFT，则每次更改参数都需要部署新的 ERC-20 合约。同时，单个 NFT 合约（集合）可以包含许多具有不同参数的 NFT（针对不同标的和范围的不同范围代币）。与 ERC-20 代币不同，具有可配置参数的 dNFT 允许无代码创建和部署，从而显着改善开发者体验。

其次，它为用户创造了更好的体验。这些 dNFT 的 SVG 文件清楚地说明了有关头寸的所有信息，并且可以在所有 NFT 市场上直接交易。SVG 图像不是静态的，而是基于链上数据和预言机动态生成的，保证了显示信息的真实性。此外，就像一个帐户一样，ERC-3525 代币也可以接收、存储和发送代币。这使得集合中的每个 NFT 都可以被细分——就像使用 ERC-20 代币一样，增强了资产流动性。

Swell Network — 用于流动性质押的 dNFT

Swell Network 允许用户通过 swNFT 质押 ETH 并接收 swETH。虽然 swETH 是质押 ETH 的流动版本，类似于 Lido 的 stETH，但 swNFT 包含用户的 swETH，可用于进入金库以从质押的 eth 中获得最大收益。

同时使用 swETH 和 swNFT 有很多好处。swNFT 允许用户在不关闭金库头寸或取消 ETH 质押的情况下交易 ETH 质押资产和扩大金库头寸。对于金库中的大额头寸，最好购买现有头寸而不是开设新头寸，因为所有金库头寸都在相互竞争以获得质押奖励。虽然 swETH 使 swETH 和 ETH 的价格比率产生差异，但 swNFT 使 ETH 和 swNFT 的价格比率差异不明显，在保持市场相对稳定的同时降低了流动性。

游戏中的 dNFT

dNFT 可以做的远不止 ERC-20 代币或静态 NFT，我们开始在游戏中看到一些创新方法。在某种程度上，游戏是开发者探索尖端技术的低风险环境。我真的很欣赏这句话：

“游戏是新兴技术的领先指标，因为它们是一个技术要求较高但风险相对较低的环境，可以在技术平台生命周期的早期探索可扩展性和可用性问题：游戏中的安全性和合规性问题相比“严格规范”的商业或金融应用程序小很多，通常迭代循环更快。因此，尽管对这方面了解较少的人可能会认为“仅”有一堆游戏的平台很无聊，但有眼光的人会认识到这是一个有价值的试验场。有很多交互模式和习惯源于早期的手机游戏，如 Doodle Jump、Cut the Rope 和 Angry Birds？”

——Gubsheep，黑暗森林的创造者，《最强的加密游戏论文》

Wolf Game

Wolf Game 是 2021 年末流行的 NFT 游戏。它实现了动态 NFT，可以根据智能合约中指定的规则对其他 NFT 窃取参数或征税。

Pak合并（灭绝游戏）

Merge 是匿名设计师 Pak 的 NFT / 游戏项目。如果这些 NFT 在同一个钱包中，它们会自动合并在一起。两个质量合并在一起成为一个更大的质量。这些变化反映在 NFT 的视觉和元数据中。一旦合并，质量就无法分离。

存在的问题

元数据的显示

在 Uniswap v3 NFT 的早期（2021 年 5 月），经常会发生一些不幸的事件，比如有人以极低的价格出售大量 LP 头寸。这是因为有些人不知道这些 NFT 代表了他们的流动性资产。同时，这也是由于当前 NFT 市场上的元数据显示不适合显示不可替代的财务头寸，也不适合复杂的基于实用程序的 NFT。

随着我们在 DeFi 和游戏中看到越来越多的 dNFT 用例，对于能够显示特征的元数据的需求将大幅提升。然而，不同类型的 dNFT 需要显示不同的元数据，这通常涉及一些围绕元数据的计算以及从预言机获取与价格相关的数据。因此，我们可能会看到 NFT 市场的市场细分。这也可能会走类似电子商务的路线——一个平台从一个占主导地位的单品类商店开始，由于客户的网络效应（NFT 市场的流动性）逐渐扩展到所有品类。

元数据与链上数据的分离

如果 Uniswap v3 LP 头寸以 x、y 和 z 价格进行交易，并且持有者根据市场情况多次更新价格范围，仅拥有 x、y 和 z 价格的时间戳数据是不够的——我们需要知道x、y 和 z 价格如何映射到持有者选择的不同范围（即将链上交易数据与动态元数据相关联以提取洞察力）。目前，链上数据和动态元数据之间没有连接。很多分析也因此无法进行。

此问题的其他示例：

1/ 我们将 dNFT 用作游戏中的角色，可以根据游戏中的活动进行升级或降级。等级信息（Level 1 - Level 10）存储在链上，并且在 NFT 交易的同时不断更新。如果没有动态的元数据快照，就不可能获得每个级别的平均价格。

2/ 健身房的会员卡有 x 次体验限制。每次使用时，元数据都会通过从 x 中减去 1 来记录使用情况。x 存储在链上，并在会员卡的使用情况不断更新。为了评估市场利益的波动，我们想要获得剩余 x 次访问的会员卡平均价格。由于 x 的每次变化都缺少元数据快照，因此不可能将交易数据与 x 关联起来。

dNFT 目前在二级市场上的交易不多，因此问题不太明显。但是，如果 NFT 变得更加复杂并且动态元数据论点成立，则必须解决这个问题。因此，我们需要在每次重大变更之后存储元数据的快照，并使元数据快照易于查询。

安全性和成本权衡

从安全的角度来看，拥有动态元数据肯定会带来更多挑战。由于元数据定义了资产，因此保护元数据与保护所有权一样重要。最常见问题是在哪里托管元数据。在中心化服务器上托管元数据更便宜，但会阻止元数据被公开访问。将元数据托管在链上可能会很昂贵，因为元数据的每一次重大变化都会产生 gas 费用。此外，所有形式的描述性属性都需要存储在链上（数字属性除外），这方面的费用都很昂贵。

结论

1/ 作为数据容器的 NFT 远不止 ERC-721 类型的艺术品和收藏品或 ERC-20 代币，其神奇之处在于复杂和动态的元数据。

2/ 金融系统中的很多东西都是不可替代的，而 dNFT 可以在这些不可替代的资产和负债的代币化方面发挥至关重要的作用。

3/ 我们需要能够更好地满足复杂元数据显示的市场，以及支持元数据和链上交易数据分析的工具。
