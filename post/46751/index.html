原文：《8 things to be excited about in web3 in 2023》by Nader Dabit

翻译：Biteye 核心贡献者 双木

编辑：Biteye 核心贡献者 Crush

Web3 已经取得了长足的进步，2023 年即将发生了很多值得兴奋的事情。今天有哪些工具、产品、协议和现实世界的用例已经准备好了呢？

在一个工作 10 年以上的开发者看来，今年有 8 件令人兴奋的事情。

01、永久存储

在 AWS，S3 是最早的也是最常用的服务之一。Web3 存储解决方案通过引入不变性和持久性，使托管存储更进一步，这是传统存储解决方案无法获得的。

这就是为什么像 Instagram 这样的公司选择 Arweave 来构建其他集中式架构无法实现的功能。

随着 Arweave 上协议更新，如 Bundlr、EXM 和 Warp，用户和开发人员的进入门槛已经大大降低，因为现在 Arweave 上有高质量、高性能的应用程序。

02、消息传递

作为开发人员，Web3 的一大价值主张是共享数据和基础设施的理念。

不幸的是，当我在 2021 年 4 月加入这个领域时，除了 DeFi、NFT、DAO 和数字支付之外，实际上没有太多可能。

虽然这些都是非常有趣的用例，但数字支付是唯一可能吸引世界上「普通」人的东西。

快进到今天 - 去年推出的最令人兴奋的新协议之一是 XMTP，一种用于实现加密和安全消息传递的协议。

如果您的应用程序需要消息传递，或者您可能只是想构建一个新的消息传递应用程序，它具有一些其他地方不存在的很酷的新功能，您使用 XMTP 不仅可以享受高质量的服务，您还继承了使用 XMTP 构建的所有其他应用程序的用户。

您不再需要引导一个全新的用户群，使用 XMTP 构建的任何应用程序中的所有用户都可以选择您的应用程序、登录，并可以从他们的其他应用程序继续他们的对话。

这已经在 Lens Protocol 生态系统和其他地方发生并获得动力。

这种由共享的、公共的、不可变的数据和基础设施支持的价值主张是 Web3 最强大、最被低估的东西，也是使用这些工具和技术进行构建的原因。

03、账户抽象（AA）

可扩展性、可访问性和用户体验是采用区块链技术的最大障碍。

帐户抽象有助于直接解决其中两个挑战（可访问性和用户体验），并迅速成为协议路线图中最优先考虑的功能之一，也是区块链社区中的热门话题。

帐户抽象启用以下功能：

1. 社交、电子邮件或任意帐户登录

2. 无 gas 元交易

3. 批量处理多个事务

4. 任意代币作为 gas 支付

5. 多重签名安全

6. 社交恢复等

像 Fuel 这样的协议正在从头开始构建，将帐户抽象视为第一等公民对待，而 EIP4337 允许现有协议将帐户抽象构建到基于 EOA 的协议中。

Biconomy 多年来一直致力于应对这些挑战，并提供易于使用的 SDK 和 API 以立即开始构建，其帐户抽象的实现基于 EIP4337。

随着无 gas 交易和越来越便宜的执行环境发展，开发人员和团队可以开始考虑像对待传统云基础设施一样对待区块链基础设施。

我们可以补贴交易，这将消除绝大多数人进入的巨大障碍 – 让他们不仅学会在正确的网络上使用正确的交易所交换正确的代币，他们还可以像使用任何其他应用程序（Web2 程序）一样，简单轻松使用我们的应用程序。

04、更好的执行环境和 L2

不久前，几乎所有网络的速度、成本、出块时间和最终确定时间都非常缓慢和昂贵，以至于大多数用例都被禁止，并且现有用例的用户体验可以说是低于标准（与集中式基础设施相比)。

今天，有些协议要么已经在生产中使用，要么将在未来约 6 个月内上市，它们提供与传统「Web2」应用程序相同或更好的用户体验，让亚秒级确认和低于 0.001 美元交易成本成为可能。

Arbitrum Nova 将其技术发展的最新步骤 Nitro 与数据可用性委员会相结合，在去中心化和安全性之间提供智能权衡，为游戏和社交媒体应用程序等用例提供令人信服的解决方案，例如 Reddit 将其用于社区积分系统。

Fuel bas 从头开始构建了一个全新的执行环境和开发人员堆栈，并且是第一个模块化执行环境。使用基于 Fuel 的 SwaySwap，从长远来看已经比我使用过的任何传统银行应用程序都更好。

作为开发人员，拥有快速且廉价的执行环境以及我在区块链世界中见过的最高质量的 DX（包括他们自己的基于 Rust 的语言 Sway，以及一套高质量的开发工具）可能是最令人兴奋的构建的地方，特别是因为它们不仅将与以太坊集成，而且可能与 @celestiaorg 等模块化堆栈的其他部分集成。

Polygon 已经做了很大，其他人很难跟上。不仅是他们交付的大量高质量 Web3 基础设施，还有无数公司，如耐克、Instagram、星巴克、Reddit、我们（Lens Protocol）和其他公司都选择了在那里建造。

还有很多事情正在发生，这些只是我个人感兴趣的一些事情。

05、更好的抽象

如果您是一名开发人员，您肯定听说过或可能使用过 Vercel。

Vercel 广受欢迎，因为它提供了可用于构建和部署 Web 应用程序的最佳用户体验以及 Serverless 等功能，这些功能非常强大，衬托出其他云和托管服务提供商的劣势。

（译者注：Serverless，又叫无服务器。Serverless 强调的是一种架构思想和服务模型，让开发者无需关心基础设施（服务器等），而是专注到应用程序业务逻辑上。）

Arweave 和 IPFS 等去中心化基础设施支持一些相同的功能，但它们提供的服务与 Vercel 并不完全相同。

借助 Fleek XYZ、Akord 和 EXM 等平台和服务，开发人员可以构建和部署应用程序，并在这些协议上利用存储功能 + Serverless 功能，而无需处理 Tokens，而只需像过去那样使用 API 密钥。

价值主张是您继承了传统基础设施的所有用例，但具有不变性和 Arweave 的永恒性。

06、社交图谱

截至目前，全球约有 49 亿社交媒体用户。据预测，到 2027 年全球将有 58.5 亿社交媒体用户。

社交功能几乎出现在我们今天使用的每个应用程序中。

在创建了一个教程视频教开发人员如何使用 Lens 进行构建，同时意识到了 Web3 社交可能成为大规模采用的关键之后，我在几个月后加入了 Lens Protocol。

与无服务器基础设施和托管服务（如 Twilio 和 AWS、GCP 等提供的服务）使开发人员能够快速构建可扩展的应用程序而无需管理后端基础设施一样，Lens Protocol 提供了用于构建具有社交功能的可扩展应用程序的 Web 基础设施。

他们不必构建、维护和迭代自己的后端和 API，而是可以专注于构建 Web 或移动应用程序，同时 Lens 团队继续迭代和改进后端基础设施。

除此之外，当他们在 Lens 上推出他们的应用程序时，他们继承了十几万（未来将是数百万）用户和生态系统，而不必从头开始启动一切。

将现在推向市场的改进用户体验与现实世界的用例（社交和消息传递）相结合，为开发人员打开了无数机会的大门，以构建独特和高质量的体验，而这些体验实际上无法用集中式技术构建，而且重要的是，这一次不是严格意义上的金融用例。

07、女巫抵抗

在这个空间中构建的其他重大挑战之一与女巫攻击和解决女巫攻击有关。

现在存在解决此问题的多种选择，最著名的是 Gitcoin Passport 和 Worldcoin。

08、其他工具和协议

Huddle 最初是作为一种产品，但此后也向开发人员开放了他们的协议，以构建类似 zoom 的体验。

借助去中心化视频基础设施网络 Livepeer，您可以创建在您的应用程序中实现视频功能和实时视频流的应用程序。

WeaveDB 是一个基于 Arweave 的 Firestore 类型的 NoSQL 数据库。

还有很多我没有涉及的事情，但对于一个精力有限的人来说，这些是我今年感到兴奋的事情。

2022 年是艰难的一年，很难预测接下来会发生什么，但对于 builders 来说，进入这个领域来，还没遇到过比现在更激动人心或更合适的时刻。
