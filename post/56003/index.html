一、项目简介

Linea 是一种创新的区块链解决方案，将强大的零知识证明功能与完整的以太坊虚拟机（EVM）等效性相结合。它允许开发者构建可扩展的去中心化应用（dapp）或迁移现有的 dapp 而无需更改代码或重写智能合约，极大地简化了开发过程。Linea 由 ConsenSys 构建，属于 zkEVM 众多类型中的 Type 2 ，它利用 zkEVM 的可扩展性属性来提供更快的交易时间和更高的吞吐量。zkEVM 是以太坊虚拟机 (EVM) 的零知识证明 (ZKP) 版本，可确保交易和智能合约执行得到验证，而不会泄露任何敏感信息。这不仅增强了安全性，而且还实现了更具可扩展性和更高效的平台。

2022 年第四季度，ConsenSys 推出的 Linea 内部测试版，为有限用户提供了早期访问权限，处理了超过 350, 000 笔交易，并部署了各种去中心化应用程序。测试网允许 Solidity 开发人员构建、测试和启动 DApp，同时大规模测试 zkEVM。

2023 年 3 月 28 日，Linea 上线公共测试网，发布 zkEVM rollup，以便对 Layer 2 扩展技术进行压力测试，在公开测试网时期，约有 550 万个独特的钱包执行了超过 4600 万笔交易。

7 月 18 日，Linea 上线主网 Alpha 版本，截至 7 月 24 日，Linea 主网拥有超过 100 个生态项目，已跨入 10, 921 枚 ETH，交易笔数达 64, 604 笔，交互地址数达 58, 344 。

根据 Linea 全球商务负责人 @hotpot_dao 介绍，主网上线后，Linea 计划着重发展 Layer 3 和 Appchain 两个方向，以满足游戏项目方对吞吐量和点对点交易的需求，同时进一步推进 Multi Prover 的合作。

二、背景团队

Linea 由 ConsenSys 公司研发设计并负责运营，ConsenSys 是一家成立于 2014 年领先的以太坊软件公司，业务遍及全球，聘请了顶尖企业家、计算机科学家、协议工程师、软件开发人员和企业交付专家。作为区块链技术领域最大和最基础的实体之一，ConsenSys 的全球人员、项目和公司网络正在为决心利用以太坊力量的企业和政府构建区块链行业的开发工具、去中心化应用程序和解决方案。该组织在 2018 年被《纽约客》称为“以太坊社区最著名、最普遍的去中心化应用程序开发者和推动者”。目前的的产品套件由Infura、Quorum、Truffle、Codefi、MetaMask和Diligence组成。

Joseph Lubin 是以太坊的联合创始人和 ConsenSys 的创始人，Lubin 在加拿大多伦多出生长大，毕业于普林斯顿大学，获得电气工程和计算机科学学位。他曾在 Tomandandy Music 的普林斯顿机器人实验室开发自主音乐作曲工具，并在私人研究公司 Vision Applications Inc. 开发自主移动机器人。Lubin 专注于密码学、工程和金融的交叉领域。

三、ZK EVM Type 解析

前文提到 Linea 属于 zkEVM 四种类型中的 Type 2 ，那么这四种类型分别是什么呢？它们各自具备哪些特点呢？Vitalik 在他的博客文章中对基于与以太坊兼容性的 zkEVM 类型进行了概述。

在阅读下文之前，需要知道的是：**zkEVM 越接近以太坊的功能，生成 zk 证明的速度就越慢，成本也越高。**另一方面，如果一个区块链或 Rollup 更符合以太坊的规范和标准，那么开发人员就更容易在其上构建应用程序并集成到以太坊生态系统中。因此，与以太坊高度兼容的区块链可能具有竞争优势。

第一类 （Type 1)：完全等同于以太坊的 Zk rollups，在所有部分（包括哈希、状态树、交易树、预编译或任何其他共识逻辑）中精确地复制以太坊。目前为止，实际上还没有任何 zkEVM 能够实现与以太坊的等效性，但 Taiko 的最终目标就是实现以太坊等效，所以理论上可以把 Taiko 划为此类。

第二类（Type 2)：完全等同于 EVM 的 Zk rollups 力求与 EVM 等效，但并非完全等效于以太坊。它们与现有应用程序完全兼容，但对以太坊进行了一些微小修改，以使开发更容易，并且可以更快地生成证明。Linea 目前属于此类。

第三类（Type 3)：几乎等同于 EVM，为了使证明生成更快，牺牲了更多元素和等效性。Polygon zkEVM 和 Scroll 目前属于此类。这个类别通常是进入第二类的跳板。

第四类（Type 4)：高级语言等效，意味着它们都接受用 Solidity 编写的智能合约，然后将其转换为另一种自定义的、适用于 Zk 的语言。zkSync 和 StarkNet 属于这个类别，尽管 zkSync 可能会随着时间推移添加对 EVM 字节码的兼容性，并最终向更高的类型 Type 3/Type 2 转移。

需要注意的是，这些不同类别并没有高下优劣之分，主要根据不同技术选择和证明生成速度来进行开发取舍，它们之间也是可以互相转化。

作为 Type 2 的 Linea，它与 EVM 完全等效，与 Type 3 和 Type 4 的项目不同，如果项目方有一个在主网上的 Dapp，无需任何代码更改就可以直接迁移到 Linea，只需进行一键转换即可使用 Linea 的 Layer 2 协议。然而，如果要转移到 Type 4 的 StarkNet，就需要一个编译器，并增加一个步骤，这可能会引入安全性问题并带来额外的开发成本。

而就目前的技术而言，Type 2 相对于 Type 1 更容易实现，目前 Linea 已经上线主网，而 Type 1 的 Taiko 尚未公布主网上线时间，可见 Type 2 或许更为现实，而 Type 1 则相对较为理想化，其项目也相对早期。

四、Linea vs 其他 zkEVM

1、资金方面：

在融资方面，Linea 并未单独进行项目融资，但其母公司 ConsenSys 已获得 7.26 亿美元的融资，资金充裕。同时，ConsenSys 旗下项目 MetaMask 是钱包赛道当之无愧的龙头，在为 ConsenSys 带来可观利润的同时也为 Linea 提供了强有力的支持。

除此之外，其他的 zkEVM 项目也获得了可观的融资：zkSync 获得了 4.58 亿美元，StarkNet 获得了 2.73 亿美元，Polygon zkEVM 获得了 4.5 亿美元，Scroll 获得了 8300 万美元，Taiko 获得了 2200 万美元。相较而言，Linea 的资金充足，并且拥有强大的背景支持，在资金方面是 zkEVM 赛道的领先者之一。

2、技术方面：

在技术方面，Linea 具备 Type 2 （目前尚未完全实现 100% 的兼容性，处于 Type 2 到 Type 3 的过渡阶段，第七部分会详细介绍项目的具体进展）本身拥有 EVM 等效性的优势，它能够完全等效于以太坊虚拟机（EVM），因此开发者可以构建可扩展的去中心化应用，或者直接迁移现有的去中心化应用而无需修改代码或重写智能合约。

此外，Linea 采用了独特的证明器（第五部分会详细介绍），在确保兼容性的前提下，尽可能提高交易速度并降低 Gas 费用。

然而，由于 zk-rollup 本身难以兼容 EVM，因此在保持高度兼容性的前提下，与 Type 4 和 Type 3 的 zkEVM 项目（例如 zkSync、StarkNet、Polygon zkEVM）相比，Linea 的交易速度相对略慢，Linea 的 gas 费用也会稍高一些。

五、Linea 有哪些亮点？

1、Linea 可以获得与 Infura、MetaMask 和 Truffle 等 ConsenSys 旗下产品的原生无缝集成：

构建者可以使用 web 3 钱包 MetaMask 轻松吸引用户，例如在 MetaMask 钱包插件的网络选择中，Linea 测试网是默认存在的，它消除了手动添加网络的用户体验障碍，一定程度上也降低了使用门槛。

此外，借助 Infura 易于使用的 API，开发者可以更快地发布 dapps 并进行扩展。开发人员可以使用 Truffle 或使用 Hardhat、Foundry 和 Brownie 等流行的开发人员环境来构建、测试、调试和部署他们的 Solidity 智能合约，与之兼容的 Ganache 和 Diligence 分别支持本地代码测试和审计。

此外，ConsenSys 原生的、无需信任的桥接器和对 MetaMask 用户友好的桥接器，也允许开发人员安全地将代币移入和移出网络。

2、独特的内部证明系统和压缩方式

  • Linea 证明流程

Linea 的技术在算术方案和内证明系统方面与其他 zkEVM 不同。为了证明交易的发生，Linea 采用了一系列步骤。它从算术化（Arithmetization）开始，将计算机程序转换为 zk 证明可以理解的数学表达式。此过程将事务转换为轨迹（trace）和一组验证计算准确性的约束（constrain）。

接着 Linea 采用内部证明系统 Vortex 和 Arcane，这些内部证明系统递归地减小证明的大小，通过优化计算和使用特定的算法，不断提高证明的效率和紧凑性。最终，通过经过多次迭代和内部优化的过程，证明被进一步压缩到最终的外部证明系统 Plonk 中。

这种设计使得 Linea 能够高效地生成和验证 zk 证明，从而确保交易的隐私和安全性。通过使用内部证明系统的递归优化和最终的压缩步骤，Linea 能够减小证明的规模并提高整体性能，为用户提供快速且高效的交易体验。这种创新的技术方法使 Linea 在 zkEVM 赛道独树一帜，并为构建可扩展的去中心化应用提供了强大的技术支持。

  • 内部证明系统：Vortex 和 Arcane
  1. 基于「格」（Lattice-based）的设计：

Vortex 是一种基于「格」的 SNARK 方案，可以在任何具有合理的二进制精度的域上运行，并能够高效地处理多种类型的查询，基于「格」的哈希与传统加密方法相比具有多种优势。 它拥有比流行的椭圆曲线密码学更快的性能，并且是后量子的，这意味着它可以抵抗量子计算攻击。「格」针对递归进行了优化，可有效实现硬件加速，并提供与 SIMD 并行性的兼容性。此外，基于格的函数避免了 SNARK（简洁非交互式知识论证）中速度和使用之间的权衡，使它们更加通用。

并且 SNARK 方案本身相较于 STARK 存在着一些优势：SNARK 的采用速度实际上比 STARK 快得多，SNARK 比 STARK 早几年被发现，这使得该技术在采用方面取得了显着的领先优势；且 SNARKs 拥有更多的开发者库、已发布的代码、项目以及积极致力于该技术的开发者；其次 SNARK 所需的 Gas 量预计仅为 STARK 的 24% ，这意味着使用 SNARK 进行交易对于最终用户来说会便宜得多；最后，SNARK 的证明大小比 STARK 小得多，这意味着它需要更少的链上存储。

2.透明设置：

Vortex 的设置过程是透明的，不需要信任设置参与者，这使得 Vortex 更加安全和可靠。

3.线性承诺方案：

Vortex 使用一种创新的线性承诺方案，可以高效地处理多个标量积（multiple scalar products）的承诺，并且证明大小比其他方案更小。

4.自递归技术：

Vortex 使用自递归技术，可以重复使用不同的 Ring-SIS 实例和不同的纠错码，从而在选择 Ring-SIS 参数和纠错码时可以灵活权衡证明大小和运行时间。

5.交互式 Oracle 证明 (IOP) 模型：

Arcane 将算术编译为交互式 Oracle 证明 (IOP) 模型，IOP 允许验证者查询预言机，这是一个可信赖的第三方，可以概率性地提供必要的信息。Linea 采用 Wizard-IOP 框架，提供比标准 IOP 模型更复杂的查询，且 Arcane 将约束集转换为多项式评估，增强了证明的数学形式。为了消除对第三方的依赖，Linea 采用密码学假设和迭代转换，用多项式承诺方案取代预言机。

  • 最终压缩步骤：Plonk

为了使证明可以直接在以太坊的 L 1 上验证，Linea 使用 PlonK 证明执行最终压缩步骤。PlonK 与 Groth 16 （Linea 原本使用 Groth 16 证明） 一样，是一个 zkSNARK 结构，利用了先进的加密技术。PlonK 的 SNARK 友好属性和基于「格」的哈希可确保快速验证并生成适合 L 1 高效验证的紧凑证明。

Plonk vs Groth 16 ：从 Groth 16 到 PlonK 的转变是由可信设置过程驱动的。每当电路发生变化时，Groth 16 都需要重复设置，而 PlonK 则执行一次设置，与电路无关。Linea 的迭代电路设计需要频繁地使用 Groth 16 重新运行可信设置，从而引发了对信任的担忧。通过采用 PlonK，Linea 保持了协议的完整性，同时为社区提供了公平竞争的信心。

在这一变革之后，将创建一个证明以供 Linea 的验证者合约在以太坊 L 1 上进行验证。成功验证证明、状态承诺和调用数据后，新的汇总状态将在 L 1 智能合约上最终确定。

以上技术为 Linea 带来了哪些好处？

简言之，前文所述的技术使得 Linea 在确保 EVM 等效性高度兼容的前提下，实现了交易速度的提升和用户 Gas 费用的减少。由于 zk-rollup 本身难以兼容 EVM，因此 zk-rollup 的兼容性与速度之间是负相关的关系。而 Linea 作为完全等效 EVM 的高兼容性 Type 2 zkEVM，其速度提升则更加困难。因此，基于"格"的 SNARK 方案 Vortex 在这方面是一项卓越而难得的技术突破。

3、多证明者（Multi- Prover）的推动

Multi Prover 是将 Linea 视为以太坊的一个客户端，而当前已经存在许多不同的客户端，如 zkSync、Polygon zkEVM 等 Layer 2 项目。只有拥有多个客户端，才能确保以太坊的安全性，因为即使一个客户端出现故障，还有其他可供使用的备用客户端。

Linea 的高级产品经理 Declan Fox 表示，为了防范潜在的风险，Linea 将推动使用多证明者（Multi Prover）方法。这意味着多个证明者（可以理解为多个不同的 Layer 2 产品）共同使用 zkEVM 的多种不同实现来验证交易。这样，如果一个证明者存在错误或变得不可用，只要其他可用的证明者达到法定数量，系统将继续可靠地运行。

ConsenSys 的全球商务负责人 @hotpot_dao 解释道：“Multi Prover 是一个产品，让三家不同的 Layer 2 去进行证明。只要这三家都通过，该产品就可以被验证并回到主网上，从而大大增强了以太坊的安全性。这也是我们认为 Linea 在构建二层协议时的一个用途。作为 ConsenSys 公司，我们主要致力于推动 Multi Prover。这也与整个以太坊或者我们未来设想的以太网方向高度契合，同时也是一个技术上的巨大挑战。”

Multi Prover 对整个 Layer 2 赛道都至关重要。而 Linea 作为 Multi Prover 的发起者、推动者甚至领导者，那么如果 Multi Prover 成功实现，Linea 也将得到市场更多的认可和信任，在未来的发展中获得更多机遇与优势。

六、生态发展

迄今为止，Linea 已经吸引了超过 100 个生态合作伙伴的加入，Linea 的主网将与这些合作伙伴组成的庞大生态系统一同推出，为用户、构建者和开发人员提供构建新一代可扩展以太坊 Dapp 的机会。

根据我们对 Linea 的长期观察，Linea 在目前生态早期着力搭建跨链桥、钱包等基础设施，布局包括 DEX、借贷等 Defi 赛道，同时深耕 Social、Gamefi 等，不断吸引用户和开发者，达到 web 3 的大规模落地。正如 ConsenSys 全球商务负责人所说的，“像社交网络需要一个网络计算，有一个冷启动问题，在二层协议上是很难实现的。所以我们先把 Defi 做好，然后再做一些小的 Gamefi，然后 Linea 希望用户们能够先体验一下我们的平台和 Network，打好基础，然后希望再有好的社交软件过来开发。”

随着主网上线，我们从 Linea 主网生态系统中选择一些当下较为火热、动作较多的项目来进行分析，其余更加详细全面的资料可以参阅官方网站。

DEFI

LineaBank

LineaBank 是建立在 Linea 上的借贷协议，其着重提供隐私保护和可扩展性。该协议赋予用户对资产的完全控制权，并通过去中心化市场消除中介机构，为用户提供具有竞争力的利率。从其项目名称就可以看出 LineaBank 是 Linea 区块链上的原生项目。此外，根据上述图表显示，LineaBank 目前是 Linea 上锁定价值（TVL）最高的项目，截至今日（7.27） TVL 已达 581 万美元。

除此之外，值得一提的是 Linea 上线主网后，LineaBank 举办了“预挖“活动，在此期间用户能够获得协议 100% 的收入。

iZUMi Finance

iZUMi Finance 是一个搭建在以太坊网络上的流动性优化协议，是 Uniswap V 3 生态项目，为区块链项目提供基于 Uniswap V 3 的“可编程性流动性即服务”业务。

iZUMi 的 LiquidBox 计划通过规划有效的激励性流动性供应计划来协助项目吸引流动性。它还提供了一个共享流动性盒，将一种代币的流动性映射到多链上，LP 可以使用原始链上的一种流动性在多链上进行挖矿。

iZUMi Finance 还在 BNB Chain 和 zkSync 上推出了 iZiswap，这是在 Uniswap V 3 集中流动性模型的基础上升级的离散流动性 AMM 。iZUMi 还在开发 iUSD，这是一种由 USDC 支持的与美元挂钩的稳定币。iUSD 用于 iZUMi 债券挖矿计划。

iZUMi 其他以用户为中心的去中心化金融产品包括无常损失保险计划，该计划为流动性提供者提供对其 LP 代币造成的无常损失的退款。它还提供固定收益机会，年利率高达 10% ，期限为 30 天。

iZUMi Finance 到目前为止已融资 5760 万美元，Mirana Ventures、Everest Ventures Group、IOSG 等投资方参投，拥有雄厚的资金和强大的资源。

如上图所示，于 7 月 11 日，iZUMi 宣布与 Linea 展开合作。截至今日，iZUMi Finance 在 Linea 的 TVL 排行榜中名列第四位，截至今日（7.27） TVL 已达 136 万美元。这意味着 iZUMi Finance 在 Linea 生态中得到了认可和积极参与，通过与 Linea 的合作，iZUMi 将进一步扩大其影响力，并为 Linea 生态系统的发展做出贡献。

Mendi Finance

Mendi Finance 是一种与 EVM 兼容的借贷协议，将在 Linea 网络上启动。

Mendi Finance 以完全去中心化、透明和非托管的方式提供点对点借贷，作为 Linea 上的原生项目，其目标是通过为货币市场提供有竞争力的激励、拥有最深度的流动性成为 Linea 上的主要借贷平台。 

上线后，用户可以在 Mendi Finance 上借出任何支持的资产，并用自己的资金作为抵押品借入这些支持的资产，其中包括 ETH、BTC、USDC、USDT 和 DAI。这些资产的抵押系数范围在 0-90% 之间，代表您可以针对特定资产借款的最高限额。如果账户的借入余额超过抵押因素的限制，则该账户将会被清算。

HorizonDEX

HorizonDEX 是一款集中流动性去中心化交易所，允许用户在自定义价格范围内分配流动性，从而使交易者能够最大限度地提高效率并最大限度地减少滑点。作为 Linea 区块链上的原生项目，HorizonDEX 在 Linea 的锁定价值（TVL）排行榜中位列第二，截至今日（7.27）其 TVL 已达到 187 万美元。

此外，如上图所示，Linea 主网上线后，HorizonDEX 举办了一项交易和提供流动性即可获得空投代币$HZN 的活动。在活动期间（7.19 ～ 8.2），每进行 1 美元的交易或提供 1 美元的流动性维持 24 小时，用户将能够获得 1 个 yHZN 代币。活动结束后，根据持有的 yHZN 数量按比例进行空投，持有的 yHZN 越多，获得的空投代币就越多。

EchoDEX

EchoDEX 是一个建立在 Linea Consensys 网络上的去中心化交易平台，同时也是 Linea 区块链上的原生项目。并且还是第一个上线 Linea 的去中心化交易所（DEX），它目前在 Linea 的锁定价值（TVL）排行榜中位列第五，截至今日（7.27）其 TVL 已超 78 万美元。

值得一提的是，其代币 ECP 目前市值已有 $ 37.4 M，代币在发行初期也引起了社区的大量关注。

并且 EchoDEX 和前文写到的两个项目一样也有其空投预期，在 7 月 19 号时他们宣布用户只要 Swap、添加流动性或者是挖矿甚至是每日签到都可以获得积分，得到获取空投的机会。

Owlto Finance 

Owlto Finance 是一个去中心化的 Cross-Rollup 桥，提供低成本、安全、快速的资产转移解决方案，智能合约由安全公司 Beosin 审核。值得一提的是，在 Linea 上线主网后，Owlto Finance 为 Linea 开启了 7 天 0 手续费跨链活动。

值得一提的是，Owlto Finance 在官方文档中表示，该 Cross-Rollup 桥与其他基于安全 Rollup 技术的 Cross-Rollup 桥不同，资产转移是在「源」和「目标」网络上的「发送者」和「流动性提供者」的 EOA 地址之间进行的，并且「发送者」不与智能合约地址进行交互。这种设计可以降低风险，因为智能合约地址可能容易受到攻击，而 EOA 地址通常更安全。

NFT

NFTs 2 ME

为了简化 Linea 网络上 NFT 创建者的创建、部署和管理过程，NFTs 2 ME 声称提供实用程序，使用户无需编写一行代码即可创建和管理 NFT，此外，NFTs 2 Me 声称该设施向任何人开放并且免费使用。

平台上的工具包括一个设计面板，允许创建者定义其 NFT 的属性。它具有人工智能驱动的艺术和图像生成工具，使用户能够在平台上生成艺术品、定义其元数据并设置铸造价格。它支持多种 NFT 智能合约标准，并允许收藏家支付任何（加密）面额的铸币费用。

NFT 创作者可以通过多种方式使用平台工具来管理生成的艺术品。NFTs 2 Me 的用户仪表板具有分析工具，可帮助用户跟踪其 NFT 的性能。用户还可以计划和执行空投、拍摄快照、设计 NFT 门控程序，并通过 NFTs 2 Me 个人资料为其 NFT 铸造活动准备白名单。  

NFTs 2 Me 是多链的，可在以太坊、Polygon、Taraxa 链和第 2 层网络 Arbitrum、Optimism、zkSync Era 和现在的 Linea 网络上使用。

ghostNFT

ghostNFT 是 Linea 网络的官方测试网项目之一，随着主网 alpha 的推出，预计将可在主网上使用。 

ghostNFT 声称利用 ERC-721 Envious 标准（ERC-721 标准的改进版本）通过 NFT 抵押为 NFT 提供额外的货币化功能。它声称通过其 NFT 2.0 项目融合 NFT 和去中心化金融，以多种方式为 NFT 创造者和收藏者赋能，并且它允许创建者为其 NFT 定义抵押计划、收集抵押品，并为其 NFT 的发布进行筹款活动，用户还可以通过“分散”功能对单个 NFT 或整个集合执行此操作。

ghostNFT 是多链的，已经可供以太坊、BNB 智能链、Avalanche 和其他几个网络上的用户使用。

Infra

thirdweb

thirdweb 是一家英国 Web 3 软件开发商，推出免费的 Web 3 开发工具，让开发人员无需编写任何代码，即可构建、启动和管理 web 3 项目，开发者只需点击几下，即可添加功能，包括 NFT、社交代币和货币、代币交易市场以及 NFT 交易等。

thirdweb 于 2022 年的 8 月 25 日以 1.6 亿美元的估值完成了 2400 万美元的战略融资，由 Haun Ventures 领投，Coinbase Ventures、Shopify、Protocol Labs、Polygon、Shrug VC 等参投，除此之外，个人投资者还包括硅谷顶级风投公司 Kleiner Perkins 的合伙人 Joseph Lacob，他也是 NBA 金州勇士队的大股东。并且在之前 500 万美元的种子轮融资中，Thirdweb 还获得了 NBA 达拉斯独行侠队老板 Mark Cuban 和美国知名企业家 Gary Vaynerchuk 的投资，总融资额达到 2900 万美元，拥有雄厚的资金力量。

此外，Linea 还发表了一个关于“如何使用 thirdweb 快速构建”的教程，教程中详细介绍了如何使用 thirdweb 的 Solidity SDK 部署智能合约，以及通过仪表板管理它，并构建一个应用程序以使用 SDK 与其交互。

QuestN

QuestN 是由Hogwarts Labs孵化的 Web 3 任务和活动平台。用户可以通过参与任务和活动来赚取代币和 NFT 徽章。

QuestN 致力于为 GameFi、DAO 和其他项目提供各种无需许可的链上和链下任务，为用户以及 Web 3 生态及其原生价值带来持续的利益。

如上图所示，到目前为止，QuestN 已经有接近 9000 万个任务被完成，发行了超过 160 万个 NFT，拥有超过 2.6 万个社区的支持。

此外，今年 6 月 1 日，QuestN 的母公司Hogwarts Labs完成了 Pre-A 轮融资，筹集了 800 万美元。

结语：

根据前文所述，可以明显观察到 Linea 的生态系统目前仍处于非常早期的发展阶段。并且就目前而言 TVL 排名靠前的多数项目主要是通过空投预期吸引了大量空投猎人的参与，从而带来了相应的 TVL 和交易量增加。

不过，Linea 背靠着强大的区块链公司 Consensys，能够得到雄厚的资源和资金支持，这为其生态系统的发展提供了坚实的基础。随着时间的推移，或许 Linea 的生态系统能够逐步完善和壮大，强大的背景支持也能够吸引更多的开发者和项目加入，为用户带来更多有用且创新的应用。

七、项目进展及未来计划

项目进展：

近期由 Linea 官方组织的「Linea Voyage」活动取得了巨大的成功，在短短的九周时间内，该活动实现了超过 3000 万笔的交易量，吸引了 520 万个独立钱包参与其中。此外，值得一提的是，在 Linea 测试网阶段，共有 550 万个独立钱包提交了超过 4600 万笔的交易，使 Linea 成为 Goerli 网络上最大且增长最快的项目之一。

如今，Linea 迈入了一个新的阶段。上周，Linea 发布了主网的 Alpha 版本。在 Alpha 阶段，Linea 官方将开始对合作伙伴和用户进行评估，监控系统，并为那些正在使用 Linea 构建新功能的人提供支持。该版本将成为 Linea 在主网上的第一个完整系统，将起到"训练轮"的作用，使用户能够在实时环境中学习并适应 Linea 的使用。

未来计划：

与此同时，Linea 团队也积极追求逐步增强系统的去中心化和信任最小化，这一过程分为五个阶段，分别为阶段 0 ～ 4 ：

阶段 0 

EVM 等价：Linea 的 zkEVM 提供了与以太坊基金会指定的 EVM 相等的功能，以确保默认的可移植性。基于 Linea 的开发者可以随时选择将他们的 dapp 转移到支持 EVM 的其他链上，如以太坊，而无需重新编写智能合约、重新安装工具或支付新费用。

发起安理会：同时，Linea 官方还将召开第一次 Linea 安全委员会会议，以监控网络、保护用户并减轻与新系统相关的风险。

此外，用于在 Linea 网络上运行本地节点的客户端软件将被公开，使任何参与者都可以选择自行验证 Linea 的状态，并最大限度地减少对 Linea 运营商的信任。

阶段 1 

开源堆栈：Linea 团队计划在 AGPL-3.0 下授权 Linea 软件堆栈，该免费软件许可证确保用户可以自由查看、分叉和修改代码。

100% EVM 覆盖率：Linea 将为所有 EVM 操作码和预编译生成计算证明，从而解锁所有用例的免信任执行。此外，算术规范将被发布、审核并接受错误奖励，以提高系统的安全性和可靠性。

阶段 2 

安理会多元化：Linea 主网 Alpha 启动期间成立的安理会在网络的监督和控制方面发挥着关键作用。为了防止权力集中和潜在偏见，Linea 团队计划扩大对理事会的参与，同时保留 8 中 6 的多重签名门槛或更严格的立即升级门槛，这一修改确保了平衡的代表性并促进更稳健的集体决策，为系统增加了额外的制衡层。

抵制审查的退出：该架构将得到完善，以防止 Rollup 运营商阻止用户提现。在这一阶段 Linea 将确保用户可以独立发起退出，从而保证其在 Linea 上的资产能够持续访问和控制。

阶段 3 

去中心化运营商：Linea 团队计划通过分散 Provers 和 Sequencer 的角色来显著增强网络参与者的信任和系统的整体完整性。目前，Provers 和 Sequencer 分别负责生成零知识证明和管理第二层交易，是 Linea 生态系统中的关键角色。此外，Linea 团队承认，这可能带来一些技术复杂性，例如潜在的延迟、交易成本的增加以及潜在的最大可提取价值（MEV）问题。然而，团队表示将致力于应对这些挑战，为社区打造一个值得信赖和高效的平台。

去中心化治理：官方团队表明 Linea 治理的民主化是持续发展的关键，这种方法培育了一个公平、透明的系统，所有利益相关者都可以参与其中，并且 Linea 的未来由其用户社区引导。这种共享治理不仅可以减轻单边决策和单点故障的风险，还可以增强网络的弹性。反过来，这促进了网络参与者之间的集体所有权和信任，从而增强了生态系统的可持续增长。

阶段 4 

官方表明路线图的最后阶段旨在进一步确保生态系统的稳健性，并通过实施多重证明者和限制治理权力来减轻与信任相关的风险。

Multi- Prover rollup：EVM 将按照以太坊路线图所述继续发展，并将由更大的以太坊社区和以太坊基金会推动。因此，为了以合理的速度对 Linea 网络进行更改并防范实施风险，官方计划整合 zkEVM 证明器的多种不同实施方式，称为 Multi- Prover rollup。

并且由于 Linea 上的每一笔交易都由 zkEVM 的多个异构实现进行验证，可以确保即使一个证明者遇到错误或变得不可用（考虑到复杂性，这并非不可能），系统也能继续不间断地运行。这种方法不仅在 EVM 规范不断发展的时期增强了对平台可靠性的信心，而且鼓励探索创新进步，同时保持卓越的系统正常运行时间水平。

限制治理权：最后，Linea 官方计划限制治理权力，并将治理限制在必要的范围内。汇总逻辑的升级将是不可变的，仅允许在 EVM 规范发生变化的情况下对验证者进行升级。用户将收到退出汇总的合理通知。这些措施旨在保持系统的稳定性和安全性，并促进整个社区的参与和信任。

八、总结

Linea 的出现目前对于 web 3 行业尚未带来较大的改变，对于 Layer 2 赛道的影响也相对微弱。然而，在 zkEVM 赛道，Linea 可以算是一个突破。作为第一个上线主网的等效 EVM 的 zkEVM 项目，Linea 本身对于 zkEVM 赛道来说是一个重要的进步。这意味着 zkEVM 的进展已经达到了与 EVM 完全等效的水平，能够实现从主网到 zk-Rollup 的无缝迁移。

此外，相较于其他 zkEVM 项目，Linea 拥有着强大的背景支持，拥有领先于其他项目的用户、资源和资金支持，使其成为一个强有力的竞争对手。这增加了其他 zkEVM 项目的紧迫感，进一步加快了整个 zkEVM 赛道的发展进程。

而不足的是，通过上文的数据分析，我们可以发现 Linea 与当前 zk-rollup 的赛道龙头 zkSync 仍存在不小的差距。同时，在生态方面与其他几个已经上线主网的 zkEVM 项目相比，Linea 的生态发展仍然相对落后。因此，Linea 还需要不断地完善和改进，才有可能超越其他 zkEVM 项目，成为 zkEVM 赛道的龙头。这需要 Linea 团队不断努力，并积极吸引更多的开发者和用户参与其中，推动整个生态系统的繁荣发展。
