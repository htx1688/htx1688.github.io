原文作者：PAUL VERADITTAKIT

原文编译：深潮 TechFlow

自比特币诞生以来，我们见证了“L1 战争”，这推动了开发尖端技术来解决区块链三难问题。对高度可扩展解决方案的需求一直存在，因为没有单一解决方案能够满足行业的需求同时确保顶级安全性。这导致了 Sui 和 Aptos 的诞生，它们利用了 Facebook 放弃的 Diem 和 Novi 项目的架构。

Aptos

Aptos Labs 由 Mo Shaikh 和 Avery Ching（来自 Diem/Novi 的高管）领导，创建了 Aptos 区块链，旨在重振 Diem 的技术并与以太坊和比特币等市场领导者竞争。Aptos 旨在提供灵活、可扩展和安全的基础设施。它利用 Diem 的架构以及尖端技术，确保高吞吐量、低延迟和可验证的状态同步。

Sui

Mysten Labs 开发的 Sui 由前 Facebook 高管 Evan Cheng 和 Sam Blackshear 管理，旨在解决当前加密网络面临的限制，以满足大规模使用的需求。与 Aptos 不同，Sui 不是 Diem 的派生版本，而是从头开始构建，充分利用固有的可扩展性和快速结算能力。Sui 旨在提供高吞吐量、低延迟和经济实惠的计算资源，以满足数十亿用户的应用需求。

Move：可扩展性语言

这两个区块链都使用了 Move，这是一种基于 Rust 的编程语言，可以实现并行处理。Move 生态系统包括编译器、虚拟机和其他开发工具。Aptos 使用了该语言的核心实现，尽管 Sui 对其进行了一些微小的修改，以更好地适应其架构。

背景

Move 是一种字节码语言，用于设计自定义交易和智能合约。Move 与其他语言（如 Solidity）的不同之处在于其资源管理能力以及对数字资产的稀缺性和访问控制的强调。稀缺性限制了资产的形成，以减少双重支付的风险，而访问控制确定所有权和资产访问。Move 的资源管理基于线性逻辑的数学思想，将资产视为绝对资源，一旦用尽就永远丢失。这种资源的特定性允许在程序存储位置之间进行安全转移，而无需隐式删除或复制，因此得名“Move”。

Sui 的 Move 变化

尽管 Aptos 的 Move 大部分复制了 Diem 的白皮书，但 Sui 添加了一个面向对象的存储系统，用于跟踪地址和交易等所有内容。

Sui 将属性分类为对象，定义如下：

  • 共享对象：可变的，没有明确的所有者，并且可以在多方交易中无需授权使用。

  • 拥有对象：拥有唯一所有者，只能由该用户更改。

  • 只读对象：没有唯一的所有者，在发布后无法更改，并可供所有用户在交易中使用。

将这些资产分类为不同的域可以减少处理和交易时间，提供 NFT、游戏产品等的可扩展性。这也是 Sui 在 NFT/游戏行业中的主要基础。

Sui/Aptos 的共识机制

共识是一种使区块链节点（验证者）就交易和区块的可靠性达成一致的过程。Aptos 和 Sui 都建立在广为认可的拜占庭容错（BFT）的概念上。共识机制的核心原则是，即使多达 1/3 的验证者变为恶意或失败，网络仍然可以正常运行。

拜占庭容错（BFT）

拜占庭容错的基本概念如下：

  1. 一个网络由验证者组成，他们共同拥有 N 个投票权，用于决定是否批准一个新的区块。

  2. N 通常取值为 3 F+ 1 ，并且必须能够承受 F 个具有投票权的验证者。这意味着最多有 F 个验证者可能是恶意的、慢速的、断开连接的等等。

  3. 在每一轮中，以随机和可观察的方式选择一个领导者。

  4. 在每个周期中，领导者提出一个新的区块，其余的验证者对其进行投票。

  5. 只要有 2 F+ 1 个投票来自可信的验证者，他们就可以就一致的选择达成一致。

任务并行化技术

Aptos 的 Block-STM

Aptos 通过 Block-STM 实现了并行化，这是受软件事务内存启发的高性能 HotStuff 算法的改进。Block-STM 识别事务之间的关系，并实现并行执行。如果一个事务在验证时失败但具有依赖关系，它可以重新执行；否则，它将被丢弃。一旦使用延迟提交方法解决了争议，一个区块中的所有事务将同时提交到区块链上。这种方法节省了时间和资源，因为事务不再需要按顺序处理。

Sui 的 Narwhal 和 Tusk

对于复杂的过程，Sui 在其执行层中采用了 Narwhal 和 Tusk 的共识技术来实现并行化。Narwhal 是一个 mempool 模块，确保提交给共识的数据可用。Narwhal 的设计使用了有向无环图（DAG），其中它的许多组件通过网络连接而不是链条连接（类似于分布式账本）。它也可以与其他共识引擎（包括 HotStuff 或 Cosmos 的 Ignite）独立使用（无需 Tusk）。

  1. 发送方（类似于领导节点）向所有 Sui 验证者广播一个事务。

  2. 发送方从 Sui 验证者那里收到投票。权重根据验证者在委托权益证明设置下对 Sui 的兴趣按比例分配。

  3. 发送方收集满足抗拜占庭多数条件的投票，创建一个证书。验证者必须通过拜占庭协议达成共识，按顺序对证书进行排序。

  4. 在依次处理每个证书后，发送方最后一次将证书发送回验证者，以完成事务的最终确认。

Sui 的交易以网络图形的形式连接，而不是在区块链中顺序堆积的一系列交易。这种基于 DAG 的数据范式，结合将交易分解为较小的部分并利用其固有属性，增加了可扩展性。Sui 的异步架构提供了对拒绝服务攻击的安全性保障，而高性能的 BFT 共识过程 Tusk 确保了交易的排序。Narwhal 和 Tusk 共同使每个验证者能够在给定时间内处理更多的交易。

产品性能

Aptos：每秒处理 160, 000 笔交易；

Aptos 以每秒处理 160, 000 笔交易（TPS）的速度引以为豪。这一成就得益于四项关键技术进展：并行交易执行、状态同步、延迟提交和协作调度。

  1. 并行交易执行 - 并行处理交易并重新执行相关进程。

  2. 状态同步 - 链数据可以由可靠的非验证者进行同步和验证。

  3. 协作调度 - 允许在调度阶段优化交易的资源分配，以加快处理速度。

  4. 延迟提交 - 在块-STM 过程完成后批量提交交易的概念。

Sui：每秒处理 120, 000 笔和“无限”TPS。

一台八核的 Macbook Pro 能够在 Sui 区块链上处理超过 120, 000 笔每秒的交易。

这在很大程度上取决于交易的类型，因为简单的交易不需要广泛的共识，并且允许横向扩展。Sui 的性能在其最大规模下可能是“无限”的。随着网络需求的发展，Sui 节点可以继续添加工作人员来完成额外的交易。

应用案例

Aptos 和 Sui 都适用于各行各业和各种应用的广泛使用案例。

Aptos 区块链的多功能架构使其适用于不同领域的应用。其主要目标是去中心化支持 Web 2.0 应用的云基础设施，促进 Web 3.0 的广泛采用。凭借可扩展性、安全性和低费用，Aptos 区块链非常适合金融、供应链管理、去中心化金融（DeFi）、游戏、身份管理等应用。该平台专注于解决现实世界的问题，使其成为开发人员和企业寻求强大且可扩展的区块链解决方案的有吸引力的选择。

作为一种无许可的第一层区块链，Sui 区块链提供即时结算、高吞吐量和低延迟，适用于各种行业和应用案例。其注重能源效率和去中心化，使 Sui 成为金融、物联网（IoT）、游戏、社交媒体、内容共享和其他延迟敏感领域的理想选择。Sui 致力于支持延迟敏感的去中心化应用，使其在区块链生态系统中成为一个强有力的竞争者。

市场表现

Aptos 已从知名风险投资公司获得了大量资金，总额约为 4 亿美元。投资者包括 Andreessen Horowitz、FTX Ventures、Jump Crypto、a16z、Tiger Global 和 Multicoin Capital 等。这种强大的财务支持凸显了投资者对 Aptos 项目及其增长和成功潜力的信心。知名公司的参与不仅提供了财务支持，还为潜在的合作和伙伴关系打开了大门，进一步巩固了 Aptos 在市场中的地位。

Sui 同样得到了主要投资者的强力支持，如 Lightspeed Venture Partners、Andreessen Horowitz 和 Redpoint。在最新的 B 轮融资中，Sui 筹集了令人瞩目的 3 亿美元，加上 A 轮融资筹集的 3600 万美元，使其总融资额达到 3.36 亿美元。巨额融资和超过 20 亿美元的高估值凸显了市场对 Sui 项目的兴趣和信心。知名投资者的参与为战略联盟和合作伙伴关系铺平了道路，为 Sui 的增长和生态系统发展做出了贡献。

此外，Aptos 和 Sui 都培育了充满活力和活跃的开发者、爱好者和用户社区。这些社区通过应用开发、参与讨论和分享想法积极地为生态系统做出贡献。这两个平台以社区驱动的方式促进创新、合作和采用，为 Aptos 和 Sui 打造了一个强大的生态系统。

总结思考

尽管目前尚不清楚哪个解决方案将成为主导市场的赢家，但 Aptos 和 Sui 在其发展过程中取得了重大进展，极大地推动了区块链设计的当前状态。Aptos 通过相对较早进入市场并吸引了大量关注而获得了市场份额，而 Sui 则受益于其创新和以安全为重点的方法。有一点是确定的：凭借其巨大的潜力来提高加密网络的可扩展性和安全性，基于 Move 的技术注定会对该行业产生持久的影响。
