ERC-6551标准由 Future Primitive团队推出，该标准提出了一种将智能合约账户绑定到每个 ERC-721 NFT 的方法。这些账户称为Token绑定账户（Token Bound Accounts，简称“TBA”）。

我们相信ERC-6551 将成为 Web3 中最具影响力的创新之一，原因如下：

  1. 它使每个 ERC-721 NFT 都可以拥有(own)/持有(hold)任意数量的任何其他代币（ERC-20、ERC-721、ERC-1155 等）。

  2. 它不需要对现有 NFT 或任何新的 NFT 合约进行任何更改即可部署。

  3. 它“自动”将这些账户绑定到所有现有的 NFT 中，而无需 NFT 所有者执行任何交易。

  4. 它实际上是可无限扩展的。

  5. 它已经支持账户抽象 （ERC-4337）。 

  6. 它为 NFT 提供了新的用例。

  7. 这是 NFT 演化的重要一步，它们将从简单的 PFP 转变为Token化的Avatar（Tokenized Avatars）。

TBA在技术上很酷，但我们实际上可以用它做什么？

TBA有许多用例，但最突出的用例是：

  • NFT 可组合性（NFT composability）

  • 链上声誉（On-chain reputation）

  • 游戏角色库存（Game avatar inventories）

NFT  可组合性

可组合性在 NFT 世界中并不是一个新概念，最近几年有许多令人印象深刻的可组合 NFT 和标准。然而，TBA 因为一个关键不同而改变了NFT的可组合格局——它们不需要对现有的 ERC-721 标准进行任何更改。

可组合性的一个实际用例是在物流和制造方面 。举一个简单的例子，想象一个拥有自己账户的汽车 NFT，该账户保存有关保险、登记注册数据、纳税数据、制造数据、零件召回等信息记录。我们在转移汽车NFT的同时，也能直接将汽车的相关数据一并转移。

与以前的可组合 NFT 不同，TBA 可以持有几乎无限数量的资产，类似于标准的外部账户 （EOA） ，并且几乎无限可扩展，这使得 TBA 对 NFT 可组合性极具影响力。

链上 声誉 （On-Chain Reputation）

TBA的另一个引人注目的用例是链上声誉领域。

让我们考虑一下使用 NFT 衡量忠诚度 的场景。使用 TBA，可以跟踪和记录个人对 DAO、组织或类似实体所做的所有贡献。通过将灵魂/代币绑定的 NFT 发送到与忠诚度 NFT 关联的钱包，它可以无缝跟踪与忠诚度相关的活动。

例如，假设您是音乐家的忠实粉丝。您从该艺术家那里获得了一张 NFT 专辑。对于您参加的每场音乐会，您都可以兑换一个出席证明 Token，该Token将存入专辑 NFT 的 TBA 中。这种机制不仅可以实现忠诚度的链上跟踪和验证，还可以随着时间的推移提高原始专辑 NFT 的价值。艺术家还可以通过简单跟踪 NFT 账户的内容来奖励参加每场音乐会的忠实粉丝。

得益于TBA，在不知道钱包地址的情况下将 NFT 空投给现有的 NFT 持有者也变得可行。此功能为培养品牌忠诚度和以透明和安全的方式为用户创造引人入胜的体验开辟了新的可能性。

游戏角色库存

我们个人最喜欢的TBA用例是游戏角色库存。

玩游戏时，玩家和角色之间存在显著区别。大多数玩家创建了许多具有独特属性、技能、成就、游戏内声誉等的角色。TBA 使每个 NFT 都有自己的角色库存。

虽然 ERC6551的官方团队Token Bound 正在与市场合作，以促进 NFT 与 TBA 的支持，但市场可能需要时间来适应并解决相关的复杂性。
