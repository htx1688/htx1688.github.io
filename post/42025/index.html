运营、设计、研发、产品人员谈他们从 Web2 进入 Web3 的一些转变和感悟。

撰文：Abby

Web3 高速发展吸引着一波又一波的年轻人，也不乏曾在大厂策马奔腾多年的互联网人。

2022 已经过去大半，那些 All In Web3 的前互联网人，现在过得怎么样呢？这次，我们采访了 bfrenz 的朋友们，包括运营、设计、研发、产品。

面对 Web3 就业机会、个人成长等问题，每个人都带来不同的观点，可谓是一场大型 「Web2 到 Web3」 的思想交流碰撞会。

特此声明，本文仅为个人经历展开的观点讨论，不代表「标准答案」。

以下，enjoy：

运营  

Eva，曾在互联网美妆公司上班

Web3 年龄：10 个月

当前职业 1：Web3 市场运营

当前职业 2：孩子妈妈

共识

刚刚我老东家群里炸了，顺利完成融资！所有人都在替老东家高兴，当然包括我。 

Web3 的世界都在说共识，参与其中，等于是项目的「投资人」，提前布局，也关注项目的发展。如果项目发展得好，我们的投入也从中收获「溢价」。

一场美好的相遇

其实我并没有很早加入 Web3。去年 10 月 NFT 很火的时候，我刚好离职看到了相关新闻。

我在海外修读艺术学，NFT 的新闻像是重新点燃起我内心深处对艺术的热爱。后来买了一些 NFT，进 holder 群和兴趣小组。在和网友面基的时候认识了一个项目的 Co-Founder ，当时他刚好有空缺的位置，愿意手把手教像我这些零经验的 Web3 新人，我觉得算是机缘巧合吧，就进了 Web3 圈。

Web3 的 「从 0 到 1 」

这 10 个月，我觉得自己改变还是挺大的。

去年的我，还是什么都不懂的状态，学着去注册各种账号，买币，学习交易。在工作上，公司也给到我很多提升自我的机会，从准备产品文案、到制定 Twitter / Discord 运营策略，与其他项目方一起办宣发活动、合作抽奖，AMA 主持等等，一步一步走过来。

我在 Web2 负责国内市场，而 Web3 更多面向海外。这里需要我从头学习运营 Twitter、Discord。这些都是蛮大的学习议程。比如 Discord 的运营学习，我会先列基础框架，根据自己产品特性再做模块与活动微调。对了，除此以外，还要研究各种各样机器人（哈哈）。

在我看来，Twitter 是用来做引流的，比如和其他项目方举办推特活动，邀请知名度比较高的嘉宾上 AMA ，就能有比较好的引流效果。当然，内容选题也很讲究。如果听众认为能从这个 AMA 里收获到东西，自然就会来听甚至主动宣传。像我们之前有一期安全主题的 AMA，邀请了慢雾以及一些安全领域比较出名的大佬，和我们 Founder 一起聊，因为内容很实用干货，接近 2000 人来参加这场 AMA，我们也会在 Galaxy 发布一些 OAT ，听完 AMA 就可以去 Mint。这样的模式，可以很顺畅从 Twitter 引流到 Discord。虽然一开始大家可能是为了薅羊毛，但是 Discord 运营得好，也是完全可以留住用户的。一句话来说，Twitter 就是引流，Discord 用来做留存。运营嘛，这里的工作都是挺细的，真正做起来工作量很大。

所谓「核心竞争力」

如果你问我，英语能力和技术能力重要吗？我觉得重要，但不是最重要。最重要的是，自我驱动力，你要真的坚定要进来 Web3，这才是最重要的。总结起来，就是 5 个字：坚定、不设限。

虽然我海外生活过，但回国后 5 年都没怎么用英语，我一开始进入 Web3 工作也依赖很多翻译软件。我是如何提升英语能力的呢？上下班的时候也会习惯去听英语 AMA ，多听欧美人是怎么讲的，似懂非懂没关系，多听就有感觉了，甚至你会突然有一天感觉自己「开窍」了，这种感觉很奇妙！

除了自我驱动力以外，「长期建设」的心态很重要。

这个不仅对 Web3，所有行业都一样，不能老想着「一夜暴富」，要对自己的职业有长期规划，这样就不会容易 FOMO。在 Web3 圈子，暴涨暴跌太常见了，这也是对自己心态的磨练。不要觉得我现在很迷茫很着急，就算现在暂时不 All In Web3 也没有关系，可以以兼职的形式或者去社区玩耍、听听播客、读白皮书、读行业专家的文章等等，方式很多，总之不要为了赚快钱，而是真的去扎稳根基，慢慢一定会有工作合适自己。等自己开始有经验了，能说出自己在 Web3 的一些思维逻辑，甚至炒小图片亏了很多钱但你总结出有用经验，一旦到了这个阶段，我相信去找一份 Web3 工作一点都不难，水到渠成。

当女性遇上 Web3 

现在很多 95 后、00 后进入 Web3 ，整体感觉都是很年轻很有活力。但你或许猜不到，我已经是一个孩子妈妈了。作为女性，我很高兴遇到了 Web3，它让我在家庭与工作之间找到了平衡。

现在我的工作只需交付结果，不用 996，不用花大量时间处理人际关系或者大量会议导致精神内耗，甚至可以申请远程办公。这让我有更多的时间，去照顾我的家庭，甚至以后我有机会做数字游民，还可以带孩子出去，白天工作，晚上陪他玩。

我很喜欢 Web3 这种自由的工作状态。

如果想和我深入交流，可以通过推特找我：@evaevawava，我也会经常分享自己的生活日常。

设计  

卡尔，曾在互联网公司负责视觉设计

Web3 年龄：6 个月（不含 2 个月「归零出局」的炒币经验）

当前职业 1 ：NFT 项目产品设计 

当前职业 2：兼职多个项目 NFT 设计

身兼多职的状态太爽了！

这个兼职的项目我画了#1035-1535，还有一个 Rainbowbaby，一个 babyape，一个骷髅，一个柴犬，都是像素风。我正职公司做的项目也有两个，不过我公司属于半匿名团队，我就不透露太多信息了。

去年年底 NFT 大火，对于很多设计师来说，像是找到自己的 「工作第二曲线」 。我自己今年也有在 NFTCN 运营个人品牌和卖数字藏品，当时还运营了一个好几百人的社群。现在国内数字藏品平台都很低迷，我的项目还在线上，但属于佛系卖完一个上一个。

从视觉设计到产品设计的转型之路

虽然我现在 Title 变了，但工作模式和以前没什么区别。

NFT 项目产品设计，主要就是负责 NFT 层级逻辑，包括：properties 稀有度，生成工具等，展开说就是测试每个层的顺序，兼容性，尽量减少最终跑出来的图的穿帮率。你可能不知道，NFT 的穿帮率其实非常高，现在你在 OpenSea 看到的那些成品 NFT 都已经是经过精挑细选的，在前期我们会删掉很多不好看的、雷同的、穿帮的。越是画风精细，部件造型多变的 NFT，穿帮率越高，像 Azuki 明显就是有很多需要适配的。设计相关的工作，主要就是输出一些运营素材，比如：MEME，二创等。 

以前作为 App 视觉设计师，设计「捏脸头像系统」是我一个很重要的工作经历。什么是 「捏脸头像系统」，用大白话讲就是有点像是 QQ 秀 。但我现在看来，它的本质其实就是一套「 NFT 角色系统」。

我觉得，国内大部分岗位与 Web2 相比都是没什么太大变化的，只是接受的内容变成了 Web3 而已，国内目前很少纯 Web3 原生的工作。除非靠智能合约，拿社区治理代币的模式运作的，才是真正的 「Web3 公司」，国内屈指可数。

无心插柳柳成荫

进入 Web3 公司，说是偶然，但也是必然。

当时我刚在上一家公司离职，开始创作自己的数字藏品，叫：PXCA，我愿称之为「像素动物园」，在小红书做宣发数据都很不错，当时这个项目的设计、宣传、运营全都是我一人包办。 

就在那时候，一个 NFT 项目方通过小红书找到我，恰好他们项目也是像素风。其实在 NFT 这个圈子，挺多合作都是在小红书找的，小红书算是国内的 NFT 的图片素材最多的地方，很多 NFT 设计师都喜欢小红书上做宣发。

自己发项目先去小红书宣传，或许是设计师吸引 Web3 项目方关注的好方法。其实这与跟现在的「约稿」没有本质的区别。

英语能力和技术能力重要吗？

这个问题其实因人而异，如果是 Discord、Twitter 社区运营，英文就肯定要好，如果是开发肯定要技术背景。最终还是看岗位需求。现在阶段， 基本的岗位技能加上一定的 Web3 知识应该就够了。公司也不可能要求你一个设计，去解释「以太坊 2.0 分叉会产生什么影响」，顶多就问问「你是怎么看待 NFT 的未来发展」。

「不要想，去做！」

如果真的想要进 Web3，我给的建议是，走进新的氛围去体验一番，你才知道做什么。

比如 Web2 的运营可能都是微博、微信、小红书、抖音，跟 Web3 的主阵地 Discord、Twitter 的运营玩法是不一样的，有一定学习成本。

关于 Web3 建设，门槛比较低的最简单的就是买个 NFT，加入到一个 NFT 项目的 DAO 中，你就可以建设了。这里更像是一群人通过 NFT，聚到一起做项目，NFT 的收入是归项目发展金库的。这就像是创业，所以参与感会很强。NFT 标准路径往往绕不开「四件套」：发 NFT、发币、发地、元宇宙。当项目做大了，就离不开发币，因此在建设的同时，你也能有不错的收获。

一句话概括：光想，或者光「学习」的作用其实不大，行动力相对更重要。

另外，再推荐一些播客，可以听听：Web3 随意门、Web3 101、Buidler Talk、CSS | 探索 Crypto 的精彩世界，在小宇宙 App 都可以收听。

最后的最后，炒币不要轻易乱碰，赔钱玩意，「暴富」和「归零」就是一念之间，2 个月「归零出局」的亲身经历，让我告诫大家千万不碰这玩意。

Web3 是新互联网吗？

Web3 不就是互联网吗？那不就反过来问，Web2 的移动互联网还有啥新鲜东西？

现在还有很多人在问是否看好 Web3。我也好奇，Web3 有啥能让人不看好的点？只能说是有很多看不懂的点，但也没必要看懂。

研发  

RoCry，曾担任某互联网公司 CTO

Web3 年龄：6 个月

当前职业 ：Web3 打杂

我先后在人人网、LeanCloud、微信、BearyChat 负责移动端 / 后端开发，多年连续创业者。上一段工作经历，在国内某个视频社交产品担任 CTO，除了 Android / iOS / Backend 早期版本实现以外，还会负责公司整体技术团队发展。今年年初正式加入 Web3 创业公司，职业：打杂。

虽然我打杂（bushi），但我依然认为现在 Web3 整体行业还在上升的趋势，应该是有不错的前景。目前看还在早期，buidler 类的比较受欢迎，比如程序员。

有什么能力可以迁移到 Web3？

在我看来，最重要的是好奇心。Web2 和 Web3 岗位没啥区别，几乎所有的能力都能用得上。另外，英文是肯定需要的，不然很多信息都拿不到一手的。（其实很多新兴行业都是）有没有技术背景倒是都行，看特长和兴趣。

给 Web3 新职场人的小建议

如果要我提一些小建议，我要说的是：不要跟风，要有独立思考的能力。

我听说有一些同学希望快速参与到 Web3 建设。我认为大家完全可以从用户开始做起，从提一些建议开始，慢慢深入核心组织。

另外，我再推荐几本书，多读、多思考总是好的：

  • 《How to DeFi》，by Darren Lau, Sze Jin Teh, Kristian Kho, Erina Azmi, TM Lee, Bobby Ong

  • 《How to DeFi: Beginner》，by Darren Lau, Sze Jin Teh, Kristian Kho, Erina Azmi, Benjamin Hor, Lucius Fang, Win Win Khor

  • 《区块链与大众之治》，作者: 威廉•马格努森

  • 《区块链核心技术与应用》，作者: 邹均 / 于斌 / 庄鹏 邢春晓

  •  《区块链实战：从技术创新到商业模式》，作者: 冒志鸿，陈俊

品霖，曾在互联网公司负责客户端研发

Web3 年龄：6 个月

当前职业：Web3 产品客户端研发、智能合约开发

我是客户端研发的，有十年多的工作经验，接触和加入 Web3 行业也是半年的时间。我将会更多地从技术的角度去介绍 Web3 行业。

如何看待 Web3 的就业前景？

Web3 行业目前是全国乃至全球融资最活跃的行业之一，很多新的创业公司，大公司也有不少开始投入大量资源做 Web3 方向，个人认为 Web3 行业的就业前景是比较乐观的。

Web3 有什么工作机会？

Web3 行业的职位需求其实跟传统移动互联网行业差别不太大，我把我知道的大致总结下。

  • 技术类：客户端、前端、后端、运维、数据分析、区块链协议、区块链应用（智能合约）、跨链区块链，其中区块链协议和应用的职位比较火热，涌入的人也很多。相对的，客户端开发会比较紧缺，因为越来越多的项目需要开发移动端应用来普及。

  • 产品类：产品经理、项目经理、产品运营等。

  • 行政人力类：远程办公支持的行政、财务、HR、猎头等。

  • 艺术类：UI / UX 设计、NFT 设计、3D 建模、音乐人等

因为 Web3 的去中心化特性，办公地点和文化也越来越去中心化，远程办公越来越多，所以如何做好远程办公的行政支持也很重要，比如不同地点的员工如何缴纳五险一金、个税，是否需要个相对固定的办公室用于定期碰面、DAO 这类新型组织的薪酬分配等等，这些问题需要不断探索和解决的，这块做得好的公司对优秀的人才会有很大的吸引力。

英语能力和技术能力重要吗？

Web3 目前还是技术驱动占比较大，有技术背景比较容易进入 Web3 。当然，可以很快理解 Web3 各种概念的学霸就算没有技术背景也是没问题的，现在不缺科普视频和文章这些学习资源。

对于英语这块，因为 Web3 发源和热门地还是以欧美等国家为主，所以能看懂英文文档和文章是很重要的。如果英文水平不够，可能会不少信息差产生。更进一步，如果想要混入核心的 Web3 圈子，还要有英文的写和说的能力。

产品  

Kent，曾在互联网公司负责产品

Web3 年龄：6 个月

当前职业 ：Web3 产品经理

我是一名互联网产品经理，有几款社交产品项目经验，今年初开始接触 Web3，现在参与一个研发中的 Web3 社交项目，到现在刚好半年，下面讲一下我对 Web3 前景的看法和一些学习经验。

我为何看好 Web3 社交项目？

互联网和移动互联网都因为底层技术的更新，带来了新的产品机会。Web3 的底层技术是区块链，数据存储和交换可实现去中心化，例如用于交换加密货币的金融协议 uniswap、又或者基于以太坊去中心化借贷协议 aave，展示出 DeFi 在没有中心监管机构之下运行的可行性。因此，社交产品等其他场景，底层数据的存储和交换也完全有能力在脱离传统互联网限制下构建新的平台和用户组织形态，也就是社交产品常说的关系链和信息交换，这是支持产品创新的技术变革，即去中心化，另一种说法是 code is law。

而 Web3 也称为价值互联网，价值可 Token 化，价值直接通过网络传播，区块链是天然的价值传播共识，而无需中心化机构获得证照或监管同意去建立金融网络，区块链更容易帮助产品和个体获得价值和交换价值，这种新的价值交换方式决定了新的社会关系组建成为可能，比如我们看到 DAO 这种新的组织形态，意味着新的社交平台出现在区块链之上。

以上两点从技术上和需求上构成了 Web3 社交行业的变革机会，我们的新项目也与此相关。

Web3 躬身入局，需要什么能力？

目前，Web3 处于早期，基础协议和设施仍在建设阶段。从传统互联网进入 Web3 ，首先需要了解的是整个 Web3 的技术框架，我认为是很必要的。学习的过程可以按照「公链 - 协议 - 应用」这样的层级去看，不同公链横向对比会发现整个结构是很像的，各个公链的应用都是基于该链的协议去实现。如果从事产品研发工作，至少需要了解 Bitcoin，Etherum。

Etherum 纵向了解智能合约、最常用的 ERC 协议，DeFi 及各类 DApp。有了对 Etherum 深入了解之后，按照 「公链 - 协议 - 应用」 这样的层级去看其他公链，就会发现模式相似之处，实质上是大同小异。现在 Web3 大多数项目属于 「协议即产品」 的阶段，需要有技术背景和英语能力，才能够深入理解背后的原理和阅读英文文档。社群内有不少整理清晰的 notion 可以作为资料索引，或者到 bilibili 搜索一些区块链相关视频学习。Web3 学习的难处在于信息量爆炸，需要提纲挈领，按一条清晰的路径去分类知识。

Web3 需要怎么样的产品经理？

现在没有一个可参照的模版。我经历过移动互联网的早期阶段，那时候一个 App 的 PM 就是产品 / 公司的负责人，我觉得 Web3 目前正需要这样人，一个有产品方向判断力、理解技术可行性、能画交互、能给产品做早期推广的 PM 。移动互联网发展到成熟阶段，PM 的工作已经不像前期那样的产品 CEO ，被细分为交互 / 运营 / 数据 / 商业化 / 中台等岗位，不同 PM 看到都是产品的一面，现在 Web3 产品能提供的对应岗位是比较初级的，或者还没有。

Web3 处于这样一个属于创业期的阶段，对于希望从事产品早期工作的 PM 来说，是一个黄金时代。

真的要 All In Web3 吗？

今年以来，很多互联网人士都想 All In Web3，但适合大多数人 All In 的机会少之又少。我反而觉得可以先以参与者的角色加入到一两个项目 / DAO 中，感受一下 Web3 的氛围和做事方式，再结合自身优势确定职业发展。最后列几个我看到的适合非技术类人员参与的角色：

  • 产品体验优化：当前 Web3 项目大多数由技术人员主导，无论用户体验还是交互设计都很 Geek ，产品经理完全可以从一名小白用户角度提出优化方案。

  • 社群运营：私域流量大家一定不陌生，到了 Web3 变成了社群运营。懂用户，与大家打成一片，能给项目带来可观的新增。

  • 用户增长和裂变：新技术需要传播手段，用户拉新和裂变这样的方式同时适合 Web3 项目从传统互联网获得流量。

  • DAO 建设管理：产品经理和项目经理拥有超强的沟通能力，同时能帮助 DAO 搭建高效而开放的管理方式，使用各种工具（如 Notion ）使得 DAO 透明化。

  •  商务合作：Web3 项目目前具有高热度的话题性，跟传统品牌结合能带动品牌形象升级。
