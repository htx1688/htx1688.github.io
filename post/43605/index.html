如今，几乎所有人都在关注视频流媒体，疫情加速了全球流媒体平台的发展。 Netflix、Disney+和HBO Max等热门视频流媒体平台受益于该行业的快速增长。但也有很多失败的尝试，包括Quibi和CNN+。

归根结底，其实是有两种类型的视频流平台：基于广告的和基于订阅的。基于广告平台的例子是YouTube，基于订阅的例子是Netflix。连接这两个平台的一线希望是什么呢？答案是中心化。随着Netflix失去其市场主导地位，订阅平台的竞争越来越激烈，但广告平台仍处于YouTube的垄断之下。

对现状持异议的人通常会感叹其封闭的算法可以审查内容。在不涉及一些层面的情况下，当前生态系统还是缺乏透明度，因此催生了“不受审查”和“去中心化”的新型平台。

问题就是这些新的去中心化平台如何将技术作为其理想的保障。技术方面的内容值得研究，本文会涉及到该部分。

什么是去中心化视频流?

去中心化视频流是指单个实体无法控制流媒体服务的内容交付。必须注意去中心化视频流和分布式视频流之间的区别，以了解和区分差异。分布式视频流意味着内容传递网络在一个实体的支持下全球传播，而去中心化则完全消除了控制。它创建了一个网络，参与者可以就该网络将做什么达成共识，增加了一层灵活度，并赋予社区更多的权力。

去中心化视频流平台通常是社区拥有的。区块链支持平台内的所有权，这是去中心化的基础，为应用程序提供了完美的基础层。区块链层是一个薄层，通过一个去中心化账本将所有部分连接起来。

所有链下交易必须向区块链层提交证明，以验证交易是否合法。区块链成为唯一的事实来源。尽管如此，由于区块链无法存储视频或图像等复杂数据源，IPFS或简单的点对点存储等链下解决方案使数据可用。

点对点技术是基于区块链视频流的先决条件

虽然区块链成为了基础层，但像IPFS或BitTorrent这样的点对点协议必须是完全去中心化的先决条件。这是因为区块链本质上是去中心化的，如果人们决定运行自己的节点，任何人都可以持有整个区块链的副本。但是对于文件系统，不使用点对点协议会比较困难。

文件系统通常是中心化的。谷歌云存储、Amazon S3和Azure存储等服务是中心化存储的有效证明。但如果要给用户一个完全去中心化的体验，是不能在去中心化的生态系统中依赖这种技术的。

去中心化视频流媒体技术问题

货币化和激励机制仍然是去中心化视频流媒体平台的突出问题。不是因为它们不能赚钱，而是与YouTube这样的中心化平台相比，去中心化的视频流媒体根本没有机会。但本节将讨论技术问题，而不是纯粹的业务模式问题。

首先想到的问题是文件存储和权限管理，但与空间相关的问题并不仅限于此。例如：

  • 在点对点网络中，用户需要保存视频文件，并为点对点连接提供良好的互联网连接。

  • 一个去中心化的生态系统也需要有保护原创艺术家创作的版权材料的必要性。

  • 制衡机制抑制了试图破坏生态系统的恶意用户。

  • 特定于内容的货币化方案。

文件存储问题

对等文件系统总是会遇到文件可用性的问题。存储是昂贵的，如果没有明确的激励措施，用户将不会在他们的系统上托管文件，更不用说保持健康的互联网连接以供其他用户访问了。

由于缺乏提供者，点对点文件共享可能会失败，但并非是创建文件共享生态系统的失败。只有当另一个同伴在线并愿意为用户提供文件时，用户才能获得该文件。因此，用户至少需要另外一个对等体来进行文件共享。当然，在这样的点对点生态系统中，当所有的对等点都断开连接时，创建者最终可以求助于一个中继节点来提供丢失的文件。

但这将重新聚焦到最初的中心化问题，主要是因为如果一个中心化的力量可以为整个社区提供服务，那么社区有什么动力能够自我可持续发展呢?在这种情况下，许多项目为持久去中心化的数据共享提供了解决方案。Arweave、Filecoin和BitTorrent等项目为同样的问题提供了不同的解决方案。

Arweave是一种永久性的去中心化数据存储，主要用于网络存档。Arweave依靠一站式支付方案来确保档案的可用性。用户预先付费在网络中存档他们的文件。

Filecoin是一种允许用户出租可用硬盘空间的协议。它作为一个建立在IPFS上的点对点文件系统，允许有多余硬盘空间的用户出租他们的系统来挖Filecoin。与此同时，Filecoin的用法听起来可能像谷歌Drive的现收现付方式。它是对等文件系统的最佳替代方案之一。

用户还可以使用IPFS来存储数据。但是普通的IPFS仍然要求用户托管用户的文件，或者如果用户想让文件在网络中保持可用，就需要为托管文件的服务付费。DTube是一个去中心的视频流媒体应用程序，是YouTube的直接竞争对手，它利用IPFS进行文件存储，并使用Steemit区块链来激励它的创造者创造高质量的内容。

BitTorrent是这四种协议中最有趣的，因为它是一种点对点协议，不依赖于那些主机节点的用户，而是使用称为播种者和下载者的概念。播种者是已经下载文件并将其提供给其他用户的用户，实质上是使用他们的网络带宽将文件上传到网络。下载者是请求文件并将文件下载到文件系统中的用户。Swarm是BitTorrent生态系统中播种者和下载者的集合。

在BitTorrent中，播种者会因对网络的贡献而获得BitTorrent币奖励。相比之下，下载者 将需要使用BitTorrent币来下载文件或提高下载速度。

虽然BitTorrent听起来是最去中心化的选择，但它依赖于一个中心化跟踪服务器来记录种子者的ip，并将其提供给文件的请求者。和其他点对点文件共享协议一样，如果没有种子程序，就无法下载文件。

Theta是一个区块链基础设施，它提供去中心化视频流，是上述三个基础设施项目中最知名的。Theta使用中心化的点对点存储解决方案，称为Theta Edge Store。作为其存储解决方案的一部分，Theta的去中心化存储与中心化云基础设施兼容，包括AWS S3和谷歌云存储。中心化云提供可靠性高的存储，而去中心化的部分在使用中心化系统的成本过高时起到容错的作用。

Theta还将拥有一个点对点共享网络，以协助传输流。然而，点对点共享网络并不存储视频，而只是帮助传递内容。

中心化解决方案不再是做事的实际方式，特别是在Web 3.0中。

内容保护

下一个主要问题将是版权问题，如果用户已经解决了持久存储的所有问题。用户不能把有版权的材料公开给所有人看。它侵犯了原始材料创作者的权利，这是p2p文件共享背后的主要问题之一。点对点文件共享背后的技术并不违法，但使用它来共享受版权保护的材料仍然是非法的。

用户不能依靠陌生人的诚实或善意来保护用户的版权。最好有一种方法可以直接保护用户的文件，同时又不妨碍用户从文件中赚钱。权限管理应用程序通常对用户的内容进行编码和加密，使其无法被任何人直接访问。但是如果想对内容进行编码，一个单独的转码者就可以转译它。

在点对点文件共享解决方案中有许多内容加密的例子。通常，那些依赖按次付费的架构需要文件编码，以防止内容从网络中流失。目前，有四个知名平台提供这些服务：LBRY、Livepeer和Theta。

LBRY是一个去中心化的内容分享平台，旨在对抗像YouTube这样的中心化内容分享服务，并创建一个用户拥有的生态系统。作为一个平台，LBRY用户在其上构建自己的应用程序，Odysee就是一个很好的例子。在将内容存储到去中心化的生态系统之前，LBRY通过将内容分成更小的块并进行加密来对其进行编码。每个数据块可以位于不同的服务器中，LBRY需要反向操作来观看视频。它将首先收集所有较小的块，解密它们，并将它们连接到单个文件中。

Livepeer是一种协议，它通过提供经济激励来存储内容，并让用户托管给可以转译内容的转码者，从而激励去中心化的视频直播流媒体。与LBRY相比，Livepeer将内容存储和代码转换过程进行货币化。存储货币化使用IPFS/Filecoin，用户必须向存储提供商付费以保存他们的数据。然而，Livepeer解决方案的新奇之处在于他们的代码转换过程(或LBRY中的解码过程)。

Livepeer实现了一种转换代码的权益证明机制。每个转码者将不得不投入一定数量的Livepeer代币来从Livepeer生态系统接收转码工作。因此，Livepeer不使用密码学，而是依靠激励机制来保护内容。转码者将可以完全访问整个Livepeer媒体库。尽管如此，需要注意的是，每个转码者都需要维护自己的声誉来获利。

Theta是上述三者中最发达的一个，其整体基础设施中有多个分布式和去中心化系统。Theta有一个元链，它将成为唯一的事实来源。虽然Theta生态系统中的所有操作都是链下的，但Theta利用零知识证明来验证提交到元链的数据。

Theta的内容加密平台严重依赖Metachain。用户必须购买NFT来进行身份验证、观看内容和查看付费内容。编码过程与Livepeer的解决方案类似。首先，一个单一的Theta Video API将作为编码和交付基础设施的抽象层。在编码基础结构中将有一个密钥服务器，存储每个内容的所有现有解密密钥。

消费者必须购买NFT，并通过 Theta Video API 向 Metachain 验证其所有权。如果验证成功，Theta Video API将检索解密密钥。一旦Theta Video API检索到解密密钥，它将连接到交付基础设施，为消费者解密内容。

内容加密是中心化和去中心化视频流媒体应用共同面临的问题。受版权保护的材料很有价值，需要加密。目前最难解决的问题是去中心化的内容保护。

创造一个无需信任的生态系统

当使用点对点视流媒体生态系统时，用户不得不信任网络中的其他计算机，相信它们会为用户提供正确的、未篡改的数据。但是在一个点对点的生态系统中，用户该如何实施这一点呢?像BitTorrent、IPFS和Arweave这样的项目是相对安全的软件。用户必须仔细检查自己从这些平台下载的内容，因为危险大多来自于文件本身。

如果存储提供商疏忽或提供恶意协议，则会削减其奖励。其实关键是要让提供者参与其中。鼓励存储提供者持久化文件，并为使用者提供正确可靠的文件。

好消息是如果我们将流式传输视频，那么接受的唯一文件类型将是视频扩展名。虽然文件可靠性仍然是一个问题，但是如果用户接收到不同的文件扩展名，就可以很容易地将其视为潜在的漏洞风险。大多数去中心化视频流平台不让用户直接访问文件；用户通过他们的web应用程序与他们进行交互。它给了用户额外的一层安全，知道用户不会在用户的设备上执行视频。浏览器会复制视频而不保存它们。

基于区块链的视频流平台为每个设备转换视频代码。因此，每个平台都需要有一种安全的方式来为使用者转码文件。在去中心化的生态系统中，代码转换本身可能成为一个问题，因为无法保证每个代码转换的可靠性。

Livepeer的权益证明转码机制类似于工作量证明，需要用户投入足够数量的Livepeer代币作为抵押品。具体方法是通过锁定代币，每个转码者都有一些代币，如果他们的行为不恰当，就会失去其代币。攻击整个生态系统以进入其中将非常昂贵，并且需要大量的代币。

创建一个无需信任的生态系统，需要激励最重要的参与者适当地执行任务。如果不对不良行为进行惩罚，整个生态系统可能会变得不可靠，或者成为恶意软件和不良行为的温床。

流媒体货币化方案

内容创作并不便宜，创作者需要通过在平台上发布内容来赚取收入。每个平台的收入来源不同；有些采用订阅模式，有些则采用按次付费和基于广告的模式。尽管并非所有的商业模式都是平等的，但在一定条件下，有些收入流比其他的更好。

举个例子来说，如果用户希望平台是免费使用的，那么YouTube基于广告的模式可能是最好的方法，而Chainflix便是一个很好的例子。Chainflix是一个去中心化的视流媒体基础设施，可以说是基于广告的流媒体的一体化软件包。

视频流内容并不便宜，需要付出很多努力来制作。Chainflix专注于激励用户创造优质内容，同时为观众提供一个免费使用的平台。它的商业模式类似于YouTube，因为它们完全依靠广告为创作者创收。

但不同之处在于，Chainflix采用去中心化架构，放弃了对其平台上发布的内容的控制，并利用PoV区块链将其广告货币化。PoV机制的好处在于，PoV 机制的好处是它可以保护系统不被寻求通过观看视频获利的机器人薅羊毛。

为了控制内容的流动，Chainflix 利用控制器将客户端与使用 API 的去中心化后端存储连接起来。Chainflix还利用人工智能来提高存储效率，并创建更智能的去中心化存储方案。Chainflix 生态系统以效率为中心，利用区块链来货币化和存储内容。其与LBRY不同，LBRY采用按次付费的模式。

LBRY的商业模式是不同的；它有自己的代币LBC。LBRY有两种收入来源，其网络内的工作证明挖矿和按次付费模式。内容创作者的主要收入来源将是按次付费模式。用户可以锁定自己的内容，并设置查看它们的最低价格。但这并不局限于按次付费。消费者也可以使用LBC给用户小费。用户不需要第三方Patreon账号来启用小费功能。

挖矿交易也可能是有利可图的。LBC的开采周期超过20年，每开采100个区块，区块奖励就会增加，在500个区块达到峰值，之后缓慢下降。然而，挖矿奖励并不局限于区块生成，因为矿工将从交易验证费中获得LBC。LBRY采用按发布付费的模式，以激励矿工验证每笔交易。它还阻止创作者向区块链发送低质量的内容。这种模式的唯一缺点是，用户需要资金(尽管是少量资金)在LBRY上发布内容。

基于广告和付费观看/订阅模式是整个行业的标准。第三个全新的选择是内容奖励。Steemit是一个基于Steem区块链的激励型公共内容平台。在基于Steem的平台上，用户发布的每一个内容都将获得奖励。内容越好，玩家获得的奖励就越多。Steem通过受欢迎程度来衡量优质内容，主要是通过好评和非好评来确定奖励。

这是一个相当新的概念，其主要缺点是需要凭空创建新的Steem代币。视频流媒体平台DTube使用了Steemit，但目前尚不清楚该平台是否仍在盈利。基于发布的内容和内容的声誉创造经济并不是一个好兆头，因为人们没有动力去创造高质量的内容。在Steemit上定义高质量内容背后的参数可以很容易地使用机器人进行干扰。甚至一项研究表明，Steemit中16%的加密货币交易是机器人。

在一定程度上，为内容创作者创造适当的激励机制是非常困难的，这取决于用户的项目所追求的模式。每一种都有其缺点，并能在其周围孕育出不同类型的社区。免费使用的网站通常会采用基于广告的系统，而付费使用的网站则不可避免地依赖于内容的质量来获得浏览量。创造内容以获得奖励的方法是比较新颖的，但它能否经受住时间的考验仍是个问题。

常见的区块链视频流基础设施

在所有区块链视频流媒体项目中，它们的大多数基础设施基本一样。每个区块链视频流基础设施通常包括：

  • 智能合约

  • 文件存储

  • 内容代码转换器

  • API层

虽然公共基础设施通常指的是这四个，但像Theta这样的项目也有适当地扩展其基础层的基础设施。但是它们是如何相互作用的呢?

通常，这个过程如下：

  • 首先，希望流式传输视频的用户将通过平台的应用程序接口和 API 层访问内容。

  • 然后，当用户找到他们正在寻找的内容时，他们将使用智能合约购买访问权限，并使用加密货币支付。

  • 之后，平台可以使用两种方法：使用NFT进行身份验证，或者无需进行任何身份验证就可以立即提供内容。

  • 最后，用户通过API层检索内容，连接到内容转码者并访问文件存储中的原始数据。内容转码者作为内容网关，根据用户需要解密内容。

虽然这一切听起来很简单，对许多人来说可能很有意义，但创建连接到去中心化存储的去中心化转码者对大多数人来说仍然是一个巨大的挑战。首先，内容创造者如何才能相信转码者是公平的?去中心化的内容转码者不会由那些有动机保护视频的公司来运行，它们可以由普通人来运行。

Livepeer试图通过创建一个基于POS的转码生态系统来解决这个问题，在这个生态系统中，任何托管转码服务器的人都必须投入足够数量的代币，才能被允许进入网络。如果一个恶意的转码者违反了生态系统的规则，他们的质押将被削减，而质押的回报也将不复存在。

将PoS与代码转换结合起来是激励良好行为的好方法。

下一个问题将是API层。API层虽然是项目中比较简单的组件之一，但仍然需要中心化。可靠性是大多数项目需要考虑的一个因素，如果API层是去中心化的，就会出现将所有数据存储在哪里的问题。

当然，我们可以说自己就可以查询智能合约。但是，如果智能合约是在像以太坊这样的平台上，并收取高额的gas费，那会怎么样呢?

创建一个新的区块链运行怎么样?当然，这是可能的，但是用户最终会在API层之后添加额外的层，从而增加项目本身的复杂性。无论如何，查询NoSQL或关系数据库比查询区块链要快得多，用户甚至可以在数据库中创建复杂的查询，而在区块链中很难或几乎不可能有效地执行这些查询。

对于视频流，用户需要高可靠性，但是用户如何确定去中心化文件系统能够胜任此任务呢?如果用户经常中途失败怎么办?

我们需要为自己提供的视频添加一个额外的层；用户可以使用CDN或者外部存储。但是将视频放在去中心化文件系统之外的外部存储上多少有点违背了初衷。就连Theta也将云提供商作为其视频流生态系统的基本构建模块。

当然，点对点存储是一种可行的替代方案，但点对点永远无法击败现代世界中开发人员可用的始终可靠的 99.9999% 的云存储。这的确是一个困境，但肯定是一个有趣的困境。

概念上的真正去中心化的视频流媒体平台可能是工程上最难实现的壮举之一。如果没有云存储、中心化API和数据库等云服务的可靠性，就很难提供一个可行的流媒体平台。

结论

总之，区块链正在用尖端技术和解决方案革新行业。本文探讨了四个去中心化视频流基础设施项目：

  • Theta

  • Livepeer

  • LBRY

  • Chainflix

与其他相关的存储区块链项目（如 IPFS、Arweave、Filecoin 和社交区块链项目）一样，Steem也鼓励内容创建。

加密货币开启了一系列新的激励模式，以在去中心化的混乱中维持秩序。许多项目都在尝试不同的模式，以找到最赚钱和最可持续的方法。但只有时间才能证明哪些项目能够延续下去。
