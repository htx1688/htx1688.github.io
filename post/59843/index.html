9 月 25 日，ETHGlobal 公布纽约黑客松 13 个决赛入围项目，覆盖游戏、Bot、NFT、账户抽象等领域。Odaily星球日报对这批项目介绍如下：

FRAMED!

FRAMED!，链上且无需信任的隐藏信息游戏。

在游戏中，一个艺术品窃贼从博物馆偷走了一幅画，玩家被随机分配为市民、小偷、警察或侦探等角色，这些角色的信息以加密值的形式存储在链上。游戏的目的是让小偷消灭市民，而市民则共同努力找出并投票选出隐藏的小偷。玩家之间存在讨论和社交推理的元素，他们必须合作并欺骗对方。

这款游戏的与众不同之处在于它能够直接在链上存储加密状态。启用 FHE 的智能合约允许在加密数据上运行计算，而无需解密。这一创新解决了隐私限制和保证问题。

Framed！由以下技术构成：包括用于构建用户界面的 React、用于游戏机制的 WAGMI 和用于网络测试的 Inco Testnet。Privvy 为社交登录功能进行了整合，确保用户能够顺利访问。该游戏采用完全同态加密技术来促进链上的私人行动和角色分配，使玩家能够在保持隐私的情况下进行互动和决策，例如对其他玩家采取行动。这项技术允许直接在链上进行加密计算操作，无需解密，为链上游戏体验设定了新标准。

F.A.S.T.

F.A.S.T.，全称是“Facial Address State Transfer”，集成了多种产品和技术的 Telegram Bot。

F.A.S.T 项目来源于黑客马拉松上的一次谈话，有人说，只要拍下某人的脸，就能把钱发送到他的钱包地址，这一定很酷。

由于存在一定的技术门槛，Web3 的采用率相当低。用户不知道自己的地址，也几乎肯定不知道别人的地址。F.A.S.T.假设，人工智能将通过以下方式实现 Web3 的超高普及率：人们将拥有更简单的聊天界面，而不是难以理解的用户体验；人工智能将能以更好的方式将人们与他们的地址联系起来。

F.A.S.T.集成多种技术和产品：Telegram、图像索引技术、AWS S 3 + DynamoDB + Lambda、Vercel（托管商）、OpenZeppelin 合约（用于部署 NFT 合约）、Airstack（使机器人能够了解经过验证的用户并创建互动社交图谱）、Polygon（主网）、Linea/XDC（多链支持）、SAFE、ENS（为经过验证的用户启用子域）、Mask。

ZeroTrustBounty

ZeroTrustBounty，零信任赏金，致力于解决传统漏洞赏金平台上的挑战。

想像一下报告一个漏洞给了一个漏洞赏金计划，然后没有得到支付。ZeroTrustBounty 通过使用 TLS 公证和 Lit 协议来解决这个问题。假设用户有一个 SQL 注入漏洞，ZeroTrustBounty 允许用户隐藏“攻击的 HTTP 请求“，但是透露经过密码学验证的 HTTP 响应，这样受影响的业务可以通过密码学验证用户的报告并了解影响。现在到了付款部分，ZeroTrustBounty 使用 Lit 协议对 HTTP 请求/响应进行加密，业务在支付给地址或持有 NFT 等之后可以解密。这样，如果业务有兴趣修复该漏洞，就可以确保付款，并消除像白帽子没有得到报酬或第三方服务窃取漏洞技术/报告的问题。它还解决了企业向第三方服务披露敏感信息的问题。

该系统使白帽和业务保持利益一致，并创建了一个正确定价的漏洞报告市场，随着越来越多的白帽和业务使用这个平台，平台可能会选择购买比第一个报告更便宜的白帽报告，从而选择自己的风险阈值并激励白帽合理定价。

Doom Arena

Doom Arena，以去中心化的方式举办游戏竞赛。

每年都有数以万计的游戏发布，游戏开发者需要找到独特的方式来吸引玩家。该项目通过允许开发者举办竞赛提供了一个解决方案，为玩家提供激励，并为开发者产生收入。

Doom Arena创建了一个去中心化的平台，用于举办游戏竞赛，为游戏开发者提供一种创新的推广游戏和吸引玩家的方式，同时为游戏推广建立可持续的经济模型。任何人都可以发起或参加竞赛，为活跃的游戏生态系统做出贡献。

具体，比赛开始后，玩家将购买一张游戏票以参与游戏，购买游戏票将增加奖金池，激励更多玩家参加比赛。玩家完成游戏后，他们将提交游戏的哈希值，并可以通过他们的 ENS 地址查看其他参与者，但无法看到他们的得分。时间限制结束后，玩家将提交游戏的最终日志以验证他们得到的分数，这也将允许每个人查看其他玩家的游戏并验证他们的分数（如果他们愿意的话）。

奖励分发时，每个玩家将获得上一位玩家奖金的 0.5 ％，直到所有玩家都得到支付。如果有剩余奖金，将归游戏主办方所有。然后等待下一位主办方启动比赛，再次开始玩游戏。

AirTracker

AirTracker，使用 AirTags 和区块链存储及消息传输进行去中心化实物资产追踪。

AirTracker 使用 AirTags 来记录你的实物资产位置的不可变记录，它从苹果设备上获取每个 AirTag 资产的位置数据，对位置数据进行加密，并将其存储在链上和 IPFS 上。位置更新还通过使用 AirStack 进行 XMTP 与 ENS 查询方式发送。

用户可以登录 Web 用户界面，查看地图上每个资产的历史位置，以及具体的时间和位置更新。

用户只需分享生成的随机标签名称（也充当位置加密密钥），就可以与任何人或应用程序共享资产位置。

SafeCreate 2 

SafeCreate 2 ，使用单链 Safe 安全部署多链智能合约。

部署有权限的多链合约非常困难。用户要么使用基于 nonce 的部署方式，并祈祷 Vitalik 永远不会丢失私钥或搞乱 nonce 的顺序，要么接受每个链都有管理员 Safe 带来的多签管理混乱。

Safe Create 2 使得在单个链上的一个 Safe 可以将智能合约部署到所有链上的相同地址上，而不需要桥接以太坊，还允许单链 Safe 在任何一个多链合约上执行管理员功能。这给项目带来了最好的双重优势- 简单易用的多链部署与单个 Safe 的安全性。

ConsciousNFT

ConsciousNFT，为 NFT 项目创始人和所有者提供了一个人工智能引擎工具集，用于共同构建丰富的内部世界，让 NFT 艺术品超越静态图片，赋予每个 NFT 独特的个性、背景故事、对话能力和叙事深度，成为一个有意识的角色。

创建挑战和任务，让创造的 NFT 参与并获得生成性 NFT 奖励。创作互动叙事，让有意识的 NFT 建立关系，参与战斗或者探索冒险。

通过 ConsciousNFT，让社区共同创作叙事，为 NFT 收藏注入生命。将经过训练的 LLM 模型上链以保留互动人物形象。

唤醒每个 NFT 内独特的意识身份，共同构建互动神话和叙事，创建挑战和任务来推动参与度，将 LLM 模型上链以保留叙事内容，赋予每个 PFP 以个性和对话能力，与社交媒体整合以实现自然发现，使 PFP 能够培养粉丝团和社交关注，将静态 NFT 艺术品转化为有意识的数字人物，通过有生命的角色身份将艺术与元宇宙相连接。

Space Guardians

Space Guardians，通过 Snaps 构建的 MetaMask Guardian 管理系统，用于卫星安全访问恢复。

在Web3中，“忘记密码”经常发生，现有的方案存在漏洞：守护者的安全可能会受到威胁；通知（减少串通事件可能性）。Space Guardians 为社交密钥恢复方案开发了一个安全解决方案，即 MM 快照+ Cryptosat 卫星。这解决了通知和审计记录的管理，从而降低串通事件的可能性。

A.S.R.

A.S.R.，全称是“Anti Social Recovery”，可以让用户在丢失签名密钥后重新取回智能合约的所有权。

如果合同的所有权属于人而不是地址，会怎样呢？A.S.R.允许用户部署智能合约，其所有权与其个人身份的唯一证明（例如，World ID）相关联。在使用 World ID 验证后，用户可以将合同部署为所有者，然后从不同的 EOA 接管所有权。

这种设计模式可以适用于许多不同的用例。虽然 A.S.R.从一个简单的可拥有合约开始，但同样的原则也可以应用于任何智能合约角色（发行者、暂停者、销毁者等）。此外，可以使用不同的个人身份证明提供者来交换 World ID。A.S.R.还考虑过如何将其应用于 ERC 20 ，使用户可以从新地址上提取他们的资金。

MEVictim Rebate

MEVictim Rebate，利用链上历史数据识别 MEV 受害者，并在 Scroll 上向受害者空投代币以访问代币门控 Uniswap v4 池，从而设计出更好的市场激励措施。

该项目旨在识别 ETH Goerli 上的 MEV 攻击受害者，并向这些受害者提供一个 ERC-721 代币，持有者通过向 Scroll 上的 Uniswap v4池提供流动性，可以获得折扣。最终，该项目通过 ETH Goerli 上用户的历史链上数据激励 Scroll 上更好的行为。

这是一个强大的新基本原理：通过对历史链上行为进行证明，并利用这个证明来使某些用户在复杂的下一代 DeFi 中获得特权。

首先，使用 ETH Goerli 交易的子图来识别潜在的 MEV 受害者（即那些被抢先交易/抢夺交易/回补交易的个人）。然后，将交易哈希和日志 ID 作为输入提供给 Axiom 电路。

Axiom 电路使用交易哈希来证明钱包地址是否确实是 MEV 攻击的受害者。如果符合条件，钱包地址将收到一个 ERC-721 代币（使用 Unlock Protocol 创建的 Nouns NFT，合约名为 MEVictim），然后，可以使用 Hyperlane Warp Route 部署将 ERC-721 代币桥接到 Scroll。

只有 ERC-721 代币的持有人才能在 Scroll 上的代币门控 Uniswap v4池进行交换。虽然我们最初希望使 ERC-721 持有者有资格获得手续费的折扣，但我们时间不足，希望将这种复杂性在未来添加。我们将代币门控实施为一个概念验证，以证明 Uniswap v4挂钩可以用于向代币持有者提供这种折扣。

RealReturn

RealReturn，是一个由 TLSNotary 和 Jomo 支持的、兼具隐私和可验证性的真实投资回报率开放排行榜。

直到今天，人们只能通过检查截图来识别，或者试图根据数字进行常识判断。除非账户持有人愿意公开分享他们的凭据，否则任何一方都无法真正证明自己。

现在，通过使用 Jomo 和 TLSNotary 的 RealReturn，可以使任何人在不向任何人或任何地方分享凭据的情况下，公开证明他们的投资回报。使用诸如 3 方 TLS，MPC 等密码技术将用户的 TLS 会话与应用服务器相关联，以便即使会话的内容仍然只对用户和应用服务器可见，公证人也可以在一侧生成一些内容，以证明该转录没有被篡改。

Abstract Wallet

Abstract Wallet，钱包抽象化使用户能够通过单击即可迅速注册，仅使用密码抽象化的智能手机即可生成密码凭证，创建智能账户，铸造 NFT，一键完成！钱包是隐形的。

该项目通过一键式操作完全无摩擦地为用户提供接入服务。结合了 Solidity FCL 库（secp 256 R 1 是最佳的 EVM 实现）来实现 WebAuthn 支持的用户运营，还结合了 Infinitism ERC 4337 入口点和自定义的支付主站来支付用户费用，使其对于完全新手来说也非常易用。

XSafe

XSafe，是一种旨在优化 Gnosis Safe 多签名框架以实现多链交易的解决方案。核心目标是减少在多个区块链网络上执行相同管理功能所涉及的时间和复杂性。通过 X-Safe，用户可以将单个签名和单个交易在多个链上传播，从而减少开销并加快工作流程。

技术特色：

  • Gnosis Safe 集成：X-Safe 是建立在 Gnosis Safe 多签框架之上的，确保它继承了 Gnosis Safe 的强大安全措施和功能，同时增加了简化多链能力。

  • 单个签名传播：传统的多签操作要求用户在每个区块链网络上单独签署每笔交易。X-Safe 通过将单个签名自动传播到所有集成的区块链中，简化了这个过程。

  • 单个交易广播：与签名传播类似，X-Safe 允许将单个交易广播到多个区块链。这对于在不同网络上执行相同的智能合约方法或管理功能非常有用。

  • 链无关：X-Safe 被设计为区块链无关，意味着它可以与任何由 Gnosis Safe 支持的区块链集成。

可以用于：

• 在多个区块链上部署相同的智能合约；

• 在存在于多个链上的 DAO 上更新治理参数；

• 为多样化的投资组合进行跨链资产管理。

通过 X-Safe，开发人员可以显著简化他们的跨链操作，使系统更加高效、安全和具有成本效益。
