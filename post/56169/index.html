引言

在活字印刷术诞生之前，如果人们想要印刷一本 1000 页的书籍，会怎么办？只有让石刻匠雕刻 1000 个满是字的石块，费时又费力，而且这 1000 个石块仅能用于这一本书籍的印刷。

然而，在有了活字印刷术后，如果想印刷一本 1000 页的书籍，就再也不需要雕刻 1000 个石块。而只需要造出有限的字库，直接从字库中拿出固定的字块进行组合拼接，就可以在很短的时间内组成需要印刷的书籍。

这就好比加入递归铭文的 Ordinals 协议，它的诞生将比特币生态的拓展性几何级的扩大，MIIX Labs 认为递归铭文会成为 BTC 生态的下一个引爆点。

一、什么是 Ordinals

想要了解什么是递归铭文，首先我们需要先明白 Ordinals 是什么。

近年来，非同质化代币（NFT）在区块链世界中引起了广泛关注。从艺术品、音乐到虚拟土地等，NFT 赋予了各种资产独一无二的数字属性。然而，这个趋势在比特币区块链上并未大规模实现，直到 Ordinals 的出现。

Ordinals 是一个使得每个比特币中的聪(sat)都能得到识别，进而实现附加数据交易的协议，这个过程被称为”记录”。此项技术的出现，不仅为比特币创造了另一种用例，而且使得比特币 NFT 的实现成为可能。

比特币的每个聪都有其唯一的编号，这些编号是根据比特币被开采和转移的顺序赋予的，所以被称为“序数”（Ordinals）。此项技术使得每个聪都能够携带额外的数据，使其独一无二。

这意味着比特币的最小单位 — 聪，不再仅仅是价值的承载者，更可以作为信息的载体。

MIIX Labs 分析认为，Ordinals 与传统 NFT 有着本质区别，NFT 通常使用智能合约在以太坊、Solana 和 BNB Smart Chain 等区块链上进行制作，有时它们所代表的资产会被托管在其他地方。

但 Ordinals 直接记录在单个聪上，然后被纳入比特币区块链的区块中。由于 Ordinals 完全驻留在区块链上，不需要侧链或单独的代币，因此，Ordinals 继承了比特币本身的简单性、不变性、安全性和耐久性。

这意味着，只要比特币还存在，就可以一直访问存储在 Ordinal 中的数据。相反，如果 NFT 所引用的数据被删除或其存储平台消失，那么 NFT 数据也会丢失。

所以，Ordinals 的出现，为了比特币带来了新的可能性，也被称之为铭文 1.0 。

二、什么是 BRC 20 

随着 Ordinals 的出现，建立在比特币网络上的一种可替代性代币的标准也开始产生。

一种基于比特币网络和 Ordinals 协议的实验性代币 Ordi，其代表了 BRC-20 代币标准的首个应用实例，某种意义上，Ordi 引爆了 BRC 20 的行情。

据 MIIX Labs 了解， 2023 年 3 月 8 日，BRC 2 标准由推特用户@domodata 受以太坊的 ERC-20 启发，基于 Ordinal 协议创建而成，她允许开发者使用 Ordinals 协议创建和转移可替代代币。

与 ERC-20 不同，BRC-20 不使用智能合约，而是利用 Ordinals 协议的 JSON 数据来部署、铸造和转移代币。

BRC-20 标准的主要特点包括：

安全性：BRC-20 代币存在于比特币区块链上，因此继承了比特币协议提供的高安全性。比特币使用工作量证明（PoW）共识机制，并拥有大量的矿工来帮助保护网络，使其成为最安全的区块链之一。

兼容性：BRC-20 代币与比特币区块链本身兼容，这使得将其整合到比特币生态系统中变得更加容易。此外，现有的比特币开发者可以方便地构建和整合 BRC-20 代币。

可替代性：BRC-20 代币被视为可替代的，这意味着它们可以在交易所上轻松交易，为新手提供了一种简单的比特币交易替代方案。

然而，MIIX Labs 分析认为 BRC-20 标准也存在一些缺点：

生态系统有限：尽管 BRC-20 正在变得越来越受欢迎，但它是一个相对较新的代币标准，生态系统相对于 ERC-20 来说较小。这意味着为构建和管理 BRC-20 代币的资源、开发者和工具较少。

潜在的高交易费用：由于 BRC-20 代币活动的激增，比特币的交易费用随着 BRC-20 交易的增长而增加。

总的来说，BRC-20 代币仍处于初级阶段，需要更多时间来揭示它们对比特币区块链的影响。这个实验阶段为未来的更好设计选择和优化改进留下了空间。

BRC-20 代币表明比特币在存储价值和交换媒介之外，仍在不断创新和发展。它们正在快速增长，并有潜力在比特币区块链的长期未来中发挥更大的作用。

三、递归铭文的诞生

MIIX Labs 分析认为，递归铭文将引领比特币链上铭文的变革，让比特币铭文从此前的 1.0 来到铭文 2.0 时代。

自 Brc 20 引起轰动开始，加密社区就开始思考，既然以 Erc 20 为参考的 Brc 20 能够获得成功，为何不能以 Erc 721 为借鉴，打造一个 Brc 721 呢？

于是乎，Brc 721 应运而生。

然而，由于大部分 Erc 721 将图片存放在中心化的托管处，需要支付管理费用，而 Brc 721 则直接将图像刻录在比特币链上，这带来了巨大的成本挑战。随着 Ordinals 的风靡，比特币链上铭刻的拥堵问题日益严重，进一步加剧了成本压力。

为了解决这一问题，Gbrc 721 被创造了出来，Gbrc 721 成功克服了链上高昂成本的难题。在开发 Gbrc 721 项目中，项目方只需上传 NFT 的组件，并将它们巧妙组合在一起。

2023 年 6 月 12 日，Ordinals 的新首席维护者 Raph 在 Github 上宣布了一项重要的进展，他将铭文“递归铭文”编号 2167 的创新合并到了 Ordinals 的代码中。

在这个版本的代码中，通过“/-/content/:inscription_id”这个语法，铭文之间可以引用彼此的内容，实现了超越 Gbrc 721 的功能，这个创新直接在链上实现，无需依赖链下解析，从根本上解决了 Gbrc 721 的问题。

因此，递归铭文诞生了，这个简单而巧妙的改动为 Ordinals 带来了无限的创造力，激发了人们的想象力。

四、玩法及代表项目

4.1 玩法

据 MIIX Labs 了解，由于目前每个递归 NFT 的项目铸造方给的代码都不一样，有 svg 格式的，也有 html 格式的，但这都没关系，只要找到项目方给的铸造文件进行下载，其铸造流程都是一样。

今天就用 Recursive Punks 为例跟大家简单介绍一下铭文铸造的方法。

步骤一： 打开 Recursive Punks 网站：https://recursivepunks.io/

步骤二： 点击 GitHub 进入该项目的代码库，当然也可以直接点击下方对应进行下载，或者通过输入对应的 ID 进行下载

步骤三： 点右上角，先登入你的 Github 账号，如果没有那就注册一下，用谷歌邮箱就可以注册， 登录后点击 svg 文件夹

步骤四：这里是做个示范，就随便点击一个，接着点击下载，在下载完成中就可以查看到下载的 svg 文件

步骤五：查验是否被铭刻过，点击打开查验网站https://bitpunks.io/Explorer/Inscriptions

步骤六：开始铸造（铭刻），点击登录：https://unisat.io/inscribe

完成支付后，剩下就是等待了，当然这个时候也可以去链上查看打包的进度，点击你支付对应的这个交易哈希即可跳转。

MIIX Labs 提醒，在进行铸造前也可以单独登录该网站查询此刻的费用是否较低，这样可以控制铸造的成本，查看实时费用网站：https://mempool.space/zh/

4.2 代表项目

1. Recursive Playground

“Recursive Playground” 是第一个可交互递归铭文 NFT 项目，正如其测试版展示的用户只需轻轻拖动鼠标，便可改变画面的状态。让铭文 NFT 从静态、只能眼观变为动态的、可实时互动的，让收藏家也成为共同创作者，可以基于此创作自己的作品。

在 “Recursive Playground” 看来，个人灵感与计算机交互碰撞以某种方式实时创作出的独特的艺术作品就是交互式艺术。

“Recursive Playground” 体验版虽然仅作为 “可交互递归铭文”NFT 的首秀，但这也不可否认其作为 NFT 里程碑的重要性。这不仅仅是一次交互的尝试，更多的是一种可能性的验证，我们对 NFT 的认知不再仅仅停留在二维世界，NFT 的三维属性得到了验证。

试想一下，如果我们尝试改变一下 “Recursive Playground” 背后的反应逻辑，将仅限于鼠标的点击交互延伸至重力系统、流体力学、乃至于时间逻辑等，那么 “可交互递归铭文”NFT 是否就拥有了更多的可玩性？

NFT 将不再仅限于通过显示屏幕来传达视觉效果，通过鼠标的交互用户可以更多地感受它的存在，用户仅需轻轻动一动手指触碰，NFT 便可根据压力反应泛起点点涟漪。

2. Fservin

引用根底特点和 JS webGL 流体模拟器，生成个人网页。据其 NFT 交互体会测试显现，用户仅需拖动鼠标，NFT 便可依据流体力学，空气动力逻辑等完结各种美轮美奂的光影效果。

3. Speculum Aeternum Miraculum

引进数学方程与时刻（区块高度)的变量，蝴蝶的颜色跟着比特币区块高度的添加而改动，完结了机器与机器间的交互。

4. Recursive Doodinals

项目提供 NFT 系列根底特点特征，玩家可经过 DIY 自制 NFT 再进行铭刻。

据项目官方介绍，Doodinals 是第一个彻底由社区创造的递归艺术藏品，社区玩家可在 io 上进行 DIY 涂鸦，所有元素特征都支撑方位、缩放以及旋转功用，而 DIY 生成的 SVG 文件将刻录在 Ordinals 上，每一个玩家的涂鸦都将有机会被收录收藏系列之中。另据官方数据显现，当前已超越 128 个 avg 文件被刻录在 Ordinals 之上。

5. Vibeland

项目是第一个经过递归完结的原创 10 K 大合集的高清系列，创始完结了单一铭文随机呈现不同的 PFP 静态高清图片，完结了链上随机性。跟着每改写一次页面，都将呈现一张全新的头像，多达一万张可供随机挑选，这在无递归调用之前是彻底不无法完结的，同时也将为未来比特币上呈现盲盒做出了衬托。

五、前景展望

MIIX Labs 认为，递归铭文的出现为比特币生态系统带来了无限的可能性，接下来我们不妨大胆展望一下递归铭文的未来场景。

1、极大降低了门槛

递归铭文极大地降低了比特币生态的创业门槛，甚至只需几行代码的修改，即可打造全新的 NFT 系列，就像 Recursive Punks 的无图版、有图版、大图版和高清大图版所展示的那样。

此外，递归铭文还显著降低了创业成本，原本需要上传数万张图像的 10 k 合集，现在只需上传 200 个组件图像，并通过代码将它们巧妙组合，即可完成一个 10 k 合集的创作。

2、丰富的表现形式

递归铭文的出现使得 Ordinals 底层铭文浏览器（如 ord.io 和 ordiscan.com）努力实现了除了最早的文本类型之外的支持，如 SVG、HTML 和 JavaScript 等多种媒体类型。

递归铭文已经可以媲美一个万能网页，几乎可以显示任何网页中的内容。并且递归铭文可以引用其他铭文，就像本地网页升级为互联网一样，极大地丰富了铭文的表现形式。

3、激发元宇宙的潜能

递归铭文的出现为比特链上的 GameFi 和元宇宙的实现带来了可能。

目前已经出现了基于递归铭文开发的小游戏 — — Ordz Games 开发的一款递归小游戏，玩家可以直接在铭文浏览器上进行点击操作，玩法类似于之前热门的“羊了个羊”。

递归铭文以其代码调用的特性，实现了小巧的体积，这一特点使得铭文的大小能够突破比特币区块大小限制的 4 MB。

由此，我们在理论上可以构建出比特币链上的元宇宙和 GameFi，甚至将复杂的三维游戏上传至比特币链。

当然，需要明确的是，递归铭文仍处于早期阶段，所有这些设想和构想都需要通过持续的探索与发现才能够实现。

六、风险与挑战

递归铭文的实现依赖于 Ordinals 协议，其实现建立在 Ordinals 协议之上。如果 Ordinals 协议没有植入或之后删除了“引用”部分的代码，那么递归铭文将无法使用。

据 MIIX Labs 了解到的信息，递归铭文的发展需要更多底层基础设施的支持，例如头部的交易所 Unisat 和 MagicEden，主要的铭文浏览器以及中心化交易所例如 OKEX、BINANCE 的支持。

截至今日，MagicEden 尚未支持上文提到的 Recursive Punks V2 版本，而 Recursive Frogs 的正常展示也是在某位社区 KOL 提议后才实现。

从短期来看，递归铭文虽可能为比特币生态带来一定繁荣。

但是，图片、电影、代码存储库、软件代码以及各类新数据铭文都有可能堆积在比特币的内存池中。

随着注册商为确认交易而出价越来越高，只想用比特币支付购买费用的普通用户将不得不支付昂贵的溢价。

因此，从长期来看，比特币网络可能变得更加拥堵，手续费也会更加昂贵。

此外，递归铭文可能会让欺诈者更容易地在比特币链上添加恶意软件。欺诈者可以将恶意软件分为多个文件，并在人们下载游戏、文件或软件时向其设备添加恶意软件。这可能导致下载者丢失资金。

七、结语

综上所述，MIIX Labs 认为递归铭文的出现引发了比特币生态系统的广泛探讨，对比特币生态和整个行业可能都会产生深远的影响。

递归铭文的出现让比特币 NFT 生态的叙事更加充满无限的想象力，并开始不断衍生出优秀的有潜力有创新的项目。

这个市场若有资本愿意推动，有项目方持续创造更新，能够吸引足够多的用户进来参与，那么，毫无疑问，这其中一定蕴藏着很多的好机会。

积极面对新鲜的事物，跟紧市场热点，把握时机埋伏具有成为热点潜力的板块。

看好递归铭文能带来无限的可能，但也需要认识到递归铭文仍处于早期阶段，在进行冒险尝试和投资之前，MIIX Labs 提醒大家更应时刻保持风险意识，充分了解其中的风险，并谨慎对待，做看得懂的赛道，把握看得懂的叙事。
