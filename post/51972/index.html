BTC NFT 的出现给人们提供了进一步探索比特币的价值多样性的机会。通过 Ordinals 协议，将 SATS 转变为独一无二的 NFT，使得比特币网络更加灵活和自由。

根据 CryptoSlam 上的数据，目前比特币 NFT 这个新兴市场近一个月的交易量已经超过 1.7 亿美元，仅次于以太坊生态的 NFT ，并且比第三名 Solana 高出了 238% ，不过与以太坊 8 亿多美元的交易量仍有不小差距。

此外，一些比特币 NFT 头部项目也出现了被刷量的情况，比特币 NFT 市场的 Washing Trade 的交易量同样也是仅次于以太坊，用户顶着如此高的成本刷交易量，也可以从侧面看出市场对于比特币 NFT 前景的看好。

目前比特币生态上的主流 NFT 项目有：

TwelveFold： Yuga Labs 发行的包含 300 件限量版生成艺术作品，每件都刻在比特币区块链上，每件 NFT 作品都是 12 x 12 网格，结合 3D 图形和手绘特征。在 3 月份公开拍卖了 288 件作品，拍卖额超过了 1600 万美元，最高一件标的拍出了 7.11 个 BTC 。

BTC DeGods： Solana 上的蓝筹 NFT DeGods 在比特币网络上发行的项目，该 NFT 系列共有 534 枚，500 枚 BTC 定价的版本以 0.333 BTC 的价格在 2 分钟内就被铸造完毕，目前地板价高达 1.1 BTC 。

Bitcoin Frogs： 目前比较火爆的一个 NFT ，由 10000 个生成青蛙组成，是比特币版的PePe NFT，每一层内所有特征的稀有性都是平等的。截止目前，Bitcoin Frogs 的地板价为 0.063 BTC，总交易额接近 300 BTC，持有地址约 4000 个。

Bitcoin Punks： 第一个使用 Ordinals 协议将原始以太坊 CryptoPunks 成功上传到比特币网络的项目，项目采用 Free Mint 的形式发行，目前地板价为 0.075 BTC 。

Bitcoin Bandits： 建立在 BTC 网络上的一个另一个头像类型 NFT 项目，采用了像素风格的设计，目标是建一个狙击比特币生态上 Alpha 项目的“强盗”社区，项目目前地板价在 0.16 个 BTC 。

虽然比特币 NFT 生态涨势迅猛，但要使 BTC NFT 获得像以太坊 NFT 那样的大规模用户采用，仍然面临一些挑战。

一个主要问题是容量限制，一个 Ordinals 文件的限制为 4MB，这导致一些艺术作品难以在比特币网络中展示。

此外，原生的OrdinalsNFT 是完全上链的，这导致存储成本非常高，目前可能无法实现大规模应用和普及。

而 BTC NFT 的市场前景依旧非常吸引人，除了常规的 BRC-20 方案之外，BRC-721 标准的引入也为 BTC NFT 带来了更多扩展性。

BRC-20 是什么？

BRC-20 是比特币网络上发行同质化 Token 的一个实验性格式标准。

在理解 BRC-20 之前首先要了解两个概念。

1、SATS： 聪，比特币上的最小单位，1 个 BTC 等于 1 亿个 SATS ；

2、Ordinals 协议： 为每一个 SATS 打编号的协议。

通过 Ordinals 协议，可以按照一定规则将 SATS 进行排序，并给它们附加上特定的信息，使得每一个原本同质化 SATS 都变成独一无二的 NFT。

这些信息可以是文字、图片或者代码等，而附加到 SATS 上之后的信息就叫做「Inscriptions」（铭文）。

可以看得出来，Ordinals 协议其实最开始是为发行 NFT 服务的。

而BRC-20 可以理解成一种特殊的 Ordinals NFT ，它通过将铭文设置为统一的 JSON 数据格式（文本），规定了包括 Token 的名称、符号、总量、小数位数等在内的 Token 信息。

以下图 ORDI 代币为例，op 字段的 deploy、mint 和 transfer 分别表示代币部署、铸造和转移的操作，tick 表示的是执行代币的名称，max 表示该代币的发行总量，lim 表示单币铸币的最大限额，amt 则是代表在铸造或转移操作中执行的代币数量。

有了这一套规则之后，开发者按照标准执行就可完成同质化 Token 的发行和后续的操作。

由于 BRC-20 协议的功能比较简单，在实际应用中依旧有许多限制，比如 BRC-20 代币仅支持输入 4 位字符、部署之后无法修改供应量和铸造量、存在双花攻击风险等等。

因此在 4 月底，OrcDAO 基于 Ordinals 又发行了一个新的协议ORC-20 ，并获得在 5 月份获得了 Binance 的支持。

ORC-20 扩展了 BRC-20 的功能、取消了一些限制，能够更好地适应项目发展的需要。

如 ORC-20 取消了 4 个字符的名称限制；增加了代币的可升级功能，让项目方可以基于代币做更多的赋能（减少供应量增加稀缺性等）；增加了一些高级功能（铸造白名单、版税等），以及通过 UTXO 降低 BRC-20 的双花风险等等。

简单来说，ORC-20 协议比 BRC-20 更加灵活，应用场景也更加丰富，用户体验上更接近于以太坊上的 ERC-20 代币。

尽管上线时间不到一个月，但 ORC-20 增长迅速，截止目前，ORC-20 的铭文数量已经超过了 26 万个，产生手续费超过了 20 个 BTC 。

BRC-721 是什么？

BRC-721 是一种在比特币网络上发行非同质化 Token （NFT）的实验性标准，它借鉴了 BRC-20 的理念和原则，并通过解决原生 Ordinals NFT 的限制来提供更多功能和灵活性。

使用 BRC-721 标准，用户可以轻松创建、铸造、转移和更新独特的数字资产，适用于广泛的应用场景，包括数字艺术、收藏品和虚拟商品等。

BRC-721 标准定义了一系列操作，如部署、铸造、转移 NFT、更新元数据等，每个 Token 都被赋予唯一的标识符，确保每个 NFT 的独特性，使其无法与其他 NFT 一对一交换。

BRC-721 与原生 Ordinals NFT 有何不同？

1、数据存储

原生 Ordinals 每个铭文都需要保存图片，会导致 Mint 费用过高、占用大量 比特币网络空间等问题；而 BRC-721 只需要在 Deploy 时指定图片等信息，Mint 操作并不需要保存图片，因此可以极大地节省 Mint 费用和网络空间；

BRC-721 可支持将图片保存在 IPFS 等链下服务中，不仅能节省 Bitcoin 空间，也能为每个 Token 提供灵活的属性信息；

原生 Ordinals NFT 无法针对 Collection 进行有效索引，而 BRC-721 通过提供类似 BRC-20 的 JSON 规范，能够索引和查找同一个 Collection 的 NFT。

2、BRC 协议的兼容性

BRC-721 采用与 BRC-20 类似的协议格式，通过 JSON 内容定义不同的功能，极大地提高了 NFT 的灵活性。例如，可以通过 Update 操作实现 Reveal 功能；通过 Tick 字段可以有效地对一个 Collection 的 NFT 进行索引。

3、NFT 生态的兼容性

ERC-721 标准的 NFT 在当前市场更受欢迎。BRC-721 采用的 Token URI 与 Metadata 规范与 ERC-721 保持一致，可以快速地与现有 NFT 生态进行适配。同时，原生 Ordinals 不支持 Trait 等字段，而 BRC-721 可以支持定义 NFT 属性和稀有度等信息。

因此相比于原生的 Ordinals NFT，使用 BRC-721 协议可以实现更多复杂的功能，将外部资源引入 Ordinals，可拓展性更强，应用场景更加丰富，同时对于开发者也更加友好。

通过将 NFT 的 Metadata 保存在 IPFS 或其他链下服务中，节省了比特币网络的空间，降低了用户的使用成本，并为每个 NFT 提供了更多的属性信息，增加了 BTC NFT 的玩法和应用场景。

希望随着时间的推移，BTC NFT 能够得到更广泛的采用，并为比特币网络的发展贡献更多可能性和创新。
