原文标题：Working as a designer in web3 — a 101 guide

撰文：Angela Ching

编译：Luna Lee，The SeeDAO 翻译公会

本文是一名 Web3 设计工作者基于自身经验撰写的一份 Web3 设计工作者指南。作者从 Web3 所需的设计师类型、设计工作对于 Web3 领域的重要性、Web3 与 Web2 设计工作的区别、设计师在 Web3 工作的收益、如何参与等角度进行了详细介绍。为想进入 Web3 工作的设计师们提供参考。

近期，有同行的设计朋友问一直问我在 Web3 中工作是什么体验——比如，设计工作在 Web3 中有啥变化吗？需要很了解技术吗？该怎么加入呢？

对于设计师而言，Web3 领域存在很多的机会，但同时也充满了风险和不确定性。为了避免反复回答同样的问题，我就偷个懒，统一为那些想进入这个领域的新锐设计师提供一份资源指南。主要对以下主题进行概述：

  • Web3 需要哪种类型的设计师？

  • 为什么设计在 Web3 领域很重要？

  • 作为设计师，在 Web3 领域的好处是？

  • 作为设计师在 Web3 工作的好处

  • 「自由职业者 / 社区 / 项目」vs「产品」vs「机构」

  • 如何参与？

所有内容都是基于我过去 6 个月在这个领域的经验。我是一名职业设计师，上午主要在 Immutable 担任产品设计师，这是一家专注于 NFT 数字所有权的公司；下午为 DAO Under 做设计；晚上的工作是 / 曾经是设计一个 NFT 工具套件，近期已卖出（出于 doxxing 原因，就不放链接了）。我拥有产品设计、插画和品牌设计的从业经验。但是一如既往，听你妈的总没错，咱们要慎重对待所有的事情。

（注：Doxing（也被称为 doxxing）是一种网络骚扰，涉及到揭露某人的个人信息——例如他们的真实姓名、地址、工作或其他识别数据——并将其公开曝光，通常发生在互联网上。）

什么是 Web3?

我曾经撰文探讨过这个话题。它并非这篇文章的重点，所以我不会深入讨论——本质上来说，Web3 是利用区块链帮助解决权力中心化和资产所有权等相关的问题——所有这些话题都被问题环绕。我强烈推荐 Cobie 写的一篇关于 Web3 的好文：《Wtf is web3》。

Web3 需要哪种类型的设计师？

与「实体」设计师相比，「数字」设计师更常见。这并不意味着「实体」设计师，比如艺术家、时装设计师、珠宝设计师等等在 Web3 领域是不存在的。大多数项目都是数字化的，直到 Web3 成为主流之前，它们之间还没有足够的相互关联性。在当前，有一些 NFT 项目确实存在「实体」效用，比如你购买 NFT 可能会得到艺术品或者服装等等。无论哪种类型的设计师，其「必要性」取决于 Web3 的类型。

所以我究竟在设计什么？

在 Web3 做设计并非你想象的那样具有跳跃性！无论是什么类型的设计师，你从始至终都需要利用核心技能来做 Web3 的设计工作。你的工作环境可能略有不同，但设计工作与环境没啥关系，你在 Web2 中练就的设计技能很重要。

这个领域真的很缺设计师

要知道，设计师在 Web3 领域是抢手货，而这是有原因的。针对不同项目的不同要求，从「必不可少」到「轻而易举被取代」都很正常，但设计师们可以在以下方面产生巨大的影响：

创造艺术卖点：

收藏家或任何会因为「看起来不错」而购买 NFT 的人，都会明白我的意思。seerlight 有篇我超爱的文章。

为产品建立合法性，便于大肆炒作：

我们可以看到有些项目的失败，仅仅是因为它们的艺术性非常糟糕。就 NFT 而言，Pixelmon 已经筹集到 7,000 万美元，却因为铸造的艺术质量不一而被认为是随时跑路的项目。在 Web3 领域，炒作一直很猖獗，某个东西如果看起来酷酷的，则有助于大肆炒作及销售。但这并不是说你的项目缺乏艺术性就一定会失败。比如 mfers 就不是艺术之作，但话又说回来，这就是它的魅力所在。

就产品而言，它看起来的好坏程度可以改变人们对其合法性的看法，进而影响它的可信度。如果产品具备一致性以及一定的美感，就表明无论该团队是雇用设计师还是聘请外包来进行设计，他们对他们的产品都做出了一定的承诺或是投入了一定的资金。在一个人们普遍因为害怕受骗而本能地怀疑、警惕的环境中，这一点极为重要。

产品可以分叉，实现令人真正愉悦的体验却很难：

这点基本上是设计师赖以生存的「面包和黄油」。当一群后端开发人员在做设计 / 前端实现时，会非常强调功能而非直观感受。区块链意味着事情能很容易被分叉或复制（如 Uniswap 的双域兑换机制），但这很可能导致同质化产品的基线看起来都一样，而且产品的可防御性很低（除非是属于技术栈的东西，如速度和安全等）。有时候一些小功能，或者那些「你甚至没有意识到你需要」的东西，反而成为让人们坚持使用一个产品的理由——这就是需要设计人员去帮助发现或者进行设计的东西。

比如，对于速度作为用户主要关注点的 NFT sniping tools （截图工具）来说，我听说大家坚持选择某个产品的原因，仅仅是他们允许用户自定义预设 gas 费。

简而言之，上图中右下角这个功能是留住用户的理由。

帮助并引导新人加入：

由于 Web3 技术仍然非常新（不禁回想到互联网刚出现那会儿），新进入这个领域的用户可能并不知道究竟发生了什么，而设计师有很大的机会可以帮助教育和引导用户。比如，创建钱包等「仪式」仍然令新人疑惑，保留「密码」（也称作「助记词」）的实体副本等行为对他们来说也非常新奇。而且，随着区块链技术越来越复杂，设计师的角色已经逐渐演变为新手们的「准教师」了。

预防资金损失，确保安全透明性：

能赚得盆满钵满，也很可能大亏特亏。由于没有中心化的公司通过客户支持的方式来帮助你，所有事情都要靠自己来解决，因此安全性对于 Web3 而言非常重要。如果你丢失了详细资料，一切都完蛋了；如果你被骗了，唯一的选择就是在 Twitter 哭诉，奢望网友们能心生善意同情你，并帮助你追踪骗子。人性就是多疑的，Web3 的用户已经看到了最好的和最坏的结果。但对于设计师来说，这意味着你要帮助设计出具备高响应性和反馈性的功能，帮助人们时刻意识到他们花了多少钱、什么时候能收到回报，应该期待什么，一切是如何计算的，并且提供预警，帮助用户避免错误的发生等等。

Web2 和 Web3 产品设计的主要区别

在我之前的文章中，我最想回答的一个问题是，在两个领域，设计的工作是否有所改变。答案很复杂：有变化但又没有完全改变。于是，我把这些变化（或没有发生的变化）主要分成三部分，包括工作范畴、产品设计和设计流程。

工作范畴

你最好是个设计百事通：

你需要准备好做整套设计甚至更多工作。除了通常的研究、内容编写、设计和研讨会之外，你可能还会涉足插画、品牌设计、动效设计等工作内容。任何与视觉相关的内容都会被纳入你的工作范围。具备插画技能或平面设计技能的设计师在这个领域很抢手，因为视觉资产在 Web3 领域中扮演着重要角色。从 Twitter 的 banner 图到 Discord 贴纸和 GIF，再到网站登陆页的图像，所有的东西都很重要。

沟通与触达：

由于远程工作和时差，设计师角色也要具备良好的沟通能力，对所有我们做的事情、对其他队友的要求等实现异步更新。而现在又更进一步，你的视觉资产必须能方便地供所有人下载。这意味着，你的 Figma 文件不能再杂乱无章，你的文件也不能再以 final_finals.png 的形式只存储在你自己的笔记本上。你的队友和 Discord 社区会不断要求你提供视觉资产，比如 logo、图片、banner 和最新的屏幕设计，因此你要整理、集中存储这些东西。我推荐大家使用 playbook， 它就像设计师版本的 Dropbox，并为所有的设计师提供 1TB 的免费存储空间。

产品设计

高响应性和细节的重要性：

任何时候用户都需要知道他们的资金放在哪里、去向何处，这些资金的状态如何，是否存在任何延迟等等——任何地方都不应该模糊不清，这些信息应该让用户「触手可及」。用户的钱包通常是他们参与和访问 Web3 领域的主要渠道，一次简单的点击错误，可能导致用户在不了解其影响的交易之中损失大笔资金；或者有时候当用户质押某个代币时，却并不知道需要在质押期结束之后才能动用代币，于是突然之间，他们的资金就丧失了流动性。

学会在技术限制中开展设计：

技术被限制在它可行的范围内——这是设计师无法控制的。从细小的用户体验，比如昂贵的 gas 费或交易速度，到基于产品所选择协议的整体行为体现，对于 Web3 设计师而言，要明白有一些限制是你必须学习，并要围绕它们进行设计的。但了解不同协议对用户体验带来的不同影响也很重要，这样你可以和其他利益攸关方进行有意义的对话，并思考更可靠的解决方案。

保持视觉效果一致性，从而确保产品看上去合法，这一点很重要：

尽管我不想这么说，但事实是很多人仅仅只通过观察项目表象就对它们发表意见，而不会尽职尽责地对其开展深入的调查。具备一些「看起来不错」的东西有时比神仙般的用户体验更为重要，你的竞争对手很可能只设定了低标准的用户体验，所以有时候如果你刚刚开始启动项目，当务之急是确保领先于他们，而非事事臻于完美。这是非常经典的 MVP 方法：使自己看起来更厉害。设计师们，请记住，在涉及视觉吸引力方面，保持一致性尤为关键，因为这展现了你们为保证视觉效果而做出的努力。

设计流程

很多人想问，Web3 的设计流程是否和 Web2 相同，简单来说，是的。就像一些 Web2 的初创公司，走一些弯路是必经之路。有些地方不适合做研究探索，只管先将产品推向市场，但有些地方则需要严格遵循双钻石设计进程。这在一定程度上取决于组织的成熟度水平以及设计的导向程度，虽然没有「全新的」Web3 革命性方法（到目前为止还没有），我们仍要坚持沿袭 Web2 关于发现、定义与构建的方法。

译者注：双钻设计模型是设计师可以在其创作过程中遵循的系统，包括四个阶段：发现（Discovery）、定义（Denifition）、发展（Development）和交付（Delivery）。

社区和你的用户一样重要：

在 Web2 时代，设计师只需要在设计中听取终端用户的意见。但现在，社区必须成为你考虑的另一大利益攸关群体。这意味着让社区参与进设计过程是很重要的，你可以采取多种形式，比如产品研究及反馈，关注 Discord 的整体情绪。我有一个在公开场合的建设效果特别好的项目，我经常会在 Figma 上直播，人们会在频道上发表评论等等。保持公开透明度，并与你的社区建立良性互动，可以为你培养强大的拥护者。提供被动或主动的参与机制，是培养这些拥护者的最好方式。

设置模板可以长期助力队友与未来合作伙伴：

为满足营销目的，Web3 领域中的很多项目都会与其他项目进行合作。所以对视觉设计师而言，你需要将视觉资产的可复用性（比如模版）放在首位。因为合作伙伴可能会变动，所以你很可能会多次使用模板。另一方面，就效率角度而言也很重要，有了这些模板，你就能快速交付你的内容。

我的朋友在 OpenGuild 发布的合作帖子示例 。

在 Web3 领域工作的好处

创新的空间，有趣的问题

作为 Web3 设计师，你正在为最先进的技术提供设计。要设计好的东西，你还需要对这个领域有深刻的理解。你可以称它是面向未来的，也可以称它是具有挑战性的，随你怎么说，但思考区块链能做的东西，真的能帮助你重新思考关于金融、市场、人类心理和社会等方面的事情。就我本人而言，我会思考以下问题：

「人们喜欢匿名，那我们如何找用户开展测试？」「用户会理解我们的激励机制吗？这样做道德吗？」「游戏化，还是不游戏化？」「我们可以用什么比喻来帮助用户理解这项新技术？」「如何更好地引导教育用户？」

快节奏

一切都发展得很快，人们期望事情发展得快。有时几乎快得让人难以接受。比如，有时候还没研究明白就推出产品了。看看你的竞争对手，你会觉得提早入局往往比完美更重要。有时你只想说「去他的」，直接用第一个方案吧，没有时间去做第二个方案了。有时，你必须采取开发人员的解决方案，因为他们很了解这个技术，而你没有时间去进一步探索。这对设计师而言可能并不具备吸引力，但这有助于帮你消化由不完美和半成品带来的不适感。当然，这种工作节奏会因为工作地点的不同而变化，一个成熟公司，相比一个不到 10 位核心成员的小项目可能会相对慢行。但无论如何，在这个领域，事情一般进展得很快，因为你正在为之建设的这个领域的人们希望如此。

跨团队协作

你将有机会向一群有趣的人学习，Web3 的工作能充分磨练你的设计技能、合作技能，提升你对该领域的理解认知以及其他软技能。我白天的工作是与产品经理、前端开发、后端开发人员以及项目经理一起协作，下午，我与很多有才华的设计师一起工作；晚上，我与数据分析师、高频交易员、Web3 营销人员、社区 MOD、动效设计师以及艺术家一起工作。

创意自由

好吧，这一点其实只实现了一半。我经历过从「用我们现有的设计组件尽可能多地进行设计」到「什么都可以，世界属于你」的所有阶段。通常情况下，团队和公司越「成熟」，设计师的自由就越少；但在那些甚至没有品牌的团队项目中，我独立负责品牌和视觉识别设计（VI），并为其他设计师提供借鉴。对于动效设计师 / 艺术家 / 3D 设计师等等而言，只要符合品牌准则，你的设计空间基本是无限的。在一个设计匮乏的空间中的好处是，很多人都抱有「什么都可以做」的心态，所以设计师有很大的自由来发挥想象力。

亮点作品集

如果你正想为你的作品集添加点新东西——这个领域有很多东西可以提供给你。从长期到短期项目，再到短线任务（也被称为「赏金任务」），所有这些都能帮助充实你的作品集。当项目结束，你将会收获与协作以及所有权相关的丰富案例经验。

见证传统角色的演变和创造

谈到这个，我脑海中浮现出了产品经理、营销人员和社区管理员几个角色。Web3 影响了设计师与这些角色的合作方式，并拓展了我们应对变化和不同沟通风格的能力，这感觉就像正在见证一场事业转折的中期阶段——超级令人兴奋。

提早入局

谈到「大科技」，我观测到的趋势是——技术第一位，设计第二位。比如，在开始考虑网页设计之前，先拥有了设计 HTML 的能力 ；或者说，在考虑如何让人们真正输入数字之前，就已经拥有了在电话里「哔哔啵啵」输入数字的能力。这通常意味着，在巨大的技术革命的初始阶段，存在一个甜蜜的「早期应用者」阶段，这是设计师们提前「入局」的最佳时机——在某些东西真正成为主流之前，推动最佳实践，做出亮眼成绩。

赚钱

不提经济激励就不现实了。虽然，这篇文章写在熊市，ETH 价格跌得让人想哭，但仍有大量资金不断注入 Web3 领域。熊市是布局建设的最佳时机。公司存有丰厚的资金，风投机构也在扎堆注资，你能赚多少钱取决于你的参与度、你为谁工作、参与项目的成功与否以及项目类型等等。没有硬性规定，支付方式可以是传统的工资，也可以是代币。

多元参与方式

作为设计师，你有四种主要的参与方式，每一种都有优缺点。

1. 代理机构：

越来越多的代理机构正专注于为 Web3 公司提供设计。与 Web2 类似，它往往接的是一次性项目，主要是为了重新设计品牌 / 视觉形象或帮助建立工具基础。在代理机构工作的优点是能服务于不同类型的客户，与外部利益攸关者开展良好的沟通，并建立起一个具备广度的专门知识库。而缺点是无法深入了解具体产品。

2. 合同工 / 自由职业者：

和代理机构类似，但你是唯一的设计师，而且需要负责所有的事情。除非你服务于更成熟的公司，否则很多工作实际上都需要资深设计师，因为基本上，他们需要你能很好地完成从合作、设计、项目推进到沟通等全部工作。这样的工作的优点包括相对较高的报酬水平，能真正地融入到团队中去，并有可能成为一个永久的角色，以及适应项目的广泛性，如果当前的项目进展顺利，也会很容易在其他项目中找到工作，并按照自己的时间表执行工作。缺点包括有限的付款方式（如用代币支付），没有提出意见想法的人，而且你的可能会经历工作淡季。

3. 公司：

还有一个选择，是你当前任职的公司正在从事 Web3。其范围可以从 Web3 原生的（极其去中心化、远程友好，并用原生代币支付），到不那么 Web3 原生的（像 Web2 公司一样工作，但产品本身是 Web3 风格的——我会亲切地称这些为 Web2.5 公司）。

4. 社区成员：

如果你加入一个 DAO 并成为会员，你也会发现也有一些设计工作可以参与。对于财力雄厚的 DAO 来说，你可能还需要通过面试程序来加入设计团队，但对于更多其他的 DAO 来说，你可以直接开始协助他们的设计工作。他们通常就像普通的产品团队一样工作——可以将其视为与一群很酷的人一起解决大学作业。这样做的好处是，你可以更靠近那些已经在这个领域的人，你的参与时间相对灵活，有很多很好的项目可供选择，同时有很大的空间来让你产生影响力，而且如果你在 DAO 工作对于这个领域的人来说是个很好的信号。缺点是在 DAO 拥有足够资金之前，你可能要无偿工作一段时间。

也有一些新模式融合了以上这几种类型，比如 Deepwork，它作为一个 DAO 运作（在治理和去中心化方面），但他们是一个有核心团队的设计机构，并仍有余地让人们参与进来，来成为他们的合同工或自由职业者。

对于技术，我需要了解多少？

简单来说… …你并不需要了解技术，但你最好能去了解一些。一个好的设计师会花时间去熟悉区块链的工作原理。这样可以使你在与他人沟通时更加简单顺畅，而不会让你感到不知所措，你的设计方案也不会那么牵强附会。话虽如此，设计工作仍然遵循无关性原则——如果你对 Web3 一无所知，利用在 Web2 中的沟通、推进、协作和设计技能也是完全合理的。

我如何参与进来？

  1.  使用 DAOlicious 寻找你感兴趣的 DAO（ P.S 如果你是来自亚太地区的设计师，不妨来个打招呼吧）。

  2. 联系已经参与该领域的设计师——Jeremy Goldberg、Yangyou、以及参与 vectorDAO 的那些人。

  3. 查看 Web3 公司的招聘网站，试试这个 (https://jobs.paradigm.xyz/jobs) 或这个 (https://web3.career/)

  4. 联系那些缺乏设计师的项目，比如 Twitter 的「兔子洞」

  5. 加入设计 discord 或 DAO，试试 web3designerz 和 deepwork
