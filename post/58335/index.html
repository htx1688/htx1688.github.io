Arbitrum vs. Optimism: 新一轮Layer3战略较量

精彩的战争在 Rollup 领域正式打响，其中 Arbitrum 和 Optimism 尤为引人注目。Optimism 以其独有的 OP Stack 开展水平扩张，而 Arbitrum 选择了以 Arbitrum Orbit 为基础进行垂直变革。

Arbitrum 向 Optimism 宣战

Arbitrum 团队推出了基于 Arbitrum Orbit 的首条 L3 链 - @XAI_GAMES，正式向 Optimism 抛出了战争的戟。XAI 的推出，在战略上对 Arbitrum 极为重要，可视为对 Optimism（以下简称 OP）的明确挑战。OP 凭借其强大的 B 端资源推动了币安、Coinbase 等基于其 OP Stack 构建的 L2 链，大力扩展了 L2 的生态版图。相较之下，Arbitrum 则显得相对低调。

OP Stack 生态版图

Arbitrum Orbit

在深入讨论 XAI 之前，必须先了解 Arbitrum Orbit。Arbitrum Orbit 是一种通用的、模块化的 L3 堆栈，它允许开发者在其基础上搭建专用的 L3 链。这些 L3 上的交易将通过 Arbitrum 的 L2（例如 Arbitrum One 或 Nova）进行结算。XAI 便是基于 Arbitrum Orbit 构建的，专门用于游戏场景的 L3 链。

Arbitrum Orbit

Arbitrum Nova 对比 XAI

Nova 本身是一个专为游戏场景设计的通用 L2，本应已能满足游戏场景的需求。那么，为什么 Arbitrum 还要推出 XAI 呢？答案很简单：相较于 Nova 这种游戏通用链，XAI 作为专用的 L3 链可以实现更高的性能，并拥有专用的计算和存储资源。这样，资源密集型的用例（如计算密集型的 AI 模型）也成为可能。

XAI

技术支持：Nitro+ BOLD+ Stylus

Nitro

XAI 将从 Arbitrum 的技术堆栈 Nitro、BOLD 和 Stylus 中受益。Nitro 是 Arbitrum One 的技术栈升级，通过在客户端软件的基础层直接编译了 Geth 的核心来实现更高的以太坊兼容性。

Arbitrum Nova vs. Nitro

BOLD

BOLD 是 Arbitrum 团队提出的一种无需许可的验证机制，旨在最小化结算状态的延迟。这种机制解决了乐观 Rollup 中的一周挑战期问题，使得从 L2 提现到 L1 变得更快。

Arbitrum BOLD

Stylus

Stylus 是 Arbitrum 开发的一个支持多语言构建应用程序的开源 SDK。开发人员在 Arbitrum 上既能使用传统的 Solidity 语言，也能使用与 WASM 兼容的语言，如 Rust、C 和 C++ 等，从而降低了 gas 成本。

Arbitrum Stylus 带来 EVM+

模块化和灵活性

BOLD 和 Stylus 也是通用的模块化组件。开发者可以在基于 Arbitrum Orbit 启动特定用例的 L3 时，原生集成 BOLD 和 Stylus。或者，在 L3 平稳运行后，通过去中心化的 DAO 治理方式，投票决定是否集成这些模块。

结语

XAI 已经开炮，拉开了 Arbitrum 与 Optimism 的战争帷幕。这场对决或许才刚刚开始，但它无疑标志着 Rollup 领域内一场精彩纷呈的竞争正式拉开了序幕。
