人们越来越依赖社交媒体获取新闻、出版和娱乐。然而，像Facebook、Tiktok和Twitter这样的中心化参与者因利用用户的数据和有偏见的审核而受到抨击。Web3解决方案已经进入该领域，竞争性的去中心化社交媒体平台现在正争相解决这个21世纪的问题。由于可扩展性和大规模采用的障碍，问题围绕着向去中心化社交媒体的伟大转型的可能性。

现状

今天，大约有 38 亿社交媒体用户，随着全球移动设备的普及，用户数量每年都在增加。

资料来源：Statista

虽然是日常生活的一部分，但目前形式的社交媒体带来的益处和它带来的成瘾性一样严重。现有的网络面临着对有偏见的节制、对用户数据的利用、令人上瘾的算法和孤立的生态系统的批评。真正的 “修复 “社交媒体需要一个完全的范式转变，而去中心化的社交媒体解决方案是我们最接近的一个机会。

从头开始，重新构想社交媒体

虽然 Web2 公司专注于将社交媒体作为单一产品，但 Web3 将社交网络的概念扩展到多个层次。基础层是一个社交图：它映射了个人资料、关注者和他们的联系。接下来是应用层，用户可以在其中消费内容并与他们的社交图进行交互。对于 Web2，将社交图视为关注者，将应用层视为提要。通过区块链，没有一个实体可以控制社交图谱。相反，你的社交图谱成为资产，而不是某家公司拥有的产品。

显然，拥有你的社交图谱的吸引力不足以说服数十亿用户放弃他们现有正在使用的社交产品。鉴于用户对社交媒体政策和高管越来越不满，还有另一个空白需要填补。除了数据所有权之外，对分散的追随者、审查制度和封闭源算法的投诉 为DeSoc（去中心化社交媒体）篡夺市场份额提供了机会。坦率地说，现在是向 DeSoc 过渡的最佳时机。现有用户需要更好的体验，而 DeSoc 可以作为加密采用的催化剂；后者需要一种能够促进其早期采用阶段的流行产品。

资料来源：Crypto.com

解决问题

虽然 Web2 关注者和连接在每个应用程序中都是孤立的，但 Web3 社交网络允许你将关注者带到任何应用程序中。想象一下，如果你可以在为品牌启动 Instagram 页面时让所有 YouTube 订阅者立即关注您，那会多么简单和节省时间。这就是将社交图谱与应用层分离的美妙之处；当应用层灵活时，你的社交图谱保持不变。借助未来的可定制性，可以轻松实现自我许可的追随者转移（仅移植选定的追随者）的可能性。

在讨论适度和审查时，拥有你的联系的想法变得更加重要。美国前总统特朗普都因为收到了 Twitter 的禁令，开始制作自己的社交媒体应用程序 Truth Social。随着这种转变，他失去了旧有平台的追随者。对于那些不是政治人物并有可能因社交媒体禁令而失去生计的个人来说，情况仍然要糟糕得多。竞争的 DeSoc 应用程序正在尝试各种审核技术，以保持在法律范围内，同时避免 Web2 过度的审核。

Lens 协议是建立在 Polygon 上的社交图层，可以说是最好的审核方法：他们只是将审核留给发布内容的应用程序层。如果用户在一个应用层被禁止或审查，他们可以简单地转移到另一个提供类似体验的协议。用户可以保留他们所有的追随者并继续，就好像什么都没发生一样。现在，用户不必将社交媒体公司变成真相和仇恨言论的仲裁者，而是可以与任何可以接受其内容的前端进行交互。真正的自由市场体验！

社交媒体难题的最后一块需要向算法求解。如今，可以说算法比黄金更有价值，流行的社交媒体应用程序防止它们的算法泄露到公共领域像防止圣杯外流一样。这些算法在设计最初就计划尽可能的让人上瘾。数据是这些公司获利的方式，但你的时间是他们获得更多数据的方式。用户目前无法控制算法。如果有一天你想看更多励志视频，或者另一天想试一个不那么令人上瘾的算法，你就没有办法了。在 DeSoc 协议中，个人可以创建开源算法或具有不同算法的多个应用程序层，以让用户更好地控制他们看到的内容。这不仅是可定制性的进步。

DeSoc三难困境

尽管 DeSoc 似乎可以解决我们使用 Web2 社交媒体的所有问题，但事实上，没有完美的解决方案。是的，DeSoc 可能在一定层面上解决了上面提到的问题，但它引入了自己的一套复杂性。我们创造了术语“DeSoc Trilemma”来强调在设计 DeSoc 协议时在安全性、可扩展性和用户体验 (UX) 方面所做的权衡。与 V神提出的“区块链三难困境”稍有不同的是，我们审查过的任何协议都无法成功优化所有三个要求。

安全性与可扩展性

首先，安全性在这个三难困境中表明了两个主体事件：区块链的去中心化以及交易的执行者。区块链的去中心化很简单：拥有更多验证者，区块链变得更加去中心化和安全。然而，这引入了安全性和可扩展性之间的权衡 . 区块链越去中心化和安全，就越难扩展。这就解释了为什么没有 DeSoc 协议可以真正存在于以太坊上，因为Gas费太高而且网络会变得太拥挤。目前，在 Zk-rollups 变得便宜且可用之前，DeSoc 协议需要在侧链、子/超网或独立的 L1 上运行。最安全的区块链并不是 DeSoc 协议的最佳家园，但公平地说，这些协议并不像 DeFi 协议那样需要以太坊的安全性。

安全与用户体验

在交易执行方面，一些 DeSoc 协议自己执行交易。例如，用户关注某人并仅签署一条消息以实现这一点。他们不支付Gas费或实际执行交易，而是这个角色落在协议本身上。这引入了安全性和 UX 之间的权衡 。混淆Gas费是 UX 向前迈出的一大步。想象一下，每次你想发帖时支付一小部分 Instagram 的 AWS 账单是多么可怕！事实上，终极用户体验是连接钱包但无需处理提示你签署合同或支付交易费用的元掩码弹出窗口。这是以向智能合约授予更多权限和对钱包的控制权为代价的，智能合约有被黑客入侵的风险。在促进大规模采用的同时，它与 Web3 的去中心化性质背道而驰。用户体验可以说是加密采用中最大的摩擦点；为了促进广泛采用，去中心化 maxis 可能需要另辟蹊径。

事实上，在安全性和执行交易的用户体验之间进行权衡时，Lens 的无gas API 是行业领导者。该 API 允许用户在区块链网络上执行交易而无需支付任何费用。它使 DeSoc 可与免费使用的 Web2 社交媒体相媲美。减少用户等待交易完成的需要，gasless API 允许他们滚动到下一个帖子并自由地继续交互。

API 允许用户简单地签署一条消息（例如，作为评论），然后中继器接收该消息以及在链上发布数据所需的相关信息，完成交易。对于中继者来说，这种方法稍微消耗更多的Gas费，因为他们必须验证签名然后将数据发布到链上。但是，它使用户的体验更加愉快和有益。应用层平台需要支付 gas 费用，这意味着 Lenster 或 LensFrens 有责任支付费用。虽然使用无Gas费 API 的缺点是用户依赖中继器在链上发布交易而不是自己完成交易，但权衡是值得的，因为社交媒体帖子不需要相同安全级别作为金融交易。

用户体验与可扩展性

用户体验和可扩展性 之间的权衡更难分析。以DeSo 为例（不要与我们一直使用的术语 DeSoc 混淆），DeSo 是专门为社交应用创建的区块链。DeSo 已经扩展到超过 150 万用户（远远超过 Lens 等竞争对手，拥有超过 50,000 个配置文件），但它缺乏 Polygon 用户可以从中受益的周边生态系统，例如游戏和 DeFi 协议。想象一下，你可以用你在Polygon上的Lens资料作为DeFi协议中的抵押品来进行贷款。这种增值的可组合性好处在DeSo或任何建立在独立区块链上的DeSoc协议上根本不存在。同样，Cyberconnect具有极大的可扩展性，有超过一百万个账户，因为它是区块链不可知的，允许你把你的社交图带到任何区块链上。然而，代价是Cyberconnect由于其设计，不能在其生态系统内利用NFT。虽然节省了计算量并简化了整个设计过程，但失去NFT的兼容性是一个巨大的损失。以Lens为例，NFT同时代表你的个人资料和你的关注。它可以实现个人资料的二次销售，创造一个全新的市场。不想创建一个拥有大量追随者的备忘录账户？只需购买一个已经迎合该受众的NFT。此外，NFTs还能为创作者带来更多收入。通过使帖子具有收藏价值，创作者可以引入排他性，从而对内容收取费用。

去中心化社交媒体能否成为现实？

去中心化社交媒体可能为用户在现有 Web2 社交媒体平台上面临的挑战提供唯一现实的解决方案。但问题仍然存在：大规模采用可能吗？从表面上看，去中心化巨头将不得不主动举白旗投降，以便优先考虑 UX 和可扩展性，以便 DeSoc 最终能够看到可持续的产品市场契合度。正确的方法似乎是社交图和应用程序层的分离，以进一步推动可定制性并避免 Web2 中存在的审核问题。最后，通过在社交媒体之外的生态系统（如游戏和金融）上构建区块链以及在设计中利用 NFT 来优先考虑可组合性，可以提供最佳用户体验，同时提供当今社交媒体设计中没有的选项。
