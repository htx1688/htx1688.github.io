作者：Jaleel、Jack，区块律动

随着 NFT 市场步入熊市，即使是市场的领头羊「无聊猿」也未能逃脱，与之形成鲜明对比的 Azuki 却表现不凡。过去三个月里 Azuki 与市场趋势背道而驰，成功跃升为最强劲的蓝筹项目。牛市高峰地板价在 30 多 ETH 的 Azuki，甚至在前几天还涨到了 17ETH。不仅如此，Azuki 社区的社区氛围也是少有的，在熊市也极具活跃度和凝聚力。

直到今夜凌晨的 Elements 发售，却让社区破了大防。除了对白名单 Mint 窗口期只有短短十分钟，导致短时间内网站流量超载而直接崩溃。发售结束后，项目方将 2 万枚 ETH 直接转走。社区讨论最多的就是 Elements 与 Azuki 一代的各种特征重复度高，这对 Azuki 一代的价值可能造成直接的稀释。

一开始社区质疑的是 Elements 与 Azuki 的高度重合性，然而随着开图的 Elements 越来越多，竟有不少持有者发现自己的 Elements 与其他持有者的长得一样，不是高度重合性，而是完全一模一样。

截止到目前，Elements 已经被发现了 5 套完全相同的图片。这五套的编号分别是：#2210 与 #10744；#1077 与 #8600；#16046 与 # 8914；#16580 与 #5613；#19697 与 #2475。不仅 Elements 的 NFT 显示的前端图片一模一样，甚至将图片保存下来后的图片命名也都一样。

随着 Elementals 开图时被发现撞图情况越来越多，团队才意识到这个问题并做出回应。Azuki 联合创始人 2PMFLOW 在社交媒体上回复社区表示，该情况系技术故障，由于数据提供商的事件日志因以太坊区块重组而过时，导致少数 Token 的元数据被错误处理。团队正在努力恢复正确的图像和元数据。

「区块重组」发生在区块链同时产生区块时，如果存在错误或由于恶意攻击将会导致临时重复的区块链。因为添加到下一个区块的矿工必须决定分叉的哪一边是正确的链或规范链，因此会发生「区块重组」的情况。一旦矿工选择了分叉或规范链，另一条链就会丢失。

回顾上一次重大的「区块重组」情况是在 2023 年 5 月 25 日，以太坊 Beacon 链经历了一次七个区块的重组，暴露出了被称为链重组的高级安全风险。Eth2（现在的共识层升级）Beacon 链上的验证者在某个客户端更新提升了特定客户端后变得不同步。 

「撞图」其实并不罕见

对于 Azuki 联合创始人 2PMFLOW 给出的「撞图情况因以太坊区块重组，系技术故障」的回复，cryptochasersco 创始人 scriptmoney.lens（@scriptdotmoney） 无法苟同，他在社交媒体上提出了自己的看法：发生撞图情况的原因是服务端监听 event 没有处理好异常，不是数据提供者。言外之意，Azuki 所说的撞图情况系技术故障似乎无法自洽。

在探讨 NFT 图片生成的技术原理时，我们需要关注的三个核心要素是：图层样式、生成算法，以及生成的 NFT 图片的上传形式。 由于这三个因素的变异性，可以产生无数种形式的 NFT。

首先，图层可以被视为预先制作好的素材，比如：10 种颜色不同的背景、10 种不同的发型、10 种不同的帽子、10 种不同的手持部件等等，它们都是各自的独立图层。一旦确定了图层的数量和样式，我们便可以预设稀有度指数、目标生成的图片数量等多种影响因素，从而创建出一套可以生成各种组合的 NFT 算法。然后，根据生成算法将各个素材图层从底部叠加至顶部，组成一个完整的 NFT。

而根据项目方事先准备的图层数量和算法复杂程度，生成的 NFT 出现撞图的概率也不尽相同。比如，当一个 NFT 项目方只准备了 3 个图层，每个图层仅有 10 多种元素，且生成算法也相对单一，那么该 NFT 系列出现撞图的概率便会非常大。但像 Azuki 这样有多图层和上百个元素，且生成算法极为复杂的蓝筹项目，即使再生成几万个新的 NFT，也很难出现撞图的情况。

当然，因为图片生成的过程是在链下完成，因此即使出现了撞图现象，也可以在图片正式上链前对其进行二次筛查，将重复的图片过滤掉。这就需要项目方在上链前对生成的 NFT 图片进行充分的筛选过程，即便是项目方对原始素材直接套用生成的情况，也应有合理的筛选过程，以确保程序运行的性能，以及所有组合都严格遵循预设的概率分布。

那么综上所述，Elementals 的问题出在哪里了呢？

首先最明显的，Elements 很可能直接套用了 Azuki 或者 BeanZ 的图层及元素甚至是生成算法。 但或许是由于该系列 NFT 的主体围绕水、火、电等有限的「元素概念」，相关的生成算法没有调试到位，导致 NFT 出现重复生成的情况。

其次，社区关于「相同图片在同一区块中 Mint」的说法似乎也不完全成立。 据一位 NFT 技术人员透露，Elementals 系列中出现撞图的 NFT 并没有前后或者相近的序号排列，表明它们在同一区块中被铸造的概率较小。因此还有一种解释，就是 Azuki 团队并没有对生成后的 Elementals 进行二次筛选，导致重复的图片被直接传至链上。

当然，不管是什么原因，像 Azuki 这样的蓝筹 NFT 项目方出现撞图事件都是不可接受的。很明显，团队在发售 Elementals 之前并没有做足充分的准备。

在眼皮底下换图，链上的图片为什么还可以修改？

针对 Elementals 铸造和开图的问题，Azuki 联合创始人 2PMFLOW 针对 Elementals 开图出现相同图像在社交媒体上回复社区表示团队正在努力恢复正确的图像和元数据。而随后 BlockBeats 也发现，在查询 Elementals 系列合约时，部分 NFT 的 tokenURL 已无法显示查询结果。同时，Elementals 系列部分 NFT 预览图在 Blur 等 NFT 交易平台也无法正常显示。Azuki 团队就这样在社区的眼前来了一次实时「换图直播」。

已经上传至链上并铸造出来的 NFT，为什么还可以更换和修改呢？这就需要对 NFT 图像的存储方式进行简单的回顾。

我们知道，今天的互联网是建立在 HTTP 协议上的，HTTP 协议是用于从网络服务器传输超文本到本地浏览器的传送协议。当我们通过 HTTP 获取网页或文件时，我们从服务器请求并获取信息。这是一个中心化的过程，存在多个问题，比如如果服务器出现故障，那么文件将无法访问；同时，如果大量用户试图同时访问某一文件，将会对服务器带宽造成极大压力。

相比之下，IPFS 采用了一种更分布式的方式。IPFS 是一个旨在创建持久和分布式存储和共享文件的协议和网络，是一种新的超媒体传输协议，地址了大量数据存储和带宽问题。IPFS 将文件和其他数据作为块存储在多个节点上，并使用唯一的哈希值来标识每个块。当用户请求某一文件时，IPFS 将从最近的节点获取块，而不是从原始服务器获取。这使得数据的访问更加快速且更加稳定，不会因为单一服务器的故障而导致数据无法访问。

正因为这些优势，IPFS 成为了如今存储 NFT 元数据的理想选择和最大共识。 NFT 包含的信息通常包括一些关于艺术作品的描述，如其作者、创建日期等，这些信息被称为元数据。由于 NFT 的不可更改性，元数据需要被永久存储，而 IPFS 的分布式特性恰好能够提供这样的服务。同时，IPFS 的分布式特性还可以防止 NFT 的元数据被篡改，确保了 NFT 的安全性。

在 IPFS 的文件系统中，每个文件将会根据内容生成一个哈希值，在 IPFS 中的文件将根据这个哈希值来进行索引。并且提前检验这个哈希值是否已经被存储过。如果被存储过，直接从其它节点读取它，不需要重复存储，一定意义上节约了空间。这极大地提升了资源存储的效率，也降低了交易费用。

而在查询 Elementals 系列的合约后我们发现，Elementals 的图片元数据并没有上传至 IPFS，而是选择存在 Azuki 自己的中心化服务器中。因此，元数据的所有权和修改权限都握在 Azuki 团队手中，可以随时进行更改。

实际上，并不是只有 Elementals 一个项目选择将图片存在自己的服务器里，许多「有钱的」蓝筹 NFT 都会这么做，以获得对项目更大的把控能力。但与 Elementals 相比，此前的 Azuki 系列的元数据就是存储在 IPFS 上的。那团队为什么要在项目的去中心化进程上「开倒车」呢？或许一开始大家就意识到，Elementals 可能会出问题。

而还要再离谱一些的是，团队在元数据修改的过程中，还犯了新的错误。scriptmoney.lens 还发现，Azuki 团队上传的修复 metadata 似乎是 file 文件，因此一访问就跳到下载界面。也就是说，服务器并没有上传正常的 json 元代码，而是直接上传了图片文件。因此当 Blur 这样的第三方交易平台在服务器上寻找 NFT 图像数据时无法正常读取，也就出现了本段开头呈现出的问题。

而当我们查看元数据指向的下载文件时，会发现这些新的图片甚至不能用「修复」这个词来形容。以 Elementals #16580 为例，截至撰稿时，该 NFT 的元数据文件与原版相比，可以说没有一处是相同的（见下图）。尽管团队此前预告过自己的换图动作，但如此大的差异，或许对许多人来说多少都有些难以接受。

不管从哪个角度评价，Elementals 的发售都是一起灾难性事件。它打击的不光是 Azuki 社区本身，也是整个头像 NFT 领域。从经验来看，人们对于 NFT 项目方的信心和信任往往是十分脆弱的，而 Azuki 的吸血鬼行为正在迅速消耗着这个行业仅存的信仰。

当然，面对社区的质疑和批评，Azuki 团队也正面做出了回应，表示会努力弥补过错，并在今日下午宣布了新的「绿豆」空投。尽管 Elementals 的场面十分混乱，但或许我们可以再留出一些信仰，给 Azuki 团队最后一次机会？
