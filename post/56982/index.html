**新东西： DePIN 是个啥？ **

De ： De centralized De ：去中心化

P ： P hysical P ：物理

I ： I nfrastructure 一：基础设施

N ： N etworks N ：网络

字面翻译：去中心化物理基础设施网络。简单说来，即是利用 Token 来激励驱动大众合力构建真实世界中的物理基础设施网络的系统。至下而上的方式，从个人价值、激励、隐私、性价比等角度出发，来搭建一套基础的信息化服务。

**至下而上的建设 **

至下而上的基础设施建设方式历史上也不是新鲜事：

在绿洲边缘，简易驿站起初只为商队提供补给（类似 DePIN 中的基础节点）。商贩和手艺人汇聚此地，驿站逐渐繁荣起来，最终演变为如长安和敦煌这样的交通与文化中心。这些自发的驿站，因其独特价值与服务，逐步连接，最终构筑了壮观的丝绸之路。整个丝绸之路的形成，就是去中心化的基础设施建设，基于各自需求与价值，体现了真实世界的去中心化协同效应。这种至下而上去中心化的数字世界的建设，也会带来的健壮性、稳定性和网络效应。但至下而上的驱动，缺乏信用和激励的方式很大的缺点就是“慢”。 Web3 去中心化的信用以及 Token 繁多的激励方式恰好就进行了补充，就能够很好的解决“快慢”效率的问题。

**人机边界的消解 **

Hashkey 为 DePIN 设计了一个精练的公式： Web3 + AIoT = DePIN 。它捕捉了一个理念：通过 Web3 承载 AGI 在物联网中创造的价值。

DePIN 可以被视为 A IOT 的下一阶段的进化。在这个新的阶段，人工通用智能（ AGI ）赋予设备大脑和意识，而 Web3 则为至下而上的 IOT 网络建设提供基础信用、价值、激励、权益、供需匹配等等。物联网设备与链上资产映射，被赋予设备不可篡改的设备身份（ DID ）和价值容器（ Wallet ）。进一步，设备采集的数据和执行行为都可以被 Token 化，并通过去中心化交易所（ DEX ）与人类或其他设备进行交换。让“机器”成为“我们”。（让机器和人、机器和机器互通，这也是 9 年前创建 QQ – IOT 为每个设备分配 QQ 号的初衷。

**边界：给 DePIN 减减负 **

从畅想回到当下。按“去中心化物理基础设施”的定义，那几乎所有的分布式网络都可以被归类为 DePIN ：

Filecoin ， Arweave ：去中心化的硬盘

以太坊：去中心化的电脑

比特币：去中心化的账本

但显然，定义覆盖过于宽泛容易失焦。

**狭义 Depin 的概念 **

Mason Nystrom 更精确地描述特定类型的网络，将“去中心化资源型网络”（ DeREN ）从“去中心化基础设施网络”（ DePIN ）中加以区分，这使得 DePIN 不再作为所有分布式网络的通用术语，而是具有特定特性和功能的网络类型。 DePINs 和 DeREN 都是通过激励机制优化硬件资源分布和使用的网络概念，但它们在处理资源可替代性和地理位置依赖性方面的应用和功能存在差异。

DePIN ：以地理位置为关键要素的网络模型，通过激励机制来部署地理位置特定的硬件设备，从而生成非同质化的资源。非可替代怎么理解？在城市各个地方都可以看到部署的移动电话基站（中国铁塔）。但同一种基站， A 基站在尖沙咀， B 基站在沙田，但由于地理位置的不同，它们并不能互相替换。在这个意义上，每个基站都是独一无二非同质化的资源。

DeREN ：一个对地理位置依赖性较弱的资源型网络，其通过激励机制建立市场，并增加现有或闲置的可替代资源的供应。这些资源是由地理位置无关的硬件生成的。在云服务领域， DeRENs 类似于云的设施即服务（ IaaS ）。例如，使用 Avware 存储，用户体验通常与用户所在地理位置的关联性较弱，服务的同质化程度高。

**不完美先驱： Helium **

虽然 Helium 有颇多非议，但无论是从 Market Cap 和知名度来看， Helium 毫无疑问是“狭义 DePIN ”的头牌和先驱。

**Helium 简介 **

Helium 是基于长距离、低速率的无线通信协议 LoRaWAN 的 DePINs 。目标是创建一个开放、去中心化的网络，任何人通过购买和部署 Helium 的热点设备来自行构建和维护网络，为需要无线数据传输的设备提供覆盖。也就是说，每个人都在做中国铁塔（为电信、移动、联通建设基站的企业）做的事情，口号也特别响亮 People’s Network (人民网络)。

**Helium 现状如何 **

几个简单的指标：

  • HNT （ Helium Token ）：截至 2023 年 8 月， NHT 价格约在 2USDT ，相较于最高峰期价格 -95%（同期比特币 -60%），在所有 Token 中表现较差。

  • 技术路线：目前已从自有的 L1 ，迁移至 Solana ，共享 Solana 上较为丰富的生态（当然也省事了）。在 LoRaWAN 之后 Helium 的新叙事是关于 5G 微基站的建设，作为运营商传统 5G 运营商网络的一个补充。

  • 覆盖情况： Helium 热点：986743 覆盖地区国家：190 个 城市：77263 个。

**优点与非议 **

先说优点： Helium 成功证明了 DePIN 类型的市场匹配（ PMF ），证明其作为一种新颖理念是受到市场接受的。其网络覆盖范围也证明了它具有实际的应用价值。然而，非议也很大：

  • 商业模式拧巴： Helium 基于 LoRaWAN ，一种偏工、农业&智慧城市级的传输协议。消费者建设和维护网络，商业机构为用户。形成了消费者（ C ） to 商业（ B ）的模式。但实际上普通人架设的网络难以满足商业需求，如网络质量、信誉、规模和价格，这影响了网络的运作效率和效益。

  • 共识设计缺陷： DePIN 对地理位置是重要因素， Helium 的证明覆盖（ PoC ）共识机制理论上确实可以保证 LoRa 网络的有效覆盖，但在实际应用中，它并未实现供需的有效匹配（还是真没什么太有价值的“需”）。也就是在沙哈拉沙漠上买几个 Helium 机器，也能挖到 HNT ，但又有什么价值呢？

  • TokenEconomy 的粗糙： Helium 项目在初始阶段，创始团队和早期投资者积累了大量 HNT ，导致社区成员只获得不到 30% 的 HNT ，这种不公平的分配模式与随时间逐渐减少的 HNT 奖励共同导致新参与者的收益低于预期，从而对项目的长期可持续性产生负面影响。与互联网软件产品不同，物联网产品具有明显的边际成本。设备终端触达消费者之前，它需要经过各级销售网络，包括各层级的代理商。当 Helium 的 HNT 价格下跌到足以动摇市场信心的程度时，代理商滞销（深圳实际有好几家公司押了上千万 Helium 设备），大量的设备导致代理商的经营困难。

  • 市场宣传污点： Helium 估计察觉到 C2 B 模式的挑战，曾误导称 Lime （一家滑板公司）是其用户。这种不实宣传和内部不当行为被 Forbes 揭露，导致声誉受损并在币安被下架，这反映了 Helium 在商业模式和公信力上的问题。

  • 生态孱弱： Helium L1 使用的是 Erlang 语言开发和编写的，非 EVM 错过了 Defi Summer 的大爆发，让其成为孤岛。好在已经整体搬家来到了 Solana 上。

尽管 Helium 存在一些问题，但仍应保持开放的态度，相信它只是在探索去中心化网络的艰难第一步。对于 Helium 早期的调整，不必过于追求“完全去中心化”和“不可篡改”的理想状态。至少，在对深圳十余家 LoRa 设备厂商的调研中，他们基本都对 Helium 都有一定了解，这已经超越了许多 Web3 / Blockchain 项目的“出圈”程度。

**未来价值： DePIN 链接带来的可能性 **

有人质疑 DePIN 只是炒作已有的概念。确实， Helium 满打满算甚至已经有 10 年的历史。然而，进步的本质常常就在于不断地挖掘和优化旧的概念，从而达到新的突破点。

**信任层：真实物理世界的可信数据 **

物联网经典模型包括三个核心层次：传感、网络、和应用。但 DePIN 为此引入了一个关键的“信任层”。这一层关注于确保数据的真实性、权益的可靠性和激励的公正性。

怎么理解这种信任关系？可以用一个理想化的遗产信托的场景来串联：

老人濒临生命尽头使用智能合约来锁定其数字遗产。他身体各部位都有传感器来实时监测他的生命体征。

（1）当医生判断老人去世的时候，按下 Trig 调用遗产主合约。

（2）遗产主合约调用预言机合约，让预言机节点验证这些传感器的数据。（3）预言机合约会使用拜占庭算法来验证各个传感器通过分布式网络传回的数据的一致性。

（4）足够的节点确认老人已去世时，遗产主合约会被触发。

（5）解锁老人数字遗产，并将资产转移到事先指定的配偶和子女的账户。（6）核心的交易被记录在链上，而传感器数据则会存储在 IPFS 上，确保数据可以被律师团队审计。

此情景中，只有传感、网络和应用都可被信任，才能保证最终结果的正义性。

**网络层 + 信任层 **

网络层与信任层的结合，会比以往更加的成熟。物联网通信网络层面技术也是不断向前迭代的。随着 WiFi7 、蓝牙 5、 UWB 等技术的日益成熟，数据传输速度得以提升，接入设备数量也得到了增加，同时时延问题也得到了有效缓解。同时，自组网技术也在不断增强。相较于 Helium 启动的 2013 年，已经存在大量支持 Mesh 网络（去中心化自组网）的方案，例如 Zigbee 、 Z – Wave 和 Thread ，以及新出现的 WIFI7 ，蓝牙 5.4 等协议，都进一步推动了 Mesh 网络方案的增多。

而 Mesh 网络的去中心化，动态扩展、自我组织、自我管理、主动纠错以及自我平衡等特性，与 DePIN 的理念不谋而合。这些原生支持 Mesh 的芯片和模组与 DePIN 结合，或许不在需要像 Helium 这样完全从零开始搭建，可以依靠简单的 OTA 来进行 DePIN 的兼容。

**信任层 + 应用层 **

高性能公链：

而在应用层，相对的基础设施也得到了全面优化，公链的吞吐能力显著提高，这使得物联网的关键数据可以通过 Roll – up 或者一些新公链以较低的成本实现上链。 Web3 的价值不再只有费用结算这一条。

资产建模（ Digital Asset Modeling ）：

资产建模（ Digital Asset Modeling ）：

物联网设备与 Web3 的结合为设备资产化打开了新局面。得益于 Web3 中新型 Token 的出现。通过使用 ERC -721、 ERC -1155、 ERC -3525 等标准，我们可以实现设备多维度的资产建模。不仅可以对设备所有权进 NFT 化，还可以将其他数字权益（如收益权、决策权）等进行细致定义，使之成为可交易的数字资产。设备运行过程中产生的数据,也可以通过预言机触发这些权益的变更。最终，设备被赋予了流动性，得到了全面的“资产化”。可以说，丰富的 Token 类型推动了设备资产化，让物联网向价值互联网转变。它开启了物联设备经济模式的新空间，值得进一步探索。

这里我们提出资产建模（ Digital Asset Modeling ）的概念，与代币化（ Token ization ）和传统证券化（ Securitization ）有所不同。它通过多种数字资产类型和金融编程,对现实世界中的资产进行抽象表达和编程描述,而不仅仅是简单地生成 Erc -20 Token 或证券。资产建模关注多维度地对资产进行拆解和重构,设计出新的数字资产关系和交互逻辑。这使得数字世界可以更贴近真实世界的复杂性。总体来说,资产建模代表了一种全新的数字资产表达和金融编程范式,它富有创造性地扩展了数字世界的范畴。

  • NFT ( ERC -721 等 ) 允许生成唯一的数字资产，定义了资产的绝对所有权。

  • SFT ( ERC -3525 等 ) 可以将一个资产进行拆分，生成可交易的 asset shards ，一定程度上保留了资产的唯一性，同时增强了流动性。

  • 复杂代币关系标准 ( ERC -6551 等 ) 试图 Model 资产的多维度属性，例如收益权、使用权等。这表达了资产不同权利的分离与重组。

总而言之，丰富的 Token 类型为物联网设备的资产化提供了可能。设这为物联网设备的经济模式打开了新的空间。整个物联网也将演变为一个崭新的具备流动性的价值网络。

**资产建模举个例子（ Digital Asset Modeling ） **

将这个共享充电宝作为一个使用资产集合的方式进行建模。

  • 所有权 / 分红权 A ：用 Erc3525 的 SFT 进行描述，金库产生收益都可以获得分红。

  • 功能挂载 B ：通过 Erc 6551 的 TBA 挂载在 SFT 上。

  • 功能 C （充电宝租赁）：每一个充电宝都是映射一个 1155 的 NFT ，结合 4907 实现 NFT 出借和抵押等功能。

  • 功能 D （大屏广告）：每日大屏的有效展示时长按分钟被拆分为一份份 3525 的 SFT ，进行自由竞价拍卖。价高者得。

  • 功能 E ：通过共享充电宝机网络，为提供“还”服务的机器支付费用，自动进行支付。

  • 功能 F ：商场发来消耗费用的 NFT 账单，自动进行支付。

  • 其他 G ：上方摄像头通过客流监控感知商场内的客流情况，通过预言机，调整充电宝租赁合约的价格。

我们看到一个原本简单的共享充电宝机器，基于 DePIN 的信任层被拆成了多种类型的资产和权益，完全透明化，去中心化。

**供应链金融新的可能性 **

当我们谈论设备数字权益在链上的拆分，实际上是在探讨一种新型的资产证明和流转方式。以 DePIN 为中介，不仅仅是一个纯粹的技术连接，而是一个金融和供应链的桥梁。在传统的供应链金融中，中小型创业公司想要生产硬件产品，往往面临巨大的资金压力。这是因为，生产消费级硬件需要大量的前期投入，而这些前期投入的回报周期可能较长。为了支持生产，公司必须寻求外部融资或借款，但这不是每家公司都能够轻易做到的。因此，许多具有潜力的硬件原型最终因为资金问题而难以实现商业化。但如果我们利用 Web3 ，把设备未来激活后的收益（如销售、租赁等）进行链上化，将其转化为一种数字资产，那么这种资产就具有了流动性。这意味着创业公司可以提前将这些预期的未来收益作为抵押，与上游硬件供应商达成合作，为生产活动提供必要的资金。

而供应商则拥有一种新型的风险保障 — — 他们不仅仅是基于创业公司的信誉进行合作，更是基于链上资产的真实性和可验证性进行决策。这种模式为中小型创业公司和供应商创造了一个双赢的合作场景。创业公司获得了资金支持，降低了生产风险；而供应商则通过持有数字资产，获得了更为稳定和可靠的收益来源。

总的来说，通过将设备的权益拆分并在链上进行表示，与供应链金融相结合，我们可以创造出一个更加灵活、高效和安全的金融环境，为硬件创新提供更强大的支撑。

**原生 RWA **

RWA （ Real World Asset ）的定义正在经历一场转变。传统上，我们将其视为链下金融产品（如债券、不动产）在区块链上的映射。但多数人的观点更为开放，认为 RWA 的范畴应更为广泛。就如欧文费雪对资产的定义：一切能够产生持续收入的事物 。

提出“原生 RWA ”的概念，是指通过 Web3 技术实现的真实世界中的事物，链上数字权益拆分。这种资产化过程能够创造出超过其单一部分之和的价值。其中， DePIN 成为 RWA 的核心组成。它通过传感层和网络层连接机器与真实世界，极大地降低了信用和交易的摩擦，从而揭示了 Web3 的真正潜力。这不仅为我们提供了一个刷新的角度来看待和利用真实世界的资产，还为区块链技术在实际应用中带来了新的机会。

变革往往源于一些微小的芽动。 DeFi 最初也只是 Web3 边缘小众的玩物，如今已经开启了新的局面。同样，类似 DePIN 这样的尝试现在还处于萌芽阶段，它试图连接数字世界和现实世界，构建新的基础设施。不用该苛求它立刻完美，而是给予鼓励和包容空间，并参与催生未来的变革。
