Windows 11在侧边栏添加了Copilot（副驾驶）AI助手。

当地时间5月23日，微软Build开发者大会拉开帷幕，成为一场以AI为中心的大型活动。微软公司就如何在其应用程序和服务中扩展人工智能发布了多项公告。

其中，微软宣布为Windows 11添加Copilot（副驾驶）AI助手，使用该功能时，就像在Edge、Office应用程序甚至GitHub中看到的Copilot侧边栏一样。

此外，人工智能开发机构OpenAI将开始使用Bing（必应）作为ChatGPT的默认搜索引擎。新的搜索功能即日起向ChatGPT Plus用户推出，并将很快通过ChatGPT中的插件为所有免费的ChatGPT用户启用。

比基本款Bing Chat功能多

“一旦打开，Windows Copilot侧边栏就会保持在您的应用程序和窗口中，始终可以充当您的个人助理。”微软Windows和设备主管帕诺斯·帕内（Panos Panay）解释道，“它使每个用户都能成为高级用户，帮助您采取行动、自定义设置并无缝连接您最喜爱的应用程序。”

Windows Copilot可以总结在应用程序中查看的内容，重写甚至解释这些内容。看起来与Bing Chat中的对话框非常相似，因此可以向它提问。

Windows Copilot可以总结在应用程序中查看的内容。

Copilot不会直接取代Windows 11任务栏上的搜索栏，而是在旁边新设一个单独的按钮，就像语音助手Cortana在Windows 10的任务栏上有自己的专用空间一样。

由于Copilot已集成到Windows中，用户还可以执行一些操作，例如让该助手“调整我的设置以便我可以集中注意力”。它比微软今年早些时候添加到任务栏的基本款Bing Chat功能要多得多。

由于Windows Copilot与Bing Chat建立在相同的基础上，微软甚至允许开发人员把为Bing或OpenAI的ChatGPT编写的插件扩展到Windows Copilot。这为Windows Copilot开放了开发人员为ChatGPT和Bing创建的许多新功能，并且未来的改进将自动转移到 Windows Copilot。

在帕内今年1月称“人工智能将重塑你在Windows上做所有事情的方式”之后，微软在过去6个月中一直暗示要在Windows中构建人工智能功能。许多人一直期待微软等到Windows的下一个主要版本推出系统级AI功能，但该公司显然正在更快推进AI计划。

微软将在6月开始公测Windows Copilot，然后将其更广泛地推送给Windows 11用户。

随着微软向OpenAI投资数十亿美元，OpenAI将Bing作为ChatGPT聊天机器人的默认搜索引擎也就不足为奇了。

微软消费者营销主管优素福·迈赫迪（Yusuf Mehdi）在23日的一篇博文中表示：“我们与OpenAI的奇妙合作关系是我们在新Bing上取得进展的基础。”“ChatGPT现在将拥有世界一流的内置搜索引擎，可通过网络访问提供更及时、更新的答案。”

就像Bing的聊天体验由GPT-4提供支持并使用引用链接一样，OpenAI的ChatGPT聊天机器人将包括搜索和网络数据，其中还包括引用。

Bing与ChatGPT的集成是微软在搜索引擎的一次胜利，此前有传言称三星将转向Bing搜索，三星最近决定继续使用谷歌。

增加插件功能

此外，微软还有一些关于其365 Copilot的重大消息：插件的添加。AI助手现在将支持3种主要类型的插件，包括Teams消息扩展、Power Platform连接器和使用ChatGPT技术的工具。用户还可以从数十种第三方插件中进行选择，例如来自Atlassian和Adobe的插件。

微软表示，它将以与ChatGPT相同的标准构建所有Copilot和Bing Chat插件。这意味着用户将能够在所有3种AI驱动的工具中使用相同的插件，并且开发人员也可以更轻松地创建插件。

微软也正在为Edge浏览器带来 365 Copilot。该工具将位于浏览器的侧边栏中，可以使用用户正在查看的网站上的内容来帮助处理Microsoft 365应用程序（Outlook、Word、Excel等）中的项目。

微软也正在为Edge浏览器带来365 Copilot。

例如，该工具应该可以帮助用户完成诸如起草电子邮件、将数据添加到电子表格、根据聊天生成状态更新等操作。该集成还将支持上述用于365 Copilot的插件。

Windows Terminal通过与GitHub Copilot的集成，也获得了一个人工智能驱动的聊天机器人。使用GitHub Copilot的开发人员现在可以直接在终端内使用聊天机器人来执行各种操作、获取代码建议和解释错误。微软还表示，它正在探索将GitHub Copilot引入其他开发人员工具，如WinDBG。

微软还引入了一个新的Dev Home仪表板，可以更轻松地设置和使用Windows开发机器。

此外，微软将人工智能生成的应用程序评论摘要引入微软商店，并设立了一个新的人工智能中心来突出使用人工智能的Windows应用程序。

将为AI生成图片打水印

另据PCWorld报道，担心其他人冒充AI生成的艺术作品的艺术家现在可以稍微松一口气了：微软已经同意为其应用程序生成的所有AI艺术作品签名，并带有一个加密水印，表明它是用一种算法制作的。

“内容出处和权威联盟（C2PA）”于2021年开始着手开发一个开放标准，用于指示数字图像的来源，以及它们是真实的还是人工智能生成的。这个问题在今年3月被推到了聚光灯下，当时AI生成的教皇穿着时尚羽绒服的图像在网上疯传，AI艺术生成器Midjourney进行了限制以防止更多类似图像生成。

C2PA规范提供了通过图标或弹出窗口告知读者哪些艺术是AI生成的，哪些不是。

作为C2PA的创始成员，微软将在Microsoft Build开发者大会上正式宣布，将对来自Bing Image Creator和Microsoft Designer的AI生成图像进行加密签名。

使用Bing Image Creator制作的图像已经在右下角包含一个小的“b”。然而，由Microsoft Designer的可视化设计工具生成的图像却没有。现在，微软将根据C2PA标准对这两种工具生成的图像进行签名，并在图像的元数据中明确披露来源。

AI图像生成器Adobe Firefly已经通过名为内容凭证的功能实现了这一点——尽管它仅在Adobe的应用程序中受支持，并且不会转移到其他应用程序（如Windows 11照片）中可见的元数据。

根据C2PA规范，除了元数据之外，消费者还会看到一个小指示器悬停在生成的图像上方或旁边。

Build大会上，微软还宣布了“Azure AI内容安全”，这是一项允许AI协助或取代内容审核员的服务。AI为AI扫描的图像打分，并标记人类监督者。微软表示，该服务将于6月推出，每扫描1000张图像收费1.5美元。微软重磅更新：Windows引入AI助手，ChatGPT默认用必应搜索
