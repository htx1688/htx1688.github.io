原文作者： Osgur Murphy O Kane， Nansen 研究员

原文编译： ChinaDeFi 

zkSync 可以说是目前最令人兴奋的 L2。它已经部署了主网「Baby Alpha」，预计将在 2023 年第一季度初推出其完整主网。 这将是第一个兼容 EVM 的 Validity Rollup，150 个项目已经表示他们将部署在 zkSync 上，包括大多数 DeFi 蓝筹股。

此外，zkSync 将在 2023 年推出第一个 L3 概念验证。到目前为止，L3 的前景纯粹是假设，看它在实践中是如何运作将会是一件非常有趣的事情。

zkSync 对 EVM 兼容性的专注是一个关键的设计选择，这可能会决定它的成败。 它有利有弊，利在于 EVM 的基础设施和生态系统。其他项目也正专注于开发新的 VM，他们希望这些 VM 能够更好地构建适合大规模采用的应用程序。虽然 zkSync 中的原生帐户抽象将提供很好的用户体验，但从长远来看，是否能与 EVM 保持一致仍有待观察。

L3 的可定制性和性能如何，以及他们在任意设计操作方式方面拥有多大程度的自主权，还有待观察。但有一件事是肯定的，zkSync 值得被密切关注，因为它接近并部署在主网上。

本报告将概述 zkSync 及其主要特性。还包括对其潜在竞争优势的评估，帐户抽象的概述，以及对 zkSync 设想的 L3 未来的分析。

简介

zkSync 是什么?

zkSync 是一个通用智能合约 Validity Rollup 平台。它专注于与以太坊社区保持一致，被大多数人认为其有潜力成为主网上第一个与 EVM 兼容的 Validity Rollup。

zkSync 今天处在哪个阶段？

来源：zkSync

10 月 28 日，zkSync 发布了他们的「Baby Alpha」，这仅限于使用主网的团队。在此之后，「Fair Onboarding Alpha」将仅限于已注册参与的项目。这意味着他们将能够在真正的用户进入之前测试他们的产品是否存在任何错误或问题。在这个阶段，所有的代码都是开源的。最后，完整的 Alpha 版本将会发布 (预计在 2023 年第一季度初)。

这种循序渐进的方法是有意义的，因为糟糕的发布可能不仅会对 zkSync，而且也会对以太坊扩展项目都有严重的负面影响。

融资

Matter Labs 已融资 4.58 亿美元，最近的 C 轮融资是由 Blockchain Capital 和 Dragonfly 共同领投了 2 亿美元。此前的融资包括 2 亿美元的生态系统基金，由 a16z 领投的 5000 万美元的 B 轮融资，以及 A 轮和 Seed 轮的 800 万美元。

zkSync 的资本非常充足，它在 L2 战争中是一支不可忽视的力量。

Starkware：总融资 2.73 亿美元 Fuel：总融资 8150 万美元

融资并不意味着成功，但 zkSync 已经筹集了大量资金来继续完成其愿景。zkSync 预计将成为第一个与以太坊兼容的实时通用 Validity Rollup，这可能会给它带来非常显著的优势。此外，它一直与以太坊社区保持一致，这可以帮助它建立一个强大的生态系统和有竞争力的护城河，而其他 Rollup 在部署到主网的时候可能就需要很努力的去克服困难。

那么它的主要特征是什么呢?

主要特征

文化

zkSync 将自己定位为专注于以太坊的扩展解决方案。它尽最大努力地与以太坊社区保持一致。当然大多数 Rollup 都与以太坊保持一致，因为它拥有很多优秀的开发人员和巨大的用户活动量。然而，有些人仍然对部署在其他链上和不同配置的可能性持开放态度（这种观点是有意义的；为什么要把自己限制在一个网络上呢？）

zkSync 完全专注于以太坊，并以证明其与以太坊社区保持一致的 5 个关键品质而感到自豪：

通用

EVM 兼容性

支持 Solidity

ETH 支付 gas

开源

来源：zkSync

这表明 zkSync 专注于与以太坊保持一致。EVM 的网络效应非常猛烈，所以这种策略至少在短期内是有意义的。然而，反对这种方法的一个论点是，EVM 有许多设计限制，而其他虚拟机和编程语言更适合未来的应用程序，这才有助于将加密技术带入主流。这是 Starkware 和 Fuel 等其他扩展解决方案正在采取的方法。然而，围绕 EVM 的基础设施的财富外溢，10 个顶级 TVL 的链中有 9 条都与 EVM 兼容，所以这并非巧合。

EVM 兼容性

zkSync 通过拥有一个适应 EVM 开发工具并与有效性证明配合良好的 VM 来实现 EVM 兼容性。它并不完全等同于 EVM，因为 Matter Labs 选择了一种更容易处理有效性证明的设计。

这种设计减轻了 Matter Labs 构建与所有 EVM 操作码兼容的系统的负担，而 EVM 操作码要复杂得多 (例如 Scroll)。

这意味着移植到 zkSync 的 dApp 可能需要进行一些调整，而不是使用与 EVM 相同的设计。

zkSync 将用 Solidity 编写的代码编译成 Yul。Yul 是一种在 LLVM(编译器框架) 中将 Solidity 编译为字节码的中间语言。这使得 zkSync 能够实现 EVM 兼容性。社区内已经讨论了未来支持 Rust 和 Javascript 等语言的计划。这可能会增加那些通过最小的更改来部署到 zkSync 的项目数量，例如已经在 Rust 中为其他区块链编写的项目。如果它能成功，就可以极大地扩展构建者可用的工具，同时保持应用程序之间的可组合性。

不同的 Validity Rollup 设计

zkSync 的 EVM 兼容性发生在语言级别，而 Hermez 和 Scroll 则更进一步，它们构建了一个可以解释 EVM 字节码的 VM(这种方法构建起来明显更复杂，长期以来被认为是不可行的)。

通过在语言级别部署，zkSync 在 EVM 执行的所有步骤中都不需要有效性证明，这应该会使证明过程更容易去中心化。完整 zkEVM 的支持者认为字节码级别的 zkEVM 是安全性方面的黄金标准。

在 zkSync 上部署

由于没有在字节码级别实现 EVM，所以 zkSync 也增加了额外的风险 (由于两个环境之间的差异)。这是否会是一个问题？如果是问题，那么会是多大的问题？这些都还有待观察。zkSync 声称 99% 的代码可以直接部署到 zkSync 2.0，而不需要任何更改。zkSync 承诺今年将开源所有东西，并且我们可以描述 zkSync 为在语言级别上与 EVM 兼容。这意味着 zkVM 将 EVM 语言 (例如 Solidity) 编译到 SNARK 友好的 VM。这给协议增加了一些编译器风险。

Token 经济学

Token 实用性是一个有问题的主题。就关于如何将令人满意的价值增值机制纳入 Token 以赋予其基本价值，在这个问题上人们还没有达成共识。其原因是由于希望项目避免落入到安全法规的范围。

例如，像 Uniswap 这样的协议（可以说是 DeFi 的金童）拥有一个基本价值不确定的 Token 。虽然它被用于治理和奖励，但它没有赋予其长期基本价值的实际增值属性 (除了治理 (其本身的价值值得怀疑))。

第 2 层 Token 学有一个额外的注意事项，就是它们是建立在另一个 Token 学协议 (例如 ETH) 之上。在顶部添加额外的 Token 会增加额外的风险和摩擦点。但是，为了发展壮大和维护生态系统采取激励措施也是很有必要的。

zkSync 选择用 ETH 支付 gas 费，这是对以太坊和以太坊社区表达忠诚的强烈信号。一些 Rollup 采取某些行动是为了安抚以太坊社区，但没有展现真正的忠诚度，用 ETH 支付 gas 是真正意义上的强烈声明。

这么做也有助于避免额外的摩擦 (考虑到交易无论如何都需要在基础层的 ETH 中结算)。使用原生 Token 作为收费 Token 可能会有损于用户体验。最终，用户体验将在竞争日益激烈的 Rollup 领域中胜出。

除此之外，Matter Labs 对 Token 的用途保持沉默。对 Rollup 的治理也有潜在的问题，因为这带来了治理捕获额外风险。就所有权而言，大多数 Token 实际上是中心化的，只有少数参与者能够影响治理决策。这增加了风险。

采用典型 PoS Token 的方法将给第二层带来额外的审查风险，因此其也被认为是不可取的。

该团队声称，三分之二的 Token 将分配给社区/生态系统，而三分之一将用于激励团队和投资者。这是一个健康的分割，因为许多 Token 都被设计为有利于内部人士，三分之一的 Token 应该为贡献者提供足够的激励。例如，Starkware 有 50% 的 Token 供应分配给内部人士，zkSync CPO Steve Newcomb 曾说过 Arbitrum 的 50% 的 Token 供应也分配给了内部人士。

护城河

先发优势有多强?

虽然率先进入市场无疑会给 zkSync 带来先发优势，但这种优势能有多强，它的可持续性又如何呢？为了探索这一点，有必要研究与 EVM 兼容的链。

在 2020/21 年的牛市中，EVM 兼容链迅速崛起，其原因是由于它们为以太坊主网的用户提供了替代方案，而同时以太坊主网的价格非常昂贵。

这就带来了以下玩家：

BNB Chain

Polygon

Avalanche

Fantom

所有这些链都发展得非常快。然而，跨 EVM 链的轻松部署使得开发人员和用户非常唯利唯图，人们可以快速地从一个链转移到下一个链，可以说没有任何忠诚度可言。以 BNB 链为例；虽然它有先发优势，但很少有项目被认为具有持久价值。

zkSync 会受到唯利是图的开发者/用户的攻击吗?

zkSync 可能会有所不同，因为它与以太坊社区紧密相连。人们普遍认为，Validity Rollup 是最有前景的扩展技术之一，而 zkSync 又与以太坊合作，特别还把 ETH 作为 gas Token ，可能会让开发人员倾向于在其上进行开发。事实上，已经有 150 个以太坊项目表示他们计划在 zkSync 上进行部署，其中包括 Aave、Uniswap、Chainlink、Curve 等。

然而，zkSync 的长期成功将取决于正在构建的其他 EVM 兼容的 Validity Rollup 的设计和用户体验，例如 Scroll。如果这些公司能够推出客观上更好的产品，那么先发优势就会变得毫无意义。然而，如果 zkSync 和它的 L3 生态系统能够在有竞争力的竞争对手推出产品之前就能产生强大的吸引力，那么它的先发地位可能会给它带来显著的竞争优势。

账户抽象

帐户抽象指的是将区块链上的两种帐户类型减少为一种的目标，这会为用户降低复杂性，因为他们将不再需要区分不同的帐户类型。

以太坊自 2017 年以来一直在努力实现它，然而，这是一项非常具有挑战性的任务，也被支持扩展路线图而被搁置。现在，zkSync(以及其他 L2，如 Starkware 和 Fuel) 将在默认情况下合并帐户抽象。从本质上讲，帐户可以实现任意的逻辑。

虽然从表面上看，这似乎并不令人兴奋，但它实际上带来了一些非常有趣的 UX 改进。

一次批准多个交易

在以太坊上使用 dApp 是令人沮丧的，因为每个链上交互都需要交易 (以及 gas 费)。这是糟糕的用户体验。通过帐户抽象，可以将多个交易捆绑在一起。下图显示了它将如何极大地改善用户体验：

社会恢复

加密领域最大的用户体验问题之一是助记词，以及忘记助记词的全部后果。

社会恢复使丢失私钥的用户可以授权一个新钱包作为合法所有者。

可以是：

硬件钱包；值得信赖的朋友/亲戚；第三方服务；甚至是以上两者的结合。

通过社会恢复，用户仍然可以保管自己的资金，并控制资产。Vitalik 是社会恢复钱包的倡导者，他说这是他保护钱包的「首选方法」。

多重身份验证

帐户抽象允许用户将其帐户设置为：1. 需要来自多个密钥的签名；2. 只有在满足指定条件时才执行交易。虽然这听起来类似于使用像 Gnosis safe 这样的多重签名，但帐户抽象可以为钱包提供比典型的多重签名更好的可定制性和可用性。

帐户抽象可以为用户提供一些自定义功能，以改善用户体验。安全级别可以根据用户的需要进行调整。

任何 Token 都可以支付 gas 费用

账户抽象可以使用任何 Token 来支付 gas。虽然目前还不确定 zkSync 是否会启用此功能，但假设它可以在后台运行，就可以将费用 Token 交换为 ETH。

会话密钥

这使得钱包可以预先批准与 dApp 交互的某些规则，这样用户就可以随心所欲地频繁使用它，也不必签署交易。这尤其适用于区块链游戏。

dApp 可以在指定的边界内自由使用，这使得用户能够在保持边界的同时最大限度地提高易用性 (并有效地降低风险)。目前，用户需要信任一个实体来代表他们签名 (或者自己签名)。

插件

账户抽象启用插件，这意味着用户可以在创建帐户后添加和删除功能。

限时交易

帐户抽象的另一个潜在特性是有时间限制的交易——用户可以创建在指定的未来时间内执行的交易。

Volition

zkSync 还旨在作为一种 Volition 来运作，被称为 zkPorter。zkPorter 允许用户在 Validity Rollup(链上数据可用性) 和 Validium(链下数据可用性) 之间进行选择。这允许用户/开发人员为他们的特定应用程序选择所需的安全性。例如，游戏应用程序可能会选择 Validium，因为 Validity Rollup 安全性的额外成本对他们来说是不必要的，而 DeFi 应用程序可能会选择 Validity Rollup，因为它增强了安全性和链上数据可用性。

zkPorter 链下数据将被发送到 Guardian Network—这将是一个由 zkSync(尚未发布) Token 保护的 PoS 网络。这增加了三分之二诚实验证者的信任假设，并展示了 Validium 和 Validity Rollup 之间的权衡。Validium 便宜得多，但安全性也较低。

L3

L3 是什么?

为了理解「Layer-3」的含义，有必要先理解 L2 Rollup。L2 Rollup 指的是区块链扩展解决方案，处理以太坊 L1 上的交易，L1 上的结算和数据可用性。L3 之于 L2，就像 L2 之于 L1。

L3 为需要在 EVM 之外进行计算的特定应用程序启用自定义扩展。普遍的共识是，L3 支持应用程序的自定义功能，并利用 L2 来实现扩展性，例如隐私。

L3 适用于可能适用于特定应用程序用例（例如游戏和企业应用程序）的 Validium。

L3 将状态和执行放在多个服务器上，这对于计算密集型应用程序来说是有必要的。预期的好处包括：

预防 MEV 的解决方案可能导致很少或没有 MEV。

可以针对应用进行优化：费用市场、硬件需求等

Fractal Hyperchain

Fractal Hyperchain 是 Matter Labs 用来描述他们在 zkSync 上 L3 的愿景的术语。所有的 Fractal Hyperchain 都将由相同的电路技术进行约束，并由相同的 prover 进行验证。其意义在于：

通过拥有相同的 prover，Hyperchain 之间有一个原生桥。桥一直是生态系统失败的关键因素，而原生的、无需信任的桥将是实现可扩展性、安全可组合性和互操作性的关键。Matter Labs 表示，在未来，Hyperchain 之间将根本没有桥。然而，这纯粹是假设。尽管有「无限」性能的说法，但任何单一的解决方案都不可能无限扩展，所以当 L3 发布时，观察它能提供多少可扩展性将是很有趣的。

L3 将是高度可定制的，设想有许多特定的应用链。

L3 用户可以根据自己的需求从 3 个数据可用性选项中进行选择：

Validity Rollup(安全性最高，成本最高)

Validium(数据链下可用性，最快，最便宜)

Volition(应用程序/用户可以在交易的 Validity Rollup 和 Validium 之间进行选择)。

zkSync 预计将在 2023 年第一季度推出 L3 概念验证，并声称一些大品牌正在想要部署在 zkSync 的 L3 上。

Matter Labs 还表示，他们的 LLVM 编译器可以支持任何现代编程语言，如 Solidity, Rust, C++等，这将打开格局。

来源：zkSync

可扩展性和 L3 的重要性

虽然 zkSync 的目的是扩展以太坊，但有点讽刺的是，目前许多现有的区块链上都有大量未使用的区块空间。这是因为目前还没有能够吸引大量用户的应用程序。L3 所带来的潜在的可扩展性和定制性为 Web3 技术的大规模采用铺平了道路。最终，这些扩展平台的成功将取决于能否开发出适合大规模采用的应用程序。

Validity Rollup 战争的主要战场可能是 prover

虽然有许多 Validity Rollup 解决方案竞相推出他们的主网产品，并试图构建最好的生态系统，但 Matter Labs 的首席产品官认为，重点应该放在哪个项目可以构建最好的 prover 上。

为了实现 L3 Hyperchain 的愿景，zkSync 的 prover 需要被业界认可为是最好的。如果发生了这种情况，它可能会促进大量构建者寻求在共享 prover 启用的可组合且可互操作的网络中部署 L3。

如果项目/网络使用相同的 prover，它们将能够彼此互操作 (由密码学保护 (理论上))。我们的愿景是，大多数链上活动将利用相同的 prover，这将使 Web3 技术的潜力取得巨大进步。

在 Bankless 播客上，Matter Labs 的首席产品官用 SSL 的比喻来描述 zkSync 的潜力。SSL 代表安全套接字层，是浏览器和服务器在互联网上进行身份验证、加密和解密的协议。在采用 SSL 之前，电子商务是小众的，许多人不相信能把他们的信用卡详细信息放在网上。然而，随着 SSL 的广泛采用和对该技术的信心，电子商务得以发展和繁荣。

如果 zkSync 的 prover 可以被接受为是一个安全可靠的通用标准，那么一个安全可互操作的 Fractal Hyperchain 组成的巨大网络的愿景可能成为现实。这当然也取决于 L3 和区块链技术本身能否实现全球规模的产品市场契合。

去中心化

通过实现 proto-danksharding 和 danksharding，以太坊将从带宽受限的环境转移到带宽充足的环境。Nansen 的报告解释了以太坊的扩展路线图。

有一种观点认为，今天的 Rollup 过度关注数据优化，而不是带宽优化，从而导致了围绕增长状态的问题。进而也带来了在未来去中心化 prover/sequencer 的挑战。

L3 / Fractal Hyperchain 可能为减少状态增长提供一个令人满意的解决方案。然而，这还有待观察，如果它们发挥了承载大量链上活动的潜力，未来可能会在 L2 层出现问题。

去中心化位于许多 Rollup 路线图的末尾，但目前尚不确定如何实现这一目标，以及多快可以完成。

可组合性

Validity Rollup 可以与结算层组合，例如以太坊 L1。为了实现这一点，需要为每个区块生成有效性证明。当然，Validity Rollup 技术还没有达到这个水平；尽管如此，它的发展速度甚至让 Vitalik Buterin 感到惊讶。这将使 L3 能够与底层 L2 合成。

然而，这需要等待下一个区块组合回来。对于大多数应用程序来说，这应该不是问题。解决这个问题的一种方法是 L2 提供 L3 交易预确认，这应该使 L3 和 L2 交易原子可组合。最终，有可能有 n 个 L3 组成相同的统一状态，一个简单的有效性证明可以验证所有 L3。这将导致完全的可组合性。

Polynya 认为，就最大化安全性、去中心化和可扩展性而言，这是一条可行的前进道路。

这是令人兴奋的，然而，这些 Rollup / L3 在实践中如何运作还有待观察。zkSync 在这方面处于领先地位，很可能会提供 L3 的第一个实例。

结论

zkSync 最新一轮融资为 2 亿美元，生态系统总融资 4.85 亿美元，使其成为资本最好的 L1/L2 之一，有不错的扩展路线图。

它与以太坊的一致性是值得注意的，用 ETH 支付 gas 费用证明了这一点。虽然这将吸引以太坊建设者，但尚不清楚他们作为通用 EVM 兼容的 Validity Rollup 的先发优势会有多强大。

原生帐户抽象可以帮助显著改善用户体验，有可能将加密技术推向主流使用。

L3 将成为 ZK-Rollup 的关键战场，zkSync 正在寻求成为托管 L3 的公认标准。他们的 L3 PoC 将在 2023 年第一季度上线，这是值得密切关注的事情。

一个关键的挑战 (目前还没有明确的解决方案) 将是去中心化的 prover。
