投资与创业

veDAO 研究院：如何判断一个 X to Earn 项目经济模型的好坏？

一个好的经济模型应该具备以下特质：通胀率低、激励高、卖压低、使用场景丰富、流动性充足、治理主要群体来自 B 端等。

当我们投资公链等需要去中心化自治的项目的时候，需要分析其经济模型是否有对维护者的激励性和 Token 的价值捕获，以保证该项目能够给用户提供持续、稳定的服务，同时让网络维护者捕获价值，减小来自维护者的抛压；当我们投资 DeFi 等涉及大量资金交易的项目时， 分析的重点应该是经济模型是如何协调 LP 与治理代币持有者的利益分配关系的，包括收入是否能够激励 LP 继续提供流动性、治理代币持有者是否能够分配合理收入等。

好的产品才是用户留存的关键。

深挖空投猎手，是「搭便车」还是优质用户？

大多数空投猎手实际上都是以太坊的早期用户，也主要是真实优质用户。高级空投猎手平均每月在以太坊上进行超过 50 笔交易，标准空投猎手每月也进行了超过 21 笔交易。

项目空投后，所有用户的活跃度都有下降的趋势，尤其是普通用户。精心设计的空投规则可以对空投猎手产生积极的影响。设计空投规则是可以参考 Optimism。好的空投需要满足两个核心条件：合理的空投规则设计和健全的生态系统。

浅谈流动性与 PoS 如何有效结合？以 Berachain、Tenet、Mangate 为例

PoS 机制在保护公链安全性的同时也占用了大量资金，导致这些资金利用率低下。而 DeFi 也常常被人诟病，流动性没有忠诚度，项目容易陷入死亡螺旋。如何将这两者结合，让流动性能够在获得收益的同时，参与到保护公链的安全？

Berachain 希望流动性能够更加可持续，并与三代币模型组成一个流动性与代币需求之间相互促进的飞轮；Tenet 采用多元化权益证明与流动性质押，希望利用其它 PoS 代币来增加自己的网络安全性；Mangate 以 LP 代币作为质押资产释放质押的流动性。这些改进的 PoS 方案能够增加资本效率，在获得区块奖励的同时，获得 DEX 的交易手续费奖励以及其它区块链的区块奖励等，增加了 PoS 资产的用途。

CeFi

解读 Chainalysis 研究报告：散户、老手和机构们，如何给交易所贡献价值？

  1. 用户分层：根据钱包的年龄和资产持有量，用户可分为早期零售、早期专业、早期机构、晚期零售、晚期专业和晚期机构 6 类。零售用户（散户）地址总数上亿，但加起来的资产数量不如机构。

  2. 资金流入 CEX 情况：晚期机构钱包所发送的总价值占比最高，为 23.6% ，晚期专业和早期零售钱包发送价值紧随其后，分别占 18.8% 和 19.0% 。

  3. FTX 中的资金流：FTX 的散户居多，但散户每周平均资金流入仅 700 美金。而机构用户数量虽少，每周平均资金流入高达 200 万美金。FTX 希望通过向在其他交易平台上活跃的个人钱包空投奖励来获取新用户。通过将目标钱包分成这些细分部分，FTX 可以根据每个钱包所属细分的预期终身流入量，精确地向每个钱包提供不同的奖励。

  4. 用户流失率：晚期专业和晚期零售钱包在 CEX 中的流失率较低，不到 1% ；相反散户流失率高达 15% 。

深入解读再质押赛道：工作原理、优缺点和潜力项目

流动性质押解决了简化质押、不束缚流动性和提高网络去中心化的主要问题。ReStake 是指涉及重复使用流动性质押代币资产（例如 stETH）以质押到网络或其他区块链平台的验证器中的活动之一。这个概念最初由 EigenLayer 引入，它最大化了流动性质押流动性的利用，并为众多其他应用程序的发展铺平了道路。这一领域的代表项目有 EigenLayer、Tenet。

作者认为，ReStaking 不是一个迅速消失的叙事，而是 DeFi 中最重要和最有前途的领域之一。因为 ReStaking 不仅帮助用户获得利润，还帮助平台提高其安全性，特别是通过促进行业其他领域的增长，并推动市场扩张。但是，这也伴随着风险，如资产损失、智能合约风险、财产价值通货膨胀和泡沫崩溃等。

Web 3.0 

起底账户抽象生态：有哪些值得关注的钱包与基础设施？

为提升用户体验，EIP-4337 被提出，旨在通过账户抽象（Account Abstraction）来解决用户体验的复杂性。EIP-4337 的实施不仅可以改善用户体验，还为以太坊带来了更多的定制性和功能，例如自定义签名选项和不同支付方式。

AA 钱包生态包括：

  • AA 钱包：Braavos、Argent、Gnosis Safe、Avocado。

  • AA 钱包基础设施：Biconomy、Gelato、Pimlico、Candide、Unipass。

如何让账户抽象赋能基础设施，服务十亿级用户？

AA 的基础设施提供商需要为其用户提供 Bundler 服务和 Paymaster 服务，并且需要整合各种 Bundler 客户端和 Paymaster 服务提供商，以确保服务的健壮性。为了优化 API 和节点客户端之间的响应性，可能需要对 AA 数据进行索引，以减少单个请求的硬件消耗。为了提供更好的用户体验，基础设施提供商还需要为用户提供更多的服务选项。

未来，随着 AA 生态系统的不断发展和公共 mempool 的出现，选择和打包 UserOperations 的策略将变得更加复杂。在不同的市场 Gas 价格和 Gas 波动性条件下，Bundlers 可能会有不同的打包策略。

比特币生态

真假朋克之争：纯链上递归朋克竟存储在 Github 上？

Recursive Inscriptions 上线后就迎来了首个递归朋克（Recursive Punks）的启动，在社区热捧下， 1 万枚递归朋克被迅速刻录完毕，后又迅速上线 MagicEden 市场，并在该平台上日内总交易额达到 8.15 个比特币，成为平台上交易总额排名第一的系列。

戏剧性的是，用户发现自己刻录的递归朋克显示为空白，检查代码后发现，这是因为递归朋克官方开发者使用了错误的递归铭文语法，导致早前使用官方 SVG 文件刻录的用户打到的铭文为无图版。于是社区发起了一个正确的修复版本，即将原来 SVG 文件中的"/-/content/:“替换成”/content/"，由此 Recursive Punks 有了两个版本，官方发布的正式版本 v1 和社区发起的 v2 版本，且 v2 版同样得到了 Recursive Punks 官方的认可，引发了真假递归朋克之争。

虽然被社区某些成员斥责为假递归朋克，存储在 github 上，甚至严格意义上这个合集都不属于是递归铭文，但它是由官方更早地发布在协议上，更早地上架 MagicEden 市场，而在比特币生态中 NFT 基础设施尚不成熟和完备的早期阶段，是否能在二级市场中上架以及是否能被索引，都决定了该系列的发展速度甚至成为其存活的关键因素。

投资这样的资产有巨大风险。

以太坊与扩容

以太坊 EIP 评审：以太坊社区如何进行开放式的治理？

以太坊的核心部分是智能合约，智能合约围绕着标准来设立。这些共识是怎样进行整合的？用一个代码来展示的话，ERC 20 就是转移。有了简化开放的共识就可以基于 ERC 20 建很多非常棒的项目，市场市值增加，这是开放整合的威力。

一个软件真正免费的话必须要四种不同的自由——自由使用、自由学习、自由分享、自由修改。

开放也有四个层次（加密货币项目都在开放的不同阶段上）：一开始最常见的最可见是开放，然后是控制、贡献，不仅仅内部可以做出贡献，外部也可以做出贡献，Github 贡献者、开源贡献者也可以贡献于项目。人们也可以进来加入治理，只要他是利益相关方及只要他愿意贡献。最后比较有哲理性的，欢迎竞争。这四种开放让以太坊非常成功。

LSD MAP 2.0 ：多维度解读 LSD 赛道发展情况

LSD MAP 2.0 

LSD 赛道已经形成初步格局。若以层级划分，SSV Network，Obol Labs 等 DVT 技术服务商可视为 L0。DVT 技术可以使验证器更稳定、更安全行使签名责任。作为层级首个发行代币的项目，SSV Network 在品牌知名度方面占据先发优势。Lido、Ankr、Coinbase 等 LST 发行商则可视为 L1。L1 主要是佣金模式，用户主要获益点来自 ETH 的 POS 收入。L2 是基于 LST 设计出的固收类产品，稳定币，收益聚合等占据 L3 层级，也就是大家常说的 LSDFi。

本文采用基于 L2 建立的产品作为 L3的 定义。引发的 Pendle War 的 StakeDAO、Equilibria、Penpie 符合基于 L3 定义；自动复投 0x Acid 的 AcidTrip；还有简化 unshETH 操作、提高收益的 gUSHer；可以看出主流还是 veToken 引发的治理权争夺。本层级有很大想象空间，不止 L2 工具和聚合治理类型项目。比如多 L2 产品的 L3，聚合策略的前端。

  • L0 是最具有技术壁垒的，但要注意代币的实际效用。

  • L1 中龙头出现，除内部或者系统性风险，时间、空间条件已经不允许有新晋 TOP 3 ，新秀会有。

  • L2 中很大一部分不具有护城河，考验团队和 BD 能力更多。基于 LST 的基础性 DeFi 策略项目越多，L2 越繁华。

  • L3 市值/流动性受限于 L2 产品发展程度，且有很大的想象空间，需要一定时间发展。

LSDFi 无疑是会促进质押率提高，质押产生的 LST 能够进入多种项目赚取多份收益减少机会成本，甚至还能赚取超额收益。而现在的 L2 层级还没有出现能承载大量 ETH 的项目，L1 中很多 LST 并未在 L2 中使用，所以 L2 仍有巨大投资机会。优质的项目应该是真实收益+应用场景+好的 Tokenomics 的结合，目前 L2 层级已经出现了 PENDLE 为首的真实收益类+veToken 类项目，基本符合以上条件。

速览以太坊扩展解决方案演变：OP Stack、Arbitrum Orbit、Polygon 2.0…

Web3 开发者目前有三个选择：

  • 在以太坊上部署，但会牺牲可扩展性。

  • 选择 L2 解决方案，但会面临对特定生态系统的依赖风险。

  • 部署并维护自己的链，但可能导致流动性碎片化。

此外，实现互操作性也是一个重大挑战。

Bankless：Optimistic Rollup 的繁荣与困境

Optimistic rollup 的安全性基于两个核心假设：在无效状态的情况下，有人提交欺诈证明；底层 L1 仍然是抗审查的。不幸的是，即使在假设的未来世界中，所有 Optimistic rollps 都有无需许可的欺诈证明，一个令人担忧的攻击向量仍然存在。尽管不太可能，但仍然有可能阻止欺诈证据的发布，同时通过验证者勾结来规避 EIP-1559 指数级增长的 gas 费。

竞争各方必须能够在 L1 级别提交欺诈证明，因为 rollup 协议将没有任何挑战解释为对其状态的隐含同意。由于在 L1 处串通而产生的欺诈证明的潜在审查会使第 2 点无效，从而使 rollup 的安全承诺无效。

Optimistic rollup 仍然容易受到攻击，并且攻击面只会随着它们的日益成功而扩大。然而，面对以太坊的社会共识可能无法拯救被利用的 Optimistic rollup 的现实，寻求替代的扩展解决方案势在必行。

Optimistic 的对应方案更容易实现，并且在今天的以太坊 L2 领域占据主导地位，但 zkRollup 可能会破坏当前的范式，提供即时确认、更快的终局性、更高的吞吐量和原生隐私。Vitalik 也指出，rollup 可能依赖以太坊来分叉和回收资金，这是一种高风险的共识应用，因此可能导致链分裂。

DAO

链上研究：获得 ARB 空投的 DAO，都用这笔钱干了什么？

绝大多数空投代币都闲置在多重签名钱包中（约占所有空投代币的 80% ），它们似乎与基金会分配空投的总体愿景一致，即赋予协议权力以负责生态系统内的治理和决策过程。

虽然有一些协议直接卖掉了全部空投来为自己的运营提供资金，但我们认为，这些资金将用于刺激协议增长并间接使整个 Arbitrum 生态系统受益。

一些卖家将自己描绘成按照社区治理运作 DAO，但他们不愿与自己的社区分享分配计划。理想情况下，这些项目将：保留代币用于未来的 Arbitrum 治理；将其价值重新分配给社区，以激励 Arbitrum 生态系统内的产品增长。

为了进一步调整激励措施，未来的治理空投分配可能会对代币的某些行为设置限制条件，例如限制出售或强制执行长期兑现。尽管如此，我们相信当前的分配是公平的，是对哪些协议更符合 Arbitrum 生态、哪些协议不符合的自然选择。
