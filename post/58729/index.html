Debank：Web3创作者收益新风口

最近，短短这一个月时间，X（前 Twitter）和 FriendTech 两个推文类社交平台的创作者收益的模式先后获得了巨大的流量关注。虽然，这两者的收益模式都有着一定缺陷，但获得如此大的关注度，从某种层面上证明了金融+社交即 SocialFi 赛道的可行性。

除了上面提到的两个平台，近期，SocialFi 赛道还有一个值得关注的事件则是 DeBank 推出了基于 OP stack 的 Debank Chain。Debank 属于稳扎稳打的做事风格，在构建出 Hi 和 Stream 两大社交功能的之后，才着手做链，Debank Chain 作为先前搭建社交功能的加密资产层。可以见得团队目标十分清晰，将 SocialFi 依据功能拆分，先完善 Social（社交）功能，再考虑用区块链技术解决 Fi（金融）端的需求。

由于 Debank 的在社交领域布局十分细致，涉及到很多巧妙设计，因此介绍 Debank Chain 之前，先聊聊 X 和 FriendTech 已经显现出的弊端，再回过头看看 Debank 能否解决这些问题。

Musk 的 X 的创作者收益模式存在三个明显问题：

  • 缺乏透明度：并没有详细说明收益的计算方法，因此内容创作者很难了解自己的真实收益。

  • 支付系统限制多：想要收到 X 的分享收益，需要注册 Strip 账号，而该账号对于申请者所在的国家/地区有着非常严格的限制，很多创作者只能眼巴巴看着收益提醒，却无法开通收款。

  • 假账号/机器人：在 X 启动创作者收益模式后，机器人账号与垃圾信息愈发猖獗，侵占了正常用户的浏览空间。如何区别真实用户与机器人将会是马斯克未来需要面对的一大挑战。

而 FriendTech 的创作者收益模型也有着一些致命问题：

  • 社交功能并不完善：网页时常会有 Bug。

  • 交易磨损过高，商业模式无法持续：因为抢跑 bot 的存在，交易磨损非常高，迄今相关 bot 已盈利超过 200 万美金。这种磨损速度，即使是再赚钱的商业模式也不能持续。

  • 经济模型粗暴，炒作导向更高：Keys 的价格曲线只与持有人数有关，并且在无人交易时创作者无法得到收益。

下面将从资产层和社交层为切入点，讲述 Debank 的社交版图，从而一窥未来 Debank 是否能解决上述同类产品出现的问题。

Debank 资产层：Debank Chain

Debank Chain 的开发以 OP Stack 为基础，主要围绕解决上述关键问题，同时保持强大的安全性：最大限度地降低 Gas 成本、提供类似于账户抽象近乎于 Web2 的体验，以及确保 L1 资产的安全。

Debank 团队修改了共识机制，使单个交易的气体成本降低了 100 到 400 倍，以适应社交互动的高频特性。

并且团队已经在链级别集成了一个类似于账户抽象的系统，让用户享受接近 Web2 的体验，同时保持与当前 EVM 标准的 100% 兼容性。

这一新账户系统支持使用专用 L2 私钥签署的交易，减少了社会场景中对 L1 私钥的使用，提高了用户 L1 资产的安全性。

Debank Chain 于 8 月 11 日启动测试网，并计划在 2024 年推出主网。目前只有 Rabby 钱包集成了 DeBank 测试网，官方并未放出公开 RPC。因此，可以极大限制了 bot 与 女巫攻击。

要注意 Debank Chain 并不是 Debank 资产页面已经上线的 Debank Layer2（又写做 Debank L2），前者还处于测试网阶段。

而后者早已上线一年有余，目前正在作为 Debank 社交产品的支付账户，但在公开渠道并未能找到详细文档或介绍，此 Debank L2 也曾作为众多空投博主埋伏的标的。

而放到今日看，已经开源的 Debank Chain 同样被定位为社交资产链，因此很有可能会成为 Debank L2 的替代品。

Debank 社交层

Debank Hi

Debank Hi 的定位是 Web3 即时通讯服务，也就是聊天工具。对于 Web3 消息应用程序来说，具有以下特点：

  • 基于 0x 地址的账户系统

  • 开放、无权限

  • 透明

其界面布局有点像 Twitter 内置的聊天，但和常见聊天工具不一样的是，Debank Hi 通过引入「注意力市场」的概念，有效规避了垃圾信息问题。

什么是注意力市场呢？

将用户对消息的注意定义为稀缺资产，设计注意力资产交易市场，发送消息就需要购买收件人的注意力，接收消息就是出售自己的注意力。利用有效的自动化定价模型来降低交易摩擦。

注意力作为一种资产，定义用户打开信息行为被视为一种注意力资产。但值得一提的是，这里的定义并不包括阅读某条信息后所花费的注意力。

不同的信息提供了不同的价值来吸引接收者的注意力，因此这里花费的注意力不能简单地定价和交易。

引入注意力市场，可以有效地防止垃圾信息，并且保证去中心化，即没有账户会被算法封锁。

虽然僵尸行为经常被指责破坏正常用户的体验，但要想在没有中心化风险的情况下封杀僵尸绝非易事。

标记和阻止潜在的僵尸地址是任何去中心化平台都不应该拥有的特权。建立一个有效的注意力市场意味着每个人都应无差别地付出代价，包括机器人。因此，僵尸不再需要区别对待。

在 Web2 中，用户不断受到信息和广告的轰炸。平台通过出售用户的注意力获取广告利润和规模增长带来估值提高的股权收益。而通过交易用户掌控自己的注意力，可以收回主动权，并获得所花费的时间和精力的补偿。

Debank Stream  

Debank Stream 是一个完全类似 Twitter 的发文系统，其特点是 Stream 账户绑定了 Debank 的资产统计老本行，凭借着资产统计的护城河，Debank Stream 可以有很多高阶的新奇玩法。

其一是引入了贡献值（Contribution Value）这个概念，就如官方宣传语「a Web3 social feed that calculates each user’s contributions to the platform itself!」所说，Stream 可以计算每个用户对平台本身的贡献！真正的 Web3 产品需要将其获得的每一点价值归功于其最初的创造者。

有了贡献值这个概念，就可以实现奖励池（Reward Pool）分配。这是一项帖子小费功能，是真实收益，不仅奖励作者，还奖励帖子发布过程中的所有贡献者。

每当在 Stream 中发布一篇文章，Debank 就会创建一个相应的奖励池。在接下来的 3 天里，任何用户都可以向该池存入资产。

每个用户在这 3 天内对帖子的贡献都会被记录下来，以确定每个人可以获得的奖励。奖励在 3 天后结算，如果资金池的资金超过了帖子的实现价值，用户将获得与其贡献相匹配的奖励，超出部分归作者所有。

如果资金池资金少于实现价值，则按每个用户对实现价值的贡献比例进行奖励。奖励通常在结算后立即支付。

Debank 团队认为是用户的时间和注意力都是有价值的。相比于 Web2 平台，在 Twitter 上花费的每一分钟，都成为它的价值基础，并最终通过广告转化为 Twitter 的利润。

而在 DeBank Stream 上，用户投入的每一分钟、发表或阅读的每一篇文章、发表的每一条评论和转贴都会被计算为价值贡献并记入您的账户。

每一个自然日，用户都会收到反映其对平台贡献的贡献记录。对于 DeBank Stream 上的每篇帖子，还将计算其实现价值（Realized value），该数据用于描述帖子在 DeBank Stream 上传播过程中所获得的用户关注价值。

在计算中，查看、评论和转发的用户数量以及这些用户的净资产等因素都被考虑在内。如此可以确定单个用户对其做出的贡献。

Twitter 上大肆流行的仿冒攻击，DeBank Stream 可以通过考虑用户净值和 TVF（Total Value of Followers，粉丝资产总值）来帮助用户鉴别内容源头的可靠性（传统的仿冒账户可以刷粉丝数，但无法增加假粉的总资产，妙啊！），这也将确保 DeBank Stream 捕捉到真实用户的关注价值。

总结

Debank 团队布局了多条细分产品线，好处是讲故事能力强，完全不缺叙事；而坏处也显而易见，不少创新型产品功能半途放弃，比如：

  • Debank 取消广受好评的 Feed 功能；Rabby 删除 Debank 入口；

  • 还有就是现如今在有正常运行的 Debank Layer 2 的情况下，却又发布了定位不明确的 Debank Chain，这些凸显了团队的决策缺少透明度。

不过这也恰好反映出 Debank 团队执行力的果断，目前 Debank 全力在优化产品的社交端，利用其 Web3 资产分析的技术沉淀，有效解决困扰 X 和 FriendTech 已久的仿冒攻击和垃圾信息问题。

而且聊天工具 Hi 中引入的注意力市场功能可以有效地提高用户使用体验，杜绝垃圾信息的侵扰。

在资产端，Debank 透过区块链可以解决传统 Web2 支付受限问题，而且通过高度定制化，带有抽象账户的 OP Stack，要远比 FriendTech 所依靠的的第三方 Layer 2 Base 更适应社交场景，有效地规避掉垃圾账户的产生。

FriendTech 能否改善经济模型还有待观察，而 Debank 厚积薄发，相信未来在 SocialFi 领域肯定会为我们带来惊喜。
