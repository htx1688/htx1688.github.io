让我们开始深入了解区块链扩容解决方案 Rollup。

什么是 Rollup？

首先，Rollup 是一种扩容解决方案，其基本思想是将一些交易数据从主链（如以太坊）移出，然后在侧链或 Layer 2 上进行处理，只在需要时将最终结果提交到主链。这种方法可以减少主链上的交易负载。目前有两种流行的 Rollup 类型：Optimistic Rollup、ZK Rollup。

尽管它们在证明方法上存在差异，但它们都有一个共同的目标：在 Rollup 成本和收入之间实现平衡。要了解它们如何做到这一点，我们首先需要对 Rollup 经济学有一个基本的了解。

Rollup 经济学的三个参与者

首先，在 Rollups 中有三个主要参与者：用户、运营者、以太坊/基础层。这些参与者中的每一个都代表了 Rollup 内部价值流动的一部分。

Rollup 的生存依赖于用户，用户在 Rollup/Layer 2 上支付 Gas 费用以执行交易，这些费用是 Layer 2 收入的主要来源之一。关于收入的详细信息将在后文中详细讨论。

用户支付的费用将流向 Rollup 的运营者，他们负责排序、捆绑、批量处理或计算那些需要有效性证明的交易。

最后，来自 Rollup 的压缩交易或消息需要在基础层上进行结算，这是所有步骤中最昂贵的部分。同时，运行一个系统不可避免地会产生成本和收入，这些成本和收入激励着每个参与方进行工作。

在短短 3 个月内，zksync 支付了超过 1300 万美元的数据可用性（DA）和验证成本，其次是支付了 830 万美元的 Arbitrum，以及支付了 650 万美元的 Optimism。但这些成本的来源是什么？导致这些成本的三个主要因素包括：

· 运营商成本（Operator costs）

· 数据可用性成本（Data availability costs，DA）

· 验证成本（Proving costs）

Rollup 成本

运营商成本： 涉及到批量交易处理的排序、交易验证、区块生成等方面的费用。由于大多数 Rollup 运营商现在是集中化的，这些成本由协议本身或合作方承担。

数据可用性成本： DA 成本是批量提交的费用。一旦运营者积累了足够的数据，它会将数据以CALLDATA的形式发布到基础层。发布数据的费用由基础层承担，数据的市场价格受 EIP-1559 规定。

验证成本： 在 zkrollup 中，L2 上的节点需要提交有效性证明，以证明更改的正确性。每当需要更改状态时，这个过程都需要产生验证成本。

Rollup 收入

现在我们了解了 Rollup 的主要成本，必须有相应的收入来抵消这些成本。Rollup 的收入依赖于两个主要领域：

  • 交易费用

  • 代币发行

交易费用： 每当用户在 Rollup 上进行交易时，都会从该交易中收取一定的费用。此外，Rollup 还可以通过拥堵费（在序列器中）以及从交易中提取的 MEV（矿工最大可提取价值）来产生收入。

代币发行： 推出原生 Layer 2 代币可以成为团队的一个重要收入来源。代币有助于支付基础设施成本，同时在运营商和投资者之间相互激励，也促进了在共享服务方面的去中心化（Layer 2 的未来）。

在不包括代币发行和融资收入的情况下，zkSync 仍然能通过从交易费用中累计获得约 2000 万美元的收入，扣除成本后获利 687 万美元，位居榜首。与此同时，Base 和 Arb 都分别获利 350 万美元，并列第二。

Rollup 经济学总结

Rollup 涉及到三个主要参与者：用户、运营商和基础层（L1）。

运行这个系统的成本包括运营商成本、数据可用性成本和验证成本（在 zkRollup 中占主要部分）。为了抵消这些成本，Rollup 的收入依赖于交易费用和代币发行。

更深入地了解用户和运营商之间的价值流动可以总结为以下方程式：

用户支付费用 = L1 数据可用性成本 + 运营商成本 + L2 拥堵费

运营商成本 = L1 数据可用性成本 + 维护运营商的成本

运营商收入 = L2 用户费用 + 序列器中的 MEV

运营商利润 = 运营商收入 - 运营商成本

通过这些基本的数学计算，我们可以估算不同 Rollup 中运营商的盈利能力。

鉴于这一点，维持预算平衡或盈余仍然是每个 L2 的主要目标。因此，许多 L2 正在尝试不同的经济设计，包括：

  • 通过策略性的方式来降低到 L1 的发送成本

  • 优化 L2 的拥堵费

结尾

目前我们仅仅触及到了 Rollup 的冰山一角。关于 Rollup 经济学，还有很多值得讨论的内容。为了更深入地了解，建议阅读《Understanding rollup economics from first principles》。
