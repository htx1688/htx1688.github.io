什么是 Web3 社交？

Web3 社交应用程序与 Web2 有何不同？

权力下放、所有权和去信任化的早期承诺是否步入正轨，还是在构建复杂技术所带来的无数权衡下动摇了？用户、创作者及其社区将会发生哪些变化？

这些是我们将在 Web3 社交系列中深入探讨的一些问题。我不是技术人员，也不是 web3 社交方面的专家，所以我会边写边学习！前几篇文章将致力于快速概述定义“web3 社交”的事物，包括项目类型、主要角色、主导叙述等等。只是为了确保我们对所关注的内容有一个总体了解。

首先，我将分享我一直在与 web3 社交项目数据库并行处理的 WIP 映射。在您的帮助下，我将不断更新此映射！但首先这里是关于为什么这个系列存在的一些背景。

该系列的背景故事和意图

在花了 2022 年一整年的时间从事心理健康项目之后，我又完全专注于 Web3，目标是作为顾问和创建者与各种项目及其社区合作。当我正在寻找新客户时，一位好朋友向我介绍了 Hadrien，他是 Sismo 的创始人兼首席执行官，这是一家总部位于巴黎的公司，应用 ZK 进行隐私和用户数据聚合。

我最好的情况是协助 Sismo 进行增长和营销，但对话中出现的结果更令人兴奋。Hadrien 建议资助我在 web3 社交网站上撰写教育系列文章。作为一个喜欢调查、学习和分享我所学到的东西的人，这是一个即使我尝试过也找不到的机会。

所以我在这里深入研究 web3 社交，试图理解概念并将许多聪明的构建者的工作分解成易于理解的部分。

我已经向许多建设者和思想家学习，并将继续这样做。其中包括来自 Yup 的 Nir、来自 Farcaster 的 Dan 和 Varun、来自 Lens 的 Bradley、来自 Zora 的 Jacob、来自 Guild 的 Raz 和 Reka、来自 SGV 的 Jakub、来自 1kx 的 Peter 和 Nich、来自 Rehash 的 Diana、来自 Converse 的 Pol、来自 XMTP 的 Matt、来自 Guild 的 Adam贝洛等等。

Web3 Social v0 现状，2023 年 7 月

我认为一张好的旧地图一开始就会很有趣。我们计划在接下来的几个月内升级和修改该映射以及相应的数据库，以便我们更详细地探索该空间。

让我们快速描述一下 web3 社交状态 (v0) 的每个类别。

协议层

协议为开发人员提供了Web3 社交应用程序的基本构建块（或原语）：发帖、评论、关注、消息传递等。因此，协议将开发人员作为最终用户。

有多个属性使 web3 社交比 web2 社交对开发人员更具吸引力：

  • Web3 协议的目标是去中心化和“可信中立”，这意味着没有中央团体能够阻止开发人员使用 Web3 工具。这与 Reddit 最近的情况类似，Reddit 显著增加了使用其 API 的成本。结果，大多数第三方开发者不得不关闭他们的应用程序。

  • Web3 社交图谱（交易历史记录和用户数据）可以公开访问，并且可供开发人员使用，而无需引导自己的社交图谱。众所周知，这是一件很难做到的事情。

  • Web3 构建者可以利用现有构建块和开源代码的主体，将想法的执行难度从零提高到 MVP。

协议的去中心化程度可能会随着时间的推移而波动，因为他们的团队必须做出中短期的权衡。一些项目正在选择不同的方法，包括渐进式权力下放或“充分权力下放”。后者基于这样的想法：协议只需要在实现其原始价值主张所需的范围内去中心化，非必要的功能不必去中心化。

Web3 社交尚处于早期阶段，关键协议仍处于开发的初始阶段。Farcaster 和 Lens 等一些公司一直部分专注于构建 web2 功能的 web3 版本，例如发帖、评论和关注。现在，Web3 更新颖的原语——例如铸造、收集和授予对基于 NFT 的服务的访问权限——正在被添加到这个组合中。总的来说，这些元素将在应用程序层启用新的用例。

XMTP 是 web3 协议的一个示例，它解决了消息传递原语的问题，并允许用户通过 Luster 和 Converse 等客户端直接在其区块链账户上跨不同应用程序接收消息。

身份层

身份层通过用户直接在去中心化标识符所有者下创建和聚合数据，为web3 社交应用程序提供更丰富的体验。这包括创建去中心化的身份、名称和头像、个人资料、交易历史、数据隐私和选择性信息共享，并提供对各种数字社交空间的访问。

这一层的一些项目也是协议，但考虑到处理身份和社交数据的项目的广度，我认为它们应该在堆栈中拥有自己的层。我觉得身份层的创新将深刻影响应用程序和客户端层创建的体验。

对 Web3 社交的期望之一是用户将控制自己的数据，并且应该能够在他们使用的应用程序中获得定制的体验，即使是第一次。为了实现这一目标，构建者需要身份层项目来提供丰富多样的数据。

POAP 是身份层项目的一个示例，该项目用于丰富聊天应用程序 Salsa 的用户体验。

客户端和应用程序层

客户端和应用程序是构建在 web3 社交协议之上的面向用户的界面和产品。他们利用可用的广度协议和身份，为收藏家、创作者和普通消费者等最终用户精心设计有价值且新颖的用例。

我最初的想法是根据应用程序对不同 web3 社交角色（如收藏家、策展人和创作者）的关注程度来对其进行细分。我最终根据应用程序的主要功能对应用程序进行了更简单的分类。有时很难对 Stealcam 等最开箱即用的体验进行分类。

Web3 社交应用程序的一个例子是 PartyBid，它使人们能够通过真正新颖的体验轻松地使用加密货币购买东西。Partybid 应用程序利用 Party 协议。

关键推动因素

这些是一系列技术，没有这些技术，协议、工具和应用程序就无法存在。一些例子是 ERC 721/1155/6551 标准，许多项目都是围绕该标准设计的。

在这个类别中，我还想包括关键的推动因素，例如在 web3 社交协议之上构建多个客户端的能力。
