今日，Ethscriptions 协议创建者 Tom Lehman（化名 Middlemarch）发推宣布，以太坊铭文虚拟机 ESC VM 在 Goerli 测试后正式上线。

在经历了市场被盗、铭文暴涨、坊间炒作的一系列事件后，走“复古风”的 ETHS，将驶向何方？刚推出的 ESC VM 有哪些新特性，它又将如何“赋能”ETHS ？Odaily星球日报将从 ETHS 的起源聊起，为你讲述一个在 ETH 上复刻比特币铭文的故事。

ETHS：以太坊上的 Ordinals

7 月初，Tom Lehman 推出了 Ethscriptions 协议。而在中文世界，这一项目更多被称之为 ETHS。Tom Lehman 曾任 Capsule 21 的联合创始人、Genius 前 CEO 及联合创始人。

这一项目的有趣之处在于，在我们的印象中，技术似乎都在向着一种更为“先进”的方向进化。如何支持更通用的计算、实现更为复杂的功能、执行更加复杂的智能合约似乎“理应”是加密世界发展的方向。而 ETHS 却有着一股“返璞归真”的气质。

与此前大火的比特币铭文（Ordinals）类似，ETHS 也引入了一种新的方法将数据写入链上 。具体来说，在以太坊的交易中，calldata 是一个“注释字段”。例如，当攻击事件后，我们常见的链上“短信聊天”通常就基于这一字段进行。而当与智能合约交互时，该字段也被用来给智能合约传递信息。

ETHS 与 NFT 类似，又有所不同。该产品使用以太坊交易的 calldata 字段存储数据，ETHS 远比 NFT 便宜 ，项目方表示其费用比 NFT 便宜了 40 倍。如此大的差距则是因为 NFT 使用了昂贵的智能合约，而 ETHS 并未使用。

在 ETHS 的设计中，他们将需要存储的数据（如 NFT）存放在这一字段中。用户使用 hexhero 等工具将 URI 数据（统一资源标识符）转换为十六进制格式。然后在链上发起一笔交易，并将十六进制数据存放进 calldata 字段，这即可变相实现数据的上链存储。

一笔在以太坊网络上的“铭文”就这样诞生了。目前，ETHS 所支持的文件大小上限约为 90 KB，且无需智能合约 。

推出 VM，为“不智能”增添“智能”？

7 月 17 日，初生不久的 ETHS 就遭到了自己的第一个重大考验。ETHS 的市场 Ethscriptions.com 遭到了黑客攻击，大量列出的铭文被盗。在这次攻击中，大约 123 个个人地址总共丢失了大约 202 个 Ethscriptions。此前，ETHS 早已经历过一次暴涨，个别 Ethscriptions 的售价一度触及 5  ETH。

本次漏洞可以追溯到两位联合创始人创建的一个智能合约，其中包含的一小段代码允许人们从市场上撤回本不属于他们的 Ethscriptions。

但随后，这一事件得到了解决。创始人表示已经与许多受该漏洞影响的用户取得联系。

7 月 25 日 ， Ethscriptions V2版本正式上线。此外还推出了一个新的面向以太坊订阅市场的协议 ESIP-2 。

新市场的上线再次引爆了 ETHS 的交易热情。OpenSea 数据显示，截至目前，其总交易量已达 638 ETH。

而在今日，ESC VM 的测试网上线，正将 ETHS 带向新的轨迹。

在 ESIP-4 提案中，以太坊铭文引入了虚拟机。这是一个建立在 Ethscriptions 之上的新协议。ESC VM 为 ETHS 赋予了计算的功能。而 ETHS 也引入被称为哑巴合约“Dumb Contracts”的合约系统，用户可通过铭文与之交互。

需要注意的是，所谓哑巴合约，正如其命名所言，该合约并不是真正的智能合约，ESC VM 也不是真正的虚拟机。

ESC VM 为哑巴合约提供了类似于 EVM 的环境。哑巴合约可以执行两个命令：部署（创建新合约）、调用（调用现有合约的状态更改函数） 。

此类合约的交互是一种较为特殊的交互方式。因为通过铭文的方式存储，它们绕过了 EVM 执行和存储成本。这也使得哑巴合约可以获得远低于智能合约的费用。但也正是由于此，此类合约并无通用性 ，也不像真正的虚拟机一样可以实现复杂的链上计算。

如何更简单的理解 ESC VM？在官方文档中，对其进行了非常简明扼要的说明——BRC-20 是一项突破性的技术，ESC VM 的目标是在 BRC-20 的基础上做到更好。ESC VM 试图对 BRC-20 进行根本性的改进。（但官方并未说明哪些改进属于“根本性”的。）

目前，在测试网部署的 ESC VM 已经可以实现基础的 DEX 功能。

“我们在玩一种很新的东西”

毋庸置疑，BRC-20 的大火启发了 ETHS，在其官方文档中也不吝对 BRC-20 的赞美。

而除此之外，NFT 市场的高度中心化，也是 ETHS 诞生的主要原因之一。团队在文档中坦言，在现在的 NFT 市场中，大量 NFT 无论是图片存储、NFT 发售，均存在着高度中心化的现象。试想一下，你已经收藏了多少图片丢失的伪 NFT 了？而在发售环节则更是可怖。

BRC-20 的公平发布解决了团队中心化的问题。而这也启发了 ETHS 的诞生 。

当比特币铭文出现之后，meme 市场为之欢呼。这是一款“根正苗红”的 meme 产品，用户可在 BTC 这一真正去中心化的 Layer 1 网络之上实现“发币”功能，且代币无预留、无增发。最原始的铭文（而非智能合约）竟然实现了现在日益进步的先进公链都难以实现的“去中心化”。

而 ETHS 也拥有相同的特点。ETHS 的团队认为，该协议彻底改变了 NFT 格局，并为完全去中心化的链上存储提供了新的解决方案。他们认为，“这可能会开辟新的创造性的表达途径，并重新定义未来我们感知数字资产的方式。”

但在被其迷人的叙事吸引之前，我们不妨把故事回到一切的起点——还记得完全去中心化的比特币吗？

当人们厌倦了没有智能合约、“落后”的 BTC 网络，ETH 诞生了。当人们发现智能合约让一切变得那么复杂，人们又回到了 BTC 网络去寻找最初的去中心化：在没有智能合约的网络上炒作略带“智能”的 BRC、在足够智能的 EVM 网络上炒作假装“不智能”的铭文、在假装不智能的 ETHS 上实现初具“智能”的哑巴合约……这恰如马克吐温所言：历史不会重演，但是会押韵。

这场不断“假装”的创新游戏，又将于何时休止呢？
