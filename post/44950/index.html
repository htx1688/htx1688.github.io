今日早间，a16z Crypto 通过官方推特分享了一些该机构自己构建的工具及概念证明 ，内容涉及零知识证明、合约安全、权力下放等多个领域。从对象上来看，这些工具及概念主要面向的是那些在漫长熊市期间仍不倦耕耘的开发者，感兴趣的朋友们不妨上手一试。

以下为 a16z 推特全文，由 Odaily 星球日报编译。

我们很高兴看到 crypto 社区出现了许多令人兴奋的新创意。在接下来的推文中，你将可以找到一些有助于你继续 build 的开发工具。

我们分享这些工具和概念证明的目的是向社区展示如何在开发、生产中使用那些最新的技术原语，并借更多开发者之手来加速这些技术原语的采用进程。

零知识证明方面

一、zk 文档（zkDocs）

zkDocs 可以被理解一个支持零知识证明的在线文档工具。它允许共同参与某项工作流中的各方集体共享并验证信息，在不暴露特定标准的前提下确保信息符合特定标准。

详细介绍：https://a 16 zcrypto.com/zkdocs-zero-knowledge-information-sharing/

Github 传送门：https://github.com/a 16 z/zkdocs

二、空投隐私保护工具（Privacy-protecting Airdrops）

顾名思义，这是一个针对空投场景的隐私保护工具。我们创建该工具的目的是利用零知识证明来保护普通用户，通过改进空投时的隐私暴漏状况来提高安全性。

详细介绍：https://a 16 z.com/2022/03/27/crypto-airdrop-privacy-tool-zero-knowledge-proofs/

Github 传送门：https://github.com/a 16 z/zkp-merkle-airdrop-contracts

合约安全方面

一、ERC 4626 性能检测器（ERC 4626 Property Tests）

这是一个用于检测 ERC 4626 （一套用于代币化金库的标准）下合约可执行性的工具，可帮助金库构建者检测那些有可能造成破坏或导致漏洞的违反标准行为。

详细介绍：https://a 16 zcrypto.com/generalized-property-tests-for-erc 4626-vaults/

Github 传送门：https://github.com/a 16 z/erc 4626-tests

二、可更改型智能合约检测器（Metamorphic Smart Contract Detector）

这是一种用来检测可更改型智能合约的工具，可抵御通过改变智能合约内容而发起的攻击。

部分智能合约在部署之后依旧可被更改，意味着开发者可以随时更改其中的代码，这对用户构成了严重威胁。本工具的作用就是允许任何人轻松检测一个特定的智能合约是否存在更改可能。

工具链接：https://metamorphic.a 16 zcrypto.com/

去中心化 & 权力下放

一、以太坊轻客户端 Helios

这是一个基于 rust 语言的以太坊轻客户端，有助于进一步推动以太坊的去中心化。Helios 提供了对以太坊区块链的完全无信任访问，且无需存储历史交易信息，需要时仅需 2 秒即可完成数据访问。

详细信息：https://a 16 zcrypto.com/building-helios-ethereum-light-client/

Github 传送门：https://github.com/a 16 z/helios

二、无需信任的链上拍卖（Trustless, on-chain Auctions）

这是“维克瑞拍卖（Vickrey auction）”的链上实现。所谓“维克瑞拍卖”，是指所有竞拍者通过密封投标的方式竞价，出价最高的竞拍者获得拍品，并支付第二高的报价。

这套工具的基础原理是，以太坊的 CREATE 2 操作码允许将竞价内容隐藏在区块链活动中。

详细信息：https://a 16 zcrypto.com/hidden-in-plain-sight-a-sneaky-solidity-implementation-of-a-sealed-bid-auction/

Github 传送门：https://github.com/a 16 z/auction-zoo

三：可信化设置（Trusted Setups）

我们开源了一个代码库，允许任何人在以太坊链上实施 Kate-Zaverucha-Goldberg（KZG）和 powers-of-tau “仪式”（Odaily 星球日报注：即以可信任的方式对数据可用性证明方案所需要的参数进行初始化）。

详细信息：https://a 16 zcrypto.com/on-chain-trusted-setup-ceremony/

Github 传送门：https://github.com/a 16 z/evm-powers-of-tau
