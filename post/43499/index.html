文/Paul Veradittakit，Pantera Capital合伙人

在 DeFi 以及 NFT 快速蹿红的带动下，去中心化应用程序（dApp）生态系统在过去几年间增势迅猛，由于 dApp 直接与区块链上的智能合约交互，因此加密钱包通常用于验证这些 dApp 中的操作，钱包也正在成为链上的身份象征，用户不仅可以通过钱包登录 dApp，还可以通过 Web3 社交网络与 Instagram 和 Twitter 等 Web2 应用进行交互，亦或者用于验证资产所有权以及获得访问一些内容的访问权限等等。钱包正在成为 Web3 世界的「身份证」。

钱包在 Web3 生态中的重要性使得该赛道成为了最拥挤的赛道之一，其中最著名的竞争者包括 Metamask、Trust Wallet、Phantom 以及 Coinbase Wallet 等。由于每个区块链可能使用不同的椭圆曲线，因此不同区块链往往也需要不同的钱包去建立身份并进行交互，这进一步增大了钱包类应用的市场需求。目前主流的钱包应用分为几类，分别是浏览器插件、浏览器（像 Brave 以及 Opera 等，内置了钱包功能）、智能合约、移动端应用以及桌面钱包。

不过问题随之产生，大量的钱包导致 dApp 的开发者需要在支持不同种类的钱包登录上需要花费很多精力，开发者必须要集成尽可能多的主流钱包以吸引更多流量，并为用户提供便利。对于早期初创团队来说，这种时间和精力的消耗对于开发核心产品来说，影响是不容忽视的。

Dynamic 通过提供简单的身份验证 SDK 和工具套件解决了这个问题，允许应用程序和 dApp 开发人员更轻松地跨链集成多个钱包，并能够快速启用像 NFT 门控等基于钱包的身份功能。

Dynamic 是什么

Dynamic 是一个基于钱包登录的身份验证和授权工具套件。它允许开发人员创建一个「面向未来」的统一的多链钱包身份验证流程。

Dynamic 的工具套件中有三个主要组件：SDK、开发人员仪表盘和一系列 API（允许定制）。开发人员把 Dynamic 钱包 SDK 嵌入到他们的网站或应用中后，即可访问 Dynamic 仪表盘进行配置。

Dynamic 提供的仪表盘允许开发人员配置启用的区块链、在注册期间捕获用户信息、为内测和 NFT 分发创建白名单以及许多其他与身份相关的功能。在 Dynamic 上进行基于身份验证的编排无需接触代码。

以 Popartcats.xyz 为例，该应用就使用了身份验证和 NFT 门控功能为其客户创建访问受限的信息投放，而 Llama 则使用 Dynamic 为其网站创建了仅限会员登陆的身份系统。

Dynamic 开发人员仪表盘

SDK 和无代码仪表盘的组合使开发人员不仅可以拥有统一的多链钱包集成框架，还可以简化用户管理和注册流程，SDK 可以直接与 dApp 前端本身的代码交互。

得益于 SDK 仪表盘结构，新功能推出后开发人员也可以更快速地上手。Dynamic 最新推出的多钱包功能可以让终端用户将 dApp 中的多个钱包组合成该 dApp 中的单个用户身份。使用仪表盘，开发人员可以在他们的 Dynamic 实现中「打开」该功能，而无需重新部署集成。

团队

Dynamic 由 Itai Turbahn 和 Yoni Goldberg 创建，这两位以色列人 15 年前在麻省理工学院相识。Itai 此前曾在产品管理岗位上工作了 7 年，曾是波士顿咨询集团的顾问。他拥有哈佛商学院的 MBA 学位和麻省理工学院的电气工程与计算机科学和经济学学士学位。Yoni 是 AQR 和 Even Financial 等公司的工程副总裁，曾任 Gilt Groupe 的技术主管。他之前曾在谷歌参与研究工作，拥有麻省理工学院计算机科学学士学位和工程硕士学位。

小结

Dynamic 完成了由 a16z 领投的750 万美元融资，并已经开始与Llama、Popartcats、Handstamp、Newton、ProofOfMerge等 alpha 客户合作构建其首个版本。该版本目前处于封闭测试阶段，并计划很快推出公开测试版。 

Dynamic 解决的问题经常被市场忽视，但随着加密钱包基础设施的快速发展，以及网站和应用程序在如何与钱包交互和管理用户方面变得愈发复杂，这个市场痛点会被快速放大。随着越来越多的团队意识到了这一问题，Dynamic 做的一切正在变得至关重要。
