原文标题：《Web3 社交赛道的族群化与场景化》

撰文：echo_z

SocialFi 浪潮有些类似海市蜃楼，有着美好的图景却缺乏坚实的用户基础。从古早的 Steemit 自建社交公链、到 Rally 发布社交代币、再到去年大火的 Monaco 发售 NFT，Web3 社交产品的努力从未停止，也从未找到真正的 PMF。

不过，随着基础设施的升级和市场动向的变化，社交产品的形态也一直在转变。梳理近期社交赛道新产品，发现行业整体有两个变化方向：

一是族群化，即基于同一个底层社交图谱而搭建一系列生态产品，产品功能各不相同、但都共享同一套社交关系 ，用户可以在生态中的任一个产品上使用已经沉淀的社交数据，类似于在抖音上和微信好友聊天，充分体现了 Web3 数据的公开性。

二是场景化，即产品往往会和 NFT 等某个具体的使用场景结合 ，作为这一场景的功能补足工具而存在，体现了 Web3 应用的可组合性。

第一个变化将会造就和 Web2 全然不同的应用图景，第二个方向则展现出社交产品开始趋于实用功能性。整体而言，「乐高积木」现象越发明显，试图用一个产品造就大而全的社交应用的尝试越来越少。

本文聚焦于能够让用户产生社交互动的产品，因此暂不包括资质证明、DID 等基础设施类产品。

1. 族群化趋势：底层图谱与生态产品

构建通用社交图谱，供其他项目在此基础之上做建设——如果要做平台型社交产品，这似乎已经是一个共识。在这一领域，也已经有多个项目在建设生态，本节将分别介绍代表项目 Lens、RSS3、CyberConnect、Farcaster。

1.1 Lens Protocol

Lens 的设计核心是以 NFT 为基础构建社交图谱，每个用户会建立一个自己的 Profile NFT，在关注他人时也会获得一个 Follow NFT，每个 NFT 都有独一无二的编号，记录了建立 / 关注的顺序。

由于这些 NFT 都可以转移或交易，因此 Lens 突破了将社交关系仅仅作为「数据」的限制，而是将社交关系封装为「资产」 。这带来的想象空间在于，可以利用 NFT 作为各类社区门控、奖励标准等等，如限制只有前 100 名关注者可以获得激励，而一旦这些 NFT 能带来收益，也就可能造就社交 NFT 的交易市场 。

有趣的是，因为 Lens 团队对于 Profile NFT 采取了饥饿营销策略，只有部分早期用户可以获得 Profile NFT、而只有领取了这一 NFT 才可以获得他人关注，现在已经形成了 Profile NFT 的交易市场，地板价 0.014E。虽然价格不高，但是也验证了社交 NFT 具有交易价值的潜力。

OpenSea 上 Lens Profile NFT 的交易界面

Lens 的另一大优势在于，由于 Aave 原班团队本身拥有的号召力、且团队在通过 Hackathon 等活动积极建立生态，基于 Lens 已经出现了一批原生项目。在 Lens 官网上，可以看到一些官方推介的项目。

来源：https://lens.xyz/#apps

其中，Lensfrens 是 Lens 团队制作，是一个简单的用来加好友的前端；Lenster 类似 Facebook，可以加好友、浏览好友动态 、加入社区讨论等等，是目前相对比较完善的一个产品；Phaver 类似一个移动端 Facebook；Alps Finance 则是围绕 Defi 话题的发帖社区，也集成了交易和转账功能。除了上述产品外，还有热门链接分享社区、视频订购平台、视频分发平台等，都是基于 Lens 社交信息的原生项目。

这些生态项目大多本身的功能切入点往往不大，聚合在一起却有很强的社区感 。例如用户通过 Lensfrens 关注了其他人之后，可以在 Lenster 上看好友动态，又可以迁移到移动端的 Phaver。前端各不相同，但是好友关系却是一致的，这确实是社交生态项目相对于 Web2 社交应用很大的体验区别。

1.2 CyberConnect

CyberConnect 是一个去中心化社交图谱协议，在构建底层社交关系这一点上和 Lens 相似，但不同点在于 Lens 的架构是基于 NFT 实现关系构建，而 CyberConnect 则是以数据库的形式沉淀社交关系，社交数据通过中间件 Ceramic 进行存储。此外，CyberConnect 部署在 ETH 和 Solana 两条链上，是两张不同的社交图谱，而 Lens 只选择了 Polygon 单链部署。

CyberConnect 为社交关系制作了前端应用，用户可以关注他人，也可以查看他人的 NFT、POAP、Galaxy 资质、Mirror 文章等。

CyberConnect 加关注及用户动态界面

CyberConnect 也与不少第三方项目正在进行合作。但和 Lens 不同的是，Lens 的生态项目都是基于 Lens 协议原生的新项目，对于社交关系的采用是排他的；而 CyberConnect 大多是与其他项目合作，提供社交关系作为一块功能补足。

来源：https://cyberconnect.me/

上图为部分 CyberConnect 官网所列举的生态项目，其中 Light.so 和 Hello World 是基于 CyberConnect 的原生项目，对于社交关系以及用户动态做了不同样式的前端展示。

左为 Light.so，右为 Hello World

而除此之外的项目，基本都是独立的第三方产品，借用了 CyberConnect 的社交关系。较为典型的是 Unipass，同时采用了 CyberConnect 和 RSS3 的社交关系。在这一场景下，CyberConnect 和 RSS3 都类似一个开放 API，供第三方项目读取用户的社交关系。相对 Lens 来说，CyberConnect 可能更难建立自己的「族群」生态，但是对于第三方项目的集成来说也更灵活。

1.3 RSS3

RSS3 致力于做一个开放的信息聚合器，实现高效且去中心化的 Web3 信息分发。目前的信息来源是基于用户地址在链上的所有动态行为，因此很像一个链上朋友圈。

RSS3 不像 Lens 和 CyberConnect 那样聚焦于底层图谱，而是自己封装了 C 端可使用的产品。目前可用的产品，一是 Web3 Pass（https://cheers.bio/），用户可以在这里编辑自己的个人资料页面，选择展示哪些动态信息，也可以生成自己的域名 RNS；二是 Revery，也就是基于用户地址动态的聚合页面，关注其他用户后就可以看到关注对象的动态，非常类似 Twitter 动态，只不过动态包含了关注对象允许公开的所有链上行为，包括在 Mirror 发文、购买 NFT、获得空投等等。

来源：https://revery.so/

不过，由于 RSS3 抓取的用户链上动态用途广泛，RSS3 也成为了一些第三方应用的数据提供方 。在 RSS3 的生态项目介绍中，可以看到双方的合作方式大多是由 RSS3 提供用户资料等数据。

来源：https://docs.rss3.io/docs/guide/ecosystem

1.4 Farcaster

Farcaster 团队低调，Twitter 仅有 2 条推文，产品一直在内部测试中。不过近日突然宣布融资 3,000 万美元，开始高调宣传。团队的两名联创 Dan Romero 和 Varun Srinivasan 均出身自 Coinbase，应该在业界有不小的影响力。

Farcater 是一个底层的社交名册 ，任何客户端都可以利用这个名册建立社交产品。协议核心分为两部分：一是在以太坊上建立的名字注册系统，目前运行在 Rinkeby 上，用户用一个以太坊地址就可以注册一个独一无二的用户名，并关联一个内容托管网址。只有拥有地址私钥，才可以更新内容托管地址，换言之相关内容的关联权掌握在用户手中。二是托管部分，用户可以选择自主托管或依靠第三方托管，实现不同程度的去中心化。

来源：https://www.farcaster.xyz/docs/the-basics

联创之一的 Dan Romero 在宣布融资的文章中附了一张产品前端，看起来也类似一个 Twitter，只是界面更简单直白。

来源：https://danromero.org/farcaster/

围绕 Farcaster，也有一些原生项目，包括存储所有产品上出现的图片的 Instacaster、搜索帖子的 Searchcaster、作为专用 RSS feed 流的 CastRSS、帮助用户在和其他网站连接钱包时预先审批敏感数据的工具 Configcaster。

由于产品没有开放使用，相关信息并不很多，但是 3,000 万美元的融资储备仍然使得这一项目值得追踪。

2. 场景化趋势：交易沟通、项目合作、数据分析

如果说上节的项目还是在试图平地而起地建设社交网络，那么本节的项目则更多地展现了社交产品的工具性。在交易沟通、项目合作和数据分析方面，分别出现了具有实用功能的新产品。

2.1 促进交易的通讯产品

Web2 的通讯产品网络效应太强大，如果直接做一款去中心化、甚至端对端加密的产品，很难能拼过 Telegram 等早已养成用户习惯的社交通讯产品 。而近期出现的 Web3 通讯产品，开始和已有的 NFT 等场景紧密结合，满足具体沟通场景的功能跨越了网络效应的关卡障碍，实现了 Web2 难以实现的体验。

Atem Network 和 Swapchat 都是基于 NFT 交易所做的通讯产品。

Atem 更注重基于身份的即时群聊 ，在其界面下可以找到热门 NFT，并进入这一 NFT 之下的群聊。可圈可点的是，Atem 会对用户钱包是否持有该 NFT 做即时验证，如果持有就可以进入持有者才可进的频道，反之就只能进入大众频道。

Atem 的 NFT 界面及群聊界面

Swapchat 虽然也有某个 NFT 的群聊，但是只有一个大众群，产品更注重点对点的沟通 。Swapchat 本身是一个扩展程序，集成在 OpenSea 上，当用户在 OpenSea 看到某个心仪的 NFT 时，可以发起和持有者的聊天。当然，持有者必须同样安装 Swapchat 扩展程序，才可以收到信息，对于网络效应仍然有较强的要求。

Swapchat 在 OpenSea 的集成

2.2 基于资质的合作产品

在最新一期 Gitcoin 的社交项目中，出现了一批职场社交网络产品，包括 Metajam、Fuse Pass、CryptoIn。Web3 新产品信息分散，通过这样的网站能够聚合项目信息、连接人与项目，职场社交赛道一定会有市场需求 ；而 Web3 又能够充分利用链上资质，使得项目方和应聘者的信息更对称，相对于 Web2 职场产品而言存在优势 。

Fuse Pass 利用 NFT 徽章、链上和链下资质来提供职场专业证明，帮助提升项目招聘过程中的透明度；CryptoIn 借助 RSS3、Arweave、Mask Network 等多个产品功能，辅助项目融资以及人才招聘。这两个产品似乎目前都还没有公开可用的版本，而 Metajam 则已经有了一定运营声量。

Metajam 类似 Web3 的 Product Hunt，项目方可以在这里展示自己的产品模式，以招徕人才和潜在融资；个体用户可以建立自己的简历证明，发现项目并应聘工作；投资人也可在这里筛选项目或协助宣传。

来源：https://www.metajam.studio/

在使用时，用户需要用钱包地址连接，可以在每个项目下留言，同时 Metajam 会同步 CyberConnect 的社交关系。个人简历部分还较为简陋，用户可以选择自己感兴趣的领域和技能标签，但是缺乏对于用户链上资质的集成。

2.3 嵌入式数据分析

链上数据分析从来不是新鲜事，但是 Nansen 等数据分析平台对于用户有一定的分析能力要求。轻量级的分析辅助工具或许能更受大众欢迎。

Blocktracker 就是一个简单直白的分析插件，对于关联了链上地址的 Twitter 账号，能够在 Twitter 界面展示这一地址的 NFT 历史收益率、钱包余额等信息，为大 V 的发言提供了更多有趣的经济证明。

总结

从以上两个趋势来看，新兴的 Web3 产品或是和其他应用相互报团形成网络效应，或是贴近某个使用场景形成功能性补足，不再像过去那样试图以一个产品完全替代 Web2 应用 。

Web2 的社交产品中，底层的社交关系和上层体验是绑定在一起的，而在 Web3 中这两个元素被分离了，这是最有趣的地方，也必然造就完全不同的产品格局。社交产品最大的壁垒在于用户迁移成本，而在 Web3，只要在同一个社交图谱协议上，用户就毫无迁移成本。也许未来会围绕几个社交图谱，形成社交产品族群，而生态内的应用会像游戏一样成为创意型产品，老产品并不能高枕无忧、小产品也总是有发展机会。
