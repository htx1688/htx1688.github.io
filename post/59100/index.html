Polygon 的目标和 Web3 的重要性

Nailwal 指出， Polygon 一直致力于为 Web3 的广泛应用打造一个稳固的基础设施。他认为，目前的区块链应用还没有实现真正的扩展，没有一个应用或链能够处理数百万的日活跃用户而不崩溃。

Nailwal 认为 ，之前人们的数字生活由中间机构控制，而这些机构在过去十几年里失去了大家的信任，导致形成了一个低信任的环境，而 Web3 旨在提供一个无需信任的环境，人们的数字生活基于去中心化的信任机制，不再受中间机构的控制。为了实现这一目标，需要一个可以容纳上亿日活的基础设施。

Nailwal 指出，Polygon 2.0 的目标是创建一个无限可扩展的区块链基础设施，开发者能够创建和扩展无需信任的应用。Web2 被描述为「信息的互联网」，主要关注数据和信息的共享，而 Web3 则被视为「价值的互联网」，重点是在去中心化的环境中创建和交换价值。

尽管 Polygon 2.0 允许增加无限的链和扩展性，但其流动性仍然是通过以太坊的结算层进行的，可以有多个链并行运行，但所有的交易和流动性都会在以太坊上进行结算。

Nailwal 回顾了 Polygon 技术的发展历程。Polygon 最初从 Plasma 早期的扩展解决方案开始，后来探索另一种扩展方法 State channels，接着转向更先进的扩展解决方案 State channels，最后决定采用 ZK 技术。

Nailwal 解释说，ZK 技术允许你通过提供一个恒定大小的证明来证明你所做的计算，而不需要提供所有的交易数据。这种证明每次验证所需的计算量都是相同的，使得 ZK 技术非常高效和可扩展。

Nailwal 和许多行业内的研究人员都认为 ZK 是为区块链和去中心化系统提供扩展性和安全性的最终解决方案，而 optimistic rollups 只是一个短期解决方案。

Nailwal 解释说，optimistic rollups 的工作方式是基于乐观的假设，链上的所有事务都是正确的，除非有人提出异议，这种方法在技术上相对简单，但也带来了挑战。比如，当用户想从 rollup 链撤回资金到主链时，需要等待七天的长时间回撤期，这段时间允许任何人检查链上的交易，并在发现任何不正当行为时提出异议。

optimistic rollups 的核心是在 off-chain（链下）环境中进行计算，与主链相比，可以处理更多的交易和操作来提高扩展性。尽管计算是在链下完成的，但所有的交易数据和状态转换的证明都必须提交到主链，这确保了链下计算的正确性和透明性。

Polygon 的多链未来愿景

Nailwal 描述了一个未来，其中存在数万个链运行在同一个系统中，这些链可以采用不同的技术和结构，包括 Layer 1、validiums 和 rollups。这种多链的结构将为开发者和用户提供更大的灵活性和选择，使他们能够根据自己的需求选择最合适的链。

Nailwal 解释了 validiums 和 rollups 之间的主要区别。Rollups 是那些将其数据放回主链（如以太坊）的链，而 validiums 则将其数据保留在链下。这两种方法都有其优点和局限性，但关键是它们都可以在同一个生态系统中互操作。

Nailwal 强调，尽管我们经常讨论区块链的去中心化特性，但真正的目标是实现 trustless 计算。用户和开发者可以信任系统的计算结果，而不需要信任任何中间实体或第三方。在这种环境中，去中心化只是实现 trustless 计算的一种手段，而不是最终目标。

Nailwal 提到，不同的链（如 Bitcoin 和 Ethereum）为不同的 trustless 计算提供了解决方案。例如，Bitcoin 提供了 trustless 的支付解决方案，而 Ethereum 则允许用户执行任何类型的通用程序。

Nailwal 认为，任何提供 trustless 计算的平台都是彼此的竞争者。这包括 ZK、optimistic rollups 以及可能出现的其他技术。他强调，目标是为那些希望在 DeFi、游戏或其他领域构建的人提供 trustless 计算。

对于初创公司，Nailwal 建议他们根据自己的需求选择最合适的链。如果他们正在构建 DeFi，他们应该选择具有更多流动性的链；如果他们正在构建游戏，他们应该选择有更多游戏社区的链。

Nailwal 谈到，为了支持多链环境，Polygon 2.0 的架构将支持 validiums、rollups 以及其他可能的链，如 Cosmos，这些链都可以在一个统一的框架下互操作。

Polygon 将从固定的 100 亿供应量转向通货膨胀模型，每年增加 1% ，这是为了激励验证者参与并继续为社区提供资金。此外，为了进一步发展生态系统，还为未来 5 到 10 年预留了 1% 的 treasury，这部分资金可以用于生态系统的增长。

Polygon 的三大治理支柱

Nailwal 详细探讨了 Polygon 的治理结构：

协议治理，涉及到核心协议和客户端开发的决策。与 Ethereum 和 Bitcoin 不同，Polygon 的核心协议和客户端开发不是由社区治理的。对于核心技术决策，Polygon 不完全依赖于 token 持有者或社区的投票。

Nailwal 认为 Ethereum 和 Bitcoin 的治理模型非常成功，特别是在处理技术细节和核心开发决策时。这种模型允许技术团队在没有过多外部干扰的情况下进行决策，确保了协议的稳定性和安全性。

系统智能合约治理，涉及到在 Polygon 网络上运行的智能合约的治理。这种治理模型允许社区对智能合约进行更广泛的审查和决策，确保合约的透明性和公正性。智能合约是区块链网络的核心组成部分，它们的行为和功能必须得到社区的信任和支持。

社区财政治理，涉及到 Polygon 社区资金的管理和分配。社区成员可以对如何使用和分配社区资金进行投票，确保资金的透明和公正使用。为了确保 Polygon 生态系统的持续发展和扩展，社区资金的正确和公正管理至关重要。

Nailwal 指出，尽管 Polygon 近年来在这在 NFT 和 DeFi 领域都取得了显著的增长，但它仍然面临着来自其他区块链项目的激烈竞争。

Nailwal 提到，Polygon 面临的一个主要挑战是在社区中形成的叙事。他认为，尽管 Polygon 在技术和实际应用上都取得了成功，但在叙事形成方面，它仍然面临着来自其他更有影响力的区块链项目的竞争。

Polygon 的业务模型

Nailwal 强调 Polygon 是一个协议，而不是一个公司。Polygon Labs 作为非营利组织，不产生利润。

Nailwal 介绍了 Polygon 协议的核心功能，即为第三方开发者提供无需信任的计算服务，开发者可以在 Polygon 网络上执行操作，而无需担心中间方的干预或不正当行为。

当开发者在 Polygon 网络上执行操作时，他们需要支付交易费用。这些费用是为了补偿那些维护和保护网络的验证者。验证者是网络的关键参与者，他们通过验证和确认交易来确保网络的安全和完整性。

Nailwal 解释，验证者为了参与网络并获得交易费用，需要抵押或「质押」代币，这些抵押的代币数量决定了验证者从网络中获得的交易费用的比例。

代币不仅仅是验证者获得交易费用的一种方式。它们还为验证者提供了激励，鼓励他们正直、诚实地行事，因为如果他们表现不佳或作恶，他们抵押的代币可能会受到惩罚或被没收。

Nailwal 表达了他对传统业务模型的欣赏，特别是那些基于实际资本和收入的模型。他认为，与加密领域的投机性质相比，这些模型更为稳定和可靠。

Nailwal 提到，尽管他们努力在加密领域建立一个去中心化的生态系统，但大部分社区成员仍然对投机感兴趣，而不是真正的价值创造。他认为，如果五年后加密领域仍然主要关注基础设施而不是应用，那么这将是一个失败。
