前言

Coinbase 正大刀阔斧地建设链上基础设施，近日宣布基于OP Stack 构建一条二层Rollup 链——Base, 一时间Crypto 市场上掀起了“Bullish on Rollup” 的呼声。

Coinbase 作为全美第一大加密交易所，此番举措或将成为Rollup 赛道发展、革新的催化剂。

背景

以太坊网络性能的局限性催生出了多样化的Layer 2扩容解决方案，Rollup 方案作为其中的翘楚领跑整个扩容赛道。以太坊通常被看成一个单体区块链网络，单体中主要由3大功能层级构成：数据可用性和共识层、结算层和执行层。

由于区块空间是有限的，所有功能层级在单体中运行消耗了大量的链上资源，导致网络拥堵、链上gas 费用飙升。Rollup 本质上是将执行层从单体区块链中分离出来的模块化扩容解决方案，将复杂运算转移到链下，再把链下执行结果传输到链上验证，在继承以太坊安全性的情况下大幅提高网络的性能。

据L2BEAT 数据显示，截止至2023.03.02，Arbitrum TVL 为34.3亿美元，份额在整个Layer 2赛道中占比高达54.25%；Optimism TVL 紧随其后，两者TVL 总和已超过80% 的市场份额。Layer 2赛道竞争格局看似已由Arbitrum 和Optimism 捷足先登，实则风起云涌，越来越多的Rollup 方案如雨后春笋般涌出，带着它们的创新叙事，企图蚕食一定的市场份额。

而其中最引入注目的，是由此衍生出的一个被称为“Rollup as a Service”的全新赛道。

顾名思义，RaaS 代表把Rollup 的部署作为一种服务，为开发者从0到1构建Rollup 提供了极大的便利。

RaaS 理念的雏形最早可以追溯到2021年Celestia 的Optimint(现更名为Rollkit), 旨在为开发者提供一整套通用软件开发套件(SDK). 由 Optimism 团队提出的OP Stack 堆栈是这一赛道的代表性项目，近期Coinbase 基于OP Stack 构建其Optimistic Rollup(ORU) Base, 这是一个起点同样也是一个契机，点燃了我对RaaS 赛道的热情和憧憬。

在讲RaaS 赛道的具体项目之前，先来捋一捋这个赛道之所以出现、兴起或者说即将兴起的原因和驱动力。

我始终认为最核心的驱动力是需求： 以太坊性能的局限性催生了扩容的需求，所以有了Rollup; 从0到1部署一条链并非易事，如果有通用的工具会事半功倍，因此出现了RaaS. 当然，上述观点成立的大前提是Rollup 本身具有需求，目前来看，通用型Rollup 的需求已经得到市场的证明；特定应用型Rollup 的需求还有待于市场验证。

一、RaaS

除了上述所说的Optimism 的OP Stack, Celestia 的Rollkit, RaaS 赛道包含的项目还有很多。从是否需要编写代码角度，RaaS 项目可分为2类：SDK 和无代码(No-Code)方案。SDK 方案即为开发者提供一整套通用的软件开发套件，让部署Rollup 和部署智能合约一样简单；无代码方案，顾名思义，即无需编写一行代码部署Rollup(一键发链).

其中OP Stack 和Rollkit 均属于SDK 方案，Dymension, Sovereign, Stackr 同属这一范畴。无代码部署Rollup 的项目包括AltLayer, Saga, Eclipse 和Caldera. 这里不会花太长的篇幅对RaaS 赛道的每个项目进行详解，仅做简答介绍。

文章更多侧重于我自己对RaaS 赛道的一些思考，希望能起到一点抛砖引玉之效。

SDK

1. OP Stack

OP Stack 是Optimism 团队为了实现模块化区块链架构的软件堆栈，将区块链网络分为共识层、结算层、执行层3大功能层级。通过将不同功能层级解耦，再以API 软件堆栈的形式把所有层级组合起来，极大地简化了模块化区块链构建的流程。

OP Stack 给Rollup 开发者带来了更大的自主性，能够切换、替换、修改不同模块层级，就像切换API 一样。Optimism 团队提出了他们认为的区块链终局形态“超级链”，即由一组高度集成的OP Stack 链组成，链之间通过共享消息传递格式进行通信，实现Rollup 链间互操作性。

2. Rollkit

Rollkit 是由Celestia 社区开发的模块化发链SDK, Rollkit 相对于Rollup 就相当于Tendermint 相对于Cosmos Layer 1. 开发者可以基于Celestia 提供的数据可用性(DA) 和共识层构建Rollup, 进一步提高可扩展性。另外，Rollkit 也带来了可定制化的优势，DA 层、排序器、执行环境、证明机制等都可以定制。

3. Dymension

Dymension 的RollApp 开发套件(RDK) 的功能与Cosmos SDK 类似，帮助开发者在Dymension 结算层上部署RollApp(基于特定应用的Rollup). 结算层扮演的角色是维护所有RollApp 的安全性，也被称为Dymension Hub, 换句话说，所有基于RDK 构建的RollApp 共享Dymension Hub 的安全性。值得注意的是，Dymension 在结算层中引入了RollApp 虚拟机(RVM), RVM 支持全新的争议解决机制，因此可以支持各种不同的执行环境。

4. Stackr

Stackr 是以太坊基金会主办的Layer 2 Community Grants 2022 22个获奖项目之一，它在130+ 项目中脱颖而出。旨在通过Stackr SDK, 让开发者更便捷地部署可定制化的App-specific ORU.

5. Sovereign

Sovereign 相对于以上4个Rollup SDK 项目而言稍显“另类”，上述SDK 方案都是以构建基于欺诈证明的ORU 为切入点，而Sovereign 一开始便专注于基于零知识证明的zk-rollup(zkRU) 的构建。零知识证明是密码学中非常难的一个分支，zk-rollup 的开发要求开发者拥有很扎实的密码学功底。Sovereign 提供的SDK 能够让没有密码学专业背景的开发者也能使用零知识证明技术开发公链，让开发zk-rollup 跟开发Dapp 一样简单。

No-Code

1. AltLayer

AltLayer 是具有代表性的无代码Rollup 解决方案之一，旨在解决单一热门Dapp 造成整个网络拥堵的问题。由于部署在同一区块链网络的不同Dapp 共享有限的区块空间，当特定Dapp 链上交易激增(如限定时间的NFT 铸造活动等)，会造成整个网络拥堵gas 费用飙升。AltLayer 通过提供部署Flash Layer 提供一次性执行层，为NFT 铸造事件提供专用的区块空间，与链上的其他项目完全隔离，在铸造活动结束后关闭执行层，进而提高网络的可扩展性。

2. Saga

Saga 是部署在Cosmos 生态上的项目，愿景是通过横向扩容打造千链生态。Saga 的架构跟上述SDK 方案Dymension 类似，Rollup在Saga 架构中被称为Chainlet. 由于基于Saga 主网构建，所有的Chainlet 共享Saga 主网提供的安全性。通过共享安全，Chainlet 无需原生代币即可运行。由于链游底层基础设施的完善和扩容成为当下亟需解决的问题，Saga 最初将专注于游戏、娱乐链等Web 3基础设施。

3. Eclipse

Eclipse 旨在为任意Layer 1提供通用的结算层，Rollup 再基于Eclipse 的结算层构建。Eclipse 为开发者提供了一个高度可组合的Rollup 框架，可以自主选择虚拟机、数据可用性层等。目前，Eclipse 支持的Layer 1包括Celestia, Polygon 和EigenLayer, 后续将支持更多的Layer 1网络。

4. Caldera

Caldera 支持“一键发ORU 链”，无需编写一行代码，将开发一条Rollup 的时间缩短至分钟级别。Cladera 链模块化堆栈由3部分组成：Caldera Chain(执行层)/ EVM 结算层/ 数据可用性层(DA). 它能基于任何EVM 兼容链作为结算层，Caldera 的执行层是基于欺诈证明的ORU, 即通过运行欺诈证明机制维护Rollup 网络的安全性。

二、Rollup 模块化架构

从Rollup 的模块化架构进行划分，大体可以分为3类：

1）通用智能合约Rollup;

即Layer 1(如以太坊/ Polygon 等)负责结算 + 共识 + 数据可用性，执行层(Rollup)从单片链中分离出来，代表性例子如Optimism/ Arbitrum.

2）结算Rollup;

即Layer 1负责共识 + 数据可用性，结算层作为独立的层级分离出来，Rollup 基于结算层构建，由此构建的Rollup 共享结算层。结算Rollup 根据功能不同又可以划分为特定应用结算Rollup 和通用型结算Rollup. 如基于Dymesion Hub 构建的RollApp, 基于Saga Chain 构建的Chainlet.

3）主权Rollup;

即Layer 1负责共识 + 数据可用性，Rollup 负责结算 + 执行，由于Rollup 可以决定其自身交易的有效性，所以我们认为它是有“主权”的。与结算Rollup 类似，根据功能不同主权Rollup 又可划分为特定应用主权Rollup 和通用型主权Rollup. 如基于Celestia Rollkit 构建的Rollup.

更具体地，Rollup 可以分为5类：通用Rollup, 特定应用 + 通用型结算Rollup, 特定应用 + 通用型结算主权Rollup. 如果再跟证明机制(零知识证明/ 欺诈证明)组合，可以衍生出5 × 2 = 10种Rollup 组合。再具体一点，开发者可以选择Celestia/ Eigen DA 做数据可用性和共识层，Dymension/ Saga/ Eclipse 做结算层，构建Rollup. 随着模块化项目的增多，由此形成的模块化Rollup 组合甚至可以达到成百上千种，而所有的Rollup 组合，都能通过RaaS 相关项目提供的基础设施轻松便捷地部署，RaaS 赛道所散发出的潜力是不容忽略的。

三、看好RaaS 赛道的一些想法

之所以看好RaaS 赛道，主要基于以下的一些投资逻辑：(仅做观点交流，没有任何投资建议。DYOR. NFA)

1) RaaS 相关项目备受资金青睐；

如Coinbase 基于OP Stack 做Rollup; RaaS 赛道相关项目获得一众明星VC 投资，如Caldera 获得红杉、蜻蜓资本领投的900万美元融资；Sovereign 获得Robot Ventures, Maven 11 等VC 740万美元融资；Eclipse 完成两轮共计1500万美元的融资，投资机构包括PolyChain, Tribe Capital 等……

2) RaaS 带来一种范式的模块化区块链底层创新；

作为核心底层基础设施，RaaS 的完善将对区块链网络架构的革新带来深远的影响。RaaS 的普及极大地降低了开发者从0到1构建一条Rollup 的门槛，准入门槛的降低将带来激励的竞争，能够很好地诠释优胜劣汰的“丛林法则”，进而倒逼Rollup 的创新。

3) Rollup 赛道尚未完全爆发，RaaS 赛道更是处于早期。

虽然目前来看，以Arbitrum 和Optimism 为代表的ORU 占据了该赛道超过80%的市场份额，但zk 系的相关项目还处在蓄势待发的状态。zkSync 和StarkNet 虽然都已上线主网，但上线时间不长且暂未迎来链上生态的大爆发。因此，现在说Rollup 赛道大局已定还为时尚早。

当然，机遇和挑战是并存的。无论是Rollup 还是RaaS, 都存在很多悬而未决的问题。如排序器的中心化(虽然目前如Rollkit/ Optimism/ Dymension 等都有提出去中心化排序器的解决方案，但目前都没有落地)、流动性碎片化、ORU 的挑战期过长等问题都有待解决。虽然存在一些问题，但都有与之对应的解决方案，只需重点关注是否能落地以及落地的效果即可。

结束语

早在公链大规模爆发之前，人们热议的焦点是未来的公链格局到底是单链还是多链。至少从目前来看，“Bullish on Multichain” 的人更胜一筹，我们除了以太坊之外，还看到了Polygon/ BNB/ Cosmos 等L1 都在冉冉升起。

而对于Rollup, 我们面临同样的一个问题是：不远的未来，Rollup 的格局到底是单个Rollup 一家独大还是Multi-Rollup 竞相繁荣？我个人更期待看到的是Multi-Rollup 的格局，借助RaaS, Rollup 在竞争中发展，不断推陈出新。但最后谁会在激烈的竞争中存活，谁又会永远地退出Rollup 的历史舞台？没有人知道，只有时间能告诉我们答案。

毕竟，只有当大潮退去，才知道谁在裸泳。
