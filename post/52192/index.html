原文作者：Cookie，律动 BlockBeats

发行成本高一直是影响比特币 NFT 市场的原因之一。尽管最近在市场上出现了比较多的 BRC-XXX 协议，但在保持比特币 NFT 的「原汁原味」与降低发行成本这二者间寻求到较好平衡的，笔者认为当属 BRC-1155 与 GBRC-721 。

BRC-1155 ：为实现比特币「Open Edition」带来可能

几个月前，以太坊上 ERC-1155 的「Open Edition」艺术类 NFT 就曾在一段时间内成为市场的「小主线」，其基建的完善以及「艺术游戏化」玩法的不断进步，最终使得「Open Edition」成功引起市场瞩目。

在比特币上实现「Open Edition」存在痛点，尤其 NFT 内容类型为大小较大的动图、音频乃至视频文件时，高昂的刻录成本只能由创作者或收藏者其中的一方来背负，这对「Open Edition」在比特币上的发展几乎是绝望的——创作者由于需要预先承担风险（付出高昂的刻录成本而无法卖出）而抗拒在比特币上发行「Open Edition」艺术作品，收藏者也同样因为高昂的刻录成本而抗拒参与。

想要让「Open Edition」在比特币上的发展成为可能，必须同时打低双方的成本。BRC-1155 的核心思路是，如果我们可以接受像 BRC-20 那样的 JSON 代码铭文对同一幅艺术品的所有权进行确认，来作为拥有一幅存储在比特币上的艺术品的方式，那么我们就没有必要花费数倍乃至更多的 Gas 来重复刻录一个图像/音频/视频/其它文件，只需要花费极低的 Gas 来获取一张「凭证」，然后让索引根据「凭证」铭文所指向的链上铭文来渲染出对应的内容即可。

BRC-20 + ERC-1155 = BRC-1155 

创作者与收藏者的成本都被大大降低。对于创作者，只需要刻录一份图像/音频/视频/其它文件铭文，作为被指向的「原版」，同时再刻录一个「部署」铭文即可。

「部署」铭文示例

上述「部署」铭文示例中的各关键词含义如下：

「p」：协议类型。必须包含的关键词，该关键词定义操作基于 BRC-1155 协议进行，帮助索引识别与处理 BRC-1155 事件。

「op」：事件类型。必须包含的关键词，该关键词定义事件类型，是 Deploy 部署、Mint 铸造、 Burn 燃烧或 Fractionalize 碎片化。

「abbrev」：BRC-1155 系列名称缩写。必须包含的关键词，该关键词定义 BRC-1155 系列名称缩写，不能超过 10 个字符。

「inscription」：索引指向。必须包含的关键词，该关键词定义 BRC-1155 系列将取得链上的哪个铭文（根据铭文序号）进行内容渲染，比如要渲染 Inscription #123 的内容，那么就填写「 123 」。

此外，创作者还可以在「部署」铭文中加入一些其它的关键词来实现确定总供应量、每张 Mint 铭文所能代表的「凭证」数量以及增加额外的描述信息等功能。（详情可参阅 BRC-1155 手册）

不仅打低了「Open Edition」的成本，BRC-1155 还将提供「Burn 燃烧」功能来实现「艺术游戏化」玩法。目前，BRC-1155 的开发者 @ivantkf 已经与比特币 NFT 项目 Bitcoin Pizzas 以及 PixelBirds Sparrow 合作成立了 BRC-1155 Foundation，索引以及更多的功能完善正在推进中。Bitcoin Pizzas 与 PixelBirds Sparrow 也已经使用 BRC-1155 完成了「Open Edition」项目发行的尝试。

GBRC-721 ：各个特征转 Base 64 上链存储，前端拼装

有些 BRC-XXX 协议通过 IPFS 来实现指向并完成文件渲染的方法被认为破坏了比特币 NFT 的「原汁原味」。GBRC-721 用了一个巧妙的方案来保持比特币 NFT 的全链上存储特性——将图片文件转换为 Base 64 字符串并写入「部署」铭文中刻录到链上，前端读取出「铸造」铭文所指向的「部署」铭文中存储的各个特征的 Base 64 字符串并重新转化为图片，组合成一个完整的图像。

「部署」铭文示例。可以看到，各个特征图片被转换成 Base 64 字符串写在了「部署」铭文中，比如蓝色的背景与机器人的天线。刻录「部署」铭文，即相当于将各个特征的图片存储到了比特币链上

「铸造」铭文示例。可以看到，前端将读取出所指向的「部署」铭文（t_ins 字段，代表着所指向「部署」铭文的 Inscription ID）中包含的各个特征，并根据特征数组（a 字段，代表着前端将根据对应的这些特征拼装出完整的图片）完成图片拼装

GBRC-721 协议由比特币 NFT 项目 DogePunks 的创始人 @0x Jerry 543 提出，他还为该协议创建了一个名为 OrdiBots 的新系列用于演示。

结语

BRC-1155 与 GBRC-721 都向我们展示了一个新的思路——BRC-20 的 JSON 代码类铭文不仅可以用于资产发行，还可以让比特币 NFT 的玩法变得更多样。同时，由于 BRC-1155 与 GBRC-721 所 Mint 出来的都是 JSON 代码文本，所以如果您参与了相关项目的铸造，到手的不会是图片，而是像 BRC-20 Token 那样的文本（Text）类型铭文。

想要获得更完善的使用体验，需要等待这些协议的索引推出，也需要等待钱包/交易市场认可这些协议，并为它们进行相应的前端适配。同时，我们也可以说，降低比特币 NFT 的成本，在现阶段的尝试来看是一个「让前端做得更多还是更少」的问题。
