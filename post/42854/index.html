9 月 21 日，在由万向区块链实验室主办的第八届区块链全球峰会——黄“金”时代主题论坛上，在 HashKey Capital 首席执行官邓超的主持下，协议实验室创始人 Juan Benet、Dfinity 创始人兼首席科学家 Dominic Williams、NEAR Protocol 联合创始人 Illia Polosukhin 通过视频连线的方式围绕“区块链技术发展”展开精彩分享，深层次介绍了区块链技术的未来发展动向。

以下是嘉宾分享内容（有删减）：

主持人：请大家对过去的一年做总体回顾？

Dominic Williams：

最令人激动的进展就是基于 IC 开发的各类项目，我们看到了一整套全新的 Web3 服务是完全基于 IC 区块链运行，而不是被部署在某个云计算的服务器之上。

而且，我也看到人们非常喜欢这样的 Web3 应用，这代表着区块链行业的转折点。IC 区块链的目标之一就是允许开发者构建一切系统、一切生态、一切服务、一切应用，完全基于区块链，基于区块链运行、基于区块链部署。基于区块链部署的时候，碳排放方面也低于基于云的部署，所以，IC 区块链与现有的区块链相比存在数量级的差距。除此之外，链钥加密学也是多年以来进行研发的。如果没有链钥加密学，很难实现我刚刚所列出的种种目标。

而现在又得到了进一步的拓展，现在有很多人基于 IC 搭建生态，将 IC 作为 Web3 的编排层。具体来说，你可以在 IC 上进行代码运行，而这样的代码直接就可以调用比特币的账本，而不需要有任何的跨链桥。

除此之外，我也看到有一些项目基于 IC 区块链打造新一代 DeFi、Social Fi、Game Fi、元宇宙的项目。

Illia Polosukhin：

去年对于加密世界来说是不同寻常的一年，对于 NEAR 来说，刚刚在里斯本召开了 NEAR 的黑客松，一共有 3000 人参与，这是非常盛大的活动。有很多有趣的项目，很多有趣的事情、应用层出不穷。所以我想给各位分享一下过去一年间重点的进展。

第一，EVM 智能合约。你可以直接将其他区块链从以太坊区块链合约部署在 NEAR 之上。相比于 Polygon 来说，费用是低 10 倍。这样的协议已经运行了 8-10 个月的时间，除此之外也有 Aurora + 允许用户免费交易，这也是基于 NEAR 搭建的。

我们也非常关注去中心化的生态。比如在不同区域有区域中心，在中国、葡萄牙、乌克兰、巴尔干、肯尼亚、印度、越南部署了区域中心，不同地区的区域中心将会汇集更多的开发者以及用户。

大家会看到在一些已经比较成熟的市场，有各类新技术，这些新技术代表着真正可以在现实世界中应用，而不仅仅只是用作投机。刚刚在葡萄牙的黑客松上，Sweatcoin 正式上线，并且推出了自己的 Token，有 20 万用户已经在他们的链上非常活跃，创建了 1000 万账户。我非常高兴看到这样的应用获得了大规模的采用，这也是加密生态希望实现的目标。

Sweatcoin 之所以可以实现这一点，主要是因为用户体验非常简单，用户并不需要理解区块链的底层逻辑，也不需要理解其他技术的复杂度，就可以使用这个应用。所以我觉得这是很好的例子，展示出我们全球战略打造去中心化的网络，但是并不是围绕节点、技术，相反是围绕人在全球范围内进行协作，一道助力贡献生态。

Juan Benet：

我们专注两件事情/两个领域：存储和使用。

在去年年初的时候是 20 TB 的数据，但现在已经达到了 180 TB，在 9 个月的时间里，实现了几倍的增长。我们对这样的数据使用情况感到非常兴奋，有很多应用利用我们的数据，人们基于 IPFS 搭建各种各样新的应用、新的组织，一共有超过 5000 个初创项目基于 IPFS 搭建，其中有一些项目已经获得了 A 轮融资，甚至是更多的融资，这是一件非常好的事情。

除此之外，我们在不断寻找跨越和 Web2 的鸿沟 ，比如和像伯克利这样的大学合作，这些大学会利用 IPFS 存储他们珍贵的数据和数据集，像纽约这样的城市也会用 IPFS 存储他们的公共数据。

第二个领域是将计算与数据相结合，使得开发者能够开发更好的应用。

我们第一个里程碑，也就是基于 WASM 的 HelpWeather FVM 虚拟机，允许其他的智能合约调用。而这将会在第一季度的时候，启动用户相关的智能合约，这是我们的第一个目标。

除此之外，IPFS 也会有大规模的数据载入计划，以及计算的项目。这一切主要是在测试网阶段，也希望能够更好地利用 GPU。

除此之外，我们也有另外一系列令人兴奋的进展，比如人们做大规模的任务的调度，基于 IPFS 实现。

总的来说，我们首先接入大量的数据，不断扩展数据，将数据和计算相结合。

主持人：你们聚焦的生态和应用是哪些领域，以及为什么选择这些领域？

Juan Benet：

现在人们在 Filecoin 生态中搭建各种各样的应用，比如说一些日常的 App 用来存储日常数据或者是企业的数据等等。同时，我们也看到很多的 Social Fi 方面的实验，比如视频分发，比如像 TikTok 这样的应用，我觉得这是令人兴奋的领域。

除此之外，在跨链方面我们必须要有很多的问题先解决掉，然后才能实现像 TikTok 这样级别的应用，或者是像其他大规模的完全去中心化的 Social Fi 应用，也许到未来几年，我们才能看到，但是开发者进行了大量探索。

我们现在接入了大量的数据集，人们就可以基于这些数据集做数据科学方面的研究，做各种各样的数据科学的实验，对大数据进行分析。

近期我对于这个方面在很多场合也进行了大量的探讨，就是基于数据建立机器学习的模型。一旦数据和机器学习相结合，这会带来爆发式的增长，我们预计这样的增长将会在未来一年会有所出现。

目前，人们应用主要是围绕使用我们的使用，检索我们的使用，并且有相关的应用去存储和检索。我们也看到一些其他的应用，利用我们的数据来做数据科学。

除此之外，也看到一些其它普遍的加密应用，比如 NFT、DeFi、GameFi、Social Fi 等等，这些已经存在于区块链领域的应用，也存在 IPFS 生态中，但是也有一些独特的应用。

对我来说，我对最终可以将 Web2 的数据全部接入 Web3 感到非常兴奋。

Dominic Williams：

我认为 IC 项目所追求的目标就是成为一个“世界计算机”。当前我们也是唯一一个真正的世界计算机区块链，你完全可以基于 IC 区块链搭建各种各样的应用，可以处理 HTTP 请求，可以直接为终端用户提供内容（从链上提供）。

同时我们可以在 IC 上搭建社交网络，所以这和其他的区块链系统以及去中心化的系统都是截然不同的。

当前，如果我们看一下 Web3 的服务，他们说自己部署在某个区块链上，实际上他们往往是在部署在 AWS 的服务之上，只是有一些 Token 在区块链上处理，或者说有一些零星的数据被放在链上。

我们看到各类不同的系统和服务在 IC 区块链上不断涌现，所以对我们来说，长期的目标就是 IC 将取代所有传统的 IT，这不仅仅是关乎 Web3。

毋庸置疑，企业家、创业者一直最关注的是 Web3，在 IC 区块链上有一个应用叫 OpenChat，它是一个消息服务，它是直接已智能合约在浏览器上所运行的，所有的智能合约都可以交互。所有的用户所发送的消息都是由智能合约进行处理，其中包括媒体信息，比如说图片和视频，而这一切数据都是在链上存储在智能合约之中，这是一个革命性的事情。

OpenChat 革命之处在于这样的服务是将控制权指定给 DAO 组织。问题在于，如果你要做 Web3，你想要有 Web3 的服务，如果你用云是可能做到的，因为 DAO 组织是由智能合约创建的。智能合约可以更新、可以配置，但是智能合约不能超越区块链本身，也不能去控制这样的账户。另外，AWS 这种付费账户本身就容易被黑客攻击。

另外，我们看到 IC 区块链上开发者的数量出现了极大的增长，现在 IC 区块链上运行的项目成百上千。我们看到最先进的一点就是很多的项目将会直接将控制权指定给 DAO 组织，或者进一步去中心化，我认为这展现的就是 Web3 的潜力。在 Web3 的世界里，用户将会成为服务的部分的所有者，同时也会成为开发团队的一员，而这样的现象当前正在发生。

如果大家感兴趣的话，可以去 OpenChat 的网站（OC.App）去查看，OpenChat 就是一个 Social Fi 的服务。

这个网站看起来就像一个 signal 或者是一个消息服务的去中心化版本。如果你深入探索，会发现 OpenChat 每一个聊天账户同时也是一个加密钱包，你可以用聊天消息去发送 ICP。未来我们也会允许你去发送其他的 Token，比如我之前提到的我们利用了链钥加密学，使得 IC 上的智能合约可以创建在其他区块链上的交易。这也就意味着可以直接和比特币账本进行交互，也需要跨链桥。换言之，你可以通过 OpenChat 发送消息，而且这个消息里面就允许你直接对比特币的账本进行改动，允许你直接基于比特币的账本去发送比特币，同样也适用于对以太坊账本的改动。

如果你想要发送以太坊或者是 ERC20 的 Token 给别人的话，你只需要创建一个消息就可以，这就是 Social Fi，这也是社交媒体的账本或者社交媒体 DeFi、Web3，一切在一个真正的世界计算机上融合之后的可能性。而且你可以用智能合约和协议，直接对它进行控制，这是一个非常关键的一步。

另外，传统的区块链来存在一个问题。由于大多数的数据存储、处理和内容的提供都是基于云计算服务的，而这些计算服务的提供者，它们自身可能会出现一些问题，比如说你将这个消息引擎搭建在 AWS 上的话，作为一个开发者，你也会有相关的一些风险。

当然，要解决这样的风险，答案就是打造 Web3 服务，这样的 Web3 服务应当是完全运行在区块链上，由 DAO 来控制，受协议的控制，这是我们真正感兴趣的方向。

我觉得很难对于这样的一个全新的 Web3 的领域进行分类，因为人们经常会做各种各样的分类。我们这样一个全新的领域实际上是包罗万象的，既有 GameFi、Social Fi、DeFi 以及元宇宙，这些是在融合在一起的，所有的这一切意味着未来 IC 区块链上的开发者将会对出很多令人惊喜的应用。

Illia Polosukhin：

我们认为在应用层方面，区块链应用有三大主要支柱，分别是：金融、社交以及第三个支柱——创造者。这三个支柱之间的交互和汇聚，将会产生真正有意思的应用，比如说“金融 + 创造者”，创造者进入，看到了一个合适的商业模式，通过与社区的合作，通过金融工具的利用，形成一个积极的、正面的循环。

从生态的角度来说，NEAR 生态的使命就是为人们提供各种各样的工具，允许开发者不断发挥创意，并且用各种各样的方式进行组合。

其中包括一些重点的领域，我们如何将 DeFi 与传统金融相结合，我们有 NEAR Pay 这样的服务，允许你直接从 NEAR 账户，直接向你的银行账户转账，或者和你的信用卡挂钩，因为我们有不同的账户模型、不同的密钥模型，你可以给其他人授权，你可以有现实世界的资产，并且将企业和链上挂钩。比如，当前有很多的市场可以把它放到链上，从而可以提升市场的有效性和流动性。我认为这也是未来会看到更多的经济发展的方向。

除此之外，我们也看到很多庞大的领域，从 GameFi、数字艺术、视频等等，我们看到这些创造者不断进入，他们迸发出各种各样的创意。比如说，我们有“Listen To Earn”这样的模式，还有一些其他相关的模式，还有一些不同的市场。

在社交方面，DAO 组织是很好的，但是它本身不过只是工具，归根结底还是由社区来治理。这些社交应用和区块链，和 NFT 集成，最后形成一些新的更加有意思的应用，最终形成完全端对端的基于链上的功能，这也是我在社交领域非常关注的一个垂直方向。不仅仅是可以搭建智能合约，你可以用 Java Script 就可以完成服务的提供，这是人们希望不断去打造的 Web3 体验。并且很多 Web2 的东西可以迁移到 Web3 之上。

我们看到这些组件和现实世界的运用进行集成，而 Java Script 也会助力这样创新的大爆发。

我们看到在 NEAR 生态中，应用现在每个月实现的增长是 20%，比如说我们现在有 800 个不同的项目。我认为未来这个数字很快就会实现爆发式的增长。

主持人：各位分享一下生态中的开发者主要是哪一类的群体？最近大量的 Web2 开发者涌入 Web3 领域，如何去更好地帮助开发者进入你们的生态开发？

Illia Polosukhin：

从工具的角度来说，我刚刚提到 Java Script，这是最好的吸引开发者的工具，这也是 NEAR 非常专注的一个领域，我们如何让开发者的开发体验尽可能简单，我们希望成为最简单的区块链。

从开发者的角度来说，我们有一系列的工具，这些工具开发之后逐渐推进，现在我们有了 Java Script，而且我们刚刚也举办了黑客松，其中有三分之一的编程语言是 Solidity，三分之一的应用是用 Java Script 编写的，而有三分之一主要是用 Java Script 和现有的智能合约进行集成，然后用 Java Script 去调用，最后做了这三分之一的项目。

目前 Java Script 有两千万左右的开发者，这是一个很大的市场，开发者的规模也是非常大的。我们认为这是很有潜力的一个领域。

另外，我们有索引器、有数据湖，以及各种各样的工具。从开发者的角度来说，在 NEAR 生态搭建的时候，你不需要思考基础设施层的东西，除此之外，我们也持续去提升用户的体验。对于新进入 NEAR 用户来说，他们不需要去理解比特币与区块链这些复杂的逻辑，对于其他生态进入到 NEAR 的生态用户来说，他们不需要在一进入 NEAR 生态就先设置一个 NEAR 账户，他们可以用现有其他链上的账户先在 NEAR 上进行尝试，逐渐熟悉。

我觉得从开发者的角度来说，Java Script 的使用程度、使用的人数也出现了极大的增长。之所以开发者不愿意加入 Web3，主要是由于他们不知道到底可以开发什么应用，有很多的想法、很多的教育，在世界各地的大学里面不断给开发者提供教育。

另一方面是他们不太理解，他们看到要进入区块链，要成为区块链的开发者是非常难的。所以我们希望让开发的体验更加简单，我们很快会推出一个相关的服务，在 NEAR 里面会给你提供一些模板、一些封装好的函数块，你直接可以调用这些函数块，就可以去搭建一个非常完整的 App。整个过程非常简单，只需要 15 秒的时间，你可以开发自己的 App，你也可以在不断探索这些 App。

除此之外，我们会推出一个安全性的沙盒，允许你在沙盒里面推出你的 App，它会有一个防溢出的阈值，从而避免万一你的 App 出了问题，会造成巨大的损失。因为我们知道最开始的想法往往不是完美无缺的，对于产品来说，需要大量的 Design，才能真正去优化它。我们让整个过程一方面快，另一方面是安全，因为很多的开发者很担心，他们不太敢推出智能合约，因为担心黑客的攻击。

所以，我们一方面让 NEAR 区块链尽可能的简单，我们为开发者提供很多好的开发者工具。

Juan Benet：

过去一年大量的开发者涌入加入 Filecoin 的生态系统，也看到一系列应用进入到 Filecoin 生态，现在有将近 1000 个不同的 App。

Filecoin 吸引开发者的方式是构建者网络，有网络帮助开发者开发新的应用，从开发者工具到黑客松，到 Grant 资金，支持加速器项目、投资者网络，帮助支持创业者开发自己的项目，创建自己的项目。

另外，也有一系列开发者工具支持开发，这样的工具是非常成功的。

我们看到大量创新潜力的出现，数字还在不断攀升。对于我们而言，关键的工作是参加业内的活动，业内活动可以让我们结识来自世界各地不同社区的人，比如说像 Filecoin 峰会就是非常重要的大型活动，会有很多新面孔加入到区块链社区中来，我们互相认识，切磋启发，想出新的点子。同时，还会组织工作坊，来分享如何进行开发，帮助一些有潜力的人成为这方面的专业人士。

Filecoin 是非常高产出的网络，不同人参与进来提供不同的资源，并因此获得回报。所以有很多来自世界不同地方的人加入进来成为了矿工。

我们也看到了在很多地区的增长，比如说在韩国、日本、新加坡、澳大利亚、南美、欧洲都有快速增长。当然，北美一直是增长的。

非洲区域也是我们持续关注的区域，我们正试图在非洲的不同地区创建社区，新加坡会是极好的例子，让我们看到如何把不同社区集结起来，以促进共同成长。

Dominic Williams：

我们看到越来越多的人加入，其中有 Web2 的开发者，也有其他区块链的开发者，他们看到了 IC 的特点和 IC 所赋予的效率，所以选择加入。我们会继续努力让新的开发者能够更快更容易地上手。最近，我们正在让社区也加入到这项工作中来。

比如说在 IC 上有 158 个样本帮助开发者快速上手，有很多教育方面的工作要做，有很多技术性的文档需要准备。但我认为，其中的重点在于要帮人们理解 IC 和其他区块链的巨大差异。

在 IC 上，智能合约叫做 Canister，其和智能合约并行是确定不可篡改的。当然，Canister 可以处理 HTTP 请求，也可以进行加密学保证的安全方式为网络用户提供服务。

区块链开发者会逐步开始了解 IC 和 Canister，一旦了解清楚后，他们就会抛弃云。

此外，它不会出现重入攻击。但是会有其他的挑战，比如说异步的问题。我们即将推出一系列新的功能，比如线上的批处理，这样人们就可以在区块链上编辑视频。还有展示的功能，这样就可以创建被区块链调用的智能合约。还有持久内存。实际上在 IC 上，每个智能合约都是 WASM 和持久内存页的集合，这样一来就没有真正的存储 API，你可以自己创建一个，或者将数据持久存储在变量中。

还有很多的不同，我们正在不断努力充技术文档，从而帮助人们更快更好地理解这个超级前沿的区块链及其所能够提供的功能。

主持人：区块链大规模采用方面现在处在哪个阶段？有哪些缺口需要填补，哪些差距？

Juan Benet：

当然，我们看到了很多方面的大规模采用，所有和静态分布相关的领域都已经看到大规模采用。比如可以看到 IPFS 每周都有 1000 万的独立访问量，这是很令人惊叹的，绝对是大规模，这个成就是在一年两年前无法触及的，当然还没有达到十亿百亿，需要达到十亿的水平，所以这是差距。

在区块链方面，所有需要使用 Web2 界面的应用都已经采纳了区块链，比如对于货币、金融工具、产权等区块链是极其有用的，使用区块链可以给他们带来很大的优势，可以接入标准的 Web2 用户界面，效果很好，已经看到了大规模采用，已经有上百万任用加密货币进行支付，以及使用像微型贷款、NFT 这类金融工具，并且交易了很多 GAS 费，形成了不同的产权。

我们已经看到了巨大的采用，但真正的差距是在消费者应用程序，消费者应用程序是完全去中心化的，完全在网络 3 基础上进行工作，并且可以拓展到传统网络 2 所需的需求，那里有巨大的差距。这些差距是因为 Web3 系统还没有在这样的规模下运行过，有各种各样的漏洞需要填补，我们正在努力解决这个问题。

包括今天在这里的三位和其他数百名开发人员，也许是成千上万的其他区块链开发人员和协议设计者都在努力解决这些问题，目前正在清除这些差距。问题是大规模的基础设施不能使用来自 Web2 的社会，因为它带有关于 Web2 的传统中心化模式的各种假设，因此你必须在完全去中心和的范式下重建技术栈的各个层次，并且能够支持完全去中心化应用程序的服务，并让他们扩展到我们所需要的吞吐量。

随着缩小差距，随着完全实现端到端的应用，我认为构建像 Twitter 和 Tiktok 这样的应用在数据使用和集散方面实现大规模应用，提供信息流和推送标记内容滥用等都是有可能的。所有的计算层必须建立在完全去中心化的方式之上。当然在这其中的不同人可以自由选择他们开发的部分。

如果说我们需要有社区导向政策的话，就需要政策来治理系统。还有很多事情要做，会在 2-3 年内看到第一批亿万规模的消费者，至少我不认为明年能够看到，如果明年能看到的话当然太好了。背后仍然是运行在 Web2 的云上，所以必须放弃 Web2 的云成为完全去中心化的应用，我认为在 2-3 年内不会发生的。通过这样方式看到应用规模越来越大的应用，但还需要一段时间，直到有完整的消费级应用。

Dominic Williams：

“区块链奇点”已经达到了，意味着不需要任何传统 IT 的情况下，一切都基于区块链上。想象一下在区块链上有社交网络，这已经在 IC 区块链上运行了。看一下当前大部分 Web2 服务和应用，可以在 IC 区块链上看到类似的，但是是 Web3 的版本，比如说完全百分之百的基于 IC 区块链上的社交媒体应用，不需要云，用智能合约直接为用户提供内容，这是当前的用户数量。所以，我认为这是第一部分用户大规模采用的好消息。

关于大规模采用，人们经常会忽略另一个维度。Web3 的“魔法”在于它能够带来数十亿的用户，是什么让它有这样的魔法？我的答案是 Token，让用户真正拥有服务的控制权。

我想再次和大家介绍一下 OpenChat 这个基于 IC 区块链的消息服务，同时也是作为你的加密钱包、你的聊天消息，或者说你的 OpenChat 账户可以作为你的加密钱包。

看起来就像是 WhatsApp，或者像 Signal 这样的 Web2 应用的 Web3 版本。提供了一定程度上的 DeFi 功能。也就是说，可以在你的账户里存储 Crypto，同时也可以通过发聊天消息的形式去发送。但我觉得这还不够让 OpenChat 实现数亿级的用户，如果要实现数亿级用户的话需要网络效应，不是一般的网络效应，而是特别的网络效应，可以挑战像 WhatsApp 已经有数十亿用户工具的网络效应。关键的工具是让用户拥有所有权。

具体解释一下，Facebook（Meta）当前正在湾区拓展自己的园区，这样可以再容纳几千员工。主要是做内容培育和销售人员。想象一下在 OpenChat 这样的应用之上，如果可以随机空投 DAO 相关的权益给用户。以奖励用户的某些行为，比如说用户帮助他宣传、推广。通过这样的方式就可以打造一支由数百万人组成的去中心化团队，很快就可以组建好团队。

但是，要实现这一点，你的服务必须要作为协议，如果说去中心化消息服务是部署在 AWS 云上，你只是用区块链存储一些数据，而且你的主体是基于 AWS 的话。很有可能你会出现各种各样的问题。比如说治理问题或者是其他的社区问题，会出现各种各样的问题。

为了达成真正的 Web3 战略，用户、社区能够拥有所有权和控制权，实现真正“病毒式”的扩展，必须要让一切都变成协议，也就是说一切必须运行在区块链上，控制权指定给智能合约，通过智能合约创建服务，然后有社区的 DAO 组织，有一些自治的运行。

一旦实现了这一点，就可以真正利用相关的区块链技术潜力，从而真正实现病毒式发展，并且真正突破实现网络效应。就像是 Web2 应用当前所享受的一样，这也是为什么我认为在明年会看到这种完全去中心化的链上社交媒体服务，会有数百万用户，而且已经看到了这一迹象、这一趋势在 IC 区块链初现端倪。

Illia Polosukhin：

我非常支持胡安和 Dominic 刚刚的观点，我们已经做了很多，但依然有很多事情要做。从控制权的角度来说，有很多潜力。比如说刚刚已经提到的 Sweatcoin 项目，现在已经达到了 2000 万的下载量，在它刚刚推出不到 3 天的时间，就达到了几百万的下载量，所以这里你看到的就是大规模采用，不管你从哪个维度来判断，它都是大规模采用。

这也就意味着一切是去中心化的，一切是很好运行的。除此之外，Web3 并不是关乎去中心化，去中心化是帮助我们实现目标的工具。而 Web3 真正代表了选择的自由，更换成本几乎是零。

所以有很多事情，并不一定需要有去中心化的服务器，因为你在转换成本为零的情况下，你的成本是非常低的，哪怕说当前我们的服务器只不过只是提供数据的节点。为什么有这么多 RPC 提供者，有这么多钱包，有这么多基础设施和应用层提供者，主要希望为用户提供选择的自由。

我们还没有做到大规模采用，但距离这个目标越来越近，期待明年能看到更多的用户，而且持续朝着这个目标前进。这也是为什么我非常欢迎看到像 Sweatcoin 这样的项目。对用户来说不需要理解区块链底层的复杂逻辑，他们只需要有一点点游戏类的资产就可以开始玩这个游戏。

对于当前的区块链生态来说，有 6000 万用户，其实他们可以被招募进来。我觉得我们是有主流采用潜力的，也就是说那些没有任何技术背景的人、并非区块链行业的人，也可以通过这些受欢迎的应用进入到这个领域。

我还想和大家分享一个非常重要的问题，就是隐私。在隐私方面我们做的不够，如果考虑社交网络，或者企业的才有。归根到底，这一切都取决于网络隐私方面我们做的怎么样。像 Tiktok 的内容是公开的，所以隐私并不是问题，但如果你用消息服务，你向朋友发送的所有消息没有隐私保护的话是非常复杂的。如果你用当前非常高透明度技术搭建的话，未来隐私是有待解决的问题。所以，我觉得隐私和区块链的透明度两者应该是有很好的融合。

主持人：你们预计在你们各自的生态以及在整个区块链领域会有哪些令人兴奋的进展？

Juan Benet：

对我们来说，未来最重要的进展就是 FVM 以及用户所定义的智能合约，明年一季度就可以上线，从而带来各种各样的可能性，还有开发者的构建者项目，Builder 项目，这样大家可以在沙盒环境中进行尝试。

第二，一旦 App 被认为是能够很好运行的 App，就可以把它迁移到主网。

第三，二层网络，也就是我刚刚一直提到的将计算和数据相结合，二层主要是负责计算。

第四，亚秒级的 CDN 交付，这也将会很快上线。

最重要的一点是第一部分会先上测试网，如果大家感兴趣的话可以去 IPFS 的 Center 上下载尝试，这就是我们非常期待的下一年进展。

这一切在没有开发者的支持下是不可能实现的，有各种各样的活动将社区聚集在一起，所以再次邀请大家和我们在新加坡的线下 meetup 见面。如果说你自己就是开发者，也可以关注各种各样的项目投资机构。

明年我们还在各个不同的大洲、各个不同的区域举办更多的线下活动，从而和更多社区有直接交流的机会。这就是我们对于 IPFS 未来一年的发展展望。

Illia Polosukhin：

在未来几天，我们就会进入到分片的下一阶段，会实现更高程度的去中心化，在明年全年也会持续推动更多的功能上线。比如说在分片及持续可扩展性，另外也会有一些持续支持用户的项目。

整个生态中，开发者的活动令人兴奋，还有很多生态基金为项目提供支持，从而确保任何有创新想法的人都可以加入社区，找到相关的社区贡献者，同时也有机会基于 NEAR 开发构建自己的项目，实现项目增长。这就是我对于明年的展望。

我预计，明年至少可以实现 10 倍的增长，不管从哪个维度去衡量，甚至至少 10 倍的增长，甚至更多，这是我所希望的。

另外，由于 JaveScript 的引入，提供了巨大难以想象的灵活性，看到很多独一无二的应用和功能在 NEAR 上上线。比如说，可以将 JaveScript 模块直接导入到现有的智能合约中，或者 JaveScript 的智能合约动态执行其他 JS 合约等等。

我非常期待生态未来的发展、增长和可持续的实验，非常期待。

Dominic Williams：

我预计明年会有进一步的增长，和社区有进一步的沟通。

IC 区块链本身是具有巨大潜力的区块链，如果看一下数据的话可以看到全天可以处理的交易量达到 5 亿，其中一部分是更新状态的交易，有一些是查询交易。也就是说在你正式调用合约之前，状态已经是确定好的，我也很期待看到明年会有进一步用户数量的增长。

另外，非常兴奋的两个领域、两个新功能：

第一，SNS 网络，相当于是一个 DAO 组织，现在在 IC 生态中已经有几百个项目使用了 SNS，从而使得将基于 IC 的项目完全去中心化，去中心化的协议也会有其他的去中心化尝试，包括像销售，这些也会引起人们未来更多的关注。

第二，比特币集成的测试网，完全基于加密学的验证，没有任何跨链桥的参与。允许在 IC 上的智能合约直接在比特币账本上进行操作。也就是说，我们可以允许人们创建比特币原生的 DeFi，另外也为人们创建了很多的可能性。

比如说在以太坊、NEAR、Avalanche 有更多的其他活动。除此之外，也看到 IC 区块链上有一些交易安全的保障，这也是非常有意思的发展方向。我认为，我们未来会看到 IC 成为 Web3 的编排层，基于 IC 可以看到很多去中心化交易所，可以进行各类交易。但是却不需要离开自己的链，也就是说你不需要用 Swap，这些是革命性的功能，现在已经成为现实。

另外，在 IC 上还有一些自制的时间切片，IC 区块链是唯一一个“一体化区块链”，没有任何切片分片，而智能合约是以并行的放运行。当前，有一些功能依然还需要进一步开发，因为有 Gas 的限制。但是，我们也引了些创新模型，比如说像反向 Gas 费用，确定性的时间分片允许多区块焦点，这点非常重要。如果你想基于 IC 去搭建像 INS 这样的社交媒体/社交网络的话，就需要你的智能合约能够做图片滤镜，当前会有 Gas 的限制，但这点非常重要。

同时，也需要进一步的灵活性。较目前来说，已经创建了无限可拓展的区块链，已经是上线的主网，能够直接处理 HTTP 请求，能够直接为用户提供服务、内容，不需要依赖传统的架构，也不需要依赖传统的云，效率也已经实现了几个数量级的提升。

到目前为止，IC 区块链将为用户、为开发者、为生态大大减少碳排放，相比于完全基于 AWS 这样的云搭建而言，这也是非常重要的革命。因为目前区块链所消耗的能源以及产生的二氧化碳太多了。

像以太坊有 400 个验证者来做 merge，也就是说数据复制了 400 遍，需要安全性，但从数学上来说你要获得安全性，400 个验证者节点也已经过于多了，是不必要的，从这个角度来说以太坊是不环保的。但是 IC 区块链允许人们不需要依赖传统的 IT 架构，不需要传统的云，可以直接在 IC 链上部署前端，也就是说你不需要依赖云，用 IC 加密学就可以。

这一切都是未来几个月、未来一年会上线的，非常期待。
