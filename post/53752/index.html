原文作者：Mia，ChainCatcher

随着比特币链上协议 Ordinals 热度持续升温，基于 Ordinals 协议的创新开始层出不穷，越来越多的研发团队与开发人员投入到了这场创新革命之中。 

近期，比特币铭文又有了新故事。比特币协议 Ordinals 新任首席开发者 Raph 近日在 Github 上将比特币协议 Ordinals 创建者 Casey Rodarmor 提出的编号 2167 的”递归铭文" 更新合并到 Ordinals 协议，最终在 0.6.2 版本正式更新上线了递归。递归铭文的概念便随之而生。这是一种新的铭刻铭文的方式，具有自我引用的特性，即递归铭文允许使用特殊的语法来请求其他铭文的内容。在这之前的铭刻方式中，每个铭文都是独立的，铭文之间没有任何关联。这将扩展了铭文更多可能性，且大大降低其交易的手续费用。

紧随其后，首款可交互递归实验铭文“Recursive Playground”体验版在比特币铭文信息平台 Ord.io 上主页登上榜首，点进铭文，用户可通过鼠标与其进行交互，体验类似于解压游戏，这就意味铭文 NFT 已经从只能用户单向查看发展成了现在可通过鼠标实时双向互动。

从“静态铭文”到“递归”再到“可实时交互”反馈，比特币生态的叙事正在变得更富有活力和想象力。“递归铭文”或许正在成为新的生态风口，而“可交互”则将赋予铭文 NFT 更多新玩法。

什么是“递归铭文”？

根据 Ordinals 知名 KOL Leonidas 的解读，“递归铭文”（Recursive inscriptions）是指在区块链上进行的一种铭刻新方式，它可实现让铭文与铭文之间相互引用、调用，它打破了此前每个铭文都是独立的、没有关联的尴尬情况，通过可组合的特性，开启了自由组合的可能。简单来说，递归铭文就是一个铭文去请求其他铭文的内容。

采用“递归铭文”这一铭刻方式，铭文通过使用特殊的“/-/content/:inscription_id”语法来请求其他铭文的内容，用户可以用更少量的代码，降低铭文的容量，也就是更低的成本在比特币链上创建铭文，并且实现更加复杂的功能。

对此，Leonidas 解释，与其为 PFP 集合单独铭刻 1 万个 JPEG 文件（这将非常昂贵），不如从集合中抽取刻录 200 个特征，然后再通过新的铭文语法来请求这 200 个特征，组合制作 1 万个铭文，每个铭文仅使用少量代码来请求特征并组合呈现图像。

实际上，递归铭文采用特殊 id 语法使用代码来找寻链上已存在的对应元素然后进行组合拼接，而无需上传和存储全部铭文内容，所以代码的体积很小，这使得铭文大小突破比特币区块大小 4 MB 的限制，不仅大大降低了成本费用，还为比特币链上带来了更多创想。

具体来说，在过去，如果要创建 1 万个 PFP 合集的铭文，需要上传 1 万个不同的图片，会因图片的清晰度不同、占用的铭文空间也不同，清晰度越高，在链上铭刻图片的费用和成本就越高。而如今的递归铭文，只需把 PFP 合集所需的关键元素如图案、人物、配饰等上传即可，使用的特殊 id 语法可调用这些元素，将已存在的对应元素进行再组合拼接就可显示为图像，无需上传整个图片。

首个交互式递归铭文 Recursive Playground

“Recursive Playground”是第一个可交互递归铭文 NFT 项目，正如其测试版展示的用户只需轻轻拖动鼠标，便可改变画面的状态。让铭文 NFT 从静态、只能眼观变为动态的、可实时互动的，让收藏家也成为共同创作者，可以基于此创作自己的作品。

在“Recursive Playground”看来，个人灵感与计算机交互碰撞以某种方式实时创作出的独特的艺术作品就是交互式艺术。

目前，“Recursive Playground”项目方仍然较为神秘，并未透露其相关 NFT 产品的发射时间、价格与总量，项目方仅公布了一张交互式 NFT 体验版和一些照片。

“Recursive Playground”体验版虽然仅作为“可交互递归铭文”NFT 的首秀，但这也不可否认其作为 NFT 里程碑的重要性。这不仅仅是一次交互的尝试，更多的是一种可能性的验证，我们对 NFT 的认知不再仅仅停留在二维世界，NFT 的三维属性得到了验证。

试想一下，如果我们尝试改变一下“Recursive Playground”背后的反应逻辑，将仅限于鼠标的点击交互延伸至重力系统、流体力学、乃至于时间逻辑等，那么“可交互递归铭文”NFT 是否就拥有了更多的可玩性？NFT 将不再仅限于通过显示屏幕来传达视觉效果，通过鼠标的交互用户可以更多地感受它的存在，用户仅需轻轻动一动手指触碰，NFT 便可根据压力反应泛起点点涟漪。

“可交互递归铭文”将为比特币生态元宇宙带来怎样的可能？

让我们再次回到“递归铭文”的解读上来，正如 Leonidas 所说的，由于递归铭文的属性，“现在将有一个庞大的软件包存储库供开发者构建”，这将解锁在 4 MB 以下永远无法完成的强大用例。就像由代码编译在一起组成复杂的软件一样，通过递归铭文而构建成的软件包储存库也可以将复杂的 3D 视频游戏完全放在比特币链上。比特币本质上将成为一个内部互联网，每个文件都可以从比特币上的其他文件请求数据，或许在不久的将来通过比特币互联网上也可以进行愉快地网络冲浪了。

“递归铭文”赋予了 NFT 突破比特币区块大小 4 MB 限制的能力，而“交互式艺术”则给予了 NFT 从视觉到触觉的升华，如果将“交互式递归铭文”衍生至元宇宙或游戏领域，未来是否会出现一种成长型的 NFT 呢？

假设将 NFT 作为一种电子宠物或是游戏道具，那么通过递归铭文的铭刻，NFT 可以在铭文元素库选择其需要的元素进行升级。就像目前网络游戏一样角色将具备成长升级的属性，同样地，这种属性也适用于元宇宙和游戏领域。此外，加上递归铭文的无限递归属性，理论上元素库可与比特币生态上的任意游戏或 NFT 兼容，由此全链游戏也成为可能，当你的 NFT 或是装备自由自在地穿梭在比特币生态的任何一款游戏之中，那么真正的比特币生态元宇宙将呈现在你的眼前。

可交互递归铭文项目有哪些？

1. Fservin：推特@fservin 

引用基础属性和 JS webGL 流体模拟器，生成个人网页。据其 NFT 交互体验测试显示，用户仅需拖动鼠标，NFT 便可根据流体力学，空气动力逻辑等实现各种美轮美奂的光影效果。

2. Speculum Aeternum Miraculum：推特 @const_quary

引入数学方程与时间（区块高度)的变量，蝴蝶的颜色随着比特币区块高度的增加而改变，实现了机器与机器间的交互。

3. Recursive Doodinals：推特 @rcsvio 

项目提供 NFT 系列基础属性特征，玩家可通过 DIY 自制 NFT 再进行铭刻。据项目官方介绍，Doodinals 是第一个完全由社区创作的递归艺术藏品，社区玩家可在 io 上进行 DIY 涂鸦，所有元素特征都支持位置、缩放以及旋转功能，而 DIY 生成的 SVG 文件将刻录在 Ordinals 上，每一个玩家的涂鸦都将有机会被收录收藏系列之中。另据官方数据显示，当前已超过 128 个 avg 文件被刻录在 Ordinals 之上。

4. Vibeland：推特@vibelandwtf 

项目是第一个通过递归实现的原创 10 K 大合集的高清系列，首创实现了单一铭文随机出现不同的 PFP 静态高清图片，实现了链上随机性。随着每刷新一次页面，都将出现一张全新的头像，多达一万张可供随机选择，这在无递归调用之前是完全不无法实现的，同时也将为未来比特币上出现盲盒做出了铺垫。

总结

正如 Recursive Playground 所诠释的那样”比特币是艺术、画布亦或是一台电脑”，可交互式递归铭文的产生为比特币生态创作者打破了容量与平面的限制，是比特币网络彻底成为了用户的创作天堂，在这里只需要轻轻触动鼠标便可创作独一无二的 svg、网站与艺术品等。

用户从简单的藏家转变为了创造者，而真正的元宇宙也需要每一个元宇宙用户的搭建与创造，交互式或许为此后元宇宙的实现提供了另一种可能。

目前，首个交互式递归铭文仅仅上线几天，这一概念才刚刚映入人们的眼帘，越来越多的交互式正在酝酿与开发。每一次伟大的技术变革都源于人们对未来技术的追求，交互式递归铭文是否能引领比特币生态的变革，亦或是仅是昙花一现的炒作？我们拭目以待。
