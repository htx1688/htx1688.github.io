区块链的各种基础设施的发展令人印象深刻。现在的无数公司、组织、项目和社区正在将区块链开发提升到一个新的水平，而 Web3 现在比以往任何时候都更接近主流。

此外，适当的区块链基础设施服务使开发人员能够获得加密领域的核心优势并毫不费力地使用 Web3。如果你想使用区块链基础设施即服务将你的业务或项目带入 Web3，那么本文会非常适合你。

什么是区块链基础设施？

区块链基础设施包括许多不同的部分。例如，区块链、dApp、智能合约平台、节点供应商、API 供应商和硬件组件等。让我们回到起点，看看这一切的开始。

2009 年比特币诞生，带来了第一个区块链。然而，虽然这个区块链网络有很多优势，但它限制了开发者。因此，区块链开发者团队创建了以太坊——第一个完全可编程的区块链。如今，我们可以看到其他几个著名的开发者区块链（又称 “基础设施区块链”）的出现，以及新的区块链也不断地被开发出来。

这些网络是所有区块链应用的基础层。没有这些网络，就没有 dApp（去中心化的应用程序）。此外，区块链基础设施包括物理设备、软件和当前区块链技术栈的所有其他层（见下图）。

但在这篇文章中，我们可以把注意力放在 dApp 开发者需要关注的技术、工具和解决方案上。毕竟，在创建一个 dApp 时，你肯定不想直接与区块链打交道。

更清晰的分类——板块和产业

除了上述区块链技术栈外，CoinDesk 的 “数字资产分类标准”（DACS）应该是一个理解区块链基础设施服务的一个不错工具。这种数字资产的分类为相对 “动荡 “的区块链行业引入了一定程度的清晰度和统一性：

上图可以看到，DACS 定义了六大板块：计算、货币、DeFi、文化娱乐、智能合约平台、数字化。虽然这六个领域都有各自的区块链基础设施服务，但在区块链基础设施即服务方面，“计算”和“智能合约平台”占据主导地位。

区块链基础设施即服务——它是什么，它是如何运作的？

区块链基础设施即服务是一种托管区块链开发平台或工具，允许用户在现有区块链网络的基础上进行建设。此外，这意味着区块链基础设施服务涵盖范围广泛一方面，他们可以专注于帮助开发人员构建第 2 层区块链并解决基础设施的底层问题。 另一方面，区块链基础设施服务可以帮助开发人员构建和部署 dApp，从而直接接触到用户。此外，区块链基础设施服务还包括所有后端工具，如数据存储，使 dApps 的创建成为可能。

一个合适的 Web3 供应商提供基础设施和区块链开发工具。你也许知道，最初的区块链应用只围绕加密货币。因此，“区块链即服务”（BaaS）一词是为了货币领域（正如你在 DACS 图中看到的那样）构建的。因此，BaaS 通常被定义为更加中心化的协议，并且将许可区块链用于狭义定义的目的。因此，我们避免交替使用 BaaS 和“区块链基础设施即服务”。毕竟，大多数领先的区块链基础设施服务与加密货币毫无关系，也不是区块链协议。

尽管如此，将区块链基础设施视为 Web3 的“软件即服务”服务还是有帮助的。 最终，可靠的区块链基础设施服务帮助企业以最小的努力启动和运行应用程序。后者确保了更高的敏捷性和更快的区块链采用。毕竟，如果企业自己开发区块链相关后端的话，他们会浪费大量的金钱和时间。

区块链基础设施服务

通过观察一些区块链基础设施服务，你会更好地理解区块链基础设施即服务的概念。因此，让我们看看计算领域的五个行业组（也是独立的新行业）。我们还将介绍其他已经在塑造区块链基础设施方面发挥重要作用的重要技术：

计算领域

根据 DACS 的文件，计算领域包括包括专注于去中心化和移除计算、云存储、数据库和网络等服务中介的协议。此外，DACS 还将计算领域划分为五个板块，都提供重要的区块链基础设施服务：

IoT — 物联网板块专注于通过将区块链网络纳入现实世界来发展 “物联网 “和Web3。基本上，这种类型的协议有助于物联网网络和区块链之间的互操作性。

预言机协议 — 预言机协议侧重于收集、组织和传输数据。他们通过实时支持“链上到链上”和“链下到链上”数据来实现这一点。通常情况下，预言机利用其网络的原生代币来支付交易成本和治理权。

隐私计算 — 这部分协议都是关于建立一个自由的、去中心化、安全即服务计算能力市场。

共享网络 — 分布式云计算开放网络允许参与者以不同的成本向买家提供能源和计算资源。

共享存储 — 共享存储协议都是为了彻底改变传统的中心化存储服务器而存在的。他们通过在一个由矿工/利益相关者组成的开源网络中分配存储责任来实现这一目标，并构建一个经济激励系统。

其他重要的区块链基础设施服务技术和工具

在下一节，你将了解更多关于构建 dApp 的最佳区块链基础设施服务。然而，即使是这样的解决方案也只能覆盖一些。因此，你要探索前面列出的协议和以下技术和工具，最终找到你的终极 Web3 技术栈。以下是领先的区块链基础架构服务技术和工具的列表：

  • 节点供应商 — 所有与区块链（去中心化数字账本）的通信都是通过节点进行的。但作为一个 dApp 开发者，你肯定不想运行节点，因为这是一个具有许多挑战的全职工作。相反，你可以使用运行节点的供应商来满足这些需求。

  • Web3 API 提供商 — 这可以说是最重要的“区块链基础设施即服务”工具/技术。毕竟，Web3 API 可以在稳定的环境中实现编码一致。此外，有各种 Web3 API 提供商提供不同的 API。在大多数情况下，这些提供商连接到节点并创建自己的区块链后端基础设施，你可以利用这些基础设施。

  • 智能合约编程语言 — 这些语言使开发人员能够编写智能合约，并且它们因链而异。

  • 智能合约开发框架和 IDE — 框架和 IDE 支持智能合约编码、编译、部署、验证和测试。

  • Web3 钱包 — 主要用于管理、发送、接收和存储加密货币。它们也在 Web3 身份验证、执行链上交易、测试 dApps 等方面也发挥着至关重要的作用。

  • Blockchain 浏览器 — 这些特殊类型的 dApp 使用户和开发人员能够探索链上数据，甚至与智能合约进行交互。
