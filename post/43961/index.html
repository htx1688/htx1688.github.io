在加密行业很多人都认可一个说法，“华人擅长应用，欧美人擅长协议”。

的确，在加密基础设施L1层公链风起云涌的过去几年，华人主创的公链确实屈指可数。但这绝不代表华人主导的公链在公链领域没有一席之地。不妨看一下Conflux。

在Conflux早期，由图灵奖得主、著名计算机科学家姚期智担任首席科学家，在其领导下经过两年潜心研发，Conflux主网于2020年10月29日上线。迄今为止，经过多次升级，Conflux主网已经稳定运行2年。

Conflux是最具创新性的公链，相比加密行业第二大公链以太坊也不逞多让。2022年加密行业最关注的“The Merge”之后，以太坊的TPS也还是维持在15左右，而凭借独特的树图账本结构和Ghast共识算法，Conflux可以在不牺牲安全性和去中心化的前提下，实际运行中TPS可以达到3000~6000。

同样，以太坊在“The Merge”之后放弃了PoW转向PoS，而Conflux早在立项之时就决定采用POW和POS混合机制，同时预防POW的51%攻击和POS的长程攻击。

在生态发展上，Conflux既具有其他公链的DeFi、GameFi、元宇宙、NFT等（见下图），还在中国市场走合规路线，可以在屏蔽用户接触数字货币的同时让用户享受到公链服务。目前，有70多个Web3相关的生态平台使用Conflux的服务，在Conflux上发行数藏的品牌/IP超过260个，发布符合标准的数字证书/藏品发行总量超过460万。

为了降低应用研发的门槛和成本，Conflux研发主体上海树图区块链研究院2022年9月在上海发布了面向Web3.0的Conflux OS操作系统。

另外值得注意的是，Conflux作为华人主导的顶尖公链 ，不仅发了通证，同时还坚守在国内，把握住了监管和合规之间的平衡。

近日，正值Conflux主网上线2周年之际，金色财经专访了Conflux创始团队成员伍鸣博士。在加入到Conflux团队之前，他曾是微软亚洲研究院系统研究院的高级研究员。

Conflux有哪些创新？过去2年Conflux有哪些发展历程？Conflux是怎么平衡监管和合规？又是怎么看待中国未来的Web3.0发展的？伍鸣博士带你读懂Conflux的这两年。

金色财经：Conflux主网上线2周年了，首先祝贺Conflux。能否介绍一下Conflux这两年的主要历程以及Conflux上的生态版图？

伍鸣： Conflux从上线到现在运行一直良好，Conflux主网经历过几次hard fork有过一系列的升级，每次升级主要是一些新的feature上线，具体我在后面详细回答。

Conflux生态基本上和所有公链生态类似，也支持DeFi、GameFi等一系列的应用，Conflux虚拟机也是兼容以太坊的，然后所以基本上所有Solidity的应用都可以部署在Conflux上。

因为一直推行的是走国内合规路线，Conflux通过像Anyweb这类应用，可以屏蔽用户接触数字货币的同时，又享受到了公链安全、稳定的服务。

Conflux树图区块链目前主要是发展国内的数藏生态。目前有包括薄盒、双镜、美匣、豹豹元宇宙、淘派、星图比特、万物大吉、阿特炭、奥数字藏品、元代、爱乐疯、上镜、元始玩家、Anyweb数连、区块积木、莫塞元宇宙、奥丁元宇宙等在内的70多个Web3相关的生态平台，合作包括中国青年报、解放日报、环球时报、西安博物院、敦煌美术院、南京市博物总馆、清华大学、航天探维、LVHM Global、麦当劳、联合利华、奈雪的茶、奥利奥、安慕希、王老吉、美的、娇韵诗、乐华娱乐、爱奇艺、开心麻花、哇唧唧哇、华策集团、欢娱影视、秦时明月、周杰伦、方文山、吴宣仪等在内的品牌/IP超过260个，发布符合标准的数字证书/藏品发行总量超过460万。

Conflux树图区块链未来会继续拓展、渗透传统web2互联网企业，希望沉淀更多从传统互联网来的相关数据。

金色财经：相比其他公链，Conflux的主要创新有哪些？

伍鸣： Conflux树图区块链主要创新有下面几个方面：

首先，Conflux的最大的创新是它的树图账本结构和Ghast共识算法。这个创新解决了公链系统共识算法的性能瓶颈，使得共识算法能够在不牺牲安全性和去中心化的前提下能够支撑上千tps的吞吐量以及极短的交易确认时间。

另一个重要的创新是Conflux网络的交易转发协议。这个协议对交易以一种混合有随机数和无随机数的短哈希作为交易的id，并通过这个id，对交易的转发行为进行控制。这使得交易转发过程中对网络带宽的有效利用率大幅提高，从而使得网络不再成为系统吞吐率的瓶颈。

再有就是我们在虚拟机状态的存储访问结构上进行了优化的设计，降低了merkle tree的访问开销。

经过了这些创新的设计，Conflux的系统才能够在实际的运行中达到3000以上的tps。另外，Conflux还创新的使用了POW和POS的混合机制来提高链自身的安全性。通过这种混合机制，Conflux能够更好的克服POW的51%攻击和POS的长程攻击。

在应用方面，Conflux也有一系列的创新机制。比如代付机制。允许DApp的项目方通过智能合约为它的用户代付gas费，从而降低用户使用门槛。这个机制目前也可以避免国内用户在使用DApp的时候触及虚拟货币，规避炒作风险。

还有Conflux独创的VM虚拟空间技术，允许Conflux在同一个账本和共识系统中支持多个独立的账户地址和虚拟机状态空间。Conflux通过这个机制可以分别支持EVM兼容的空间以及增加一些不兼容的特定优化的空间。而且，Conflux是第一个能够允许用户通过智能合约以DAO投票的方式对区块链系统参数进行无需硬分叉的修改的公链系统。

金色财经：今年9月Conflux发布了面向Web3.0的Conflux OS操作系统。能否介绍一下Conflux OS是什么？为何推出Conflux OS？Conflux OS主要应用场景有哪些？

伍鸣： Conflux OS是遵循Web3.0计算范式，管理和调度Web3.0底层计算资源的网络操作系统，其运行的载体不是单台计算机或智能手机，而是整个Web3.0网络。粗略来说，Conflux OS最核心的计算资源不是来自于本地的 CPU、内存、硬盘等硬件，而是 Web3.0 网络中存在的公共服务，包括区块链共识系统提供的数据存储和计算能力、IPFS 提供的数据存储服务、第三方服务商（基于 TEE、MPC、FHE 等可信计算技术）提供的计算服务等。Conflux OS 通过管理和调度这些来自于 Web3.0 网络公共服务的资源保障数据存储和计算的中立性和可信性，并以此帮助 Web3.0 应用实现用户对于身份、数据和算法的自主权。

之所以需要一个Conflux OS，是因为调用区块链共识系统完成数据存储和计算任务是不同于传统的基于中心化服务器（包括分布式系统和云计算）的全新计算模型，技术门槛较高，特别是在涉及适配多个区块链共识系统时将更为复杂。让应用开发者直接面对Web3.0底层的共识存储和共识计算，针对各个区块链系统开发应用，必然浪费大量精力，不是一个好的解决方案。所以，为了降低应用研发的门槛和成本，让应用开发者更专注于业务逻辑，应当从技术上统一管理底层分布式的、多来源的异构计算资源，为应用开发者提供封装好的接口。

Conflux OS操作系统分为四个层次：

1.用户客户端：形式上是运行在用户本地（手机、PC）的程序，为用户提供访问 Web3.0 资源和应用的入口，包括 Web3.0 浏览器、应用市场、个人账户和数字身份管理等功能，并为运行其他 Web3.0 应用提供图形化的操作界面。目前 Conflux OS 主要提供手机端 App 作为用户客户端。

2.生产力工具：面向Web3.0 应用运营者的生产力工具，包括 NFT 生产/发行套件、DAO 社区套件等，通过图形化界面实现开箱即用和无代码开发，降低生产 Web3.0 产品的技术门槛。套件还提供可扩展的 API 接口，支持轻量级的定制开发，丰富产品形态。Conflux OS还将陆续推出面向其他应用场景的生产力工具。

3.开发环境和基础服务：面向Web3.0 应用开发者，Conflux OS将以开源形式提供UI交互层的前端组件库和面向中、后台业务的开发框架，SDK 或 Web API 形式的中间件，以及测试网络、沙盒环境等基础服务。Web3.0 应用开发者可在已有的开发框架内实现自定义的业务逻辑，以低代码开发的形式快速高效地完成开发任务。

4.底层基础设施：管理和调度来自 Web3.0 网络的计算资源，包括在选择和定位区块链共识节点、基于区块链系统读写数据和执行计算任务。Conflux OS 将先在底层接入树图公有链和树图联盟链作为承载 Web3.0 应用的共识存储和共识计算的基础设施，未来还将接入以太坊、IPFS 等其他 Web3.0 资源。

金色财经：Conflux作为华人主导顶尖公链 ，发了通证的同时还坚守在国内，Conflux是如何做到监管和合规平衡的？

伍鸣： 我觉得要想在国内发展的好，需要有真正的技术创新，因为中国是鼓励区块链的技术发展的。如果一个项目走在世界区块链技术发展的前列，那它就具有受到中国政府支持的潜力。

另外一点就是要为国内的用户找到合规使用的方案。比如在Conflux的平台上我们通过一些安全的私钥托管的技术能够让用户在不触及虚拟货币的前提下使用公链，那就具有了合规性。我觉得坚持这两个方面的原则，就有很大的机会在国内蓬勃发展。

金色财经：前不久在新加坡举行的Token2049后，很多参会人士哀叹中国人是Crypto犹太人，对中国未来的Web3发展比较悲观。Conflux怎么看？

伍鸣： 我觉得Web3在中国还是会有很大发展机会的。

首先来看看人们为什么会担忧中国有可能错过Web3？主要是源自中国政府对基于去中心化的数字货币所带来的监管问题的担忧。但其实去中心化对于中国政府来讲并不只有坏的一面。去中心化永远是个双刃剑，用好了它可以杀敌，用不好则会伤己。去中心化技术的发展也一直是朝着让这把双刃剑更好用的方向在前行。

中国政府从来没有完全否认去中心化的价值。如果完全否认，那中国政府为什么还要支持区块链技术的发展呢？中国科技部每年十四五规划的区块链专项有大量的政府资金的支持。如果完全否认，那中国为什么还鼓励联盟链的发展呢？虽然联盟链不是完全去中心化的，但至少是半中心化的。如果完全否定去中心化，大可以叫停所有区块链的技术研发，而回到之前的完全中心化的状态。

那我们再来看什么是Web3呢？我觉得web3包括但不仅仅是去中心化的数字货币，它还有去中心化存储，去中心化计算，而这些方面的去中心化，对于提升中国的国际竞争力是很有好处的。因为大国之间的竞争其实就是大的中心化实体之间的碰撞与对抗。那么在这个过程中，去中心化的手段对于实力较弱的一方是一个很有利的与对方抗衡的工具。

举个例子，由于今年的俄乌战争，Github对俄罗斯的开发者发起了制裁，禁掉了一些俄国开发者的账户。另一个事件是，美国的一个叫做科睿唯安（Clarivate）的科技数据索引公司在今年3月份宣布停止对俄罗斯的所有商业活动。也就是俄罗斯的科学家和研究人员将不能使用该公司的产品来查找来自世界的最新科研成果。我们都知道，在全球的政治地位上，中国和俄罗斯是相似的。今天发生在俄罗斯上的事情，未来也有可能发生在我们中国。那么如果这样的事情在中国发生，那一定也会对中国的科技进步带来很大的影响。那么这种制裁本质上是限制信息的传播，于是在这种情况下，信息的传播如果构建在去中心化的基础设施之上，将会对我国非常有利的。所以，我觉得中国一定会以某种形式拥抱web3的。

我们在web2时代和国外差距很小，甚至可以说超前。中国有非常强大的web2的系统，应用，和用户基础。而web3的发展是在web2的基础上，是为web2注入了新的元素和模式。因此我认为中国的web3应用一定会有很好的发展。

To C的应用场景在web3的整个市场里是占有很大比重的，甚至比在web2时代的比重还要大。这也主要是由数据的去中心化所带来的一个必然的趋势。那中国的人口优势为To C的应用发展提供了很好的土壤。因此，中国web3应用的发展是具有很好的先决条件的。

另外，中国强大的web2产业基础培育出来非常多的优秀的系统，应用，以及产品工程师和运营人员。这些人才的储备也是推动中国web3应用发展的生力军。这些中国的技术团队，一方面可以和海外的商业团队结合，参与到海外的web3项目中去练兵。另一方面也可以寻找到国内正在转型的投资机构的支持，在国内合规的框架下搭建生态项目及产品。从而将web3生态应用带到更多的传统web2的用户面前。

在当前监管模式下找到让web3应用场景触达用户的合理的方式。在当前的中国政策下，虚拟货币是不予以支持的，这个状态会持续较长的一段时间。那在这样的框架下，如果我们不能够将web3的应用模式推广到web2的用户那里去，那中国就很可能错失web3革命。中国的Web3如果想要有生命力，必须能够在一定程度上摆脱对虚拟货币及其构成的庞氏游戏规则的依赖。能够为传统的web2用户提供实际的价值。能够被广大的web2用户所接受。这样才能够使Web3的应用和用户规模达到web2的量级。中国的Web3才能繁荣的发展。因此我非常建议这样的作法。就是在与用户的接口上为用户过滤掉虚拟货币的访问，但是仍然使用完全去中心化的区块链后端作为信息的载体。这样才能在未来web3的用户体量和合规机制都达到一个很好的状态的时候，Web3的系统及应用能够做好充足的准备，迎接美好的机遇。

那么找到这样一个合理的方式会不会太理想化，会不会是不可实现的？我觉得不会。其实我觉得现在反而是一个非常好的时机。因为现在中国的政策不仅仅是对虚拟货币不友好，其实对传统互联网的web2也不友好。大家看看现在蚂蚁，阿里，滴滴的各种遭遇就应该明白这一点。但是Web3却是一个新的浪潮，国家反而在鼓励探索。在这样一个背景下，我们反而更应该把Web2的基础设施和应用在Web3的加持下重新构造一遍。这也是web3的创业者重新挑战web2的商业巨头的一个非常好的机会。

金色财经：未来web3发展会有国内和国际之别吗？如果有的话，区别会在哪里？

伍鸣： 肯定是有区别的，区别主要是在不同监管环境下的应用模式的区别。

在国内，我们需要用符合国内合规的模式来发展web3的基础设施和应用。其实我觉得国内的环境反而会更加好一些，因为如果没有非常好的监管，或者是在比较开放的场景里面，必然是鱼龙混杂。国外的环境下会有很多庞氏炒作的东西在里面，会把真正有价值的东西给掩盖掉。在国内做web3更能够沉淀到Web3本身的基本价值上。

金色财经：Conflux怎么看未来的web3发展？

伍鸣： 未来web3的发展大有前途。由去中心化技术带来的生产关系的改变，以及元宇宙带来的全新的应用场景，人们的生活方式会朝着更加数字化，更加高效便捷的方向发展。伴随着这种需求，行业里有大量基础设施建设和应用场景创新的挑战等待着我们去攻克，去完成。

说到具体方向，我个人觉得有几个方面。一个是区块链的基础设施公有链应该朝着一个更加标准化和模块化的方向发展。

这个行业里肯定是一个多链的世界，多链之间需要互操动性，所以建立一个通用的通信标准，对于这个行业发展是非常有利的。

另外，现在有一个趋势就是就是一个庞大的公链系统里面的一些功能模块，慢慢拆分出来变成独立服务，这些服务本身就构成了一个类似于区块链的系统，可以去为不同的区块链提供支持。这个模块化的发展，我觉得也是一个促进这个行业技术进步的这样一个因素。

从功能布局的角度讲，一个区块链代表了分布式账本。但是在去中心化的web3世界里，要承载原来web2的那些基础设施，就还需要去中心化的存储，去中心化的计算，也就是所有的互联网技术架构，要慢慢加入去中心化元素，也会变成未来元宇宙场景的核心支撑技术。
