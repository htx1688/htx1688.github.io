今天，我们很高兴地宣布推出 Base 的测试网，这是一个以太坊第 2 层 (L2) 网络，为任何人在任何地方提供一种安全、低成本、对开发人员友好的方式来在链上构建去中心化应用程序。我们与 Base 的目标是让 onchain 成为下一个 online 以及将 10 亿用户带入加密经济。为了实现这一目标，Base 将作为 Coinbase 链上产品的大本营和任何人都可以构建的开放生态系统。

我们正在 Coinbase 内部孵化 Base，利用我们过去十年构建加密产品的经验，并计划随着时间的推移逐步去中心化区块链。为了支持去中心化的道路，我们正在构建基于Optimism 的 OP Stack，愿景是创建一个由 Optimism 提供支持的标准、模块化、Rollup Agnostic的超级链。我们不打算发行新的网络代币。

开发人员已经可以在 base.org 上在 Base 进行构建。在这篇文章中，我们将更深入地探讨为什么要构建 Base 以及我们计划如何加速加密经济。

我们为什么要建立Base？

从 Coinbase 成立之初，我们的秘密总体规划就一直清晰且一致：创建一个开放的金融系统，以增加全球经济自由度。在过去十年中，我们已经完成了我们设想的前三个阶段：协议（比特币、以太坊）、建立交易所（Coinbase Exchange）以充当数字货币的桥梁，以及创建去中心化应用程序或“dapps”（Coinbase 和 Coinbase 钱包）。在过去的一年里，除了继续我们对前三个阶段的投资，我们越来越关注如何解锁最后阶段：开放金融系统的应用程序。特别是，我们问自己需要什么来创造一个环境，让成千上万的人真正有用出现可以将10亿+用户带入全球加密经济的 dapp？

在我们觉得可以为客户回答这个问题之前，我们想说说这对我们自己来说是什么。目前，大多数 Coinbase 产品都没有写成智能合约，它们主要利用其他 web2 公司使用的相同应用程序基础设施。 虽然它们允许客户访问加密货币，但它们并不是在由加密货币启用的链上平台上作为 dapps 本地构建的。 在过去的几年里，我们慢慢地改变了这一点：USDC、Coinbase 钱包和 cbETH 是原生链上产品的例子，它们为 Coinbase 带来了显著的用户活动和收入。 展望未来，我们希望帮助我们的团队构建更多类似的产品，以充分利用链上环境的全部功能。 我们假设，如果能够让自己的团队做出这种改变，就能很好地推动其他开发者也这样做。

当我们与内部团队讨论如何做到这一点时，我们看到他们正在努力解决两个看似简单的问题：

1、他们应该如何构建 dapps？ 有许多用于编写智能合约的开发人员工具包——EVM、 Solana 的 Rust、Cosmos-SDK 等等——我们的团队不确定他们应该使用其中的哪一个来构建他们的应用程序。 而且因为我们没有一条清晰的“宽敞道路”，所以我们没有使团队能够以安全的方式轻松开始快速开发的基础设施。

2、他们应该在哪里部署 dapp？ 一旦他们选择了一个工具包，他们就可以在许多不同的链上部署——例如，使用 EVM，他们可以部署到以太坊 L1、以太坊 L2 或其他提供 EVM 执行环境的链。 我们的团队不确定他们应该选择其中的哪一个，也不确定他们如何确保该环境与其他 Coinbase 产品兼容。

我们发现这两个问题在很多情况下会在团队开始之前就让他们放弃。 因此，在 2022 年上半年，我们运行了一个内部流程来回答这些问题，并为内部链上开发铺平了道路。 在与外部建设者和内部团队进行广泛对话后，我们在三个主要决策上达成了一致：

1、EVM 作为我们 dapps 的主要开发者平台。 EVM 是当今的主导环境，绝大多数开发人员、活动和流动性都发生在 EVM 环境中。 此外，EVM 已经在整个 Coinbase 产品套件中得到了重要支持，这意味着默认情况下将支持新产品。

2、拥有最高锁仓价值 dapp 的以太坊第 1 层 (L1)。 以太坊已成为最成熟、安全和去中心化的 EVM 链。 这使其成为部署对安全性要求最高的链上应用程序的理想环境，例如价值数十亿美元的大型机构或企业用例。

3、以太坊第 2 层 (L2) 适用于需要扩展的 dapp。 以太坊优先考虑安全性和去中心化，这导致高昂的费用使其不适合旨在覆盖数十亿用户的应用程序。 为了解决这个问题，以太坊扩展策略是启用由以太坊保护的链，以实现更便宜的交易（通常称为“第 2 层”或 L2）。 我们将 Ethereum L2 确定为我们部署未来 Coinbase 应用程序的地方，旨在为我们的整个客户群提供服务。

虽然这些决定帮助我们的团队更快地开始构建，但他们也看到了下一个机会：以太坊 L2 今天仍然太昂贵（交易价格为 0.05-0.25 美元）并且有许多 L2，这意味着我们的团队仍然需要在不同选项之间选择。 2022 年下半年，我们开始投资 EIP4844，它将 L2 费用降低 10-100 倍，我们开始了另一个流程来确定我们应该选择哪个 L2 作为 Coinbase dapps 的主要家园。

当我们与在以太坊 L2 上构建的团队会面时，我们被他们的创造力和洞察力以及他们为扩展以太坊带来的协作心态所震撼。 我们也对他们采用的多种方法感到兴奋，每种方法都为如何最好地扩展加密经济带来了新的知识。 通过我们的对话，我们更好地理解了阻碍 L2 安全性、规模和采用的主要挑战。 我们还针对未来以太坊 L1 和 L2 的架构改进了我们的mental模型。

特别是，我们以这样的论点开始了这一过程，即单个 L2 将成为“主导”并逐渐吸收所有链上活动，从而形成近乎垄断的局面。 有了这个愿景，我们最初将问题视为“选择正确的”，如果我们选择“错误的”，我们就有可能将 Coinbase dapps 和用户从生态系统中的大部分活动中切断。

我们以一个非常不同的论点结束了这个过程：会有许多具有重要活动的 L2，作为不同生态系统的“枢纽”，并逐渐增加它们的互操作性，直到它们形成一个共同扩展以太坊的“巨网”或“超级链”。

带着对未来的愿景，我们问自己：在一个拥有许多 L2 的世界中，我们能否通过自己构建一个来加速 Coinbase 的使命和更广泛的加密经济？ Coinbase 之前曾两次（2018 年、2020 年）考虑过推出一条链，但每次我们都决定不这样做。 经过慎重考虑，我们决定同意并推出 Base，因为：

1、Base 将加速 Coinbase 链上产品的开发。 通过在我们构建的地方创造更多的确定性，我们消除了开始之前一直困扰团队的问题。 我们创建了一个基基础层，我们可以围绕其他开发工具分层，使在链上的构建变得异常容易。 通过 Base，我们致力于让团队更容易在链上而不是链下构建。

2、Base 将把我们的客户带入加密经济。 通过在 Coinbase 轨道上创建一个具有 Coinbase 级安全性且完全在链上的环境，我们将能够将更多的活动带到链上。 我们认为将用户及其资产带到 Base 是他们访问更广泛的加密经济的垫脚石。 重要的是，我们不希望 Base 成为一个孤岛——我们希望它成为一座与以太坊 L1、其他 L2 甚至其他 L1 生态系统（如比特币、Solana 和 Cosmos）深度连接的桥梁。

3、Base 将增加 Coinbase 对核心加密基础设施的投资。 通过对 Base 的投资，我们现在有更大的动力来帮助扩展 Base 构建的基础平台。 我们从 EIP4844 的工作开始，我们预计投资只会增加。 我们很高兴能够回馈过去十年使业务成为可能的核心基础设施。

我们将 Base 视为我们秘密总体规划（第 3.5 阶段）的关键组成部分，并相信它将解锁下一代 dapp，将数十亿用户带入全球加密经济。

我们如何建立 Base 以加速更广泛的加密经济？

从本质上讲，区块链与今天的 Coinbase 产品有很大不同——特别是，它是一个平台，而我们迄今为止构建的大多数东西都是产品。 构建一个具有去中心化和规模化愿景的加密平台意味着我们需要建立在这些令人难以置信的去中心化平台之上。 在我们建立 Base 时，四个关键原则影响了我们的决策，它们将在未来几年继续为我们提供指引。

一座桥梁，而不是一座孤岛

我们正在设计 Base 以优化对以太坊 L1、其他 L2 以及其他 L1 生态系统（如比特币、Solana 和 Cosmos）的轻松、安全访问。 我们鼓励他们从 Base 开始，但通往一切：我们将 Base 视为用户进入加密经济的“桥梁”。 这是一种易于使用的默认链上体验，可以访问其他链上的产品。 随着 Base 与其他链的互操作性，我们将继续支持跨 Coinbase 产品的尽可能多的链。

开源

全球加密经济的基础软件应该完全开源并免费提供。 Base 与 Optimism 合作，建立在麻省理工学院许可的 OP Stack 之上。 我们作为第二个核心开发团队加入，致力于 OP Stack，以确保它是一项对所有人都可用的公共产品。 我们将此工具包视为一个开放平台，任何人都可以贡献、分叉和扩展以帮助加密经济规模扩大。

去中心化

去中心化对于加密经济保持开放、全球化和人人可及至关重要。 虽然我们已经开始在 Coinbase 内部孵化 Base，但我们坚定地致力于在未来几年朝着完全去中心化的方向发展。 随着 Coinbase 为 OP Stack 带来的资源增加，我们有信心 Base 将在 2023 年从第 0 阶段升级到第 1 阶段 rollup，并在 2024 年完成第 2 阶段 rollup。

为了所有人

Base 的愿景是成为一个开放的生态系统，以 Coinbase 产品、用户和资产为种子。 我们将与更广泛的社区合作，使区块链活力满满，我们很高兴拥有一个令人难以置信的社区，他们将为使 Base 成为一个强大的生态系统做出贡献。

Base上的建设者

这是我们与 Base 的旅程的第一天，我们对已经建立的令人难以置信的社区感到无比兴奋。 如果您对我们的方法有任何想法、反馈或问题，可以加入我们的Discord进行讨论。

我们要如何推进？

我们现在的重点是与加入 Base 生态系统的构建者和开发者一起运行一个成功的测试网。 我们希望这是一个迭代过程，并期待共同努力使 Base 取得成功。

在接下来的几周内，我们将分享我们的主网路线图，并为在 Base 上构建提供更多开发人员资源——敬请期待。
