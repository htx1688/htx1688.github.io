原文作者：Yinan

近日，比特币协议 Ordinals 新任首席维护者 Raph 近日在 Github 上将比特币协议 Ordinals 创建者 Casey Rodarmor 提出的“递归铭文” 2167 号更新合并到 Ordinals 代码中。

递归铭文是指在区块链上进行的一种铭刻方式，具有自我引用的特性。在之前的铭刻方式中，每个铭文都是独立的，铭文之间没有关联。但通过引入递归铭文，这种情况即将改变。

递归铭文允许使用特殊的"/-/content/:铭文 ID"语法来请求其他铭文的内容。这个简单的改变就有了无限的可能。

例如，与其单独为一个 PFP 集合铭刻 10, 000 个 JPEG 文件(这将非常昂贵)，不如从集合中铭刻 200 个特征，然后再通过新的铭文语法来请求这 200 个特征，组合制作 10, 000 个铭刻，每个铭刻使用少量代码请求特征并通过编程呈现图像，这将极大地降低交易费用，同时突破铭文图片大小的限制。

通俗的说，熟悉 NFT 人都清楚，一个 10000 个 NFT 合集，大部分都是由十几个或者几十个部件组成，以人物 PFP 为例，一般包含背景、帽子、发饰、眼睛、挂饰、衣服、裤子、鞋子等部件，然后再由这些部件进行排列组合成 10000 张图片。

而递归铭文就是只需要把这些几十个部件上传上去，再通过语法对这些部件进行调取，然后再进行一个组合，之前方式是先组合再上传，如今递归铭文改变了这一方式，这将在很大的程度上节省费用，并提供了更多的可能性。

而后面这些上传后的部件又将形成一个素材库，所有人都能够进行调取，艺术家以及创作者们将能够对此进行一个很好的后期创作，打开想象力。

可能性

可组合性的 NFT/铭文合集：像之前的 Moonbird 发行之后，出来了 AIMoonbird 这种对蓝筹项目的二次创作，递归铭文的出现将对艺术创作的可组合性将大大提高，比如说艺术品，数字资产，链上游戏等可能。

去中心化的网站：不就之前，casey 回答了社区成员的提问，铭文类型新增了 “JS”和“CSS”类型，这也就意味着未来可能出现存在于去中心化网络上永不消失的动态网站。

NFT 的碎片化：既然可组合那么是否可进行拆解，对铭文进行碎片化，然后再进行一个更多的操作。

域名系统：随着铭文可组合性的提升，域名是否不单单只是链接一个地址，而可以挂载更多的东西，挂在网页还有指向包含的 NFT 等等。

总结

目前，递归铭文提出只有 3 天，也算是一个社会实验，但是引起了加密社区一大部分人的关注讨论，是否能够引起 BTC 生态的下一波热潮，需要时间来鉴定，但是往往热点在哪里，机会就孕育在哪里。

新的事物出现，需要时刻保持敏锐的嗅觉，保持对新事物的好奇心以及探索能力，适当的研究以及参与，将有不一样的收获。
