金色财经报道，8月12日消息，Cosmos开发团队informal发布了Cosmos跨链安全的链间安全代码v0.1，该版本为链间安全代码v1的预发布版本。

链间安全是一项允许 Cosmos Hub 与其他区块链共享安全性的功能。在技术角度上看，该功能的上线意味着 Cosmos Hub 验证器集将获得验证Cosmos Hub 连接的其他区块链的能力。

在跨链结构里，Cosmos Hub 连接的其他区块链被称为“消费者链”，由 Cosmos Hub 数十亿美元的质押 ATOM 保护。

预发布后，将在未来几个月与不同的 Cosmos 利益相关者合作进行广泛的审计、测试。一旦发现错误、修复后，会发布 v1 且部署到 Cosmos Hub。

9 月初会为这个进程开启一个私有测试网，包含一小部分 Cosmos Hub 验证器，作为将在 v1.0 发布后开始的公共测试网的试运行。

通过查验github，可以发现V0.1 预发布版本中包含了如下内容：

完整规格ICS 底层的 CCV 协议

协议的实施

验证器集更改数据包和确认：使验证器集在消费者链上保持同步

削减数据包：惩罚 Hub 上的验证者停机和消费链上的双重签名

分发数据包：允许消费链向 Hub 发送可变百分比的费用和通货膨胀作为安全支付

Cosmos SDK 修改：对 Cosmos SDK 模块的小修改

可以发现，预发布版本主要为测试作准备，也因此，会启动下面的测试和审计过程：

单元测试：与代码并存并测试各个功能。

集成测试：启动一个模拟 Hub 和消费者链的 6 节点网络，在每次代码更改时运行。

差异测试：Python 中的协议模型与真实系统一起进行测试，以发现最复杂的错误。

测试网： 3 个小型私有测试网，首先在开发团队中，然后扩展到包括一些验证者

私有测试网：发现任何操作错误或困难，并为公共激励测试网做准备

Informal Systems审计：审计实践是 Informal 的一个完全独立的团队，将仔细检查代码并进行测试，以发现任何安全漏洞。

激励性测试网：一个有奖品的激励性公共测试网将把协议暴露给一个大型的、Hub 大小的验证器集，并奖励完成任务和发现错误的验证器

以上过程在路线图中可以推测出，在 Cosmos Hub 上最终部署链间安全代码的时间为2023 年 1 月，该时间段，会有多个消费者链入驻运行，例如QuickSilver、Neutron等。

链间安全的主要部分是跨链验证CCV，是指与供应商和消费者区块链之间的质押和削减通信相关的链间安全子集。提供者区块链将质押更改传达给消费者区块链，而消费者区块链可以将削减证据传达给提供者区块链。

CCV 的types文件夹包含提供者链和消费者链使用的类型和相关函数，consumer模块包含消费者链运行的代码，provider模块包含提供者链运行的代码。目前测试应用程序可能无法正常运行，请暂时依靠 IBC 测试套件编写单元测试。以下为CCV技术运作关系图。
