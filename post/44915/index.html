编译：深潮 TechFlow

2021 年，加密货币迎来了牛市，以太坊变得拥挤，用户为此支付数百美元的 Gas 费。

解决方案是什么？部署更新更快的链！像 Solana、Binance Smart Chain 和 Avalanche 这样的 “以太坊杀手 “出现了。他们都开展了流动性挖矿，从以太坊中吸走了流动性。

尽管过去很多人都在支持以太坊，但在那段高需求时间人们放弃了它。

Alt L1 链暂时解决了市场对扩展性的需求。这么多具有不同标准的链的出现，也带来了跨链桥的需求。同时跨链桥也带来了麻烦：不仅不方便，它还很危险。我们经常能看到跨链桥被黑客攻击的新闻，大家肯定不喜欢这样。

以太坊试图通过 Rollups 以不同的方式解决扩展性问题。将交易转移到一个单独的（Rollup）链上处理，但继承主网的去中心化和强大的安全性。

截至 10 月，截至 10 月，Layer 2 已经在交易执行方面超越了以太坊主网。

但仍有一个问题。每个新链或 Rollup 本身就是一个整体链，有它自己独立的语言和设计结构。

其结果是什么？

一个异构的 Web3 生态系统。这就是为什么我们需要类似 Hop 这样的协议来在 Rollup 链之间 “跳跃”。

如果 Web3 要被主流采用，我们就需要和这种持续的网络切换说再见。人们不希望处理零散的、异步的链。这就像每次要进行银行间转账时都必须兑换货币一样。

如果说跨链桥是一次性创可贴式的解决方案，那么，模块化的 Rollup 开始为更全面的解决方案铺平道路。

怎么办？

Optimism 背后的开发者有一个直击问题根源的答案：OP Stack。

目前的难题

今天，大多数链都在追求自己的模块化战略。

  • Optimism、Arbitrum 和 Starknet 在执行层上有所不同，但他们通过将结算、共识和数据可用性层外包给 Ethereum 主网从而共享相同的结算、共识和数据可用性层。

  • Metis 和 Celestia 选择自己的数据可用性层，同时仍然使用以太坊作为其结算和共识层。

  • 基于 StarkEx 的 validium 链，如 Immutable X 或 rhino.fi，通过运行它们自己的相对中心化的数据可用性委员会，用一组预选节点来托管交易数据，以执行类似的事情。

每条链都运行着自己的策略，在执行、结算和数据可用性层的设计上存在差异。

如果这些链共享一个标准化的开源代码库，而不是今天的孤岛式产业链/Rollup 系统，会如何？

这就是 Optimism 的 OP Stack 的作用：一套模块化的基础乐高积木，用于在以太坊上构建更具表现力和精确的 Rollup 链，这在今天的单体 L2 上是不可能的。

OP Stack 是一套标准化的开源模块，可以组装起来建立一个自定义的链—— Optimism 将其称之为 “OP-链”——以服务于任何特定的区块链用例。

让我们把它分解：

模块是任何开发者都可以插入到 OP Stack 中的数据位，以创建一个 L2、L3 或 L4。“标准化"意味着对一个模块的标准有共识，而且所有人都可以实现。开源意味着它可以免费提供给任何人进行迭代和请求。

使用 OP Stack，你不会被束缚在一个特定的证明系统或技术中。开发人员有能力在一个链的不同执行、共识、结算和数据可用性层中弹出模块，就像切换 API 一样。

dYdX 离开以太坊，选择 Cosmos 应用链，因为他们希望对他们的链的共识层有更大的模块化。OP Stack 解决了这个问题。

OP Stack 旨在允许以比当前工作更容易的方式分叉代码 ，因为开发人员可以很容易地抽象出区块链的各个组成部分，并通过插入不同的模块来修改它。

也许 Optimistic rollup 想要将自己改造为 ZK-rollup。 没问题！ 只需将其欺诈证明模块切换为结算层的有效性证明模块即可。

也许某个链想使用 Celestia 作为其数据可用性层？没问题！把 Ethereum 换成 Celestia 作为数据可用性层就行。

想要把执行层的 EVM 换成另一种虚拟机，比如 FuelVM？在运行的链上交换执行层很难，但这在 OP Stack 的技术上是可行的。

也许你想把 Minecraft 作为一个 L2 rollup 运行，但链上游戏在主网上的计算量太大？事实上，有人已经做到了。它被称为 OP Craft，其背后的团队 Lattice 通过将自己的执行模块引入到 L2 rollup 运行的执行槽中，然后在共识层中修改 Plasma 以增加可扩展性。

因此，OPCraft 在以太坊上作为其自己的 L2 rollup（一个 OP 链）存在，游戏中的每一个动作都是作为链上交易执行的，并回滚回以太坊主网。这是 Minecraft 在 EVM 兼容区块链上的扩展！ 就像任何其他 rollup 一样，开发人员可以通过节点访问它并在其上部署智能合约。

Lattice 通过利用 Optimism Bedrock rollup 架构做到了这一点。Bedrock 是 OP Stack 的第一个实现，OP Stack 是 Optimism 使用的模块集合。Bedrock 使用以太坊虚拟机作为执行层，使其与 EVM 等效，并使用 Cannon——其在结算层上的交互式故障证明系统。

而更多疯狂的分叉正在到来。0xPARC 通过将 Bedrock 的执行引擎替换为 Game Boy 模拟器来构建 Game Boy rollup。

而且都是在链上进行的。

OP Stack 的开放花园

今天，模块化区块链的主要问题是碎片化加剧，随着开发人员做出自己的设计选择和权衡，碎片化程度就会增加。这种碎片化问题类似于 Web2 的围墙花园，只是在 Web3 它是偶然的。

OP Stack 通过从开放花园的理念出发，在 Web3 的建设中解决了这种逐渐分裂的问题。所有 OP 链都可以享受原子跨链可组合性，只要 OP 链自愿选择同一个共享排序器，即在每个 OP 链上生成块的唯一实体。

OP 链可能不想运行他们自己的定序器，所以他们可以支付一定的费用来使用他们信任的 Optimism 共享定序器。这为 Optimism 开辟了另一种盈利模式，不仅仅是目前 Optimism 链上的 dApp。

最终，以太坊上的任何用户都可以从生态系统的任何一端向对方发送交易。没有更多的网络切换或跨链桥！

这一愿景催生了 Optimism 的 “超级链 “的新兴结构，数百/数千条 OP 链将在 Optimism 上完全互操作，并通过相同的技术结构连接。

启动 Rollups 并不会比启动 ERC20 代币更难，这加快了 Web3 实验和创新的速度。

这不仅仅是互操作性。

随着 OP Stack 共享模块所带来的更高的灵活性，开发人员能够反复回收以前开发人员使用过的可重用代码，从而使代码更强大，更能抵御黑客攻击和错误。

例如，当 Lattice 团队构建 OPCraft 时，他们设计的每个区块的 Gas 限制比 Optimism 自己的链高得多。在这种不同的配置中，他们发现了某些以前不明显的 Bug。

让 OP Stack 更加普及

总而言之，OP Stack 是对以太坊通过模块化进行扩展的颂歌。

Optimism 正在建立一个完全开源的 Rollup 链生态系统，而 OP Stack 是建立这一愿景的基础。但是，不仅仅是 Rollup，OP Stack 还可以被利用来建立治理和身份模块，让开发者有能力轻松地从头设计他们的链。

OP-链不是 Cosmos 应用链。它们也不是单链。它是按相同的技术标准合并的超级链。

当所有的事情都完成后，Rollup 生态系统将如同成千上万的花朵一样在以太坊上绽放。
