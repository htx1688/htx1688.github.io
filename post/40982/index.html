随着 Magic Eden 崛起，Solana NFT 社区成员越来越担心平台在其发展过程中变得过于中心化。 

撰文：Andrew Hayward

在 Solana NFT 领域，没有比 Magic Eden 更大的玩家了。该市场于去年秋天启动，通常占据 Solana 所有交易总量的 90% 或以上。在今年 6 月的最新一轮风投融资中，它的估值已达到 16 亿美元。

但是随着 Magic Eden 的崛起，Solana NFT 社区的成员（包括创造者和收藏者）越来越担心平台在其发展过程中变得过于中心化。他们指出，最近的更新限制了第三方聚合器和工具的访问，以及 Magic Eden 托管用户 NFT 的方式，可能会让用户的资产容易受到攻击。制造 Solana NFT 工具的 Zion Labs 的创始人 Marty 告诉 Decrypt ：「人们应该意识到黑客可以获得 Magic Eden 的密钥，并 ‘rug’ 他们的每个 NFT。如果它是去中心化的，代码是开源的，这就不会发生。」

在发给 Decrypt 的回复中，Magic Eden 并没有特别提到基于托管的交易模式的风险，但它表示，它认为目前对用户来说，替代方案不太安全。该市场计划在未来采用无托管系统，但「该技术还不够安全」。

Magic Eden NFT 托管模式受质疑

关于 Magic Eden 将用户列出的 NFT 资产保存在托管钱包中的争论正在升温。Magic Eden 托管所有上线的资产，而不是允许它们留在用户自己的钱包中，用户的 NFT 通过市场智能合约保存在托管钱包中。这种做法在 Solana NFT 市场早期很常见，但后进入 Solana 生态系统的公司，如 OpenSea 和 Hyperspace，则并没有采取这种方法。

上周三，OpenSea 在推特上发起**「反对 Solana 市场托管 NFT 」** 的话题，虽然没有直接点名 Magic Eden，但目标很明显。OpenSea 当时在推特上写道：「我们相信，托管用户的 NFT 的市场会限制选择和效用，并危及安全性。」

Metaplex 的拍卖协议使 Solana 能够在不需要市场托管资产的情况下进行 NFT 交易。一位不愿透露姓名的 Metaplex 消息人士向 Decrypt 证实，Magic Eden 的市场合约是基于拍卖行的早期版本，是一个免许可的点对点交易系统。然而，Magic Eden 对该合约代码以及基于 Metaplex 的 Candy Machine 的 Mint 工具的启动平台合约进行了重大修改。Magic Eden 也将他们与社区的其他成员隔离开来。该消息人士表示：「它们是 Metaplex 提供的开源技术的封闭源代码和许可衍生品。」

这种方法增加了 NFT 交易者的潜在风险。闭源软件不能被社区审计，或从漏洞奖励计划中受益。甚至连 Metaplex 都不知道 Magic Eden 的市场合约代码中的内容。如果 Magic Eden 的托管钱包被盗，会发生什么？或者，如果 Magic Eden 突然宕机，就像最近几个月其他一些加密公司在最近的市场崩盘中那样，会发生什么？Metaplex 消息人士表示，截至上周，「中心化」托管钱包持有约 18 万个 NFT。

在回答 Decrypt 的问题时，Magic Eden 的联合创始人兼首席技术官 Sidney Zhang 表示，市场计划在某个时候过渡到无托管模式，但在他的团队看来，目前的解决方案还不够安全。他写道：「我们正在积极探索无托管模式，并计划转向无托管模式，但我们认为，其他市场目前使用的实现无托管模式的智能合约是不安全的。这种转变会带来许多安全问题，我们希望谨慎行事，以确保我们的用户不会因为没有及时更新的列表而无意中丢失资产。」

Magic Eden 近期的多项调整

除托管模式外，Magic Eden 还迎来很多新变化：越来越多的审查，审查其平台运行的方式，以及第三方应用程序在其基础上构建的方式。

上周，由于用户 「Pland」 在 Twitter 上疯传 Magic Eden 的托管模式话题：由于最近的智能合约更改，Magic Eden 「不再是一个免许可的 Dapp 了」，大多数用户没有注意到，但确实对生态系统有很大的影响。智能合约持有为 Dapp 和 NFT 资产提供动力的代码。与 Decrypt 交谈过的开发者表示，合约变更使得 Magic Eden 必须签署其市场上发生的每一笔交易，这与之前的情况不同。其结果是，一些聚合多个市场列表的第三方应用程序以及可用于购买特定 NFT 的所谓「狙击机器人」工具遭到破坏。

Magic Eden 向 Decrypt 承认了合约的变更，解释说交易现在需要两个签名：一个来自最终用户，另一个来自 Magic Eden 提供的 API 密钥。API 密钥用于验证希望访问应用程序或服务的开发者和第三方程序。 像 OpenSea 这样以以太坊为中心的市场也有 API 系统。

Magic Eden 联合创始人兼首席工程官 Zhuojie Zhou 告诉 Decrypt：「 这一改变的推出是为了保持核心网站的可靠性，并减少可能危及用户列表和交易的机器人行为。我们非常欢迎生态系统参与我们的 API 程序。」 Solana Labs 最近做出了一些改变，试图提高网络的稳定性。

Zhou 表示，Magic Eden 已经向开发者提供了 300 多个 API 密钥，包括像 Tensor 和 NFT Soloist 这样的聚合器，以及像 Exodus 和 Slope 这样的钱包应用开发商。他还指出，Solana 钱包 Phantom 的开发者要求 Magic Eden 拥有一个 API 来验证交易是否来自其服务器。「我们相信支持正式的开发者生态系统，从而实现安全可靠的市场的目标，」Zhou 补充道，「我们保持开放的态度，根据合作开发者的需求来发展 API。」

Magic Eden 的强制性「反内卷行动」

然而，Solana 领域的一些开发者认为这种转变是拒绝去中心化原则。NFT 市场聚合器 Hyperspace 的一名代表告诉 Decrypt：「我们很惊讶他们这么做，因为这是完全中心化的，对终端用户没有什么好处。因为它增加了对他们服务器的依赖，从而导致交易失败率的增加。」

这位不愿透露姓名的人士表示，在合约更改之前，Magic Eden 联系了 Hyperspace，并威胁说「如果不改变 Hyperspace 的平台，为它们服务，就会关闭 Hyperspace。」据称，Magic Eden 想让 Hyperspace 向 Magic Eden 提供「独家上线信息，并且只通过他们的 API 进行操作。

Magic Eden 的一名代表否认在讨论中威胁他们：「我们鼓励我们的合作伙伴尽可能深入地与 Magic Eden 整合，以便提供尽可能全面的技术和运营支持。不幸的是，Hyperspace 对这样的合作不感兴趣，并一直处于敌对状态。」

Hyperspace 表示，它发现了 Magic Eden API 的解决方案，并继续提供聚合列表，但其他聚合器（如 CoralCube）显然因此失去了功能。「从那时起，他们一直在尝试并积极研究如何阻止我们，」Hyperspace 代表声称。

「CoralCube 曾经有一个迁移列表按钮，但是 Magic Eden 最近转向 Web2 的中心化平台。没有 Magic Eden 集中签名，不能删除项目。这就是为什么我们删除了迁移按钮，现在 NFT 卡在 Magic Eden 的托管中。」

Solana 的一些构建者告诉 Decrypt，他们认为 Magic Eden 的举动是有意为之，目的是将近几个月来获得关注的 NFT 聚合器排除在外。它最终让 Magic Eden 控制谁可以进入其上线列表并从其流动性中受益。Hyperspace 一直在公开反对这一严格的反竞争行为，因为这违反了开放网络的原则。

Magic Eden 新功能受到激烈抨击

此外，Magic Eden 在实施新功能时，似乎受到了外部 Solana 应用程序的启发，因此受到了抨击。上周，Magic Eden 的上线功能允许项目在 NFT 删除之前创建用户允许列表，因为与 Blocksmith Labs 的 Mercurytool 非常相似而受到了抵制。

匿名 NFT 收藏者 Topo Gigio 告诉 Decrypt 关于 Magic Eden 新增的功能：「这似乎是一种不由分说的尝试，将任何可以做得更好的人排除出去。」 同时，Zion Labs 的 Marty 声称：「Magic Eden 正在‘将风险投资作为武器’，迅速扩张成为一体化的 Solana NFT 资源。」

Magic Eden 的 Zhou 回应称，Magic Eden 是一家「用户至上的公司」，它主要根据用户的请求来增加功能。他声称平台上的扩展功能是为 NFT 收藏者服务的，并拒绝了关于中心化的辩论。Zhou 称：「这次对话不是关于集权和去中心化，从来都不是。自我们成立以来，在 Magic Eden 不断发展的市场经验基础上已经存在合作工具，我们不打算改变这种方式。」

对于 Web3 领域的一些参与者来说，围绕 Magic Eden 的整体对话主要是关于中心化与去中心化的，包括该领域的主要参与者应该如何处理诸如资产托管、开源代码以及区块链资产和协议的可组合性等问题。在继续使用第三方托管和以 API 为中心的更改之间，Magic Eden 的决定最近并不适合所有人。但 Magic Eden 仍然是 Solana 收藏者买卖 NFT 的最佳选择。

对 Magic Eden 的批评越来越多，但许多 NFT 项目是否会选择在其他地方发行，以及知名收藏者是否会选择公开立场并退出 Magic Eden 市场，还有待观察。Topo Gigio 就是其中之一。这位 NFT 收藏者在推特上表示，他将放弃流动性，并声称他将不再使用这个市场，并指出 Magic Eden 的托管政策和合约发生了变化。「所有的流动性都在 Magic Eden，我的退出对他们来说无关紧要，」他告诉 Decrypt，「我很乐意把我的高价值资产转移到其他地方，即使交易量较小。」
