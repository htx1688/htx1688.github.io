作者： Callum@Web3CN.Pro

随着以太坊生态的日益庞大，单体区块链的弊端也越来明显，链上拥堵、交易成本高、不可定制等问题逐渐暴露出来，为解决这些性能的局限性，催生出了多样化的扩容解决方案。

目前以太坊的扩容方案分为链上扩容（on-chain）和链下扩容（off-chain）。链上扩容是对区块链本身的性能提升，对以太坊本身进行改造来得到更好的扩展性；链下扩容是与第一层主网分开，在无需更改现有以太坊协议的情况下实现更高的扩展性。当前链上扩容只有分片这一种方案，链下扩容虽然有很多，但目前主要采用Rollup方案。

Rollup的本质是将执行层从单体区块链中分离出来的模块化扩容方案，它将运算转移到链下，再把链下的执行结果传输到链上验证，在继承以太坊安全性的情况下提高网络性能。 在这个过程中，一个新的赛道——Rollup as a Service（RaaS）逐渐衍生出来。RaaS把Rollups的部署作为一种服务，提供可定制的Rollups，为开发者构建Rollup提供更加便捷和灵活的服务。

本文将介绍什么是RaaS、梳理RaaS相关项目并分析RaaS的机遇和挑战。

什么是RaaS

在了解RaaS之前，我们先简单介绍一下Rollup技术原理。它是将大量的交易信息打包成单个交易来进行处理的技术，可以在以太坊网络上提高交易吞吐量、降低交易费用，同时提高网络的去中心化程度。以太坊的扩容需求催生了Rollup的发展，而且还区分了通用型Rollup和特定应用型Rollup。通用型Rollup比如Arbitrum、Optimism，这些项目允许在其链上部署dapp生态，但依赖于Layer1的结算层、数据可用层和共识层。特定应用型Rollup比如Loo它pring、dydx，是作为rollup部署在区块链上的单个应用程序，不需要单独的验证器集合，消耗更少。因此不管是单一的区块链开发还是特定应用开发，都对Rollup工具有很高的需求，这就慢慢衍生出了RaaS市场。

简而言之，Rollup as a Service (RaaS) 就是一款用于推出新的应用程序定制的 Rollup 的产品。该产品类似于software-as-a-service(SaaS)产品，用户不需要维护或托管服务产品的任何部分，因此无需复杂的软件和硬件管理。通过RaaS，开发人员可以获得 SDK 或仪表板来启动特定应用程序的汇总。开发人员不必担心寻找操作汇总的节点，也不必担心构建或维护任何代码，它会提供工具和服务来部署Rollup产品。这种产品提供了强大的抗审查能力、实现安全的交易处理、保证区块链网络的可持续运行。

**RaaS相关项目 **

目前市场上的RaaS项目主要可分为3类：SDK、无代码方案和共享排序器。

SDK

SDK方案为开发者提供一整套通用的软件开发套件，让部署Rollup和部署智能合约一样轻松。代表产品有OP Stack 、Rollkit、Sovereign SDK等。

OP Stack ： 是Optimism 团队为了实现模块化区块链架构的软件堆栈，OP Stack把Layer2分成了共识层、执行层、结算层三层，然后把三层标准化，让开发者很容易的组装，简化了模块化区块链的构建流程。

Rollkit： 是由Celestia社区开发的模块化发链SDK，Rollkit框架具有一个模块化节点，可以运行汇总并公开ABCI兼容的客户端接口，可以在任何 ABCI 兼容的区块链应用程序中用作 Tendermint 的替代品。默认情况下，节点使用 Celestia 作为数据可用性 (DA) 层，但可以集成其他数据可用性层。

Sovereign SDK： 是一个开放、互连的Rollup生态系统，是第一个消除零知识复杂性的rollup框架，使所有开发人员能够无缝部署可在任何区块链上运行的可互操作和可扩展的Rollup。SovereignSDK的目标是简化zk-rollups的创建，类似Cosmos SDK 对Layer1所做的事情。

这些SDK项目提供了用户友好且易于使用的方案，简化了Rollup的部署，也让 RaaS 市场日益成熟。

无代码方案

无代码方案就是无需编写代码就可部署Rollup，为开发人员提供了易于高效部署可扩展Rollup的工具和资源。代表产品有AltLayer、Eclipse、Caldera等。

AltLayer： 是基于Optimistic Rollups构建的扩展层，可以与以太坊等Layer1连接，也可以与 Arbitrum 、Optimism等Layer2连接。AltLayer支持多链和多VM，默认支持EVM 和 WASM。因此，AltLayer 不会被束缚在单一的 Layer1 或 Layer2，而是可作为一个模块化和可插拔的扩展方案，用于所有 EVM 和 WASM 兼容链。

Eclipse： 是基于Cosmos生态的主权Rollup，为开发者提供了一个可定制的Rollup框架。作为模块化区块链的基础设施，它能够为任意公链提供通用的结算层方案。旨在成为一个通用Layer2平台，与多个Layer1区块链兼容。目前，Eclipse 支持以太坊虚拟机和Solana 虚拟机。

Caldera： 是基于Optimistic Rollups构建的可定制Rollup框架，它专注于构建高性能、可定制和特定于应用程序的Layer1区块链。这些定制的区块链提供高吞吐量、低延迟和可定制的功能，用于优化去中心化应用程序的性能和用户体验。Caldera Chains可以选择所有与 EVM 兼容的链。

无代码方案RaaS项目提供了简易、高效、易用的解决方案来部署Rollup，每个方案都有特定的功能（比如支持多链和虚拟机）来满足特定的需求，无代码方案的发展也帮助推动了Rollup技术。

共享排序器

排序器是Layer2架构中非常重要的组件，通过在链下汇集许多Layer2上的用户交易，并将它们作为一个集合的单个交易提交到主链Layer1上，来让Layer2高效运行。共享排序器本质上是一个兼容Rollup的一组排序器，它可以为不同的 Rollup 提供服务。目前以太坊上的所有二层 Rollups使用的都是中心化排序器，但一些 RaaS 项目正在开发去中心化排序器，代表产品有 Dymension和Espresso。

Dymension： 是一个建立在Cosmos上的主权Rollup，是一个模块化区块链项目，旨在通过 Dymension Chain（结算层）、RDK（RollApp 开发工具包）和 IRC（Rollup 间通信）简化了部署RollApp的过程。RollApp是特定于应用程序的Rollup，RollApp的安全性和商定的应用程序状态由Dymension 结算层维护，该结算层要求RollApp排序器抵押 DYM 代币以作为生态系统的一部分参与。

Espresso： Espresso Sequencer 支持 Layer2的去中心化。它处理Rollup交易的去中心化排序和数据可用性，充当汇总及其底层Layer1平台之间的中间件。Espresso Sequencer 被设计为一个可以部署任何zk-VM或optimistic VM的平台。最终，Espresso 还可以通过将 zk-VM 和optimistic VM  同时复制到多个 Layer1 来充当互操作层。

去中心化的排序器节点可以同时处理多个Rollup程序，并使用代币作为激励。在不同链上对交易进行排序并创建区块，也让跨链Rollup更加安全。

**RaaS的优势与挑战 **

未来加密行业多链多Rollup是一个大趋势，随着项目越来越多，他们也会去寻找性能更高、成本更低甚至是可以做一些定制服务的公链，RaaS的发展就为其奠定了基础。

首先，从可定制化角度来看。 RaaS将Rollup模块化，可以让开发者更加灵活地构建和使用基于Rollup的应用程序，满足项目特定的业务需求，从而为整个生态系统带来更多的创新。

其次， 从性能上看，Rollup 技术作为一种 Layer 2 解决方案，本身就可以提高以太坊网络的吞吐量和降低成本。 RaaS就可以最大限度地确保开发者在降低费用的前提下安全部署应用程序。

最后，从商业模式来看，RaaS 的发展前景非常广阔。 随着链上应用的不断增长，以太坊主网拥堵问题将更加严重，因此对RaaS的需求也会增长。

当然，Rollup本身就是在技术条件的限制下发展较为缓慢，RaaS赛道更是在早期阶段。比如安全性、中心化和桥接漏洞以及流动性碎片化等问题，但随着技术的进步，Rollup也在不断改进，主要的是项目今后能否落地和实现。借助RaaS，Rollup也不断发展，实现赛道和生态的大爆发。
